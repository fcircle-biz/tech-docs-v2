2.1 大規模言語モデル（LLM）とは何か

Claude Code の根幹にあるのは、大規模言語モデル（Large Language Model、LLM） と呼ばれる AI 技術です。Claude もその一つです。

LLM はインターネット上の膨大なテキストデータを学習し、「次にどんな言葉が来るか」を繰り返し予測することで、自然な文章を生成できる AI です。単純に「暗記して答える」のではなく、文章の文脈・構造・意味を理解したうえで応答を生成します。これがコードの理解や生成にも非常に役立ちます。

大量のデータで学習

ウェブ上の文章・コード・書籍など膨大な情報を学習しています。

自然言語を理解・生成

日本語・英語などの自然な言葉で指示でき、自然な言葉で応答します。

コードも得意

プログラムコードの読み書き・デバッグ・説明も高い精度でこなします。

用語: 「Claude」と「LLM」の関係

Claude は Anthropic が開発した LLM（大規模言語モデル）の名前です。GPT（OpenAI）や Gemini（Google）なども同じく LLM の一種です。Claude Code はその Claude を使って開発作業を自動化するツールです。

2.2 トークンとコンテキストウィンドウ

LLM を使ううえで理解しておきたい重要な概念が、トークンとコンテキストウィンドウです。これらを知ることで、Claude Code がなぜ大きなファイルを何度かに分けて処理するのかが理解できます。

トークンとは

LLM はテキストを「トークン」という単位に分割して処理します。英語では1単語がおおよそ1トークン、日本語では数文字がまとめて1トークンになります。「Claude Code は便利です」という文章は、5〜10トークン程度に分割されます。

トークン分割のイメージ（英語の例）

Claude Code is very useful

※ 実際の分割はモデルによって異なります

コンテキストウィンドウとは

LLM が「一度に把握できる情報量の上限」をコンテキストウィンドウ（context window）と呼びます。たとえば「20万トークンのコンテキストウィンドウ」を持つモデルは、一度の会話でおよそ20万トークン分（数十万文字）の情報を参照できます。

コンテキストウィンドウの「窓」のたとえ

コンテキストウィンドウは「AI が一度に見られる窓の大きさ」のようなものです。窓が大きいほど、たくさんのコードや会話履歴を参照しながら作業できます。Claude Code は長い会話や大きなファイルを扱う際に、この制限を意識して動作します。

実践上のポイント

コンテキストウィンドウが大きくなると処理時間やコストが増えます。Claude Code は必要なファイルだけを読み込むようにして、効率よく作業を進めます。第8章では、会話の長さを管理する「コンテキスト管理」についても学びます。

2.3 「エージェント」と「ツール使用」の概念

Claude Code が「エージェント型」と呼ばれる理由は、ツール（道具）を使って自律的に作業を進めるからです。ここでいう「ツール」とは、AI がファイルを読んだり、コードを書いたり、コマンドを実行したりするための操作の仕組みのことです。

Claude Code が使う主なツール

ツール名 できること
ファイル読み込み プロジェクト内のファイル内容を読み取り、コードを理解する
ファイル編集 ファイルの内容を追加・変更・削除する
コマンド実行 ターミナルコマンド（テスト実行・ビルドなど）を実行して結果を取得する
検索 ファイル内のテキストやコードのパターンを検索する
ファイル作成 新しいファイルを作成する

たとえば「ログイン処理のバグを直して」という指示を受けたとき、Claude Code は次のようにツールを組み合わせて使います。

Claude Code のツール使用の流れ（概念例） 
● [ファイル読み込み] src/auth/login.js を読んでいます…
● [検索] "token" に関連する処理を探しています…
● [ファイル編集] 有効期限チェックの抜けを修正しています…
● [コマンド実行] npm test を実行して動作を確認しています…
● 修正が完了しました。変更内容を確認しますか？

安全への配慮

コマンドの実行やファイルの変更など、影響の大きい操作にはあなたの確認・許可が必要です。Claude Code は勝手に重要な変更を適用しません。詳しくは第10章（権限と設定）で学びます。

2.4 エージェントの基本ループ

第1章でも紹介したエージェントの基本ループを、ここではより詳しく見ていきましょう。Claude Code は指示を受け取ってから完了まで、次の4段階を繰り返します。

flowchart TD A["あなたの指示"] --> B["1. 読む
（コードとコンテキストを把握）"] B --> C["2. 計画する
（やるべきことを考える）"] C --> D["3. 編集する
（ツールを使って変更・実行）"] D --> E["4. 検証する
（テストや確認で結果をチェック）"] E -->|"問題あり"| C E -->|"完了"| F["報告する"]
1

読む — コードとコンテキストを把握する

「ファイル読み込み」ツールを使い、関連するコードファイル・設定ファイル・エラーログなどを読んで状況を理解します。会話の文脈（コンテキストウィンドウ内の情報）も参照します。

2

計画する — やるべきことを考える

読んだ情報をもとに、「どのファイルをどう変更すれば指示を実現できるか」を考えます。複数の変更が必要な場合は順序も決めます。

3

編集する — ツールを使って変更・実行する

「ファイル編集」「コマンド実行」などのツールを使って実際に作業を進めます。コードの修正・新規ファイルの作成・コマンドの実行を行います。

4

検証する — 結果をチェックする

テストを実行したり、変更後のコードを確認したりして、期待通りに動いているかを検証します。問題があれば計画に戻ってやり直します。

ループが繰り返される理由

プログラムの修正は1度でうまくいかないこともあります。Claude Code は検証で問題を見つけたら自分で計画を立て直し、別のアプローチを試みます。このループがあるからこそ、人間が細かく指示しなくても複雑な作業を完遂できます。

2.5 補完型・チャット型・エージェント型の整理

第1章で紹介した3つのタイプを、ここでは基礎知識を踏まえてより詳しく整理します。それぞれのタイプが LLM をどのように使っているかを意識すると理解が深まります。

タイプ LLMの使い方 ツール使用 作業の実行
補完型 入力の次を予測して提案 なし 人間が行う
チャット型 質問に回答・コード例を生成 限定的 人間が行う
エージェント型
（Claude Code）		 読む・計画・編集・検証を反復 積極的に使用 AI が自律的に進める

補完型とチャット型は LLM を「回答機械」として使うのに対し、エージェント型の Claude Code は LLM を「推論エンジン」として使います。計画を立て、ツールを選び、結果を評価して次の行動を決める、という連続した判断ができる点が根本的な違いです。

重要ポイント

Claude Code の強みは「ツールとループの組み合わせ」にあります。LLM 単体では実際の作業はできませんが、ツールを使う仕組みと繰り返しのループが加わることで、具体的な開発タスクを完遂できるエージェントになります。

2.6 この章の用語まとめ

LLM（大規模言語モデル）

膨大なテキストデータを学習し、自然言語を理解・生成する AI モデル。Claude もその一種。

トークン

LLM がテキストを処理する最小単位。英語では1単語ほど、日本語では数文字がまとめて1トークンになる。

コンテキストウィンドウ

LLM が一度に参照できる情報量の上限。トークン数で表される「AI の作業机の大きさ」のようなもの。

エージェント

目標に向けてツールを使いながら自律的に行動する AI の仕組み。Claude Code はエージェント型。

ツール使用

AI がファイル読み込み・編集・コマンド実行などの外部操作を行う仕組み。これによりエージェントは実際の作業が可能になる。

基本ループ

「読む → 計画する → 編集する → 検証する」という繰り返しのサイクル。Claude Code はこのループで作業を完遂する。

理解度確認クイズ

  1. Q1. 「コンテキストウィンドウ」とは何ですか？ 日常的な言葉で説明してください。

    回答を表示

    LLM が一度に扱える情報量の上限のことです。「AI が一度に見られる窓の大きさ」のようなもので、この範囲内にある情報（コード・会話履歴など）だけを参照しながら作業します。窓から外れた情報は参照できません。

  2. Q2. Claude Code が「エージェント型」と呼ばれる理由を、「ツール使用」という言葉を使って説明してください。

    回答を表示

    Claude Code はファイルの読み書き・コマンド実行・検索などの「ツール」を使って、自律的に作業を進めることができるからです。LLM が回答を返すだけでなく、ツールを組み合わせて実際の開発作業を完遂するためエージェント型と呼ばれます。

  3. Q3. エージェントの基本ループを4つのステップで説明してください。また、検証で問題が見つかったときはどうなりますか？

    回答を表示

    「①読む（コードと状況を把握）→ ②計画する（やることを考える）→ ③編集する（ツールで変更・実行）→ ④検証する（テスト・確認）」という4段階です。検証で問題が見つかった場合は②計画する に戻り、別のアプローチを試みます。うまくいったら結果を報告して完了します。
