ホーム Claude Code 第3章
第3章 所要時間: 約60分

インストールと初期設定

Claude Code を実際にあなたの環境に導入します。必要な前提環境を確認し、インストールから起動・サインイン・動作確認まで、一歩一歩手順に沿って進めていきましょう。

本教材が前提とするバージョンについて

本教材は Claude Code v2.1 系を前提に記述しています。Claude Code は更新が速く、コマンド名やオプションが変わることがあります。手元の環境は claude --version でバージョンを、claude doctor（セッション中は /doctor）で状態を確認し、相違があれば公式ドキュメントの最新情報を優先してください。

この章で学ぶこと

  • Claude Code の導入に必要な環境（OS・Node.js・アカウント）を確認できる
  • npm または代替手段で Claude Code をインストールできる
  • 初回起動・サインイン・動作確認の手順を実行できる
  • よくあるインストールトラブルとその対処の見当をつけられる

3.1 必要な環境を確認する

インストール前に、必要な環境がそろっているかを確認しましょう。以下の4点を押さえておけば、スムーズに導入できます。

対応OS

macOS・Linux に加え、ネイティブ Windows（PowerShell / コマンドプロンプト）も正式にサポートされています。Windows では ネイティブと WSL2 のどちらも利用でき、チームの標準やシェルの互換性に合わせて選べます。

Node.js 18 以上

npm 経由でインストールする場合に必要です。node --version で確認できます。18 未満の場合はアップグレードが必要です。

ターミナル

コマンドラインが使える環境が必要です。macOS は「ターミナル」、Linux は標準のシェル、Windows は PowerShell・コマンドプロンプト、または WSL2 のターミナルを使います。

Anthropic アカウント

Claude.ai の Pro または Max プラン、あるいは Anthropic Console の API キーが必要です。どちらか一方があればサインインできます。

npm 以外のインストール方法

npm 以外に、ネイティブインストーラも提供されています。Node.js 環境を用意せずに導入したい場合や、OS ごとのパッケージマネージャを好む場合はそちらを利用することもできます。詳細は 公式ドキュメント を参照してください。

Node.js のバージョン確認は以下のコマンドで行えます。

terminal — Node.js バージョン確認 
node --version
# v20.x.x のように 18 以上であれば OK

3.2 Claude Code をインストールする

npm が使える環境では、次の1行でインストールできます。-g はグローバルインストール（PC 全体で claude コマンドが使えるようになる）を意味します。

terminal — npm でインストール 
npm install -g @anthropic-ai/claude-code

npm でインストールするときの注意

  • sudo npm install -g は避ける（権限まわりのトラブルの原因になります）。権限エラーが出る場合は npm のグローバルインストール先をユーザー権限のフォルダに変更します。
  • 更新は npm update -g ではなく npm install -g @anthropic-ai/claude-code@latest を使うのが確実です。
  • npm パッケージはお使いの OS 向けのネイティブバイナリを導入します。インストールにはプラットフォーム別の optional dependencies が必要なため、それらを除外する設定（例: --no-optional）は使わないでください。

インストールが完了したら、バージョンを確認してコマンドが正しく認識されているか確認します。

terminal — バージョン確認・環境診断 
# バージョンを確認する
claude --version

# 環境診断（詳細な情報を表示）
claude doctor

claude doctor について

claude doctor コマンドを実行すると、インストール状況・Node.js バージョン・認証状態・接続状況などをまとめて診断できます。何か問題が起きた際に最初に試すと原因の特定に役立ちます。

インストールから動作確認までの流れ

flowchart TD A["Node.js 18以上を用意する"] --> B["npm install -g @anthropic-ai/claude-code"] B --> C["claude --version で確認"] C --> D["プロジェクトフォルダへ移動"] D --> E["claude コマンドで起動"] E --> F["サインイン認証を行う"] F --> G["動作確認（簡単な指示を送る）"] G -->|成功| H["セットアップ完了！"] G -->|エラー| I["claude doctor で診断"]

3.3 起動と初回サインイン

インストールが完了したら、実際に Claude Code を起動してみましょう。まず作業したいプロジェクトのフォルダに移動してから claude コマンドを実行します。

terminal — Claude Code を起動する 
# プロジェクトのフォルダに移動
cd /path/to/your-project

# Claude Code を起動
claude

初回起動時は認証フローが自動的に始まります。サインインには2つの方法があります。

Claude.ai アカウントでログイン

ブラウザが開き、Claude.ai の認証画面にリダイレクトされます。Pro または Max プランが必要です。

個人利用・入門向け

API キーで認証

Anthropic Console で発行した API キーを入力します。チームや CI/CD など、アカウントが使いにくい場面で便利です。

チーム・自動化向け

重要ポイント: 認証は一度だけ

一度サインインすれば、認証情報はローカルに保存されます。次回以降は claude コマンドを実行するだけで即座にセッションが始まります。

3.4 動作確認

サインインが完了すると、ターミナルにプロンプト（>）が表示されます。簡単な指示を送って、Claude Code が正常に動いているか確認しましょう。

terminal — 動作確認の例 
$ claude
> このプロジェクトについて教えて

● プロジェクトのファイル構成を確認しています…
● package.json・README を読んでいます…
● このプロジェクトは Node.js 製の〇〇アプリケーションです。
  主なエントリポイントは src/index.js で…（以下、説明が続く）

動作確認チェック

  • claude --version でバージョンが表示される
  • claude コマンドでプロンプトが表示される
  • 簡単な質問（「このプロジェクトについて教えて」など）に対して回答が返ってくる

3.5 うまくいかない時のポイント

インストール後につまずきやすい代表的な問題と、最初に確認することをまとめます。詳細なトラブルシューティングは第14章で扱います。

よくある問題と対処

「command not found: claude」と表示される

PATH が通っていない可能性があります。npm prefix -g で表示されるパスの bin フォルダ（npm のグローバルインストール先）を、シェルの設定ファイル（.bashrc / .zshrc）の PATH に追記してください。その後 source ~/.zshrc で反映します。claude doctor でも導入状況を診断できます。

インストール時にエラーが出る

まず node --version で Node.js が 18 以上かを確認してください。バージョンが古い場合は Node.js をアップグレードしてから再実行します。

サインインができない・認証でエラーになる

claude doctor を実行して認証状態を確認しましょう。Pro/Max プランになっているかの確認と、インターネット接続も見直してください。

やってみよう 3-1: インストール〜動作確認を通しで行う

以下の手順をターミナルで順番に実行し、Claude Code を動かせる状態にしましょう。

手順

  1. 1 node --version を実行して Node.js が 18 以上かを確認する。
  2. 2 npm install -g @anthropic-ai/claude-code を実行してインストールする。
  3. 3 claude --version でバージョンが表示されることを確認する。
  4. 4 任意のプロジェクトフォルダに移動して claude を起動し、認証フローに沿ってサインインする。
  5. 5 プロンプトが表示されたら「このプロジェクトについて教えて」と入力して、回答が返ってくることを確認する。

ゴール

Claude Code からプロジェクトについての説明や回答が返ってきたら成功です。次の章では、この状態からさらに基本操作を学んでいきます。

理解度確認クイズ

  1. Q1. npm 経由で Claude Code をインストールするために必要な Node.js の最低バージョンは何ですか？

    回答を表示

    Node.js 18 以上が必要です。node --version で確認でき、18 未満の場合はアップグレードが必要です。

  2. Q2. Claude Code のサインインには、どのような認証方法がありますか？2つ答えてください。

    回答を表示

    ① Claude.ai の Pro または Max プランのアカウントでブラウザ経由でログインする方法と、② Anthropic Console で発行した API キーを入力して認証する方法の2つがあります。

  3. Q3. インストール後に「command not found: claude」というエラーが出た場合、最初に確認すべきことは何ですか？

    回答を表示

    npm のグローバルインストール先が PATH に通っていない可能性が高いです。npm prefix -g で表示されるパスの bin フォルダを確認し、シェル設定ファイルの PATH に追記することで解決できます。また、claude doctor で環境診断を行うことも有効です。
