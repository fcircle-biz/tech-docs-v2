インストールと初期設定

いよいよ Codex を実際にインストールしましょう。この章では、公式インストーラ（npm・Homebrew も可）を使ったインストール方法、対応プラットフォーム（macOS / Windows / Linux）の確認、初回のサインイン設定、そして動作確認までを手順に沿って丁寧に進めます。

この章で学ぶこと

  • Codex がサポートするプラットフォーム（macOS / Windows / Linux）を理解する
  • 公式インストーラ・npm・Homebrew のいずれかで Codex をインストールできる
  • ChatGPT アカウントと OpenAI API キー、2 種類のサインイン方式の違いを理解する
  • codex --version 等で動作確認し、最初のセッションを起動できる

3.1 対応プラットフォームと事前準備

Codex CLI はネイティブのターミナル環境で動作します。以下の3つのプラットフォームが公式にサポートされています。お使いの環境を確認してから先に進みましょう。

macOS

最も多くの開発者が使う環境。公式インストーラ・Homebrew・npm のいずれでも簡単にインストールできます。

Linux

Ubuntu / Debian / Fedora などの主要ディストリビューションに対応。公式インストーラまたは npm でインストールします。

Windows

PowerShell 上でネイティブに動作します。Linux 互換の環境が必要な場合は WSL2（Windows Subsystem for Linux 2）も選択できます。

Windows をお使いの方へ

Codex CLI は Windows でもネイティブに動作します（PowerShell から実行）。Windows 用のサンドボックスも利用できます。Linux ネイティブの開発環境が必要な場合は WSL2（Windows Subsystem for Linux 2） を使う選択肢もあります。WSL2 のセットアップ方法は Microsoft 公式ドキュメントを参照してください。

インストールは必ず公式の入手元から

Codex を装った偽パッケージ・偽リポジトリが出回ることがあります。インストール時は公式インストーラ（chatgpt.com/codex）、公式 npm パッケージ名（@openai/codex）、公式 GitHub リポジトリ（openai/codex）であることを必ず確認してください。発行元が不明なパッケージは認証トークンを盗む恐れがあります。

事前に必要なもの

インストール前に以下を確認してください。

  • Node.js 18 以上（npm 経由でインストールする場合） — node -v でバージョンを確認できます
  • Homebrew（macOS で Homebrew を使う場合） — 未インストールなら brew.sh から入れましょう
  • ChatGPT アカウント（Plus / Pro / Business / Edu / Enterprise などの対象プラン）または OpenAI Platform の API キー — どちらか一方があれば OK
terminal — Node.js バージョン確認 
# Node.js のバージョンを確認（18 以上が必要）
node -v
# v20.11.0 などと表示されれば OK

# npm のバージョンも合わせて確認
npm -v
# 10.x.x などと表示されれば OK

3.2 Codex のインストール

インストール方法は複数あります。お使いの環境や好みに合わせて選んでください。特にこだわりがなければ公式インストーラが手軽です。

インストール方法の選択フロー

flowchart TD A["Codex をインストールしたい"] --> B{"お使いの環境は？"} B -->|"macOS / Linux"| C["公式インストーラ（推奨）
または npm・Homebrew"] B -->|"Windows"| D["PowerShell インストーラ（推奨）
または npm"] C --> G["動作確認へ"] D --> G

方法 A: 公式インストーラ（推奨）

OpenAI 公式のインストールスクリプトを使う方法です。Node.js が無くても導入でき、現在の主要な導線です。

terminal — 公式インストーラ 
# macOS / Linux: 公式インストールスクリプト
curl -fsSL https://chatgpt.com/codex/install.sh | sh

# Windows（PowerShell）: 公式インストーラ
powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"

# インストール完了後、バージョンを確認
codex --version

方法 B: npm でインストール（全プラットフォーム共通）

Node.js の npm（パッケージマネージャ）を使う方法です。macOS・Windows・Linux のいずれでも使えます。

terminal — npm でグローバルインストール 
# Codex CLI をグローバルインストール（-g: システム全体で使えるようにする）
npm install -g @openai/codex

# インストール完了後、バージョンを確認
codex --version

EACCES エラーが出た場合

npm install -g 実行時に権限エラーが出ることがあります。その場合は sudo npm install -g @openai/codex を試すか、npm のグローバルディレクトリをホームフォルダに変更する設定を行ってください（npm 公式ドキュメント参照）。

方法 C: Homebrew でインストール（macOS）

macOS で Homebrew を使っている場合は、こちらの方法が手軽です。Codex は cask として配布されています。

terminal — Homebrew でインストール 
# Homebrew で Codex をインストール（cask）
brew install --cask codex

# インストール完了後、バージョンを確認
codex --version

どれを選べばよいか？

どの方法でインストールしても機能は同じです。特にこだわりがなければ公式インストーラが手軽です。Node.js プロジェクトで普段から npm を使っている方は npm、macOS で Homebrew を使い慣れている方は Homebrew を選ぶとよいでしょう。

3.3 初回のサインイン

Codex を初めて起動すると、認証（サインイン）を求められます。サインイン方式は 2種類 あります。どちらか1つを設定すれば OK です。

方式 1: ChatGPT アカウント

  • ChatGPT Plus / Pro / Team などの有料プランが必要
  • ブラウザで OAuth サインインするだけで簡単
  • API 利用料は ChatGPT サブスクリプションに含まれる

方式 2: OpenAI API キー

  • OpenAI Platform（platform.openai.com）で発行した API キーを使う
  • 従量課金（トークン数に応じた料金）
  • 使用するモデルを細かく指定したい上級者向け

方式 1 の手順: ChatGPT アカウントでサインイン

初心者には最もシンプルな方法です。ブラウザで操作するだけで完了します。

terminal — Codex を起動してサインイン 
# Codex を起動（初回は認証フローが始まる）
codex

# → ターミナルに URL が表示される
#   ブラウザでその URL を開き、ChatGPT アカウントでログイン
#   承認すると認証完了 — ターミナルに戻って操作できるようになる

方式 2 の手順: API キーを設定する

API キーを使う場合は、環境変数 OPENAI_API_KEY に設定します。

terminal — API キーを環境変数に設定 
# API キーを環境変数にセット（sk-... の部分を実際のキーに置き換える）
export OPENAI_API_KEY="sk-..."

# 永続化するには ~/.bashrc または ~/.zshrc に追記する
echo 'export OPENAI_API_KEY="sk-..."' >> ~/.zshrc
source ~/.zshrc

API キーの取り扱いに注意

API キーはパスワードと同じ秘密情報です。コードや Git リポジトリに直接書かず、環境変数や秘密管理ツールを使って管理してください。流出するとAPIを不正利用される恐れがあります。

3.4 動作確認

インストールとサインインが完了したら、いくつかのコマンドで正しく動作することを確認しましょう。

terminal — 動作確認コマンド一覧 
# バージョン確認（インストール済みかどうかのチェック）
codex --version
# 例: 0.140.x などのバージョン番号が表示されれば OK

# ヘルプを表示（使えるオプションを確認）
codex --help

# 対話型セッションを起動（終了は /exit または Ctrl+C）
codex

動作確認チェックリスト

  • codex --version でバージョン番号が表示された
  • codex を実行してターミナル UI（TUI）が表示された
  • 「こんにちは」などと入力して AI から応答が返ってきた

よくあるトラブルと対処法

「codex: command not found」と表示される

→ npm のグローバルディレクトリが PATH に含まれていない可能性があります。npm root -g で確認し、~/.bashrc や ~/.zshrc に PATH を追加してください。

認証エラー（Unauthorized）が表示される

→ API キーの場合は値が正しいか確認。ChatGPT アカウントの場合は再度 codex を起動して認証フローをやり直してみてください。

Node.js のバージョンが古いと言われる

→ nvm（Node Version Manager）を使って Node.js 18 以上に更新してください。

やってみよう 3-1: Codex をインストールして動かしてみる

この章で学んだ手順を実際に自分の環境で試してみましょう。詰まったら焦らず、トラブルシューティングを参考に一つずつ確認しましょう。

手順

  1. 1 node -v でバージョンを確認し、18 未満なら更新する。
  2. 2 公式インストーラ（または npm install -g @openai/codex）を実行する。
  3. 3 codex --version でバージョンが表示されることを確認する。
  4. 4 codex を起動してサインインし、「こんにちは」と入力して AI が応答することを確認する。
  5. 5 /exit または Ctrl+C でセッションを終了する。

理解度確認クイズ

  1. Q1. npm を使って Codex をインストールするコマンドは何ですか？

    回答を表示

    npm install -g @openai/codex です。-g フラグによりシステム全体（グローバル）にインストールされ、どのディレクトリからでも codex コマンドを使えるようになります。

  2. Q2. Windows で Codex を使うにはどうすればよいですか？

    回答を表示

    Codex は Windows でもネイティブに動作します。PowerShell 上で公式インストーラ（irm https://chatgpt.com/codex/install.ps1 | iex）または npm を使ってインストールします。Linux 互換の開発環境が必要な場合は WSL2（Windows Subsystem for Linux 2） も選択できます。

  3. Q3. Codex の初回サインイン方式を2つ答え、それぞれの違いを説明してください。

    回答を表示

    ① ChatGPT アカウント: ChatGPT Plus / Pro / Team などの有料プランを持っていれば、ブラウザの OAuth サインインで簡単に認証できます。API 利用料はサブスクリプションに含まれます。
    ② OpenAI API キー: OpenAI Platform で発行した API キーを OPENAI_API_KEY 環境変数に設定します。従量課金制で、使用モデルを細かく指定できます。
