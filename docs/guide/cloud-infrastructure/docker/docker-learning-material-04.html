NginxでWebサーバーを動かす：ポートの理解

この章では、Nginxというwebサーバーをコンテナとして起動し、ブラウザからWebページを表示します。コンテナに名前を付ける方法と、PCとコンテナの通信を結びつける「ポートフォワーディング」という仕組みを、図を使って丁寧に理解します。

この章で学ぶこと

  • Nginxとは何か、Webサーバーの役割を理解する
  • -p 8080:80 の意味（ホスト側ポートとコンテナ側ポート）を説明できる
  • --name でコンテナに名前を付けて起動できる
  • ブラウザからコンテナのWebページにアクセスして動作を確認できる

4.1 NginxとWebサーバーとは

インターネット上のWebページは、どうやって私たちのブラウザに届くのでしょうか。その仕組みを担うのが Webサーバー（ウェブサーバー）です。Webサーバーとは、ブラウザから「このページを見せてください」というリクエスト（要求）を受け取り、HTMLやCSS・画像などを返送するソフトウェアのことです。

Nginx（エンジンエックス）は、世界中で広く使われているWebサーバーソフトの1つです。高速で軽量なことで知られており、大規模なWebサイトから個人の開発環境まで、幅広い場面で利用されています。

高速・軽量

多くのリクエストを同時にさばける設計で、応答速度が速いのが特長です。

世界標準

世界の大規模サイトの多くが採用しており、情報や事例が豊富です。

Dockerで手軽に使える

インストールや設定不要。Docker Hubのイメージを取り寄せるだけで動かせます。

通常、Nginxを使うには自分でソフトをインストールして設定ファイルを編集する必要があります。しかしDockerを使えば、Docker HubにあるNginxイメージを取り寄せるだけで、すぐにWebサーバーを動かせます。この手軽さがDockerの大きな魅力の1つです。

4.2 Nginxコンテナを起動する

それでは実際にNginxを起動してみましょう。ターミナルを開き、以下のコマンドを入力してください。このコマンドは、これまでの章で使った docker run に、新しいオプションが加わっています。

ターミナル 
# Nginxコンテナを起動する
# --name でコンテナ名を指定し、-p でポートをつなぐ
docker run --name my-nginx -p 8080:80 nginx

コマンドの各部分には、それぞれ意味があります。1つずつ確認しましょう。

部分 意味
docker run イメージからコンテナを作って起動する基本コマンド
--name my-nginx コンテナに「my-nginx」という名前を付ける。名前を付けると、停止・削除のときに名前で指定できて便利。
-p 8080:80 ポートをつなぐ（詳しくは4.4節で解説）。左側がPC側、右側がコンテナ側のポート番号。
nginx 使うイメージの名前。Docker Hubから自動で取り寄せる。

ターミナルが戻ってこない！これは正常です

コマンドを実行すると、ターミナルがログ（記録）を表示したまま入力待ちの状態に戻りません。これは フォアグラウンド実行 と呼ばれる状態で、コンテナがずっと動き続けていることを意味します。
止めたい場合は Ctrl + C を押します。別のコマンドを試す場合は、別のターミナルを開いてそちらで操作しましょう。

ターミナルにログが流れ始めたら、Nginxが正常に起動しています。次は、ブラウザで動作を確認しましょう。

4.3 ブラウザで確認する

Nginxが起動した状態で、ブラウザのアドレスバーに以下のURLを入力してアクセスしてみましょう。

ブラウザのアドレスバー 
http://localhost:8080

動作確認：ブラウザでアクセスする

ブラウザに以下の内容が表示されれば成功です。

Welcome to nginx!

このメッセージがNginxのデフォルトページです。

【スクリーンショット：ブラウザに Welcome to nginx! が表示された画面】

ページが表示されたということは、あなたのPC上のDockerコンテナの中でNginxが動いており、そのNginxがブラウザからのリクエストに返答できた、ということです。

コンテナが本当に動いているか、別のターミナルを開いて確認することもできます。次のコマンドを実行してみましょう。

別のターミナル 
# 動作中のコンテナ一覧を確認する
docker ps

実行すると、以下のような一覧が表示されます。my-nginx という名前のコンテナが Up 状態であることを確認できます。

docker ps の出力例 
CONTAINER ID   IMAGE   COMMAND                  CREATED        STATUS        PORTS                  NAMES
a1b2c3d4e5f6   nginx   "/docker-entrypoint.…"   2 minutes ago  Up 2 minutes  0.0.0.0:8080->80/tcp   my-nginx

PORTS 列に 0.0.0.0:8080->80/tcp と表示されています。これが「PC側の8080番ポートをコンテナの80番に転送中」という状態を表しています。次の節で、この仕組みを詳しく理解しましょう。

4.4 ポートとは・-p 8080:80 の意味【重要】

この節は、Dockerを使ううえで非常に重要な概念です。「ポート」という言葉をしっかり理解しましょう。

ポート（port）とは、コンピュータが通信をやり取りする「窓口」を区別するための番号です。1台のコンピュータにはポートが65535個あり、それぞれに番号が振られています。たとえばWebの通信は通常 80番、セキュアなWebは 443番、というように、よく使われるポート番号はあらかじめ決まっています。

-p 8080:80 の意味 — 4つのポイント

  • 1 書き方は -p ホスト側:コンテナ側。左がPC、右がコンテナのポート番号です。
  • 2 左側の 8080 は、あなたの PC（ホストマシン）側のポート番号です。ブラウザはここにアクセスします。
  • 3 右側の 80 は、コンテナ内部のNginxが使っているポート番号です。Nginxはデフォルトで80番を使います。
  • 4 Dockerが橋渡し（転送）をしてくれます。ブラウザが8080番にアクセスすると、Dockerがそれをコンテナの80番に届けます。

コンテナは独立した空間なので、そのままではPC側からコンテナ内部のNginxに通信できません。-p オプションで「窓口をつなぐ」ことで、初めてブラウザからコンテナ内のWebサーバーにアクセスできるようになります。下の図でその流れを確認しましょう。

図4: ポートフォワーディングの仕組み（-p 8080:80）

flowchart LR Browser["ブラウザ
localhost:8080にアクセス"] Host["ホスト（あなたのPC）
8080番ポートで受信"] Docker["Dockerが転送
8080 → 80"] Container["コンテナ内のNginx
80番ポートで待機"] Browser -->|"http://localhost:8080"| Host Host --> Docker Docker -->|"ポート80へ転送"| Container Container -->|"Webページを返す"| Browser

ポート番号は自由に変えられます。たとえば -p 9090:80 とすれば、http://localhost:9090 でアクセスできます。ただし、同じポート番号を複数のコンテナで同時に使うことはできません。複数のコンテナを動かすときは、PC側のポート番号（左側）をそれぞれ別の番号にする必要があります。

ポート番号についての豆知識

1024番以下のポートは「よく知られたポート（ウェルノウンポート）」と呼ばれ、OSの管理下に置かれます。そのためローカル開発では、ぶつかりにくい8080番・3000番・8000番などを使うのが一般的です。

4.5 コンテナを停止・削除する

Nginxの動作を確認したら、コンテナを片付けましょう。今回はコンテナに「my-nginx」という名前を付けているので、長い英数字のIDではなく名前で操作できます。

コンテナを停止する

まずコンテナを止めます。フォアグラウンドで動いているターミナルとは別のターミナルで実行してください。

ターミナル（別ウィンドウ） 
# 名前を指定してコンテナを停止する
docker stop my-nginx

docker stop は、コンテナに「終了してください」と伝えて穏やかに止めるコマンドです。実行すると my-nginx と表示され、停止したことがわかります。フォアグラウンドのターミナルも自動で終了します。

コンテナを削除する

停止しただけではコンテナはまだ残っています。完全に削除するには docker rm を使います。

ターミナル 
# 停止済みのコンテナを削除する
docker rm my-nginx

削除後に docker ps -a を実行すると、my-nginx がリストから消えていることを確認できます。これでコンテナが完全に片付きました。

この章で使ったコマンドのまとめ

コマンド 意味
docker run --name my-nginx -p 8080:80 nginx nginxイメージから「my-nginx」という名前のコンテナを作り、PCの8080番をコンテナの80番につないで起動する
docker ps 動作中のコンテナ一覧を確認する
docker stop my-nginx my-nginxコンテナを停止する
docker rm my-nginx 停止したmy-nginxコンテナを削除する

よくあるエラーと対処法

エラー: port is already allocated

PC側の8080番ポートがすでに別のプロセスやコンテナに使われている場合に発生します。

対処法：

  • docker ps で動いているコンテナを確認し、不要なものを docker stop で止める
  • または -p 8081:80 のように別のポート番号（左側）を使う

ブラウザでページが表示されない

ブラウザにアクセスしても「接続できません」などと表示される場合はコンテナの状態を確認しましょう。

確認ポイント：

  • docker ps を実行してコンテナが Up 状態か確認する
  • URLが http://localhost:8080 と正確か確認する（https ではなく http）
  • ポート番号が -p で指定したホスト側の番号と一致しているか確認する

コンテナを止め忘れている

作業が終わったのにコンテナが動いたままになっていると、PCのリソースを使い続けます。

対処法：

  • docker ps で動作中のコンテナを確認し、不要なものは docker stop で止める

ターミナルが入力待ちに戻らない

これはエラーではなく正常な動作です。コンテナがフォアグラウンドで動き続けている状態です。

対処法：

  • 別のターミナルを開いてそちらで他のコマンドを実行する（推奨）
  • Ctrl + C を押すとコンテナが停止してプロンプトが戻る

この章のまとめ

  • NginxはWebサーバーソフトウェアの1つ。Dockerを使えばインストール不要で起動できる。
  • --name でコンテナに名前を付けると、コマンドでの操作がわかりやすくなる。
  • ポートは通信の窓口番号。-p 8080:80 で「PC側8080番 → コンテナ側80番」とつなぐ（ポートフォワーディング）。
  • コンテナは docker stop で停止、docker rm で削除する。

確認問題

  1. Q1. docker run -p 8080:80 nginx のうち、8080 はどちら側のポート番号ですか？また、80 はどちら側のポート番号ですか？

    回答を表示

    8080 はホスト（PC）側のポート番号です。80 はコンテナ内部（Nginx）のポート番号です。ブラウザは PC側の8080番にアクセスし、DockerがそれをコンテナのNginxが待機している80番に転送します。

  2. Q2. Nginxコンテナを起動したとき、ターミナルが入力待ちに戻らないのはなぜですか？どう対処しますか？

    回答を表示

    コンテナがフォアグラウンドで動き続けているためです。これは正常な状態です。他のコマンドを実行したい場合は別のターミナルを開いて操作します。コンテナを止めたい場合は Ctrl + C を押すか、別のターミナルで docker stop my-nginx を実行します。

  3. Q3. --name でコンテナに名前を付けるメリットは何ですか？

    回答を表示

    名前を付けることで、停止・削除・ログ確認などの操作を、長くて覚えにくいコンテナIDの代わりに、わかりやすい名前（例: my-nginx）で実行できます。複数のコンテナを扱う場合に、どのコンテナか一目でわかるようになります。
