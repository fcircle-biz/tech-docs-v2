第5章 所要時間: 約50分

Dockerfileで自分のWebページをイメージ化する

これまでDocker Hubにある既存のイメージを使ってきました。この章では、Dockerfile（ドッカーファイル）という「手順書」を自分で書き、オリジナルのHTMLページを含む独自のDockerイメージを作ります。「既存のレシピに自分のアレンジを加えて、新しいレシピを作る」イメージです。

この章で学ぶこと

  • Dockerfileがイメージを作るための「手順書」だと理解する
  • FROM と COPY の役割を説明できる
  • docker build コマンドでオリジナルイメージを作れる
  • 自作HTMLページをコンテナで起動してブラウザに表示できる

実行前チェック（コマンドを打つ前に確認）

  • Rancher Desktop が起動しているか（タスクトレイ／メニューバーのアイコンを確認）
  • docker version が成功するか（Client と Server の両方が表示される）
  • 同じ名前のコンテナ（my-web-container）が残っていないか（docker ps -a で確認。あれば削除）
  • 8080番ポートを他のコンテナが使っていないか（docker ps で確認）

5.1 Dockerfileとは

第1章で「イメージはレシピ、コンテナは出来上がった料理」と学びました。では、その「レシピ」を自分で作るにはどうすればよいでしょうか。そのための書き物が Dockerfile（ドッカーファイル） です。

Dockerfileは、イメージを作るための手順書です。「どのイメージを土台にするか」「どんなファイルを追加するか」「起動時に何をするか」といった指示を1行ずつ書いていきます。Dockerはこの手順書を上から順に読み、指示どおりにイメージを組み上げます。

たとえ話: 既存レシピに自分のアレンジを加える

「Nginxを使って自分のWebページを配信したい」という場合、Nginxの環境を1からセットアップする必要はありません。Docker Hub にある公式のNginxイメージを土台（ベース）にして、そこに自分のHTMLファイルを追加するだけでよいのです。イメージで言えば「有名シェフのレシピを借りて、自分の好みの材料を1つだけ加えた新しいレシピを作る」ようなものです。

図5-1: Dockerfileがイメージになり、コンテナになる流れ

flowchart LR DF["Dockerfile
（手順書）"] -->|"docker build"| IMG["オリジナルイメージ
（自分のHTMLを含む）"] IMG -->|"docker run"| CTR["コンテナ
（Webサーバーが起動）"] CTR -->|ブラウザでアクセス| BW["http://localhost:8080"]

5.2 ファイルを準備する

まず、作業用のフォルダとファイルを用意します。今回は my-docker-web というフォルダを作り、その中に Dockerfile と index.html の2つのファイルを置きます。

今回のフォルダ構成 

my-docker-web/
├── Dockerfile       ← イメージを作る手順書
└── index.html       ← 表示したいHTMLページ

まず、ターミナルで作業フォルダを作ってその中へ移動しましょう。フォルダを作るコマンドです。

ターミナル 
# 作業用フォルダを作成して移動する
mkdir my-docker-web
cd my-docker-web

次に、表示するHTMLファイルを作ります。内容はシンプルなWebページです。テキストエディタで my-docker-web フォルダの中に index.html を作り、以下の内容を貼り付けて保存してください。

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>はじめてのDocker</title>
</head>
<body>
  <h1>Dockerで表示しています</h1>
  <p>このページは、自分で作成したDockerイメージから表示されています。</p>
</body>
</html>

続いて、Dockerfileを作ります。同じフォルダの中に Dockerfile（先頭大文字・拡張子なし）というファイルを作り、以下の2行を書いて保存してください。

Dockerfile 
# 土台にするイメージを指定する（公式のnginx）
FROM nginx
# 手元のHTMLファイルをNginxの配信フォルダにコピーする
COPY index.html /usr/share/nginx/html/index.html

FROM nginx

このイメージの土台（ベースイメージ）を指定します。FROM nginx と書くと、Docker Hub にある公式のNginxイメージをベースにします。「有名シェフの基本レシピを借りる」部分です。Dockerfileには必ず最初に書く必要があります。

COPY

手元のファイルをイメージの中にコピーする命令です。COPY コピー元 コピー先 の形式で書きます。ここでは手元の index.html を、Nginxがファイルを配信するフォルダ /usr/share/nginx/html/ にコピーします。

重要ポイント: Nginxのファイル配信フォルダ

Nginxは /usr/share/nginx/html/ に置かれたファイルをWebとして配信します。このパスはNginxの設定で決まっている固定の場所です。そこに自分の index.html をコピーすることで、アクセスしたときに自作ページが表示されます。

補足：FROM nginx とバージョンタグ（実務のヒント）

学習用には FROM nginx のままで十分です。nginx だけ書くと「最新版（latest）」が使われます。

一方で実務では、いつビルドしても同じ結果になるように、FROM nginx:1.27 のようにバージョンタグを指定することがあります。「最新版」は時間が経つと中身が変わるため、バージョンを固定して実行結果を安定させるねらいです。

nginx のような Docker Hub の公式イメージは、Nginx の配布元が用意・メンテナンスしている信頼できるイメージです。自分でゼロから環境を作らなくても、公式の「すぐ動く土台」を安心して再利用できるのが利点です。

5.3 イメージをビルドして動かす

ファイルの準備ができたら、Dockerfileからイメージを作る（ビルドする）操作に進みます。ビルドとは「手順書どおりに材料を組み合わせて、イメージという完成品を作る」ことです。

STEP 1: イメージをビルドする

my-docker-web フォルダにいる状態で、次のコマンドを実行します。

-t my-web は作るイメージに 「my-web」という名前（タグ）をつけるオプションです。最後の .（ピリオド）は「今いるフォルダのDockerfileを使う」という意味で、省略できない重要なポイントです。

ターミナル — my-docker-web/ 内で実行 
# Dockerfileから「my-web」という名前のイメージを作る
# 末尾の「.」は「このフォルダのDockerfileを使う」という意味（必須！）
docker build -t my-web .

よくあるミス: 末尾の「.」を忘れる

docker build -t my-web と書いてピリオドを省略すると、「ビルドコンテキスト（Dockerfileのある場所）」が指定されないためエラーになります。必ず末尾にスペース＋ピリオドをつけることを忘れないようにしましょう。

コマンドを実行すると、Dockerが手順書（Dockerfile）を読み、Nginxイメージを取得してファイルをコピーする処理を行います。完了すると「Successfully built ...」や「Successfully tagged my-web:latest」のようなメッセージが表示されます。

画面で確認するポイント

ビルドの最後に「Successfully tagged my-web:latest」（または「Successfully built ...」）のようなメッセージが表示されれば、イメージの作成は成功です。

STEP 2: コンテナを起動してブラウザで確認する

ビルドしたイメージ my-web からコンテナを起動します。-d でバックグラウンド起動し、--name でコンテナに名前をつけ、-p 8080:80 でポートをつなぎます。

ターミナル 
# my-webイメージからコンテナをバックグラウンドで起動する
# -d: バックグラウンドで動かす（ターミナルがすぐ戻る）
# --name: コンテナに「my-web-container」という名前をつける
# -p 8080:80: PCの8080番とコンテナの80番（Nginxの標準ポート）をつなぐ
docker run -d --name my-web-container -p 8080:80 my-web

コンテナが起動したら、ブラウザで http://localhost:8080 を開きます。

動作確認

ブラウザに次のように表示されれば成功です。

Dockerで表示しています

このページは、自分で作成したDockerイメージから表示されています。

「Dockerで表示しています」という見出しが見えればOKです。あなたが書いたHTMLが、自作のDockerイメージを通じてWebページとして配信されています。

画面で確認するポイント

ブラウザに「Dockerで表示しています」という見出しと説明文が表示されれば成功です。自分で書いた index.html が、自作イメージを通して配信されています。

STEP 3: コンテナを停止・削除する

確認が終わったらコンテナを停止します。停止コマンドでコンテナの実行が止まります。その後、不要であれば削除します。

ターミナル 
# コンテナを停止する
docker stop my-web-container

# 停止したコンテナを削除する
docker rm my-web-container

この章で使ったコマンドの整理

コマンド 意味
docker build -t my-web . 今いるフォルダのDockerfileから「my-web」という名前のイメージを作る（末尾の . を忘れずに）
docker run -d --name my-web-container -p 8080:80 my-web my-webイメージからコンテナをバックグラウンドで起動し、PCの8080番をコンテナの80番に接続する
docker stop my-web-container コンテナを停止する
docker rm my-web-container 停止したコンテナを削除する

5.4 補足: 他のDockerfile命令

この章では FROM と COPY の2つだけを使いました。Dockerfileにはほかにもよく使われる命令があります。ここでは名前と概要を簡単に紹介します。深く学ぶのは後の章や実務で必要になったタイミングで構いません。

よく使われるDockerfile命令（参考）

  • FROM 土台にするベースイメージを指定する。必ず先頭に書く。
  • COPY 手元のファイルやフォルダをイメージの中にコピーする。
  • RUN イメージを作る過程でコマンドを実行する（例: パッケージのインストール）。
  • CMD コンテナ起動時にデフォルトで実行するコマンドを指定する。
  • WORKDIR 以降の命令を実行する作業ディレクトリを指定する。
  • EXPOSE コンテナが使うポート番号を明示する（あくまでドキュメント的な役割）。

よくあるエラーと対処法

Dockerfileのファイル名が間違っている

ファイル名は 先頭が大文字の「D」 で始まる Dockerfile です。拡張子はありません（Dockerfile.txt や dockerfile は認識されません）。docker build を実行したときに「unable to prepare context: unable to evaluate symlinks in Dockerfile path」と表示された場合はファイル名を確認しましょう。

docker build の末尾の「.」を忘れた

docker build -t my-web とだけ書いて . を省略すると、「"docker build" requires exactly 1 argument」などのエラーになります。最後にスペースとピリオドを必ずつけてください。

COPYするファイル名が実ファイルと違う

Dockerfile内の COPY index.html ... は、実際に手元に index.html というファイルが存在している必要があります。ファイル名の大文字小文字や拡張子のスペルが1文字でも違うと「COPY failed: file not found」エラーになります。ファイル名を見直してください。

同じ名前のコンテナがすでに存在する

docker run -d --name my-web-container ... を実行したとき「Conflict. The container name "/my-web-container" is already in use」と表示された場合、同じ名前のコンテナが残っています。起動中なら停止してから、停止中ならそのまま削除し、再実行しましょう（起動中／停止中の詳しい手順は第8章でも解説します）。

ターミナル 
# 既存のコンテナを停止してから削除する（停止中なら stop は省略可）
docker stop my-web-container
docker rm my-web-container

# 削除できたら、もう一度バックグラウンドで起動する
docker run -d --name my-web-container -p 8080:80 my-web

この章のまとめ

  • Dockerfile はイメージを作るための手順書。上から順にDockerが読み、イメージを組み上げる。
  • FROM は土台にするベースイメージの指定。COPY は手元のファイルをイメージにコピーする命令。
  • docker build -t my-web . でDockerfileをビルドしてイメージを作る。末尾の . は必須。
  • 自作イメージからコンテナを起動すると、自分が書いたHTMLページをブラウザで確認できる。
  • 他にも RUN・CMD・WORKDIR・EXPOSE などの命令があり、より複雑なイメージを作るために使う。

確認問題

  1. Q1. Dockerfileの役割を一言で説明してください。また、Dockerfileを使ってイメージを作るコマンドは何ですか？

    回答を表示

    Dockerfileは「イメージを作るための手順書」です。Dockerがこの手順書を上から順に読み、イメージを組み上げます。イメージを作るコマンドは docker build です。名前をつける場合は docker build -t イメージ名 . のように書きます（末尾の . を忘れずに）。

  2. Q2. Dockerfile内の FROM nginx と COPY index.html /usr/share/nginx/html/index.html は、それぞれ何をしている命令ですか？

    回答を表示

    FROM nginx は「Docker HubにあるNginxの公式イメージを土台（ベースイメージ）にする」という命令です。COPY index.html /usr/share/nginx/html/index.html は「手元の index.html を、Nginxがファイルを配信するフォルダにコピーする」命令です。これにより、コンテナ起動後にブラウザでアクセスすると自作ページが表示されます。

  3. Q3. docker run を実行したとき「Conflict. The container name "/my-web-container" is already in use」と表示されました。どう対処しますか？

    回答を表示

    同じ名前のコンテナがすでに存在しているため、起動できない状態です。docker ps -a で状態を確認し、起動中（Up）なら docker stop my-web-container で停止してから docker rm my-web-container で削除します（停止中（Exited）ならそのまま docker rm でOK）。その後、もう一度 docker run -d コマンドを実行します。
