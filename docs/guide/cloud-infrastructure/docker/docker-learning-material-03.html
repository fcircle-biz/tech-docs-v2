ホーム Docker 第3章
第3章 所要時間: 約45分

はじめてのコンテナ：hello-worldと基本コマンド

この章では、いよいよ実際にコンテナを起動します。最初に動かすのは hello-world という小さなイメージです。たった1つのコマンドを打つだけで、Dockerが自動でイメージを取り寄せ、コンテナを作って実行してくれます。その後、コンテナの状態を確認するコマンドと、後片付けのコマンドをひと通り学びます。

この章で学ぶこと

  • docker run hello-world で最初のコンテナを起動できる
  • コンテナが起動するまでの流れ（イメージ検索→取得→作成→実行→停止）を説明できる
  • docker ps・docker ps -a・docker images でコンテナとイメージの状態を確認できる
  • docker stop・docker rm でコンテナを片付けられる

3.1 はじめてのコンテナを動かす

環境の準備ができたら、いよいよコンテナを起動してみましょう。最初に実行するのは hello-world という名前の特別なイメージです。このイメージは、Dockerが正しく動いているかを確認するためだけに用意された、非常に小さなテスト用イメージです。

ターミナル（またはコマンドプロンプト）を開いて、次のコマンドを入力してみてください。

hello-world イメージからコンテナを作成して実行します（手元になければ Docker Hub から自動で取得します）。

ターミナル 
docker run hello-world

コマンドを実行すると、しばらくして次のようなメッセージが表示されます。

出力結果（例） 
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
...
Hello from Docker!
This message shows that your installation appears to be working correctly.

「Hello from Docker!」と表示されれば成功です。「Your installation appears to be working correctly.」は「Docker の環境が正しく動いています」という意味です。

【スクリーンショット：「Hello from Docker!」の出力結果】

ターミナルに上記の出力が表示された画面を残しておきましょう。うまく表示されない場合は、3.4節の「よくあるエラー」を確認してください。

3.2 何が起きたのか — コンテナが動くまでの流れ

たった1つのコマンドを打っただけで、裏側ではいくつかの処理が順番に行われていました。「docker run hello-world」を実行したとき、Dockerは次の5つのステップを自動でこなしています。

  1. ① イメージを手元（ローカル）から探す — まず「hello-world」というイメージが自分のパソコンに保存されているか調べます。
  2. ② 手元になければ Docker Hub から自動取得する — 初回は手元にないため、Docker Hub から自動でダウンロードします。「Unable to find image ... locally」というメッセージがこの処理を表しています。
  3. ③ イメージからコンテナを作る — ダウンロードしたイメージ（設計図）をもとに、新しいコンテナ（実体）を作成します。
  4. ④ コンテナを実行する — 作ったコンテナを起動し、「Hello from Docker!」というメッセージを表示します。
  5. ⑤ 実行が終わるとコンテナは停止する — hello-world はメッセージを表示するだけのコンテナです。表示が終わると、コンテナは自動で停止します（削除はされません）。

図1: docker run 実行時の流れ

flowchart TD A["docker run hello-world を実行"] --> B["ローカルでイメージを探す"] B --> C{"手元にある?"} C -->|ある| E["イメージからコンテナを作成"] C -->|ない| D["Docker Hub から自動取得"] D --> E E --> F["コンテナを実行する"] F --> G["メッセージを表示"] G --> H["コンテナが停止する"]

図2: Docker Hub・イメージ・コンテナの関係

flowchart LR HUB["Docker Hub
（インターネット上の保管庫）"] IMG["ローカルのイメージ
（手元のレシピ）"] C1["コンテナ1
（起動した実体）"] C2["コンテナ2
（同じイメージから別途起動）"] HUB -->|"pull（取得）"| IMG IMG -->|"run（起動）"| C1 IMG -->|"run（起動）"| C2

2回目以降は速い

イメージは一度ダウンロードされると手元に保存されます。2回目以降に同じイメージを使うときは、Docker Hub から取得する必要がなくなるため、素早く起動できます。

3.3 コンテナの状態を確認する

コンテナを動かしたら、今どんな状態になっているかを確認できるようになりましょう。よく使う確認コマンドは3つあります。

動作中のコンテナを確認する: docker ps

docker ps は、現在起動中のコンテナの一覧を表示するコマンドです。「ps」はプロセス（Process Status）の略で、動いているコンテナをプロセスとして管理しています。

現在起動中のコンテナの一覧を表示します。

ターミナル 
docker ps

hello-world コンテナはメッセージを表示した後にすでに停止しているため、docker ps では何も表示されません（または空のテーブルが表示されます）。起動中のコンテナがあるときのみ、ここに行が表示されます。

停止済みも含めてすべて確認する: docker ps -a

docker ps -a（-a は all の略）は、停止済みのコンテナも含めたすべてのコンテナを表示します。hello-world のように「実行して終了したコンテナ」はここで確認できます。

停止済みも含めたすべてのコンテナの一覧を表示します。

ターミナル 
docker ps -a
出力結果（例） 
CONTAINER ID   IMAGE         COMMAND    CREATED         STATUS                     NAMES
a1b2c3d4e5f6   hello-world   "/hello"   2 minutes ago   Exited (0) 2 minutes ago   confident_pascal

各列の意味を確認しておきましょう。

列名 意味
CONTAINER ID コンテナに自動で割り当てられる識別番号（ID）
IMAGE コンテナの元になったイメージ名
STATUS Exited (0) は正常終了。Up は起動中
NAMES コンテナに自動でつけられる名前。コマンドで指定することもできる

手元のイメージを確認する: docker images

docker images は、自分のパソコンに保存されているイメージの一覧を表示するコマンドです。コンテナを削除してもイメージは残るので、このコマンドで手元にあるイメージを確認できます。

手元に保存されているイメージの一覧を表示します。

ターミナル 
docker images
出力結果（例） 
REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
hello-world   latest    d2c94e258dcb   12 months ago  13.3kB

hello-world イメージはたった 13.3kB ほどしかありません。このテスト用イメージがいかに小さいか分かります。TAG の「latest」は「最新版」を意味します。

動作確認

以下をすべて確認できたら、この節の内容はOKです。

  • docker ps -a の出力に hello-world が Exited として表示される
  • docker images の出力に hello-world イメージが表示される

3.4 コンテナを片付ける

コンテナを使い終わったら、きちんと片付けましょう。停止しただけのコンテナはパソコンに残り続けます。使わないコンテナを削除することで、パソコンをすっきり保てます。

コンテナの片付けには、まず「停止」、次に「削除」の順に行います。

動作中のコンテナを停止する: docker stop

docker stop は、起動中のコンテナを停止するコマンドです。コンテナ名またはコンテナIDを引数に指定します。コンテナ名やIDは docker ps -a で確認できます（NAMES列 または CONTAINER ID列）。

hello-world はすでに停止しているので、このコマンドの練習は次章以降（起動し続けるコンテナ）で行います。ここでは構文だけ覚えておきましょう。

起動中のコンテナを停止します。<コンテナ名またはID> の部分に実際の値を入れて実行します。

ターミナル 
# コンテナ名を使う場合
docker stop confident_pascal

# コンテナIDを使う場合（先頭数文字でも可）
docker stop a1b2c3d4e5f6

停止済みコンテナを削除する: docker rm

docker rm は、停止済みのコンテナをパソコンから完全に削除するコマンドです。「rm」は remove（削除）の略です。こちらもコンテナ名またはコンテナIDを引数に指定します。

今回の hello-world コンテナはすでに停止しているので、そのまま削除できます。docker ps -a でコンテナ名を確認して実行してみましょう。

停止済みのコンテナを削除します。コンテナ名は docker ps -a の NAMES 列で確認できます。

ターミナル 
# コンテナ名で削除する場合
docker rm confident_pascal

# コンテナIDで削除する場合
docker rm a1b2c3d4e5f6

削除後に docker ps -a を実行すると、hello-world コンテナがリストから消えていることを確認できます。イメージ自体は残っているので、再度 docker run hello-world を実行すれば、また新しいコンテナを作ることができます。

コンテナとイメージの削除は別々

docker rm はコンテナを削除しますが、イメージは削除しません。イメージを削除したいときは別途 docker rmi（remove image）コマンドを使います。今は覚えなくて大丈夫ですが、「コンテナとイメージは別々に管理されている」ことを押さえておきましょう。

この章で学んだコマンドの整理

コマンド 意味
docker run hello-world hello-world イメージからコンテナを作成して実行する（手元になければ自動取得）
docker ps 現在起動中のコンテナの一覧を表示する
docker ps -a 停止済みも含めたすべてのコンテナの一覧を表示する
docker images 手元に保存されているイメージの一覧を表示する
docker stop <名前またはID> 起動中のコンテナを停止する
docker rm <名前またはID> 停止済みのコンテナを削除する

3.5 よくあるエラーと対処法

コマンドを実行したときにエラーが出てしまった場合は、以下を参考に確認してみましょう。

エラー: docker: command not found

「docker コマンドが見つからない」というエラーです。Dockerがパソコンにインストールされていないか、正しく認識されていない状態です。

対処法:

  • ・第2章の手順に従って Rancher Desktop のインストールを確認する
  • ・ターミナルを一度閉じて開き直し、再度試す
  • ・Rancher Desktop が起動中かどうかを確認する

エラー: Cannot connect to the Docker daemon

「Dockerデーモン（バックグラウンドで動くプログラム）に接続できない」というエラーです。Rancher Desktop が起動していないときに発生します。

対処法:

  • ・タスクバー（またはメニューバー）に Rancher Desktop のアイコンがあるか確認する
  • ・アイコンがなければ Rancher Desktop を起動し、完全に立ち上がるまで数分待つ
  • ・起動後、再度コマンドを実行する

困ったとき: コンテナIDや名前が分からない

docker stop や docker rm でコンテナを指定するとき、IDや名前が分からなくなることがあります。

対処法:

  • ・docker ps -a を実行し、CONTAINER ID または NAMES 列の値を確認する
  • ・CONTAINER ID は先頭の3〜4文字だけでも使用できる（例: docker rm a1b2）

この章のまとめ

  • docker run hello-world で最初のコンテナを起動できた。イメージが手元になければ Docker Hub から自動で取得される。
  • コンテナ起動の流れは「イメージを探す → 取得 → コンテナ作成 → 実行 → 停止」の5ステップ。
  • docker ps で起動中のコンテナ、docker ps -a で全コンテナ（停止済み含む）、docker images で手元のイメージを確認できる。
  • docker stop でコンテナを停止し、docker rm で削除する。コンテナを削除してもイメージは残る。

確認問題

  1. Q1. docker run hello-world を初めて実行したとき、「Unable to find image 'hello-world:latest' locally」と表示されました。これはどういう意味で、この後 Docker は何をしますか？

    回答を表示

    「hello-world イメージが手元（ローカル）に見つからなかった」という意味です。この後、Docker は自動で Docker Hub から hello-world イメージをダウンロード（pull）し、そのイメージからコンテナを作って実行します。

  2. Q2. docker ps では何も表示されないのに、docker ps -a では hello-world コンテナが表示されました。なぜこのような違いが生じるのでしょうか？

    回答を表示

    docker ps は現在起動中のコンテナのみを表示します。hello-world はメッセージを表示した後に自動で停止するため、起動中のコンテナには含まれません。一方、docker ps -a（-a は all の略）は停止済みコンテナも含むすべてを表示するため、Exited 状態の hello-world が表示されます。

  3. Q3. コンテナを docker rm で削除しました。その後 docker images を実行すると hello-world イメージはまだ表示されています。なぜでしょうか？

    回答を表示

    docker rm はコンテナ（実体）を削除しますが、イメージ（設計図）は削除しません。コンテナとイメージは別々に管理されているため、コンテナを削除してもイメージは手元に残ります。イメージを削除したい場合は、別途 docker rmi コマンドを使います。
