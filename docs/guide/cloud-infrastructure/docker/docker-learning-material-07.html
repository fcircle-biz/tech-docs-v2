Docker Composeで起動手順をまとめる

毎回入力していた長い docker run コマンドを、設定ファイルにまとめておく方法を学びます。Docker Compose を使えば、コンテナの起動方法をファイルで管理できるので、ミスが減り、チームでも共有しやすくなります。この章では compose.yml の書き方と、起動・停止のコマンドを実際に体験します。

この章で学ぶこと

  • Docker Compose が何か・なぜ使うのかを理解する
  • compose.yml を書ける（services / image / ports / volumes）
  • docker compose up と docker compose down で起動・停止できる

実行前チェック（コマンドを打つ前に確認）

  • Rancher Desktop が起動しているか（タスクトレイ／メニューバーのアイコンを確認）
  • docker version が成功するか（Client と Server の両方が表示される）
  • 前の章で起動したコンテナが残っていないか（docker ps -a で確認。残っていれば停止・削除）
  • 8080番ポートを他のコンテナが使っていないか（docker ps で確認）

7.1 Docker Composeとは・なぜ使うのか

第4章から第6章にかけて、コンテナの起動にはこんなコマンドを使ってきました。

ターミナル（これまでのコマンド例） 
# オプションが多くなると、1行がとても長くなる
docker run -d --name my-nginx -p 8080:80 -v ./html:/usr/share/nginx/html nginx

このコマンドは、コンテナに名前をつけ、ポートをつなぎ、フォルダをマウントしています。今は1行ですが、オプションが増えるほどコマンドは長くなります。毎回手入力するとミスが起きやすく、他の人と共有するのも大変です。

この問題を解決するのが Docker Compose（ドッカー コンポーズ） です。Docker Compose は、コンテナの起動方法を compose.yml（コンポーズ ワイエムエル）という設定ファイルにまとめておき、短いコマンド1つで起動できる仕組みです。

ファイルに残る

起動方法が compose.yml に書いてあるので、後から見直せます。

チームで共有できる

ファイルを渡すだけで、全員が同じ方法でコンテナを起動できます。

コマンドが短くなる

長い docker run の代わりに docker compose up だけで起動できます。

図7: docker run の長いコマンドを compose.yml にまとめるイメージ

flowchart LR A["長い docker run コマンド
（毎回手入力・ミスしやすい）"] -->|Docker Composeで置き換え| B["compose.yml
（設定ファイルに記述）"] B -->|docker compose up| C["コンテナが起動
（短いコマンドで完了）"]

Docker Composeはすでに入っています

Rancher Desktop や Docker Desktop をインストールした環境では、Docker Compose は追加インストール不要です。docker compose（スペースあり）とタイプすれば、そのまま使えます。

7.2 compose.ymlを作る

実際に compose.yml を作ってみましょう。まず、作業用フォルダを用意します。

今回作るフォルダ構成 

compose-sample/
├── compose.yml        ← 起動設定を書くファイル
└── html/
    └── index.html     ← Nginxで表示するページ

まず、このフォルダとファイルを作ります。ターミナルで次のコマンドを順番に実行してください。

ターミナル 
# 作業フォルダを作って移動する
mkdir compose-sample
cd compose-sample

# Nginxで表示するHTMLファイルを置くフォルダを作る
mkdir html

次に html/index.html を作ります。内容は第6章で作ったものを流用してもかまいません。

html/index.html 
<!DOCTYPE html>
<html lang="ja">
<head><meta charset="UTF-8"><title>Compose サンプル</title></head>
<body>
  <h1>Docker Compose で表示しています！</h1>
</body>
</html>

続いて、このフォルダの直下に compose.yml を作ります。これが今回の主役です。

compose.yml 
services:       # 起動するコンテナの一覧を書く場所
  web:          # このサービスにつけた名前（自由に決められる）
    image: nginx  # 使うイメージ（nginx）
    ports:
      - "8080:80"  # PCの8080番をコンテナの80番につなぐ（-p と同じ）
    volumes:
      - ./html:/usr/share/nginx/html  # バインドマウント（-v と同じ）

compose.yml の各項目を理解する

compose.yml は YAML（ヤムル）という形式で書きます。YAMLとは、設定を書くためのファイル形式で、インデント（字下げ）が重要です。半角スペース2つで1段下げて構造を表し、タブ文字は使えないので注意してください。

項目 意味 docker run との対応
services 起動するコンテナ（サービス）の一覧を書く場所 —
web サービスにつけた名前（自由に決められる） --name に相当
image 使うDockerイメージの名前 最後の引数（nginx）
ports 公開するポートを「ホスト:コンテナ」の形式で指定 -p 8080:80
volumes マウント設定を「ホストパス:コンテナパス」で指定 -v ./html:/usr/share/nginx/html

YAMLはインデントが命

YAML ファイルは半角スペースの字下げ（インデント）で構造を表します。スペースの数がずれているだけでエラーになります。タブ文字は絶対に使わず、必ず半角スペースで揃えましょう。

補足：image: nginx とバージョンタグ（実務のヒント）

学習用には image: nginx のままで十分です。ただし nginx だけだと「最新版」が使われ、時間が経つと中身が変わることがあります。実務では、いつ起動しても同じ結果になるように、次のようにバージョンタグを指定することがあります（第5章の Dockerfile と同じ考え方です）。

services:
  web:
    image: nginx:1.27  # バージョンを固定して安定させる

7.3 起動と停止

compose.yml ができたら、コンテナを起動してみましょう。必ず compose.yml があるフォルダ（compose-sample）の中でコマンドを実行します。

コンテナを起動する

次のコマンドを実行すると、compose.yml の内容どおりにコンテナが起動します。

ターミナル（compose-sample フォルダ内で実行） 
# compose.yml を読んでコンテナを起動する（ログが流れ続ける）
docker compose up

コマンドを実行すると、Nginxのログがターミナルに表示され続けます。この状態でターミナルは「ログ表示モード」になっているので、他の操作はできません。

画面で確認するポイント

docker compose up を実行すると、Nginx のログがターミナルに流れ続けます。web-1 | ... start worker process のような行が表示されていれば、起動は成功です。

ブラウザで http://localhost:8080 を開いてみましょう。html/index.html の内容が表示されれば成功です。

動作確認

  1. 1 docker compose up を実行し、ログが流れ始めることを確認する
  2. 2 ブラウザで http://localhost:8080 を開く
  3. 3 html/index.html の内容が表示されれば成功

バックグラウンドで起動する（-d オプション）

docker compose up はログが流れ続けてターミナルを占有します（フォアグラウンド）。ログを表示せずにすぐターミナルを返してほしい場合は、-d オプション（デタッチモード＝バックグラウンドで動かすモード。第4章参照）を使います。

ターミナル 
# バックグラウンドで起動する（ターミナルがすぐ戻る）
docker compose up -d

-d を付けるとコンテナはバックグラウンドで動き続け、ターミナルはすぐに次のコマンドを入力できる状態に戻ります。

コンテナを停止・削除する

Docker Compose で起動したコンテナは、docker compose down でまとめて停止・削除できます。

ターミナル（compose-sample フォルダ内で実行） 
# Composeで起動したコンテナを停止して削除する
docker compose down

docker compose down を実行すると、Compose で起動していたコンテナが停止し、続いて削除されます。compose.yml や html/ フォルダのファイルは消えません。また起動したくなったら、同じフォルダで docker compose up -d を実行するだけです。

この章で使うコマンド まとめ

  • docker compose up … compose.yml の内容どおりにコンテナを起動する（ログが表示される）
  • docker compose up -d … バックグラウンドで起動する（ターミナルがすぐ戻る）
  • docker compose down … Compose で起動したコンテナを停止して削除する

7.4 補足: 複数コンテナをまとめて管理する

この章では Nginx コンテナ1つだけを扱いましたが、Docker Compose の本領は複数のコンテナをまとめて管理するときに発揮されます。

複数コンテナの管理に強い

本章では1つの Nginx コンテナだけを扱いましたが、Compose は Web サーバーとデータベースなど、複数のコンテナをまとめて管理するときにもよく使われます。たとえば次のようなイメージです（本章では作りません）。

services:
  web:
    image: nginx
  db:
    image: mysql
    # → これだけで2つのコンテナが一緒に起動できる

docker compose up -d の1コマンドで、Web サーバーとデータベースを同時に起動できます。

7.5 よくあるエラーと対処法

Docker Compose を使い始めたときに起きやすいエラーをまとめました。エラーが出たときは、まずここを確認してみましょう。

YAMLのインデントミス

症状: yaml: line X: did not find expected key などのエラーが出る

対処: compose.yml を開き、スペースのずれを確認します。タブ文字（Tab キー）は使えません。半角スペース2つで1段ずつ揃えてください。

compose.yml の場所が違う

症状: no configuration file provided: not found というエラーが出る

対処: docker compose コマンドは、compose.yml があるフォルダで実行する必要があります。cd compose-sample で正しいフォルダに移動してから実行してください。

docker compose（新）と docker-compose（旧）の違い

症状: 検索すると docker-compose up（ハイフンあり）が出てくるが、動かない

説明: 古い記事では docker-compose（ハイフンあり）が使われています。本教材では現在の標準である docker compose（スペースあり） を使います。

ポートがすでに使われている

症状: Ports are not available: address already in use というエラーが出る

対処: 前に起動したコンテナがまだ動いています。別のターミナルで docker compose down または docker stop <コンテナ名> で止めてから、再度 docker compose up を実行してください。

この章のまとめ

  • Docker Compose は、長い docker run コマンドを compose.yml にまとめておく仕組み。ミスが減り、チームで共有しやすい。
  • compose.yml には services / image / ports / volumes を書く。YAML はインデント（半角スペース2つ）が重要。
  • docker compose up で起動、docker compose up -d でバックグラウンド起動、docker compose down で停止・削除。
  • Compose は複数のコンテナをまとめて管理するときにも使われる（本章では1つだけ）。

確認問題

  1. Q1. Docker Compose を使うと、どのような問題を解決できますか。

    回答を表示

    長い docker run コマンドをその都度手入力するとミスが起きやすく、他の人への共有も大変です。Docker Compose を使うと、起動方法を compose.yml ファイルにまとめておけるので、ミスが減り・コマンドが短くなり・チームで共有しやすくなります。

  2. Q2. compose.yml の ports: - "8080:80" は、docker run のどのオプションに対応しますか。

    回答を表示

    docker run の -p 8080:80 オプションに対応します。PCの8080番ポートをコンテナの80番ポートにつなぐという意味は同じです。

  3. Q3. Compose で起動したコンテナを停止して削除するコマンドは何ですか。また、どのフォルダで実行する必要がありますか。

    回答を表示

    docker compose down コマンドを使います。compose.yml が置いてあるフォルダ（この章では compose-sample フォルダ）の中で実行する必要があります。別のフォルダで実行すると「設定ファイルが見つからない」エラーになります。

  4. Q4. docker compose up で yaml: ... did not find expected key のようなエラーが出ました。何が原因で、何を確認すればよいですか？

    回答を表示

    YAML のインデント（字下げ）のずれが原因のことが多いです。compose.yml を開き、タブ文字を使っていないか、半角スペース2つで段が揃っているかを確認します。YAML は字下げで構造を表すため、スペースの数がずれるだけでエラーになります。
