Rancher DesktopでDocker環境を準備する

この章では、実際に Rancher Desktop（ランチャー デスクトップ）をインストールし、ターミナルから docker コマンドが使える状態を作ります。コマンドを打つ前の「環境準備」の章です。画面を見ながら一緒に進めましょう。

この章で学ぶこと

  • Rancher Desktop とは何かを理解する
  • Windows・Mac に Rancher Desktop をインストールできる
  • 初期設定で Container Runtime として dockerd（Moby） を選べる
  • ターミナルから docker version を実行して動作確認できる

2.1 Rancher Desktop とは何か

Rancher Desktop（ランチャー デスクトップ）は、WindowsやMacのパソコンにDocker環境を手軽に用意するためのアプリです。インストールするだけで、ターミナルから docker コマンドが使えるようになります。

Rancher Desktop は無料で使えるオープンソースのソフトウェアです。商用利用にも制限がなく、本教材ではこれを使って学習を進めます。

無料で使える

商用利用含め完全無料のオープンソースソフト。ライセンス料は不要です。

Windows・Mac 対応

どちらのOSでも同じ手順で docker コマンドを使えます。

標準の docker コマンド

普段使いの docker run や docker pull がそのまま使えます。

Docker Desktop を使っている方へ

すでに Docker Desktop をインストール済みの方でも、本教材で使う基本的な docker コマンドはほぼ同じです。そのまま学習を進められます。この章のインストール手順はスキップして、2.5 の動作確認から始めてください。

2.2 ダウンロードとインストール

まず公式サイトからインストーラーを取得します。公式サイトのURLは https://rancherdesktop.io/ です。

画面で確認するポイント

https://rancherdesktop.io/ を開くと、ページ上部に「Download」ボタンがあります。そこから、お使いのOSに合ったインストーラー（Windowsは .exe、Macは .dmg）を選べることを確認しましょう。

Windows の場合

WSL2 について（Windows限定の事前知識）

WindowsでDockerを動かすには、WSL2（Windows Subsystem for Linux 2）という仕組みが必要です。WSL2 は「Windows の中で Linux を動かす仕組み」で、Dockerはこの Linux 環境の上で動きます。難しく考える必要はなく、「Dockerを使うのに必要な土台」と覚えておけば十分です。

Windows 11 や最新の Windows 10 では WSL2 があらかじめ有効になっている場合が多いですが、有効でない環境では手動で有効化が必要です（後述）。

  1. 1 公式サイト（https://rancherdesktop.io/）を開き、「Download for Windows」をクリックしてインストーラー（.exe ファイル）をダウンロードします。
  2. 2 ダウンロードした .exe ファイルをダブルクリックし、インストーラーを起動します。画面の指示に従って「次へ」を押し続けてインストールを完了させます。
  3. 3 インストール完了後、PCを再起動します。初回インストール時は再起動が必要なことがあります。

画面で確認するポイント

インストーラーを起動すると、「次へ（Next）」で進んでいくウィザード画面が表示されます。特に変更する項目はないので、基本は初期設定のまま進めて問題ありません。

WSL2 が有効でない場合の対処

インストール中や起動時に「WSL2 が必要です」というエラーが出た場合は、以下の手順で WSL2 を有効にします。

PowerShell を管理者として開き、次のコマンドを実行します。

PowerShell（管理者） 
# WSL2 を有効にするコマンド（管理者 PowerShell で実行）
wsl --install

コマンドの実行後、PCを再起動してください。再起動後にRancher Desktopを再度起動すると正常に動作します。

Mac の場合

  1. 1 公式サイトを開き、お使いのMacに合わせてインストーラーを選択します。Intel Mac の場合は「Intel」版、Apple シリコン Mac（M1/M2/M3 など）の場合は「Apple Silicon」版をダウンロードします。どちらかわからない場合は、画面左上の「」→「このMacについて」でチップの種類を確認できます。
  2. 2 ダウンロードした .dmg ファイルを開き、Rancher Desktop のアイコンを「アプリケーション」フォルダへドラッグします。これでインストール完了です。
  3. 3 アプリケーションフォルダから Rancher Desktop を起動します。

Mac のセキュリティ警告について

初回起動時に「開発元を確認できません」という警告が出ることがあります。その場合は「システム設定」→「プライバシーとセキュリティ」を開き、「このまま開く」を選択してください。

2.3 初回起動と初期設定

Rancher Desktop を初めて起動すると、いくつかの設定を選ぶ画面が表示されます。ここでの設定が最も重要なポイントです。

画面で確認するポイント

初回起動では、設定を選ぶウィザード画面が表示されます。この中に、次に説明する Container Runtime の選択欄が含まれています。

重要：Container Runtime の選択

設定画面で Container Runtime を選ぶ項目が出てきます。Container Runtime（コンテナランタイム）とは、コンテナを実際に動かす裏側の仕組みのことです。ここでは必ず dockerd（Moby） を選んでください。

他に「containerd」「nerdctl」という選択肢が表示されることがありますが、これらは別のツールセットを使う上級者向けの選択肢です。初心者の方は dockerd（Moby） だけを選べば問題ありません。

画面で確認するポイント

Container Runtime の選択欄で、dockerd（Moby） が選ばれている状態にします。「containerd」が選ばれていないことを確認しましょう。

初期設定の各項目について

設定項目 選択する値 理由
Container Runtime dockerd（Moby） 標準の docker コマンドが使えるようになる
Kubernetes チェックを外す（無効） この教材では使わない。起動が速くなり、PCへの負荷も軽くなる

設定が完了したら、画面の「Apply」または「Confirm」ボタンを押します。Rancher Desktop がバックグラウンドで準備を始めます。初回は数分かかることがあるので、タスクバー（Windowsの場合）またはメニューバー（Macの場合）のアイコンが落ち着くまで待ちましょう。

2.4 ターミナルの開き方

Docker コマンドはターミナル（黒い画面）から実行します。まだ使ったことがない方のために、ターミナルの開き方を確認しておきましょう。

Windows の場合

  • Windows Terminal（推奨）: スタートメニューで「Windows Terminal」と検索して起動
  • PowerShell: スタートメニューで「PowerShell」と検索して起動

Mac の場合

  • ターミナル.app: Spotlight（⌘+スペース）で「ターミナル」と検索して起動
  • または「アプリケーション」→「ユーティリティ」→「ターミナル」

ターミナルを初めて使う方へ

ターミナルは文字でコンピューターに命令するための道具です。コマンドを入力して Enter キーを押すと実行されます。ひとつずつ確認しながら進めましょう。

2.5 Docker が使えるか確認する

Rancher Desktop の準備ができたら、ターミナルを開いて Docker が正しく動いているか確認します。2つのコマンドを実行しましょう。

1. docker version — バージョン確認

このコマンドは、インストールしたDockerのクライアントとサーバー（Dockerデーモン）のバージョン情報を表示します。Dockerデーモンとは、Dockerの命令を受け取って処理する、裏で常に動いているプログラムのことです（「デーモン」はこうした常駐プログラムを指す言葉です）。クライアントとサーバーの両方のバージョンが表示されれば、Dockerは正しく動いています。クライアントだけが表示される場合は、Rancher Desktop がまだ起動中の可能性があります。

ターミナル 
# Dockerのバージョン情報（クライアントとサーバー）を表示する
docker version

実行すると、次のような出力が得られます（バージョン番号はお使いの環境によって異なります）。

出力例 
Client:
 Version:           27.x.x
 OS/Arch:           linux/amd64
 ...

Server: Docker Engine - Community
 Engine:
  Version:          27.x.x
  OS/Arch:          linux/amd64
  ...

2. docker info — 稼働状況の確認

このコマンドは、Dockerの動作状況や設定の詳細な情報を表示します。コンテナや起動中のプロセス数、ストレージの設定など、Dockerが今どんな状態かを一覧で確認できます。

ターミナル 
# Dockerの動作状況・設定の詳細を表示する
docker info

動作確認：準備OKの判定基準

  • docker version の出力に 「Client:」と「Server:」の両方が表示される → Docker は正常に動いています
  • docker info がエラーなく実行でき、大量の情報が表示される → 環境は準備完了です

画面で確認するポイント

ターミナルで docker version を実行したとき、出力に「Client:」と「Server:」の2つのまとまりが両方表示されていれば成功です。

2.6 よくあるエラーと対処法

環境構築でつまずきやすいポイントをまとめました。同じエラーが出たら、以下を確認してみてください。

エラーメッセージ 原因 対処法
docker: command not found Rancher Desktop が未インストール、またはインストール後にターミナルを再起動していない ターミナルを一度閉じて再度開く。それでも出る場合は Rancher Desktop を再インストール
Cannot connect to the Docker daemon Rancher Desktop が起動していない、または起動途中 Rancher Desktop アプリを起動し、タスクバー・メニューバーのアイコンが落ち着くまで数分待ってから再実行する
WSL2 が有効でない（Windows） Windows に WSL2 がインストールされていない 管理者 PowerShell で wsl --install を実行し、PCを再起動する

「Cannot connect to the Docker daemon」が出た場合

このエラーは「Dockerのサーバー（デーモン）に接続できない」という意味です。多くの場合、Rancher Desktop が起動していないか、まだ準備中です。

  1. Rancher Desktop アプリが起動しているか確認する
  2. タスクバー（Windows）またはメニューバー（Mac）のアイコンが安定するまで待つ
  3. 数分待ってから docker version を再度実行する
  4. それでも解決しない場合は Rancher Desktop を再起動する

インストール後は PC を再起動する

Rancher Desktop のインストール後は、念のため PC を再起動してから動作確認することを推奨します。特にWindows環境では、WSL2 の有効化やパス設定が再起動で完全に反映されます。

やってみよう 2-1: 2つのコマンドを実行して確認する

Rancher Desktop のインストールと初期設定が完了したら、ターミナルから以下を順番に実行してみましょう。

手順

  1. 1 ターミナルを開く（Windows: PowerShell / Mac: ターミナル.app）
  2. 2 docker version を実行し、「Client:」と「Server:」の両方が表示されることを確認する
  3. 3 docker info を実行し、エラーなく情報が表示されることを確認する
  4. 4 エラーが出た場合は、2.6のよくあるエラー表を参照して対処する

この章のまとめ

  • Rancher Desktop は無料で使える Docker 環境構築ツール。Windows・Mac 両対応。
  • 初期設定では Container Runtime に dockerd（Moby） を選ぶ。Kubernetes は今回不要（無効化でOK）。
  • Windows では WSL2（Windows で Linux を動かす仕組み）が必要。有効でない場合は wsl --install で有効にしてPCを再起動。
  • docker version で Client と Server の両方が表示されれば、Docker 環境の準備は完了。
  • 「Cannot connect to the Docker daemon」エラーは Rancher Desktop が未起動のサイン。アプリを起動して数分待ってから再試行する。

確認問題

  1. Q1. Rancher Desktop の初期設定で Container Runtime として選ぶべきものはどれですか。

    回答を表示

    dockerd（Moby） を選びます。これにより、標準の docker コマンドが使えるようになります。「containerd」「nerdctl」は上級者向けの別ツールセットで、初心者はこれらを選ぶ必要はありません。

  2. Q2. docker version を実行した際、Docker が正常に動いていると判断できるのはどのような出力が得られた場合ですか。

    回答を表示

    「Client:」と「Server:」の両方のバージョン情報が表示された場合です。Client だけが表示されて Server の情報が出ない場合は、Rancher Desktop がまだ起動中か起動していない状態です。

  3. Q3. Windows 環境で「WSL2 が必要です」というエラーが出た場合、どのような手順で対処しますか。

    回答を表示

    PowerShell を管理者として開き、wsl --install を実行します。その後、PC を再起動します。再起動後に Rancher Desktop を起動すると正常に動作します。
