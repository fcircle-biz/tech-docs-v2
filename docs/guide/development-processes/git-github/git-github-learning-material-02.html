ホーム Git/GitHub 第2章
第2章 所要時間: 約60分

環境構築とGitHubの準備

この章では、Git・VS Code・GitHubを使うための準備を整えます。インストールから初期設定、さらにGitHubと安全につなぐためのSSH接続の設定まで、手順に沿って進めることで「自分のPCでGitコマンドを実行できる」「GitHubにSSHでつないでリポジトリを扱える」状態を目指します。

この章で学ぶこと

  • Git をインストールし、git --version でバージョンを確認できる
  • VS Code をインストールし、統合ターミナルを開ける
  • GitHubアカウントを作成し、ブラウザからログインできる
  • git config でユーザー名とメールアドレスを設定できる
  • SSH鍵を作成して GitHub に公開鍵を登録し、ssh -T git@github.com で接続を確認できる
  • 自分のPCでGitコマンドを実行でき、GitHubにログインしてリポジトリを扱う準備ができている

2.1 Gitのインストール

まず最初に、自分のPC に Git をインストールします。OSによって手順が少し異なりますので、それぞれ確認してください。

Windows

  1. ブラウザで git-scm.com を開く
  2. 「Download for Windows」ボタンをクリック
  3. ダウンロードされたインストーラー（.exe）を実行
  4. 画面の指示に従い「Next」を押し続けてインストール完了

macOS

  1. ターミナルを開く（Launchpad → ターミナル）
  2. git --version と入力して Enter
  3. インストール確認ダイアログが表示されたら「インストール」を選択
  4. （Homebrew 導入済みの人向け）brew install git

補足: インストール時の選択肢で迷ったら

Windows のインストーラーは選択画面がいくつも出ますが、基本的には初期設定（デフォルト）のままで進めて大丈夫です。特別な指定がない限り「Next」を押し続けて問題ありません。
macOS の brew install git は Homebrew（macOS用のパッケージ管理ツール）を導入済みの人向けです。導入していない場合は、上の手順のように git --version で表示されるダイアログからインストールするか、git-scm.com の公式サイトや Xcode Command Line Tools からも導入できます。

インストールの確認

インストールが終わったら、ターミナル（またはコマンドプロンプト）を新しく開き直して、以下のコマンドを実行してください。バージョン番号が表示されれば成功です。

ターミナル / コマンドプロンプト 
# インストールされたGitのバージョンを確認する
git --version
出力例 
git version 2.45.0

ターミナルは「開き直す」のがポイント

インストール前に開いていたターミナルでは、新しくインストールした Git が認識されないことがあります。インストール後は必ずターミナルを閉じて新しく開き直してから確認してください。「command not found」と表示されたときは、まずこれを試しましょう。

2.2 VS Code のインストール

VS Code（Visual Studio Code） は、無料で使えるコードエディタです。このガイドでは VS Code の統合ターミナルを使って Git コマンドを実行します。ファイルの編集とターミナル操作を1つの画面で行えるため、初心者にも扱いやすい環境です。

1 公式サイトへアクセス

ブラウザで code.visualstudio.com を開き、「Download」ボタンをクリックします。お使いのOSが自動的に判定されてインストーラーがダウンロードされます。

2 インストーラーを実行

ダウンロードしたファイルをダブルクリックして実行します。Windows の場合は「次へ」を押し続け、最後に「インストール」をクリックするだけです。macOS の場合はアプリをドラッグして Applications フォルダへ移動します。

3 VS Code を起動する

インストール後、VS Code を起動します。Windows はスタートメニュー、macOS は Launchpad または Applications フォルダから開けます。

統合ターミナルを開く

VS Code には統合ターミナル（エディタ内に埋め込まれたターミナル）があります。これを使うと、エディタを離れずにコマンドを実行できます。

開く方法 操作
メニューから 上部メニュー「ターミナル」→「新しいターミナル」をクリック
キーボードショートカット Windows: Ctrl + `（バッククォート）／ macOS: Ctrl + `

「` 」（バッククォート）はどこにあるの？

バッククォートは ` という記号のキーです。日本語キーボードでは @ キーのすぐ左隣にあります。つまり Ctrl を押しながら「@ キーの左隣のキー」を押す、と覚えると迷いにくいです。どうしても見つからないときは、メニューの「ターミナル」→「新しいターミナル」から開く方法が確実です。

2.3 GitHubアカウントの作成

次に、コードをインターネット上で管理・共有するためのプラットフォーム GitHub のアカウントを作成します。無料プランで十分に使い始められます。

1 GitHub にアクセス

ブラウザで github.com を開きます。右上の「Sign up」ボタンをクリックしてください。

2 必要事項を入力

  • ユーザー名: 英数字とハイフンで設定（公開されます。本名でなくても可）
  • メールアドレス: 連絡用のメールアドレス
  • パスワード: 安全な強度のパスワード（15文字以上、または数字・小文字・大文字を組み合わせた8文字以上）

3 メール認証・ログインを確認

登録したメールアドレスに確認メールが届きます。メール内のリンクをクリックして認証を完了させます。その後、github.com にログインできることを確認しましょう。

ユーザー名は慎重に選ぼう

GitHubのユーザー名はプロフィールURLや共同作業時に表示されます。あとから変更できますが、既存のリンクが壊れることもあります。チームや採用担当者が見ることもあるため、シンプルで覚えやすい名前を選ぶことをおすすめします。

2.4 Gitの初期設定

Git をインストールしたら、最初にやることが1つあります。自分の名前とメールアドレスを登録することです。この情報はコミット（変更の記録）に付与されます。「誰が変更したか」が履歴に残るため、特にチーム開発では必須の設定です。

ターミナル / VS Code 統合ターミナル 
# ユーザー名を設定する（"Your Name" を自分の名前に変える）
git config --global user.name "Your Name"

# メールアドレスを設定する（GitHubの登録メールと揃えると良い）
git config --global user.email "you@example.com"

user.name はコミット履歴に表示される「あなたの名前」、user.email は同じく履歴に残る「あなたの連絡先メール」です。"Your Name" と "you@example.com" の部分は、必ず自分のものに置き換えて実行してください。

補足: 名前とメールはどう決める？

  • user.name は GitHub のアカウント名と完全に一致していなくても大丈夫です。履歴に表示される表示名なので、自分だと分かる名前で構いません。
  • user.email は、GitHub に登録したメールアドレスを使うのがおすすめです。メールアドレスを公開したくない場合は、GitHub が用意する noreply メール（数字+ユーザー名@users.noreply.github.com の形式）も使えます。

設定後、正しく登録されたか確認するには以下のコマンドを使います。

設定の確認 
# 現在の設定をすべて表示する
git config --global --list
出力例 
user.name=Your Name
user.email=you@example.com

設定の仕組み: --global オプションとは

flowchart LR A["git config --global"] --> B["ホームディレクトリの
.gitconfig ファイルに保存"] B --> C["そのPC上の全リポジトリで
共通して使われる"]

--global なしで設定すると、そのリポジトリだけに適用されます（複数のメールを使い分けるときに便利）。

メールはGitHubの登録メールと揃えよう

Git の user.email を GitHub のアカウント登録メールと同じにすると、GitHub上でコミット履歴があなたのアカウントと自動的に紐付けられます。違うメールにするとコミットが「誰かわからない人の変更」として表示されることがあります。

2.5 VS Code の統合ターミナルで Git を動かす

環境が整ったら、VS Code の統合ターミナルから Git コマンドを実行できるか確認しましょう。これ以降の章ではすべてこの統合ターミナルを使います。

1 VS Code を起動してフォルダを開く

VS Code を起動し、「ファイル」→「フォルダーを開く」で適当なフォルダを開きます（デスクトップ等でかまいません）。

2 統合ターミナルを開く

メニューの「ターミナル」→「新しいターミナル」をクリック（または Ctrl + `）。画面下部にターミナルパネルが表示されます。

3 Git が使えるか確認する

ターミナルに以下のコマンドを入力して Enter を押します。

VS Code 統合ターミナル 
# バージョン確認
git --version

# 設定確認
git config --global --list

動作確認: 成功の判断基準

  • git --version で git version 2.x.x のようなバージョンが表示される
  • git config --global --list で設定したユーザー名とメールが表示される

2.6 GitHubへのサインインを確認する

最後に、ブラウザから GitHub にサインインできることを確認しましょう。第4章でリモートリポジトリへ push するときに必要になります。

github.com をブラウザで開く

右上の「Sign in」をクリックし、2.3 で登録したユーザー名（またはメール）とパスワードを入力してログインします。

ダッシュボードが表示されればOK

ログイン後、右上に自分のアイコンが表示されていれば準備完了です。まだリポジトリはありませんが、これで GitHub を使う土台が整いました。

二要素認証（2FA）の設定も検討しよう

GitHub はアカウントのセキュリティ向上のため、二要素認証（2FA）の設定を推奨しています。設定は「Settings」→「Password and authentication」から行えます。スマートフォンの認証アプリ（Google Authenticatorなど）を使うと安全です。

2.7 GitHubとSSHで接続する準備をする

第4章以降で、手元のリポジトリを GitHub に送る（push）には、GitHub に対して「これは確かにあなたですよ」と本人確認をする必要があります。このガイドでは、その方法として SSH 方式 を使います。少し準備が必要ですが、一度設定すれば以降はとても楽になります。

SSH方式とは

SSH（エスエスエイチ） とは、自分のPCと GitHub のようなサービスとの間で安全に通信するための仕組み（認証方式）です。SSH を使った接続方式（SSH方式）で GitHub とつなぐと、push / pull / clone のたびに GitHub のパスワードを入力しなくても、安全に本人確認ができます。

重要ポイント: この教材ではSSH方式で統一します

GitHub への接続にはいくつか方式がありますが、このガイドでは GitHub 接続をすべて SSH 方式で統一します。これ以降の章の手順も、すべて SSH 方式を前提に進めます。ここで行う設定は、最初の一度だけで大丈夫です。

① SSH鍵を作成する

SSH方式では、本人確認に SSH鍵ペア（本人確認に使う「秘密鍵」と「公開鍵」の、2つで1組の鍵）を使います。まずは次のコマンドで、自分専用の鍵ペアを作成します。-C の後ろは自分用の目印なので、GitHubに登録したメールアドレスに置き換えてください。

ターミナル（Windowsは Git Bash / macOS / Linux 共通） 
# SSH鍵ペアを作成する（メールアドレスは自分のものに置き換える）
ssh-keygen -t ed25519 -C "GitHubに登録したメールアドレス"

実行するといくつか質問が表示されます。次のように進めて大丈夫です（表示は環境によって多少違っても問題ありません）。

  • 鍵の保存場所を聞かれたら、基本的には何も入力せず Enter を押せばOKです。
  • パスフレーズは設定してもかまいませんが、学習用であれば Enter を押して空のまま進めても大丈夫です。

作成が終わると、2つのファイルができます。id_ed25519 が秘密鍵（自分だけが持つ鍵）、id_ed25519.pub が公開鍵（GitHubに登録してよい鍵）です。

注意: 秘密鍵は絶対に他人に渡さない

秘密鍵 id_ed25519（末尾に .pub が付かないほう）は、絶対に他人に見せたり、GitHubに貼り付けたりしてはいけません。GitHubに登録するのは、末尾が .pub の公開鍵だけです。

② 公開鍵の内容を確認する

次に、GitHubに登録するための公開鍵の中身を表示します。cat はファイルの中身を表示するコマンドです。

ターミナル（Git Bash / macOS / Linux 共通） 
# 公開鍵（.pub のほう）の中身を表示する
cat ~/.ssh/id_ed25519.pub

ssh-ed25519 から始まる長い1行の文字列が表示されます。この1行全体をコピーしておきましょう（次のステップで GitHub に貼り付けます）。

補足: コピーするのは「.pub」のほう

コピーするのは id_ed25519（秘密鍵）ではなく、id_ed25519.pub（公開鍵）の内容です。表示された ssh-ed25519 ... の行を、先頭から末尾まで1行まるごとコピーします。

③ GitHubに公開鍵を登録する

コピーした公開鍵を、GitHub のアカウントに登録します。ブラウザで GitHub を開き、次の順に進めます。

  1. GitHub にログインする
  2. 右上の自分のアイコンをクリックして「Settings（設定）」を開く
  3. 左メニューの「SSH and GPG keys」を開く
  4. 「New SSH key（新しいSSH鍵）」ボタンを押す
  5. Title に、自分が分かる名前を入力する
  6. Key の欄に、先ほどコピーした公開鍵を貼り付ける
  7. 「Add SSH key」ボタンを押す

補足: 登録するときのポイント

  • Title は「My PC」「School Laptop」など、自分がどのPCの鍵か分かる名前で大丈夫です。
  • Key には ssh-ed25519 から始まる公開鍵を貼り付けます。
  • 秘密鍵（id_ed25519）は絶対に貼り付けないでください。

④ SSHで接続できるか確認する

最後に、設定がうまくいったかを確認します。次のコマンドは「GitHub に SSH でつないで、本人確認だけを試す」コマンドです。

ターミナル（Git Bash / macOS / Linux 共通） 
# GitHubにSSHで接続できるか確認する
ssh -T git@github.com

初回接続時には「本当に接続してよいか？」という確認メッセージが出ることがあります。その場合は yes と入力して Enter を押します。

成功したときの表示例 
Hi ユーザー名! You've successfully authenticated, but GitHub does not provide shell access.

Hi ユーザー名! You've successfully authenticated... のような表示が出れば成功です（ユーザー名 の部分には自分のGitHubユーザー名が入ります）。これで GitHub と SSH でつなぐ準備が整いました。

うまくいかないとき

Permission denied (publickey). と表示された場合は、鍵の登録がうまくいっていない可能性があります。あわてず、第7章「よくあるトラブルと最低限の対処」のSSH接続エラーの手順を参照してください。

手順

  1. 1 VS Code の統合ターミナルを開き、git --version を実行してバージョンが表示されることを確認する。
  2. 2 git config --global user.name "自分の名前" と git config --global user.email "自分のメール" を実行する。
  3. 3 git config --global --list を実行し、設定した名前とメールが表示されることを確認する。
  4. 4 ブラウザで github.com を開き、作成したアカウントでサインインできることを確認する。
  5. 5 SSH鍵を作成して GitHub に公開鍵を登録し、ssh -T git@github.com を実行して「You've successfully authenticated」と表示されることを確認する（手順は 2.7 を参照）。

理解度確認クイズ

  1. Q1. Git をインストールした後、「command not found」と表示された場合、まず何を試すべきですか？

    回答を表示

    ターミナル（またはコマンドプロンプト）を一度閉じて、新しく開き直します。インストール前に開いていたターミナルには新しいパスが反映されていないため、開き直すことで Git が認識されるようになります。

  2. Q2. git config --global user.email に設定するメールアドレスは、どこで使ったメールと揃えると良いですか？

    回答を表示

    GitHub のアカウント登録に使ったメールアドレスと揃えることが推奨されます。同じメールにすることで、コミット履歴が自動的にあなたの GitHub アカウントに紐付けられます。

  3. Q3. VS Code の統合ターミナルをキーボードショートカットで開くとき、どのキーを使いますか？

    回答を表示

    Ctrl + `（バッククォート）です。日本語キーボードでは @ キーの左隣にあります。見つからない場合は、メニューの「ターミナル」→「新しいターミナル」から開くこともできます。

  4. Q4. このガイドでは、GitHubとの接続にどの方式を使いますか？また、その設定がうまくいったか確認するコマンドは何ですか？

    回答を表示

    SSH方式を使います。SSH鍵を作成して公開鍵を GitHub に登録したあと、ssh -T git@github.com を実行し、「You've successfully authenticated」と表示されれば接続の準備は完了です。
