プログラムに判断させる - 条件分岐

現実の世界では、状況によって行動を変えることが当たり前です。プログラムも同じです。この章では「もし～ならば」という条件に応じてプログラムの動きを変える条件分岐を学びます。if 文・比較演算子・論理演算子・switch 文を使いこなせるようになると、実用的なプログラムが一気に書けるようになります。

この章で学ぶこと

  • 条件分岐とは何か、なぜプログラムに「判断」が必要かを説明できる
  • 比較演算子（== != < > <= >=）を使って条件式を書ける
  • if 文・if-else 文・else if を使って多分岐処理を実装できる
  • 論理演算子（&& || !）で複数の条件を組み合わせられる
  • switch 文の使い方と if-else との使い分けを理解する

このページのコード例の読み方

各コード例の右上のラベルで「コードを書く場所」を示します。

  • 完全なプログラムそのまま .java ファイルに貼り付けて実行できます。
  • mainメソッド内に記述public static void main(String[] args) { ... } の中に書いて実行します。

3.1 条件分岐とは — 分かれ道のたとえ

日常の生活を思い浮かべてください。「明日雨なら傘を持っていく、晴れなら持っていかない」という判断を、私たちは自然に行っています。プログラムも同じです。条件に応じて処理を変える仕組みを条件分岐といいます。

道路の分かれ道にたとえると分かりやすいでしょう。一本道だけでは目的地（=結果）は1つしかありませんが、分かれ道に標識（=条件）を置けば、状況に応じて異なるルートを選べます。これが条件分岐の本質です。

なぜ判断が必要か？

「点数が60点以上なら合格・未満なら不合格」「ログインしているならマイページを表示・していないならログイン画面へ」など、実際のアプリは状況によって動きを変えます。条件分岐がなければ、プログラムはいつも同じ結果しか出せません。

3.2 比較演算子と boolean

条件分岐を書くには、まず「条件を比べる道具」が必要です。Javaでは比較演算子を使って2つの値を比較し、その結果は必ず true（真） か false（偽） の2種類の値になります。この true/false を扱うデータ型を boolean（ブーリアン） といいます。

演算子 意味 例（score = 75） 結果
== 等しい score == 75 true
!= 等しくない score != 100 true
< より小さい score < 60 false
> より大きい score > 60 true
<= 以下 score <= 75 true
>= 以上 score >= 80 false

注意: = と == を混同しない

= は「代入」（変数に値を入れる）、== は「比較」（等しいか調べる）です。条件式で = を使うとコンパイルエラーになるので注意してください。これは初心者が最もよくやるミスのひとつです。

3.3 if 文・if-else 文・else if による多分岐

比較演算子で条件を作れるようになったら、いよいよ if 文でプログラムに判断させます。if 文は「もし条件が true ならば、この処理をする」という構文です。

図: if-else の処理の流れ

flowchart TD A([プログラム開始]) --> B{"条件式は true?"} B -->|"true（はい）"| C["if ブロックの処理を実行"] B -->|"false（いいえ）"| D["else ブロックの処理を実行"] C --> E([次の処理へ]) D --> E

図の説明: 条件式の結果が true か false かによって、if ブロックと else ブロックのどちらか一方だけが実行され、その後は共通の処理へ合流することを示している。

基本: if 文と if-else 文

まずは点数で合格・不合格を判定する例を見てみましょう。

IfExample.java — if-else の基本 mainメソッド内に記述 
int score = 75; // テストの点数

// もし60点以上なら合格、そうでなければ不合格
if (score >= 60) {
    // 条件が true のときに実行される
    System.out.println("合格です！");
} else {
    // 条件が false のときに実行される
    System.out.println("不合格です。");
}

応用: else if で3段階以上に分岐する

「合格・不合格」だけでなく「優・良・可・不可」のように3段階以上に分けたいときは、else if を使います。条件は上から順に評価され、最初に true になった箇所だけが実行されます。

GradeJudge.java — else if による多段階分岐 mainメソッド内に記述 
int score = 82; // 判定したい点数

if (score >= 90) {
    System.out.println("優");      // 90点以上
} else if (score >= 75) {
    System.out.println("良");      // 75点以上90点未満
} else if (score >= 60) {
    System.out.println("可");      // 60点以上75点未満
} else {
    System.out.println("不可");    // 60点未満
}
terminal — 実行結果 
良

重要ポイント: 条件の順番に気をつける

else if は上から順番に評価されます。「90点以上」を先に書き、「75点以上」はその次に書くことで、「90点以上かつ75点以上」という重複を避けられます。条件の広い・狭いを意識しながら並べましょう。

3.4 論理演算子 — 複数の条件を組み合わせる

「点数が60点以上かつ出席日数が10日以上なら合格」のように、複数の条件を組み合わせたいことがあります。このときに使うのが論理演算子です。

&& かつ（AND）

左右の条件がどちらも true のときだけ true になる。一方でも false なら false。

|| または（OR）

左右の条件のどちらかが true なら true になる。両方 false のときだけ false。

! ではない（NOT）

条件の結果を反転する。true を false に、false を true にする。

LogicExample.java — 論理演算子の使い方 mainメソッド内に記述 
int score = 70;    // 点数
int attendance = 12; // 出席日数

// && (かつ): 点数が60以上 かつ 出席が10日以上
if (score >= 60 && attendance >= 10) {
    System.out.println("総合合格です");
}

// || (または): 点数が90以上 または 特別賞あり
boolean specialAward = false;
if (score >= 90 || specialAward) {
    System.out.println("優秀賞対象です");
}

// ! (ではない): まだ合格していない
boolean passed = false;
if (!passed) {
    System.out.println("まだ合格していません");
}

短絡評価（ショートサーキット）について

&& では左側が false なら右側は評価しません（どうせ false になるから）。|| では左側が true なら右側は評価しません（どうせ true になるから）。これを「短絡評価」といいます。重い処理を右側に書くと効率的です。

3.5 switch 文 — 値による分岐

「曜日ごとに違う処理をしたい」「メニュー番号に応じて動作を変えたい」など、特定の値に対して複数の処理を振り分けたい場合は、switch 文が便利です。if-else を縦に並べるよりもスッキリ書けます。

SwitchExample.java — 曜日で分岐するswitch文 mainメソッド内に記述 
int day = 3; // 1=月曜, 2=火曜, 3=水曜...

switch (day) {
    case 1:
        System.out.println("月曜日");
        break; // break がないと次のcaseも実行される
    case 2:
        System.out.println("火曜日");
        break;
    case 3:
        System.out.println("水曜日");
        break;
    default:
        // どのcaseにも当てはまらないとき
        System.out.println("それ以外の曜日");
        break;
}
terminal — 実行結果 
水曜日

if-else と switch の使い分け

構文 得意なケース
if-else 範囲判定（60以上・未満など）や複雑な条件に向いている
switch 特定の値（整数・文字列など）への完全一致で複数の処理を振り分けるときに向いている

注意: break を忘れないこと

switch 文の各 case の末尾には必ず break; を書きましょう。break を書かないと、次の case の処理も続けて実行される「フォールスルー」が起きます。これは意図しないバグの原因になります。

やってみよう 3-1: 点数判定プログラムを作ろう

変数 score に点数を入れ、else if を使って「優・良・可・不可」を表示するプログラムを作ります。条件分岐の書き方と順番を実際に体験しましょう。

手順

  • 1 新しいファイル GradeChecker.java を作成し、int score = 82; のように変数を宣言する。
  • 2 else if を使って「90点以上→優」「75点以上→良」「60点以上→可」「それ以外→不可」と表示するコードを書く（3.3 のコード例を参考に）。
  • 3 実行して結果を確認したら、score の値を 55・75・95 などに変えて、判定が正しく切り替わるかテストする。
  • 4 （チャレンジ）&& を使って「80点以上かつ出席10日以上なら特別表彰」という条件もさらに追加してみよう。

理解度確認クイズ

  • Q1. int x = 50; のとき、x >= 50 の結果は true と false のどちらですか？ また、x != 50 の結果はどうなりますか？

    回答を表示

    >= 50 は「50以上」なので x = 50 はちょうど50以上、よって true です。!= 50 は「50と等しくない」ですが x は 50 なので等しい、よって false です。

  • Q2. 次のコードを実行するとどの行が表示されますか？ 理由も合わせて説明してください。
    int age = 20; boolean hasTicket = false;
    if (age >= 18 && hasTicket) { ... "入場可" ... } else { ... "入場不可" ... }

    回答を表示

    「入場不可」が表示されます。&&（かつ）は両方の条件が true のときだけ true になります。age >= 18 は true ですが、hasTicket は false なので、全体の条件は false になり、else 側が実行されます。

  • Q3. switch 文で break; を書き忘れると何が起きますか？ また、どの case にも当てはまらないときに実行されるキーワードは何ですか？

    回答を表示

    break を書き忘れると、一致した case の処理の後に続く case の処理も続けて実行されてしまいます（フォールスルー）。意図しない動作につながるため必ず書きましょう。どの case にも当てはまらないときに実行されるのは default です。
