エラーに対処する - 例外処理

プログラムは予期せぬ事態（例外）で止まることがあります。この章では、Javaの例外処理の仕組みを学び、try-catch-finally を使ってプログラムを安全に継続させる方法を習得します。エラーに強いコードを書くための最初の一歩です。

この章で学ぶこと

  • 例外（Exception）とは何か、コンパイルエラーとの違いを説明できる
  • try-catch 構文を使って例外を受け止め、プログラムを継続させられる
  • 代表的な例外の種類を知り、getMessage() でエラー情報を取得できる
  • finally ブロックの役割と throws / throw の基本を理解できる

12.1 例外（Exception）とは

プログラムを実行していると、予期せぬ問題が起きることがあります。これを例外（Exception）と呼びます。例えば、「0で割り算をしてしまった」「配列の範囲外にアクセスした」「"abc" を数値に変換しようとした」といった状況です。

まず、よく混同しがちなコンパイルエラーとの違いを整理しましょう。コンパイルエラーは「書き方のミス」で、プログラムを実行する前に検出されます。一方、例外は「実行中に起きる問題」です。書き方は正しくても、実行してみて初めて発覚します。

種類 いつ起きる 例
コンパイルエラー 実行前（コンパイル時） セミコロン忘れ・型の不一致
例外（Exception） 実行中（Runtime） 0除算・配列範囲外・変換失敗

例外が発生すると、そのままではプログラムが強制終了してしまいます。次のコードで確かめてみましょう。

ExceptionDemo.java — 未処理の例外 
public class ExceptionDemo {
    public static void main(String[] args) {
        int a = 10;
        int b = 0;
        // 0で割ると例外が発生してプログラムが止まる
        int result = a / b;
        System.out.println("結果: " + result); // ここには到達しない
    }
}
terminal — 実行結果（エラー） 
Exception in thread "main" java.lang.ArithmeticException: / by zero
    at ExceptionDemo.main(ExceptionDemo.java:6)

「スタックトレース」とは

エラー発生時に表示される at ExceptionDemo.main(ExceptionDemo.java:6) のような情報をスタックトレースと呼びます。「どのファイルの何行目で問題が起きたか」が書いてあるので、バグ修正の手がかりになります。読めるようになると非常に便利です。

12.2 try-catch — 例外を受け止める

例外が起きてもプログラムを止めないようにするのが try-catch 構文 です。「危険かもしれない処理」を try ブロックで囲み、例外が発生した場合の処理を catch ブロックに書きます。

TryCatchDemo.java — 基本構文 
public class TryCatchDemo {
    public static void main(String[] args) {
        try {
            // 危険な処理をここに書く
            int result = 10 / 0;
            System.out.println("結果: " + result);
        } catch (ArithmeticException e) {
            // 例外が起きたらここに来る
            System.out.println("エラー: " + e.getMessage());
        }
        // プログラムはここに続く
        System.out.println("プログラムは継続中");
    }
}
terminal — 実行結果 
エラー: / by zero
プログラムは継続中

catch に書いた処理が実行されたあと、プログラムは止まらずに続きます。try-catch の処理の流れを図で確認しましょう。

flowchart TD A([開始]) --> B["try ブロックの処理を実行"] B --> C{"例外が
発生した?"} C -- いいえ（正常） --> D["try ブロックの
残り処理を続ける"] C -- はい（例外発生） --> E["catch ブロックへジャンプ"] E --> F["catch の処理を実行
（エラーメッセージ表示など）"] D --> G["try-catch の外の処理へ"] F --> G G --> H([終了])

図の説明: try ブロックの処理中に例外が発生すると catch ブロックへジャンプして処理し、発生しなければ catch を飛ばします。どちらの場合も最後は try-catch の外の処理へ進む、という流れを表しています。

重要ポイント: try-catch の動き

try ブロックで例外が起きた瞬間、その行以降の try の処理はスキップされて catch へジャンプします。例外が起きなければ catch ブロックはまったく実行されません。

12.3 例外の種類と getMessage()

Javaにはたくさんの種類の例外があります。よく出会う代表的なものを覚えておきましょう。

例外クラス名 起きる状況 例
ArithmeticException 算術エラー（0除算など） 10 / 0
ArrayIndexOutOfBoundsException 配列の範囲外アクセス arr[10]（要素が5個のとき）
NumberFormatException 文字列を数値に変換失敗 Integer.parseInt("abc")
NullPointerException null の参照を使おうとした str.length()（str が null のとき）

catch で受け取った例外オブジェクト（上の例では e）には、エラーの詳細情報が入っています。e.getMessage() を呼ぶと、何が起きたかを示すメッセージ文字列を取得できます。

ExceptionTypes.java — 複数の例外を確認 
public class ExceptionTypes {
    public static void main(String[] args) {
        // 文字列 → 数値への変換失敗を catch する
        try {
            int num = Integer.parseInt("abc"); // 数値でない文字列
        } catch (NumberFormatException e) {
            // getMessage() でエラーメッセージを表示
            System.out.println("変換エラー: " + e.getMessage());
        }

        // 配列の範囲外アクセスを catch する
        try {
            int[] arr = {10, 20, 30}; // 要素は 0〜2 のみ
            System.out.println(arr[5]); // 範囲外！
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("範囲外エラー: " + e.getMessage());
        }
    }
}
terminal — 実行結果 
変換エラー: For input string: "abc"
範囲外エラー: Index 5 out of bounds for length 3

12.4 finally — 必ず実行されるブロック

try-catch に finally ブロックを追加すると、例外が発生したかどうかにかかわらず、必ず最後に実行される処理を書けます。ファイルを閉じる・接続を切るなど「後片付け」に使います。

FinallyDemo.java — finally の動作確認 
public class FinallyDemo {
    public static void main(String[] args) {
        try {
            System.out.println("処理を開始します");
            int result = 10 / 0; // 例外が発生
            System.out.println("この行は実行されない");
        } catch (ArithmeticException e) {
            System.out.println("例外をキャッチ: " + e.getMessage());
        } finally {
            // 例外の有無に関係なく必ず実行される
            System.out.println("後片付け処理（finally）");
        }
    }
}
terminal — 実行結果 
処理を開始します
例外をキャッチ: / by zero
後片付け処理（finally）

finally の使いどき

実際のプログラムでは、ファイルを開いたら必ず閉じる、データベース接続を使ったら必ず切断するなど、「使ったら片付ける」処理を finally に書くのが定番です。例外が起きても起きなくても後片付けを確実に行えるため、バグや資源の無駄遣いを防げます。

発展コラム: try-with-resources で後片付けを自動化する

ファイルやデータベース接続のように「使い終わったら必ず閉じる（解放する）」必要があるものをリソースと呼びます。先ほどの finally で閉じることもできますが、Java には閉じ忘れを防ぐ便利な書き方「try-with-resources」があります。try の直後の丸括弧でリソースを用意すると、処理が終わったとき（途中で例外が起きたときも）に自動的に閉じてくれます。

TryWithResources.java — リソースの自動クローズ 
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class TryWithResources {
    public static void main(String[] args) {
        // try の括弧内で開いたリソースは、処理後に自動で close される
        try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
            String line = br.readLine();
            System.out.println(line);
        } catch (IOException e) {
            System.out.println("読み込みに失敗しました: " + e.getMessage());
        }
        // ここに来た時点で br は自動的に閉じられている（finally で close を書かなくてよい）
    }
}

今は「こういう便利な書き方がある」と知っておくだけで十分です。ファイル操作などを本格的に学ぶときに、改めて使いこなしていきましょう。

12.5 複数の catch と Exception で広く受ける

1つの try ブロックに対して、複数の catch を書くことができます。発生した例外の種類に応じて異なる処理を行いたいときに便利です。

MultiCatch.java — 複数の catch 
public class MultiCatch {
    public static void main(String[] args) {
        try {
            String s = null;
            System.out.println(s.length()); // NullPointerException
        } catch (NullPointerException e) {
            System.out.println("null エラー: " + e.getMessage());
        } catch (Exception e) {
            // 親クラス Exception で残り全ての例外を広く受け止める
            System.out.println("その他のエラー: " + e.getMessage());
        }
    }
}

catch の並び順に注意

複数 catch を書くときは、より具体的な例外を上に、より広い例外（Exception）を下に書きます。上から順番にチェックされるため、広い例外を上に書いてしまうと、具体的な catch に到達しなくなります。

12.6 throws と throw の触り

例外処理には try-catch で「その場で受け止める」以外に、「呼び出し元に例外を投げる（伝える）」という方法もあります。

throws（宣言）

メソッドのシグネチャに書き、「このメソッドは○○例外を投げる可能性があります」と宣言する。呼び出し元に処理を委ねます。

throw（発生）

メソッド内部で意図的に例外を発生させる命令。入力値チェックなどで「この値はおかしい！」と知らせたいときに使います。

ThrowsDemo.java — throws と throw の例 
public class ThrowsDemo {
    // throws: このメソッドは例外を投げる可能性があると宣言
    static int divide(int a, int b) throws ArithmeticException {
        if (b == 0) {
            // throw: 意図的に例外を発生させる
            throw new ArithmeticException("0で割ることはできません");
        }
        return a / b;
    }

    public static void main(String[] args) {
        try {
            System.out.println(divide(10, 0));
        } catch (ArithmeticException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}

やってみよう 12-1: 例外を安全に処理しよう

配列の範囲外アクセスや0除算を try-catch で受け止め、エラーが起きてもプログラムが続行されることを確かめましょう。

手順

  • 1 要素数3の int 配列を用意し、arr[10] にアクセスするコードを書く。try-catch（ArrayIndexOutOfBoundsException）でエラーメッセージを表示する。
  • 2 10 / 0 の計算を別の try-catch（ArithmeticException）で囲み、「0で割り算しました」と表示する。
  • 3 両方のエラーが表示されたあと「プログラム終了」と出力されることを確認する。エラーが起きてもプログラムが止まらないことを体験しよう。

期待する実行結果イメージ:

範囲外エラー: Index 10 out of bounds for length 3
0除算エラー: / by zero
プログラム終了

理解度確認クイズ

  • Q1. 例外（Exception）とコンパイルエラーの違いを、「いつ起きるか」に着目して説明してください。

    回答を表示

    コンパイルエラーは、プログラムを実行する前のコンパイル時に検出される「書き方のミス」です。一方、例外は書き方は正しくても実行中（Runtime）に発生するエラーで、0除算・配列の範囲外アクセスなど実際に動かしてみて初めて分かります。

  • Q2. try-catch の catch ブロックに書いた e.getMessage() は何を返しますか？

    回答を表示

    catch で受け取った例外オブジェクト e が持つエラーメッセージ文字列を返します。例えば ArithmeticException（0除算）なら / by zero という文字列が返ります。これを表示することで、ユーザーや開発者に何が起きたかを伝えられます。

  • Q3. finally ブロックはどのような状況でも実行されますか？ どんな用途に向いていますか？

    回答を表示

    はい、例外が発生した場合でも発生しなかった場合でも、必ず実行されます。ファイルを閉じる・データベース接続を切断するなど「使ったら必ず後片付けする」処理を書くのに向いています。こうすることで資源の無駄遣いやデータ破損を防げます。
