繰り返し処理 - ループ

同じ作業を何度も手でやるのは大変です。コンピュータが得意なのは、こうした繰り返し作業を自動でこなすこと。この章ではJavaの「ループ（繰り返し）」構文を身近な例で学び、九九の表示といった実践的なプログラムを自分の手で動かしていきます。

この章で学ぶこと

  • ループとは何か、手作業との違いを説明できる
  • for 文の3要素（初期化・条件・更新）を理解して使える
  • while 文と do-while 文の違いを説明できる
  • break と continue でループを制御できる
  • 二重ループ（ネストしたループ）を使って九九の表を表示できる

このページのコード例の読み方

各コード例の右上のラベルで「コードを書く場所」を示します。

  • 完全なプログラムそのまま .java ファイルに貼り付けて実行できます。
  • mainメソッド内に記述public static void main(String[] args) { ... } の中に書いて実行します。

4.1 ループとは — 繰り返しを自動化する

「1から10までの数字を画面に表示したい」とします。前章までの知識では、System.out.println(1); から System.out.println(10); まで10行書くことになります。これが1から100まで、1000までになったらどうでしょうか。

ループ（繰り返し処理）を使えば、「この処理を○回繰り返す」「条件が満たされる間ずっと繰り返す」といったことを、短いコードで実現できます。料理でいえば「材料を10個切る」手順で「包丁を10回動かしてください」と書けばよいところを、1行で指示できるようなものです。

ループなし（手書き）

同じ命令を100回書く必要があり、修正も100か所になる。数が増えるほど現実的でなくなる。

ループあり（自動化）

「100回繰り返す」と1回書くだけ。修正も1か所で済む。コンピュータが得意な作業を任せられる。

Javaには主に3種類のループ構文があります。for文・while文・do-while文です。それぞれ得意な場面が異なりますが、この章で一通り習得しましょう。

4.2 for 文 — 回数が決まっているループ

for文は「何回繰り返すかが決まっている」場合に最もよく使われるループです。書き方は次のとおりです。

for文の基本構文 mainメソッド内に記述 
// for（初期化; 条件; 更新）{ 繰り返す処理 }
for (int i = 1; i <= 10; i++) {
    System.out.println(i); // iの値を表示
}
要素 役割 例
初期化 ループ開始前に1回だけ実行。カウンタ変数を用意する。 int i = 1
条件 毎回の繰り返し前に確認。true なら続行、false なら終了。 i <= 10
更新 処理実行後に毎回行う。カウンタを進める。 i++

for文の実行フロー

flowchart TD A(["開始"]) --> B["初期化: int i = 1"] B --> C{"条件チェック: i <= 10"} C -->|"true（続行）"| D["処理を実行"] D --> E["更新: i++"] E --> C C -->|"false（終了）"| F(["ループ終了"])

図の説明: for文の実行順序を示すフローチャートで、初期化→条件チェック→処理→更新のサイクルが条件が偽になるまで繰り返されることを表しています。

実際のコード例で動きを確認しましょう。1から10の合計を計算する定番プログラムです。

Sum1to10.java — 1から10の合計 完全なプログラム 
public class Sum1to10 {
    public static void main(String[] args) {
        int sum = 0; // 合計を入れる変数（初期値0）

        // i が 1 から 10 になるまで繰り返す
        for (int i = 1; i <= 10; i++) {
            sum = sum + i; // 合計にiを足していく
        }

        // ループを抜けた後に合計を表示
        System.out.println("1から10の合計: " + sum);
    }
}
実行結果 
1から10の合計: 55

用語: カウンタ変数とインクリメント

カウンタ変数（ここでは i）はループの回数を数える変数です。i++ は「i = i + 1」の短縮形でインクリメント（1つ増やす）と呼びます。慣習的に変数名は i・j・k を使うことが多いです。

4.3 while 文 — 条件が真の間繰り返す

while文は「条件が満たされている間、繰り返し続ける」ループです。for文と違い、「何回繰り返すか」が最初から分からない場合に向いています。

while文の基本構文 mainメソッド内に記述 
// while（条件）{ 繰り返す処理 }
int count = 1; // カウンタを外で用意する

while (count <= 5) {
    System.out.println("カウント: " + count);
    count++; // 忘れると無限ループになるので注意！
}

for文とwhile文は、多くの場合どちらでも同じ処理を書けます。使い分けの目安は次のとおりです。

for文が向く場面

繰り返す回数が決まっている場合。「5回繰り返す」「配列の全要素に対して行う」など。

while文が向く場面

繰り返す回数が不定な場合。「ユーザーが正しい入力をするまで繰り返す」など。

注意: 無限ループに気をつけよう

while文で count++ のような更新を書き忘れると、条件が永遠に true のままになり、プログラムが止まらなくなります（無限ループ）。プログラムが止まらなくなったら、VSCodeのターミナルで Ctrl + C を押して強制終了しましょう。

4.4 do-while 文 — 最低1回は実行される

do-while文はwhile文の変形で、処理を先に1回実行してから条件をチェックするのが特徴です。条件に関わらず「最低1回は実行したい」場面に使います。

DoWhileExample.java mainメソッド内に記述 
int n = 1;

do {
    System.out.println("実行中: n = " + n);
    n++;
} while (n <= 3); // 処理の後に条件を確認する

// 出力: 実行中: n = 1
//       実行中: n = 2
//       実行中: n = 3

do-while のポイント

while文は「条件が最初から false なら1回も実行しない」のに対し、do-while文は「条件が false でも必ず1回実行する」違いがあります。メニュー表示・入力検証など「まず実行してから確認する」処理に向いています。

4.5 break と continue — ループを制御する

ループの途中で「もうやめたい」「今回だけスキップしたい」という場面があります。そんなときに使うのが break と continue です。

break — ループを途中で抜ける

break はその時点でループを終了し、ループの外にある次の処理へジャンプします。「5が見つかったら探すのをやめる」といった場合に使います。

BreakExample.java mainメソッド内に記述 
for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        System.out.println("5が見つかったので終了！");
        break; // ループをここで抜ける
    }
    System.out.println("チェック中: " + i);
}
// 出力: チェック中: 1  チェック中: 2  ...
//       5が見つかったので終了！

continue — 今回だけスキップする

continue はその回の残りの処理をスキップして、次の繰り返しへ進みます。「偶数だけ表示する（奇数はスキップ）」といった場合に使います。

ContinueExample.java — 奇数をスキップして偶数だけ表示 mainメソッド内に記述 
for (int i = 1; i <= 10; i++) {
    if (i % 2 != 0) {
        continue; // 奇数ならスキップして次のiへ
    }
    System.out.println("偶数: " + i);
}
// 出力: 偶数: 2  偶数: 4  偶数: 6  偶数: 8  偶数: 10

break と continue の違いまとめ

  • break: ループそのものを終了する。ループの外に出る。
  • continue: 今回の繰り返しだけスキップ。ループ自体は続く。

4.6 ネストしたループ（二重ループ）で九九の表を作る

ループの中にさらにループを書くことをネスト（入れ子）といいます。外側のループと内側のループが組み合わさることで、表のような2次元的な処理が可能になります。代表的な例が九九（かけ算）の表です。

外側ループで段（1の段〜9の段）を制御し、内側ループで各段の計算（1〜9を順番にかける）を行います。

MultiplicationTable.java — 九九の表 完全なプログラム 
public class MultiplicationTable {
    public static void main(String[] args) {
        // 外側ループ: 1の段〜9の段
        for (int i = 1; i <= 9; i++) {
            // 内側ループ: 各段で1〜9をかける
            for (int j = 1; j <= 9; j++) {
                // printf で桁をそろえて表示
                System.out.printf("%3d", i * j);
            }
            System.out.println(); // 1段分終わったら改行
        }
    }
}

実行すると以下のように九九の表が表示されます。

実行結果（九九の表） 
  1  2  3  4  5  6  7  8  9
  2  4  6  8 10 12 14 16 18
  3  6  9 12 15 18 21 24 27
  4  8 12 16 20 24 28 32 36
  5 10 15 20 25 30 35 40 45
  6 12 18 24 30 36 42 48 54
  7 14 21 28 35 42 49 56 63
  8 16 24 32 40 48 56 64 72
  9 18 27 36 45 54 63 72 81

用語: System.out.printf とは

System.out.printf("%3d", 値) は「最低3文字分の幅で整数を表示する」命令です。数字の桁をそろえて表示したいときに便利です。println が改行つきで表示するのに対し、printf は改行なしで書式指定できます。

やってみよう 4-1: ループを使って表示しよう

ループの感覚をつかむために、以下の課題に取り組んでみましょう。2つのうちどちらか（または両方！）に挑戦してください。

課題A: 1〜100の偶数だけ表示

  • 1 for文で1〜100を順番に処理するループを書く。
  • 2 i % 2 == 0（2で割った余りが0 = 偶数）のときだけ表示するif文をループ内に加える。
  • 3 実行して 2, 4, 6, … 100 と表示されるか確認する。

課題B: 好きな段の九九を1行で表示

  • 1 好きな段（例: 7の段）を変数 dan に入れる。
  • 2 for文で1〜9まで繰り返し、dan × i の結果を表示する。
  • 3 段を変えて（例: 3の段、5の段）動作を確認する。

理解度確認クイズ

  • Q1. for (int i = 0; i < 5; i++) のループは何回実行されますか？ また、最後に i はいくつになりますか？

    回答を表示

    5回実行されます（i = 0, 1, 2, 3, 4 のとき実行）。条件が i < 5 なので i = 5 になった時点でループを抜けます。ループを抜けた後の i の値は 5 です。

  • Q2. ループの途中で特定の条件が満たされたときにループを完全に終了するには、どのキーワードを使いますか？ また、今回の繰り返しだけスキップして次の繰り返しに進むには？

    回答を表示

    ループを完全に終了するには break、今回だけスキップして次の繰り返しへ進むには continue を使います。break はループの外に飛び出し、continue は現在の繰り返しの残りをとばして次の繰り返しへ移ります。

  • Q3. while文と do-while文の違いは何ですか？ 「条件が最初から false でも1回は処理を実行したい」場合はどちらを使いますか？

    回答を表示

    while文は先に条件を確認し、false なら1回も実行しません。do-while文は先に処理を実行してから条件を確認するため、条件が false でも必ず1回は処理が実行されます。「最低1回は実行したい」場合は do-while文 を使います。
