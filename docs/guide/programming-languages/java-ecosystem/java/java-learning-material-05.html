第5章 所要時間: 約90分

データをまとめて管理する - 配列

これまで1つずつ変数を用意してデータを管理してきました。しかし「100人分のテスト点数を扱いたい」となったとき、変数を100個も書くのは大変です。この章では複数のデータをひとまとめにして管理できる「配列」という仕組みを、番号付きロッカーのたとえを使って丁寧に学んでいきます。

この章で学ぶこと

  • 配列が「なぜ必要か」をロッカーのたとえで理解し、宣言・初期化の書き方を覚える
  • インデックス（添字）で要素にアクセス・代入できる。length で配列の長さを取得できる
  • for文と拡張for文（for-each）を使って配列の全要素を処理できる
  • 2次元配列のイメージをつかみ、表のようなデータを扱う感覚を得る

このページのコード例の読み方

各コード例の右上のラベルで「コードを書く場所」を示します。

  • 完全なプログラムそのまま .java ファイルに貼り付けて実行できます。
  • mainメソッド内に記述public static void main(String[] args) { ... } の中に書いて実行します。

5.1 配列とは — 番号付きロッカーのたとえ

たとえばクラスの生徒5人のテスト点数を変数で管理しようとすると、score1、score2、score3、score4、score5 と5個の変数が必要です。これが100人になったら変数を100個も用意しなければなりません。とても大変ですよね。

そこで登場するのが配列（はいれつ）です。配列は「番号付きロッカー」のようなものです。ロッカー室に100個の扉が並んでいて、それぞれに番号（0番・1番・2番…）が書いてあるイメージです。1つの名前（配列名）で全ロッカーをひとまとめに呼べ、番号を指定すれば好きなロッカーを開けられます。

用語: インデックス（添字）

配列の各マスにアクセスするための番号をインデックス（添字）と呼びます。Javaの配列はインデックスが 0 から始まります。5個の配列なら 0・1・2・3・4 が有効なインデックスです。「1番目のマスはインデックス0」と覚えておきましょう。

5.2 配列の宣言・初期化

Javaで配列を作る方法は主に2つあります。「サイズを指定して作る方法」と「最初から値を入れて作る方法」です。

方法1: サイズを指定して宣言する

new int[5] のように書くと、int型の値が5個入るロッカーが作られます。作った直後は全マスが 0 で初期化されます。

ArrayDeclare.java — サイズ指定で作る mainメソッド内に記述 
// int型を5個入れられる配列を作る
int[] scores = new int[5];

// 作った直後は全マスが 0 になっている
// scores[0]=0, scores[1]=0, ... scores[4]=0

方法2: 最初から値を入れて初期化する

{}（波括弧）の中に値をカンマ区切りで並べると、サイズを書かなくても配列を作れます。値の数が自動的にサイズになります。

ArrayInit.java — 初期値を指定して作る mainメソッド内に記述 
// 5つの値を入れた配列を作る（サイズは自動で5になる）
int[] scores = {80, 90, 75, 85, 92};

// scores[0] は 80、scores[1] は 90 …
System.out.println(scores[0]); // 80 と表示される
書き方 例 使い場面
サイズ指定 new int[5] 後から値を入れたいとき
初期値指定 {80, 90, 75} 最初から値が決まっているとき

5.3 要素へのアクセスと代入、配列の長さ

配列のマス（要素）を読み書きするには 配列名[インデックス] の形で書きます。また、配列が何個の要素を持っているかは 配列名.length で取得できます。

ArrayAccess.java — アクセスと代入 mainメソッド内に記述 
int[] scores = {80, 90, 75, 85, 92};

// 読み取り: インデックス1の値を表示
System.out.println(scores[1]); // 90

// 代入: インデックス2の値を変更
scores[2] = 100;
System.out.println(scores[2]); // 100

// 配列の長さを取得
System.out.println(scores.length); // 5

注意: ArrayIndexOutOfBoundsException

配列のサイズを超えたインデックスにアクセスすると ArrayIndexOutOfBoundsException（配列の添字が範囲外）というエラーが発生してプログラムが止まります。5個の配列なら有効なインデックスは 0〜4 です。scores[5] はエラーになるので注意しましょう。

5.4 配列とループ — for文で全要素を処理する

配列と for 文を組み合わせると、全要素を順番に処理できます。scores.length を上限にすることで、配列のサイズが変わっても同じコードが使えます。合計や最大値を求める処理の典型パターンを見てみましょう。

ArrayLoop.java — 合計と最大値 mainメソッド内に記述 
int[] scores = {80, 90, 75, 85, 92};
int total = 0;
int max = scores[0]; // 最初の値を最大値の初期値に

for (int i = 0; i < scores.length; i++) {
    total += scores[i]; // 合計を加算
    if (scores[i] > max) {
        max = scores[i]; // 最大値を更新
    }
}
System.out.println("合計: " + total); // 合計: 422
System.out.println("最高点: " + max); // 最高点: 92
flowchart TD A([開始]) --> B["total=0, max=scores[0]"] B --> C["i=0"] C --> D{"i < scores.length?"} D -- はい --> E["total += scores[i]
max を更新"] E --> F["i++"] F --> D D -- いいえ --> G["合計・最大値を表示"] G --> H([終了])

図: for文で配列の全要素を処理する流れ

重要ポイント: 「i < scores.length」と書く理由

i <= scores.length（以下）と書くと1つ多くなり、最後に範囲外アクセスのエラーが起きます。「未満（<）」で書くのが正しい書き方です。インデックスは0から始まるため、length と同じ値のインデックスは存在しません。

5.5 拡張for文（for-each）— より簡潔な書き方

インデックスを意識せず全要素を順に取り出したいだけなら、拡張for文（for-each文）を使うと簡潔に書けます。for (型 変数名 : 配列名) という形で、配列の先頭から末尾まで自動的に繰り返してくれます。

ForEach.java — 拡張for文 mainメソッド内に記述 
int[] scores = {80, 90, 75, 85, 92};
int total = 0;

// s には先頭から順に各要素の値が入ってくる
for (int s : scores) {
    total += s; // インデックスを書かなくてよい
}
System.out.println("合計: " + total); // 合計: 422

通常のfor文

インデックスを使いたいとき（scores[i] のように番号で処理を変えるなど）に向いている

拡張for文（for-each）

全要素を順番に読み取るだけでよいときに便利。コードが短くなり、読みやすい

5.6 2次元配列 — 表のようなデータ

配列の中にさらに配列を入れると2次元配列が作れます。表（行と列）をイメージしてください。たとえば「3クラスの生徒、各クラス4科目の成績」を管理したいときに使います。

int[行数][列数] のように2つのサイズを指定します。要素へのアクセスは 配列名[行][列] です。

Array2D.java — 2次元配列の基本 mainメソッド内に記述 
// 3行 × 4列 の2次元配列（3人 × 4科目のイメージ）
int[][] grades = {
    {80, 75, 90, 85}, // 1人目: 国語・数学・英語・理科
    {70, 95, 80, 60}, // 2人目
    {90, 85, 70, 88}  // 3人目
};

// 2人目（行=1）の数学（列=1）を表示
System.out.println(grades[1][1]); // 95

2次元配列はこの章では触りだけ

2次元配列の使い方はまだ入口です。「行・列でデータを管理できる」という感覚をつかんでおけば十分です。本格的な使い方はオブジェクト指向（第7〜9章）でコレクション（ArrayList等）を学ぶ際に改めて整理します。

やってみよう 5-1: 5人のテスト点数から合計・平均を求めよう

配列と for 文を組み合わせて、5人分の点数の合計と平均を計算して表示するプログラムを作ります。

手順

  • 1 ScoreCalc.java を作り、5人分の点数を配列 int[] scores = {72, 88, 65, 95, 80}; で定義する。
  • 2 for 文か拡張 for 文を使って全要素を合計し、System.out.println で合計を表示する。
  • 3 合計を scores.length で割って平均を求め、表示する。（ヒント: 平均を小数で出したい場合は (double) total / scores.length と書く。）
  • 4 余裕があれば最高点・最低点も表示してみよう。

期待される出力例

合計: 400
平均: 80.0

理解度確認クイズ

  • Q1. int[] nums = {10, 20, 30}; と宣言したとき、nums[1] の値は何ですか？ またその理由を、インデックスの仕組みを使って説明してください。

    回答を表示

    20 です。配列のインデックスは 0 から始まるため、nums[0]=10、nums[1]=20、nums[2]=30 と対応しています。「1番目」という感覚でインデックス 1 を指定すると実際には2番目の値が返ります。「インデックスは0始まり」を常に意識しましょう。

  • Q2. 配列のサイズを5にして作ったとき、有効なインデックスの範囲はどこからどこまでですか？ また、範囲外を指定するとどうなりますか？

    回答を表示

    有効なインデックスは 0〜4（5個）です。scores[5] のように範囲外のインデックスにアクセスすると ArrayIndexOutOfBoundsException というエラーが発生し、プログラムが強制終了します。for 文の終了条件に i < scores.length（未満）を使うことでこのエラーを防げます。

  • Q3. 通常のfor文と拡張for文（for-each）はどう使い分けますか？ それぞれ向いているシーンを説明してください。

    回答を表示

    通常のfor文は、インデックス番号を使って処理を変えたいとき（例: 偶数番目だけ処理する・配列の要素を書き換えるなど）に向いています。拡張for文（for-each）は、全要素を先頭から順番に読み取るだけでよいとき（例: 合計を計算する・全要素を表示するなど）に向いており、コードが短く読みやすくなります。
