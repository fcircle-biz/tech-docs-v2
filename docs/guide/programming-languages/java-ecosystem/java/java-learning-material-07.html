設計図からモノを作る - クラスとオブジェクトの基本

この章では、Javaプログラミングの核心である「オブジェクト指向」の入り口に踏み込みます。「クラス（設計図）」と「オブジェクト（実体）」という2つの概念を、日常的なたとえを通じて理解し、実際にクラスを定義してオブジェクトを生成・利用するコードを書いてみましょう。

この章で学ぶこと

  • オブジェクト指向とは何か、なぜ必要なのかを説明できる
  • クラス（設計図）とオブジェクト（実体）の関係を理解する
  • フィールドとメソッドを持つシンプルなクラスを定義できる
  • new 演算子でオブジェクトを生成し、ドット記法でフィールド・メソッドにアクセスできる

7.1 オブジェクト指向とは

これまでの章では、変数や条件分岐・繰り返し・メソッドといった「プログラムの基本構造」を学んできました。これらだけでも小さなプログラムは書けますが、規模が大きくなるにつれて「どこに何を書けばいいのか分からなくなる」「同じような処理があちこちに散らばる」という問題が起きます。

この問題を解決する考え方が、オブジェクト指向（Object-Oriented Programming / OOP）です。オブジェクト指向では、現実世界にある「モノ（Object）」を手がかりにプログラムを整理します。たとえば「車」というモノは、色や速度というデータ（状態）を持ち、走ったり止まったりという振る舞い（動作）を持ちます。

データ（フィールド）

モノが持つ「状態」や「属性」。車なら「色」「速度」「燃料の残量」などです。

振る舞い（メソッド）

モノができる「動作」や「操作」。車なら「走る」「止まる」「加速する」などです。

このように「データ」と「振る舞い」をひとかたまりにして管理することで、大きなプログラムでも「車に関することは Carクラスを見ればわかる」という整理ができます。これがオブジェクト指向の基本的な考え方です。

なぜオブジェクト指向が必要か

10行のプログラムなら普通の書き方で十分ですが、1万行を超えると「あの処理はどこに書いた？」「バグを直したら別の場所も壊れた」という問題が出てきます。オブジェクト指向は、プログラムを「責任ごとに部品化」して管理しやすくする技術です。Javaはオブジェクト指向を中心に設計されており、これからの章でその仕組みを少しずつ学んでいきます。

7.2 クラスとオブジェクトの関係

オブジェクト指向で最も重要な概念が、クラスとオブジェクト（インスタンス）の関係です。この2つを理解するために、「たい焼き」を例に考えてみましょう。

たとえ: たい焼きの型とたい焼き

クラス ＝ たい焼きの型

「どんな形か」「何を入れるか」という設計図・型。型そのものはたい焼きではありません。

オブジェクト ＝ 実際のたい焼き

型を使って作られた実体。1つの型から「あんこ入り」「クリーム入り」など、複数のたい焼きが作れます。

プログラムの世界に置き換えると、クラスは「このオブジェクトはどんなデータ（フィールド）と動作（メソッド）を持つか」を定義した設計図です。オブジェクト（インスタンス）は、その設計図に基づいてメモリ上に作られた実体です。

重要なのは「1つのクラスから、複数のオブジェクトを作れる」ことです。たい焼きの型が1つあれば何個でもたい焼きを作れるように、Carクラスが1つあれば car1、car2、car3 … と何台でも車オブジェクトを作れます。

重要ポイント

クラスは「設計図」です。クラス定義を書いただけでは、フィールドの値を持つ個別のオブジェクト（実体）はまだ作られません。new を実行すると、そのオブジェクト専用のフィールド領域がメモリ上に確保されます。なお、プログラムの実行時にはクラスの情報自体も JVM に読み込まれますが、ここでは「個別のデータを持つ実体は new で作られる」と理解しておけば十分です。オブジェクトのことを「インスタンス（Instance）」とも呼びます。

7.3 クラスの定義

Javaでクラスを定義する基本的な書き方を見てみましょう。クラスには「フィールド（データを入れる変数）」と「メソッド（動作を定義する処理）」を書きます。まず、シンプルな Car クラスを例に確認します。

Car.java — クラスの定義 
// Carクラス（設計図）の定義
public class Car {

    // フィールド: 車が持つデータ（色・速度）
    String color;   // 色を表すフィールド
    int speed;      // 速度（km/h）を表すフィールド

    // メソッド: 車の動作（走る）
    void run() {
        speed = 60;  // 速度を60に設定
        System.out.println(color + "の車が時速" + speed + "kmで走ります");
    }

    // メソッド: 車の動作（止まる）
    void stop() {
        speed = 0;
        System.out.println(color + "の車が停止しました");
    }
}

クラスの書き方のポイントをまとめます。

  • class クラス名 { } — クラスの宣言。ファイル名は クラス名.java にします（Car.java）。
  • クラス名は大文字ではじめる（例: Car, Dog, Book）のがJavaの慣例です。
  • フィールドとメソッドの詳細（コンストラクタ・アクセス修飾子など）は第8章以降で詳しく学びます。今は「設計図の中にデータと動作を書く」という感覚をつかんでください。

複数クラスとファイル構成

Javaでは通常、1つのクラスを1つの .java ファイルに書きます。Car.java に Car クラスを、Main.java に main メソッドを書いて、プログラムを複数ファイルで構成するのが一般的です。

7.4 オブジェクトの生成 — new 演算子

クラス（設計図）を定義しただけではオブジェクトは存在しません。実際にオブジェクトを作るには、new 演算子を使います。

Main.java — newでオブジェクトを生成 
public class Main {
    public static void main(String[] args) {
        // new Car() でCarオブジェクトを生成し、変数myCar に代入
        Car myCar = new Car();

        // 2つ目のオブジェクトも作れる（型は同じ、でも独立した実体）
        Car yourCar = new Car();
    }
}

構文の読み方を確認しましょう。Car myCar = new Car(); は次のように読めます。

部品 意味
Car（左側） 変数の型。「Carクラスのオブジェクトを入れる変数」という意味。
myCar 変数名。このオブジェクトを参照するための名前。
new Car() オブジェクトの生成。Carクラスの設計図を元に実体をメモリ上に作る。

7.5 オブジェクトの利用 — ドット記法

生成したオブジェクトのフィールド（データ）に値を設定したり、メソッド（動作）を呼び出したりするには、ドット記法（.）を使います。

Main.java — ドット記法でフィールド・メソッドにアクセス 
public class Main {
    public static void main(String[] args) {
        Car car1 = new Car();   // 1台目のオブジェクトを生成
        car1.color = "赤";      // フィールドに値をセット
        car1.run();             // メソッドを呼び出す

        Car car2 = new Car();   // 2台目のオブジェクト（car1とは別の実体）
        car2.color = "青";
        car2.run();
    }
}
terminal — 実行結果 
赤の車が時速60kmで走ります
青の車が時速60kmで走ります

car1 と car2 は同じ Car クラスから作られましたが、それぞれ独立したデータを持っています。car1.color を変えても car2.color は変わりません。これが「1つの型から複数の独立した実体を作れる」という意味です。

ドット記法のルール

オブジェクト名.フィールド名 でフィールドの値を読み書きします。オブジェクト名.メソッド名（） でメソッドを呼び出します。ドット（.）は「〜の」という意味で、car1.color は「car1 の color」と読めます。

この章で学んだクラスとオブジェクトの知識を使って、Dog クラスを自分で定義し、2匹の犬オブジェクトを作って情報を表示してみましょう。

手順

  • 1 Dog.java を作り、name（名前）と breed（犬種）というフィールドと、bark()（「ワン！」と表示する）メソッドを定義する。

  • 3 各犬に異なる名前と犬種を設定し、それぞれの情報（名前・犬種）を System.out.println で表示してから、bark() を呼び出す。

理解度確認クイズ

  • Q1. クラスとオブジェクトの違いを「たい焼き」のたとえを使って説明してください。

    回答を表示

    クラスは「たい焼きの型（設計図）」で、どんな形・どんな中身かを定義したものです。オブジェクトは「実際のたい焼き（実体）」で、型を元に new を使って作ったものです。1つのクラス（型）から、複数のオブジェクト（たい焼き）を作ることができます。

  • Q2. 次のコードは何をしていますか？ Car myCar = new Car();

    回答を表示

    new Car() で Car クラスの設計図を元にオブジェクト（実体）をメモリ上に生成し、Car 型の変数 myCar にそのオブジェクトを代入しています。これにより myCar を通じてそのオブジェクトを操作できるようになります。

  • Q3. car1 と car2 が同じ Car クラスから作られた場合、car1.color = "赤" と設定すると car2.color も「赤」になりますか？

    回答を表示

    なりません。car1 と car2 はそれぞれ独立したオブジェクトであり、別々のメモリ領域に自分専用のフィールドを持っています。片方のフィールドを変更しても、もう一方には影響しません。
