オブジェクトの中身を理解する - フィールドとメソッド

前章でクラスとオブジェクトの基本を学びました。この章ではクラスの中身をより深く掘り下げ、オブジェクトが「データ（フィールド）」と「動作（メソッド）」をどう持つかを理解します。さらに、オブジェクト生成時に呼ばれる特別な仕組み「コンストラクタ」と「thisキーワード」も学び、実践的なクラス設計の基礎を固めます。

この章で学ぶこと

  • フィールド（インスタンス変数）とは何か、オブジェクトごとに独立した値を持つことを理解する
  • メソッド（インスタンスメソッド）を定義し、フィールドを読み書きできることを理解する
  • コンストラクタを使ってオブジェクト生成時にフィールドを初期化できる
  • thisキーワードで「自分自身のオブジェクト」を指す意味と使い方を説明できる

8.1 フィールド（インスタンス変数）とは

クラスの中で宣言する変数をフィールド（またはインスタンス変数）と呼びます。フィールドはオブジェクトが「持つデータ」を表します。たとえば車（Car）クラスなら、「色」や「スピード」がフィールドにあたります。

フィールドの最大の特徴は、オブジェクトごとに独立した値を持つ点です。Car クラスから2台の車を作ると、それぞれが自分の color や speed を別々に記憶します。1台の色を変えても、もう1台には影響しません。

Car.java — フィールドの例 
public class Car {
    // フィールド（インスタンス変数）: オブジェクトごとのデータ
    String color;   // 色
    int speed;      // スピード（km/h）
}

オブジェクトごとに独立したフィールドを持つイメージ

flowchart LR subgraph car1["car1 オブジェクト"] f1["color = 赤
speed = 0"] end subgraph car2["car2 オブジェクト"] f2["color = 青
speed = 100"] end C["Car クラス（設計図）"] --> car1 C --> car2

用語: インスタンス変数とは

「インスタンス」はオブジェクトの別名です。クラスから生成された実体を「インスタンス」と呼ぶため、オブジェクトが持つ変数を「インスタンス変数」とも言います。フィールドとインスタンス変数はほぼ同じ意味で使われます。

8.2 メソッド（インスタンスメソッド）とは

クラスの中で定義する処理のまとまりをメソッドと呼びます。オブジェクトに属するメソッドは特にインスタンスメソッドと言い、オブジェクトの「動作・振る舞い」を表します。Car クラスなら「走る（run）」や「止まる（stop）」がメソッドにあたります。

メソッドの大きな特徴は、同じクラスのフィールドに直接アクセスできる点です。フィールド名をそのまま書くだけで読み書きできます。

Car.java — メソッドの追加 
public class Car {
    String color;   // フィールド: 色
    int speed;      // フィールド: スピード

    // メソッド: 走る（フィールドを書き換える）
    void run() {
        speed = 60;  // フィールドに直接アクセス
        System.out.println(color + "の車が走り始めた！");
    }

    // メソッド: 止まる
    void stop() {
        speed = 0;
        System.out.println(color + "の車が止まった。");
    }
}

重要ポイント: クラスとは「データ＋動作」のセット

クラスはフィールド（データ）とメソッド（動作）をひとまとめにした設計図です。「データを持ち、そのデータを使って動ける」ことがオブジェクト指向プログラミングの核心です。

8.3 フィールドとメソッドを持つクラスの完全な例

実際にフィールドとメソッドを持つクラスを作り、main メソッドから呼び出してみましょう。ここでは「人物（Person）」クラスを例に、名前と年齢を持ち、あいさつを表示するメソッドを実装します。

Person.java 
public class Person {
    // フィールド（名前・年齢）
    String name;
    int age;

    // メソッド: あいさつを表示する
    void greet() {
        // フィールドをメソッド内で直接使える
        System.out.println("こんにちは、" + name + "です。" + age + "歳です。");
    }
}
Main.java — main から呼び出す 
public class Main {
    public static void main(String[] args) {
        // オブジェクト生成
        Person p1 = new Person();
        p1.name = "田中";   // フィールドに値を設定
        p1.age  = 25;

        Person p2 = new Person();
        p2.name = "鈴木";
        p2.age  = 30;

        p1.greet();  // 田中さんのあいさつ
        p2.greet();  // 鈴木さんのあいさつ
    }
}
terminal — 実行結果 
こんにちは、田中です。25歳です。
こんにちは、鈴木です。30歳です。

フィールド = データの保管庫

各オブジェクトが自分だけの値を保持します。p1.name と p2.name は別々の場所に格納されます。

メソッド = オブジェクトの動き

greet() を呼び出すと、そのオブジェクトのフィールド（name/age）を使って処理を実行します。

8.4 コンストラクタ — オブジェクト生成時に呼ばれる特別なメソッド

コンストラクタは、new でオブジェクトを生成する瞬間に自動的に呼ばれる特別なメソッドです。主な用途はフィールドの初期値を設定することです。前の例では生成後に個別に値を代入していましたが、コンストラクタを使うと一発で初期化できます。

コンストラクタには次の2つのルールがあります。

ルール1: 名前はクラス名と同じ

Person クラスならコンストラクタ名も Person です。大文字・小文字まで一致させます。

ルール2: 戻り値の型を書かない

void も含め、戻り値の型を書きません。書いてしまうと普通のメソッドと見なされます。

Person.java — コンストラクタを追加 
public class Person {
    String name;
    int age;

    // コンストラクタ: クラス名と同じ名前・戻り値の型なし
    Person(String name, int age) {
        this.name = name;  // thisについては次の8.5で解説
        this.age  = age;
    }

    void greet() {
        System.out.println("こんにちは、" + name + "です。" + age + "歳です。");
    }
}
Main.java — コンストラクタを使って生成 
public class Main {
    public static void main(String[] args) {
        // コンストラクタに値を渡して生成（フィールドが同時に初期化される）
        Person p1 = new Person("田中", 25);
        Person p2 = new Person("鈴木", 30);

        p1.greet();
        p2.greet();
    }
}

new Person（"田中", 25）を実行したときの流れ

flowchart LR A["new Person（「田中」, 25）"] --> B["コンストラクタが呼ばれる"] B --> C["this.name = 「田中」
this.age = 25"] C --> D["Personオブジェクト完成！"]

よくある間違い: void を書いてしまう

void Person(...) と書くとコンストラクタではなく「Person という名前の普通のメソッド」になります。new 時に自動呼び出しされず、フィールドが初期化されないままになる事故が起きます。戻り値の型は必ず省略してください。

8.5 thisキーワード — 「自分自身のオブジェクト」を指す

this（ディス）は「今このメソッドが実行されているオブジェクト自身」を指すキーワードです。最もよく使うシーンは、コンストラクタの引数名とフィールド名が同じときです。

たとえば Person(String name, int age) というコンストラクタでは、引数名 name もフィールド名 name もどちらも同じです。name = name; と書くと、コンピュータは引数の name を自分自身（引数）に代入しようとするだけで、フィールドが更新されません。そこで this.name と書くことで「フィールドの name」を明示します。

this キーワードの使い方 
public class Person {
    String name;  // フィールド
    int age;      // フィールド

    Person(String name, int age) {
        // this.name = フィールドのname、右辺のname = 引数のname
        this.name = name;  // 正しい: フィールドに引数を代入
        this.age  = age;   // 正しい: フィールドに引数を代入
        // name = name; と書くと引数に引数を代入するだけで無意味！
    }
}
書き方 意味 結果
this.name = name; フィールドのnameに引数のnameを代入 正しく初期化できる
name = name; 引数のnameに引数のnameを代入（何もしない） フィールドが更新されない

Tips: 引数名を変えれば this は不要

引数名とフィールド名を意図的に変えれば（例: n と name）、this なしでも name = n; と書けます。ただし、引数名を意味のある名前にしてフィールドと揃えた上で this を使うほうが読みやすいコードになります。

コンストラクタを持つ Book（本）クラスを作り、フィールドを初期化してから内容を表示してみましょう。

理解度確認クイズ

  • Q1. フィールドとローカル変数の違いは何ですか？ オブジェクトとの関係で説明してください。

    回答を表示

    フィールドはクラスの中（メソッドの外）に宣言し、オブジェクトごとに別々の値を保持します。オブジェクトが存在している間はずっと値を記憶しています。一方、ローカル変数はメソッドの中で宣言し、そのメソッドの実行中だけ存在して終了すると消えます。フィールドはオブジェクトの「状態（データ）」を表すのに使います。

  • Q2. コンストラクタに必ず守らなければならない2つのルールを述べてください。

    回答を表示

    1. 名前がクラス名と完全に一致すること（大文字・小文字も含め）。
    2. 戻り値の型を書かないこと（void も含め不要）。これらを守らないと、コンストラクタではなく普通のメソッドと見なされ、new 時に自動呼び出しされません。

  • Q3. Person(String name, int age) の中で this.name = name; と書く理由は何ですか？

    回答を表示

    コンストラクタの引数名（name）とフィールド名（name）が同じため、name だけではどちらを指すか区別できません。this.name と書くことで「自分自身のオブジェクトのフィールド name」を明示し、右辺の name（引数）の値をフィールドへ正しく代入できます。this は「今このコードが実行されているオブジェクト自身」を指すキーワードです。
