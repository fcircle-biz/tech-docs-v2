Spring Data JPAプロジェクトの作成

この章では、Spring Boot で JPA を使うためのプロジェクトを一から作成します。必要な依存関係を追加し、パッケージ構成を整え、組み込みデータベース H2 の設定を行うことで、「すぐに動く開発環境」を整えます。プロジェクトの土台となる部分なので、各設定の意味をきちんと理解しながら進みましょう。

この章で学ぶこと

  • Spring Web・Spring Data JPA・H2 Database・Thymeleaf の4つの依存関係それぞれの役割を説明できる
  • controller / service / repository / entity のパッケージ分け構成を理解する
  • H2 Database が「なぜ学習用に向いているか」を説明できる
  • application.properties の各設定（H2 接続・SQL 表示・DDL 自動生成・H2 コンソール）の意味を理解する
  • アプリを起動して H2 コンソールにブラウザから接続できる

2.1 使用する依存関係

Spring Boot プロジェクトは、Spring Initializr（start.spring.io）で生成するのが一般的です。本教材では以下の4つの依存関係（Dependency）を追加します。それぞれの「役割」を先に理解しておくと、なぜその依存が必要なのかがわかります。

依存関係 役割 なぜ必要？
Spring Web Web アプリとして動かす基盤 Controller・REST API・組み込み Tomcat が使える
Spring Data JPA JPA・Hibernate を Spring で使いやすくする Repository インターフェースだけで CRUD が使える
H2 Database メモリ上で動く軽量 DB インストール不要・依存追加だけで使える学習用 DB
Thymeleaf HTML テンプレートエンジン Controller からデータを受け取り画面に表示する

Spring Initializr での選び方

start.spring.io を開き、「Project」を Maven、「Language」を Java、「Spring Boot」を 3.x 系（最新安定版）に設定します。「Artifact」は jpa-learning などの名前を入力してください。その後、「Dependencies」の「ADD DEPENDENCIES」ボタンから上記4つを検索して追加し、「GENERATE」をクリックすると ZIP がダウンロードされます。

生成された ZIP を展開すると、ルートに pom.xml（Maven の場合）が含まれます。4つの依存関係がすでに記述されている状態です。以下に主要な依存の抜粋を示します。

pom.xml（依存関係の抜粋） 
<dependencies>
    <!-- Web アプリとして動かす基盤（Tomcat 内蔵） -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <!-- Spring Data JPA（Hibernate が内部で使われる） -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    <!-- H2: メモリ上で動く学習用 DB -->
    <dependency>
        <groupId>com.h2database</groupId>
        <artifactId>h2</artifactId>
        <scope>runtime</scope>
    </dependency>
    <!-- Thymeleaf: HTML テンプレートエンジン -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-thymeleaf</artifactId>
    </dependency>
</dependencies>

2.2 プロジェクト構成 — パッケージを役割で分ける

Spring Boot プロジェクトでは、クラスを役割ごとにパッケージ（フォルダ）で分けて管理します。本教材では以下の4つのパッケージを使います。全クラスを1か所に置くと、ファイル数が増えるにつれて「どのクラスが何をするのか」がわからなくなります。役割ごとに分けることで、コードが読みやすく・保守しやすくなります。

プロジェクト構成（src/main/java 配下） 

src/main/java/com/example/jpalearning
├── JpaLearningApplication.java   ← アプリ起動クラス（自動生成）
├── controller                    ← 画面の入出力を担当
├── service                       ← 業務処理を担当
├── repository                    ← DB 操作（データアクセス）を担当
└── entity                        ← テーブルに対応する Java クラス

controller

ブラウザからのリクエストを受け取り、画面（View）にデータを渡す。Spring MVC の @Controller クラスを置く。

service

登録・更新・削除などの業務処理を担う。Controller と Repository の橋渡し役。@Service クラスを置く。

repository

DB との読み書きを担当。JpaRepository を継承したインターフェースを置く。

entity

テーブルの1行を表す Java クラス。@Entity を付けたクラスを置く。

リクエストが届いてから DB に保存されるまでの流れ

flowchart LR A["ブラウザ"] -->|リクエスト| B["Controller
（入出力）"] B --> C["Service
（業務処理）"] C --> D["Repository
（DB 操作）"] D --> E[("H2 Database")] E --> D D --> C C --> B B -->|レスポンス| A

重要ポイント: Controller から直接 Repository を呼ばない

初心者のうちは「Controller から Repository を直接呼べばいいのでは？」と思うかもしれません。しかし業務処理（入力チェック・複数の DB 操作など）を Controller に書いてしまうと、コードが肥大化して管理しにくくなります。Controller → Service → Repository の流れを守ることで、役割が明確になり、テストも書きやすくなります。

2.3 H2 Database — 学習に最適な組み込みデータベース

H2 Database は、Java で作られた軽量なリレーショナルデータベースです。最大の特徴は「組み込み（Embedded）モード」で動けること——つまり、MySQL や PostgreSQL のように別途インストールや起動が不要で、依存関係に追加するだけですぐ使えます。

インストール不要

pom.xml に依存を追加するだけ。環境構築の手間がかからない。

メモリ上で動作

アプリ起動中のみデータが存在し、停止するとリセットされる。毎回クリーンな状態で学習できる。

ブラウザで管理

H2 コンソール（/h2-console）でブラウザから SQL を実行してデータを確認できる。

H2 はあくまで学習用

H2 の「メモリモード」はアプリを停止するとデータが消えます。本番環境では MySQL・PostgreSQL などの永続化 DB を使います。ただし Spring Data JPA では設定を変えるだけで本番 DB に切り替えられるため、学習中は H2 で JPA の使い方を身につけることに集中しましょう。

2.4 application.properties の設定

Spring Boot の設定は src/main/resources/application.properties に書きます。本教材では H2 の接続設定・JPA の SQL 表示・テーブル自動生成・H2 コンソールの有効化という4種類の設定を行います。

src/main/resources/application.properties 
# H2 データベース接続設定
spring.datasource.url=jdbc:h2:mem:jpalearning
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# JPA / Hibernate 設定
spring.jpa.show-sql=true
spring.jpa.hibernate.ddl-auto=update

# H2 コンソールの有効化（ブラウザから DB を確認できる）
spring.h2.console.enabled=true

各設定が何をしているのかを、以下の表で確認しておきましょう。

設定キー 値の例 意味
spring.datasource.url jdbc:h2:mem:jpalearning mem: でメモリ上に DB を作る。jpalearning はその DB 名。
spring.datasource.driver-class-name org.h2.Driver H2 用のドライバクラス。Spring Boot が自動検出することも多い。
spring.datasource.username sa H2 のデフォルトユーザー名。パスワードは空で OK。
spring.jpa.show-sql true JPA が自動生成した SQL をコンソール（ログ）に表示する。学習時に実際の SQL を確認できる。
spring.jpa.hibernate.ddl-auto update Entity クラスの定義をもとにテーブルを自動生成・更新する。
spring.h2.console.enabled true ブラウザから H2 DB を操作できるコンソール画面を有効にする。

ddl-auto の選択肢を理解しよう

spring.jpa.hibernate.ddl-auto には複数の値があります。学習段階では以下の2つを覚えておけば十分です。

  • update — 起動のたびに Entity の定義と既存テーブルを比較し、差分があれば更新する。既存データは消えない。
  • create — 起動のたびにテーブルを作り直す（既存データは消える）。毎回クリーンな状態で試したいときに使う。

2.5 アプリの起動と H2 コンソールの確認

設定が完了したら、アプリを起動して H2 コンソールに接続できることを確認しましょう。この「起動確認」を最初に行っておくことで、後の章で Entity や Repository を追加したときに問題が起きた場合の切り分けがしやすくなります。

ターミナル（プロジェクトルートで実行） 
# Maven の場合（pom.xml が存在するディレクトリで実行）
./mvnw spring-boot:run

# または IDE の「Run」ボタンで JpaLearningApplication.java を実行しても OK

起動に成功すると、ターミナルに以下のようなログが出力されます。Started JpaLearningApplication と表示されれば OK です。 

  .   ____          _            __ _ _
 /\\ / ___'_ __ _ _(_)_ __  __ _ \ \ \ \
...
Started JpaLearningApplication in 2.345 seconds

起動後、ブラウザで http://localhost:8080/h2-console を開くと、H2 コンソールのログイン画面が表示されます。以下の値を入力して「Connect」をクリックしてください。

項目 値
JDBC URL jdbc:h2:mem:jpalearning
User Name sa
Password （空のまま）

起動確認チェックリスト

  • ターミナルに Started JpaLearningApplication が表示された
  • http://localhost:8080/h2-console をブラウザで開けた
  • JDBC URL・ユーザー名を入力して「Connect」を押し、H2 コンソールの管理画面が表示された

よくあるトラブルと対処

  • H2 コンソールが開かない → spring.h2.console.enabled=true が設定されているか確認。
  • 「Connection refused」エラー → アプリが起動できていない可能性。ターミナルのエラーログを確認。
  • ログイン失敗 → JDBC URL が application.properties の datasource.url と一致しているか確認。

やってみよう 2-1: プロジェクトを作成して起動する

実際に Spring Initializr でプロジェクトを生成し、設定して起動してみましょう。H2 コンソールに接続できればこの章のゴール達成です。

手順

  1. 1 start.spring.io で Maven / Java / Spring Boot 3.x を選択し、依存関係に Spring Web・Spring Data JPA・H2 Database・Thymeleaf を追加して ZIP を生成する。
  2. 2 ZIP を展開し、IDE またはテキストエディタで開く。src/main/java 配下に controller・service・repository・entity パッケージを作成する。
  3. 3 src/main/resources/application.properties に 2.4 の設定を追記する。
  4. 4 ./mvnw spring-boot:run（または IDE の Run）でアプリを起動し、ブラウザで http://localhost:8080/h2-console を開いてログインできることを確認する。

確認ポイント

H2 コンソールにログインできた後、左のツリーに テーブルが何も表示されない のは正常です。Entity クラスをまだ作っていないので、テーブルは存在しません。第3章で Entity を作成すると、テーブルが自動生成されます。

理解度確認クイズ

  1. Q1. H2 Database は何が「便利」で学習に向いていますか？ 2点挙げてください。

    回答を表示

    ①インストール不要——依存関係に追加するだけですぐ使える。②メモリ上で動作——起動するたびにクリーンな状態になるため、学習中に気軽に試せる。（H2 コンソールでブラウザから確認できる点も加点対象）

  2. Q2. spring.jpa.show-sql=true を設定する目的は何ですか？

    回答を表示

    JPA（Hibernate）が自動生成した SQL をコンソール（ログ）に出力するためです。「Repository のメソッドを呼んだとき、裏でどんな SQL が実行されているか」を目で確認できるようになり、学習の理解が深まります。

  3. Q3. spring.jpa.hibernate.ddl-auto=update と create の違いは何ですか？

    回答を表示

    update は Entity の定義と既存テーブルを比較して差分を更新します。既存のデータは保持されます。create は起動のたびにテーブルを作り直します（既存データは消えます）。H2 のメモリモードではどちらも起動ごとにリセットされますが、学習段階では update を使っておくと、将来ファイルモード H2 や MySQL に切り替えたときに安全です。
