総合演習：部署・社員管理アプリ

ここまで学んだ Entity・Repository・Service・関連（@ManyToOne）・Spring MVC 連携をすべて組み合わせ、関連テーブルを扱う部署・社員管理 Web アプリケーションを完成させます。部署マスタを使って社員登録画面で部署を選択し、社員一覧に所属部署名を表示する——実務に近い機能を一から作り上げましょう。

この章で学ぶこと

  • これまで学んだ全要素（Entity / Repository / Service / Controller / Thymeleaf）を1つのアプリとして統合できる
  • 部署一覧・社員一覧・社員登録・社員編集・社員削除・社員検索の6機能を実装できる
  • @ManyToOne による関連を使い、社員一覧に所属部署名を表示できる
  • 全体のクラス構成（entity / repository / service / controller）を俯瞰して説明できる

11.1 作成する機能の全体像

この総合演習で完成させるアプリケーションは、部署マスタと社員情報を管理する Web アプリです。登録された部署を選択しながら社員を登録・編集でき、社員一覧には所属部署名が表示されます。まず作成する機能の一覧を確認しましょう。

機能 説明 URL
部署一覧 登録済み部署を表示する GET /departments
社員一覧 社員名・メール・所属部署を表示する GET /employees
社員登録 社員情報と所属部署を登録する GET /employees/new（フォーム）

11.2 全体のクラス構成を俯瞰する

アプリを実装する前に、全体のクラス構成を改めて確認しておきましょう。第3〜10章にわたって1つずつ作ってきた部品が、どのように連携して動くかを図で俯瞰します。

構成のポイントは3つあります。1つ目は Employee エンティティが Department への参照（@ManyToOne）を持つこと。2つ目は EmployeeService が DepartmentRepository を直接呼ばず、必要な場合は DepartmentService 経由で部署データを取得する（または Controller が両方の Service を呼ぶ）こと。3つ目は Controller が Thymeleaf テンプレートに Model を渡し、ブラウザへ HTML を返すことです。

Department（id, name）
Employee（id, name, email + @ManyToOne Department）

Repository 層

DepartmentRepository / EmployeeRepository（JpaRepository を継承・クエリメソッド）

Service 層

DepartmentService / EmployeeService（@Transactional・業務ロジックを集約）

Controller 層

DepartmentController / EmployeeController（@Controller・Thymeleaf へ Model を渡す）

11.3 部署一覧・社員一覧の実装

まず読み取り系（一覧表示）から実装します。DepartmentController は全部署を取得して表示するだけのシンプルな構成です。EmployeeController の一覧では、employee.getDepartment().getName() のようにして所属部署名を画面に表示します。

DepartmentController.java 
package com.example.jpalearning.controller;

@Controller
@RequestMapping("/departments")
public class DepartmentController {

    private final DepartmentService departmentService;

    // コンストラクタインジェクション
    public DepartmentController(DepartmentService departmentService) {
        this.departmentService = departmentService;
    }

    // 部署一覧を表示する
    @GetMapping
    public String list(Model model) {
        model.addAttribute("departments", departmentService.findAll());
        return "departments/list"; // templates/departments/list.html
    }
}
社員一覧メソッド（EmployeeController.java 抜粋） 
// 社員一覧 + 検索（キーワードがある場合は部分一致検索）
@GetMapping
public String list(@RequestParam(required = false) String keyword,
                   Model model) {
    List<Employee> employees = (keyword != null && !keyword.isBlank())
        ? employeeService.searchByName(keyword)   // 部分一致検索
        : employeeService.findAll();              // 全件取得
    model.addAttribute("employees", employees);
    model.addAttribute("keyword", keyword);
    return "employees/list";   // templates/employees/list.html
}

Thymeleaf で所属部署名を表示する

社員一覧テンプレート（list.html）では th:text="${emp.department.name}" のように書くと、JPA が @ManyToOne の関連をたどって部署名を取得してくれます。

templates/employees/list.html（テーブル行部分の抜粋） 
<!-- th:each で全社員を繰り返し表示 -->
<tr th:each="emp : ${employees}">
    <td th:text="${emp.id}"></td>
    <td th:text="${emp.name}"></td>
    <td th:text="${emp.email}"></td>
    <!-- @ManyToOne 関連をたどって部署名を表示 -->
    <td th:text="${emp.department.name}"></td>
    <td>
        <a th:href="@{/employees/{id}/edit(id=${emp.id})}">編集</a>
        <form th:action="@{/employees/{id}/delete(id=${emp.id})}"
              method="post">
            <button type="submit">削除</button>
        </form>
    </td>
</tr>

11.4 社員登録・編集の実装

社員の登録・編集フォームには「所属部署」を選ぶドロップダウンが必要です。フォームを表示する GET メソッドで、全部署リストを Model に渡します。Thymeleaf 側では th:each でドロップダウンを生成し、選択した部署の ID を送信します。

社員登録メソッド（EmployeeController.java 抜粋） 
// 登録フォームの表示（全部署リストをドロップダウン用に渡す）
@GetMapping("/new")
public String newForm(Model model) {
    model.addAttribute("employee", new Employee());
    model.addAttribute("departments", departmentService.findAll());
    return "employees/form";  // templates/employees/form.html
}

// 登録フォームの送信（POST /employees。第10章とURLをそろえる）
// ※ クラスに @RequestMapping("/employees") があるため、パスなしの @PostMapping は POST /employees になる
@PostMapping
public String create(@RequestParam Long departmentId,
                     @ModelAttribute Employee employee) {
    Department dept = departmentService.findById(departmentId);
    employee.setDepartment(dept);   // @ManyToOne の関連を設定
    employeeService.save(employee);
    return "redirect:/employees";
}

重要ポイント: 部署の紐づけ方

フォームから送信されるのは 部署 ID（数値）です。Controller で departmentService.findById(departmentId) を呼んで Department オブジェクトを取得し、employee.setDepartment(dept) でセットしてから保存します。こうすると JPA が外部キー（department_id）を自動で設定してくれます。

EmployeeService.java（代表メソッド抜粋） 
@Service
@Transactional  // クラス全体にトランザクションを適用
public class EmployeeService {

    private final EmployeeRepository employeeRepository;

    public EmployeeService(EmployeeRepository employeeRepository) {
        this.employeeRepository = employeeRepository;
    }

    public List<Employee> findAll() {
        return employeeRepository.findAll();
    }

    // 社員名の部分一致検索（第6章クエリメソッド）
    public List<Employee> searchByName(String keyword) {
        return employeeRepository.findByNameContaining(keyword);
    }

    public void save(Employee employee) {
        employeeRepository.save(employee);   // 新規・更新を自動判定
    }

    public void delete(Long id) {
        employeeRepository.deleteById(id);
    }
}

11.5 社員検索の実装

社員名の部分一致検索は第6章のクエリメソッドで実装済みです。ここでは検索フォームを一覧画面に組み込む方法を確認します。11.3 の list メソッドで keyword パラメータを受け取り、空の場合は全件表示、入力がある場合は部分一致検索の結果を表示する、という1つのメソッドで兼用するシンプルな設計です。

templates/employees/list.html（検索フォーム部分） 
<!-- 社員名で検索するフォーム（GET /employees?keyword=... を送信） -->
<form th:action="@{/employees}" method="get">
    <input type="text" name="keyword"
           th:value="${keyword}"
           placeholder="社員名を入力"/>
    <button type="submit">検索</button>
    <!-- キーワードがある場合だけ「クリア」を表示 -->
    <a th:if="${keyword != null and keyword != ''}"
       th:href="@{/employees}">クリア</a>
</form>

よくあるミス: キーワードが null になる

@RequestParam String keyword とすると、検索フォームを使わずに GET /employees を開いたときに 400 Bad Request になります。@RequestParam(required = false) にして 省略を許可するのを忘れずに。

やってみよう 11-1: 発展課題

基本機能が完成したら、以下の発展課題に挑戦してみましょう。いずれも実務でよく求められる機能で、これまで学んだ知識を組み合わせて実装できます。

  1. 1

    部署ごとの社員数を表示する

    部署一覧画面に「在籍社員数」列を追加しましょう。@Query で SELECT d, COUNT(e) FROM Department d LEFT JOIN Employee e ON e.department = d GROUP BY d のような JPQL を試してみてください。

  2. 2

    部署でも絞り込み検索できるようにする

    現在の「社員名での部分一致検索」に加え、ドロップダウンで「部署を選択して絞り込む」機能を追加しましょう。findByDepartment または findByDepartmentId クエリメソッドを活用できます。

  3. 3

    登録・編集フォームに入力バリデーションを追加する

    社員名・メールアドレスの未入力チェックを @NotBlank・@Email アノテーション（Bean Validation）で実装し、エラーメッセージを Thymeleaf で表示してみましょう。

  4. 4

    部署の追加・削除もできるようにする

    部署一覧画面に「部署を追加」フォームと「削除」ボタンを追加しましょう。ただし社員が在籍している部署を削除しようとするとどうなるかを確認し、削除前チェック（在籍社員数 == 0 のみ削除可）を実装してみてください。

理解度確認クイズ

  1. Q1. 社員登録フォームで部署を選択したとき、Controller はどのような処理で Employee と Department を紐づけますか？

    回答を表示

    フォームから送信された 部署 ID（数値）を @RequestParam Long departmentId で受け取り、departmentService.findById(departmentId) で Department オブジェクトを取得します。そのオブジェクトを employee.setDepartment(dept) でセットしてから employeeService.save(employee) を呼ぶと、JPA が外部キー（department_id）を自動で書き込みます。

  2. Q2. 社員一覧の Thymeleaf テンプレートで所属部署名を表示するには、どのような式を書きますか？

    回答を表示

    th:text="${emp.department.name}" と書きます。emp は Employee オブジェクト、.department で @ManyToOne の関連をたどり、.name で部署名を取得します。JPA が関連先のデータを自動的に取得してくれるため、SQL の JOIN を自分で書く必要はありません。

  3. Q3. 検索フォームがない状態で GET /employees にアクセスすると 400 エラーになります。なぜですか？また、どう直しますか？

    回答を表示

    @RequestParam String keyword はデフォルトで 必須のため、クエリパラメータ keyword がないリクエストに対して Spring が 400 Bad Request を返します。@RequestParam(required = false) String keyword に変更してパラメータを省略可能にし、keyword == null または空文字の場合は全件取得、そうでない場合は部分一致検索と分岐させることで解消できます。

① entity パッケージ

entity/Department.java 
package com.example.jpalearning.entity;

import jakarta.persistence.*;

@Entity
@Table(name = "departments")
public class Department {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    public Department() {}   // JPA 必須のデフォルトコンストラクタ

    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
entity/Employee.java 
package com.example.jpalearning.entity;

import jakarta.persistence.*;

@Entity
@Table(name = "employees")
public class Employee {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    @Column(nullable = false)
    private String email;

    // 多対1: 多数の社員が1つの部署に所属する（Employee → Department）
    @ManyToOne
    @JoinColumn(name = "department_id")
    private Department department;

    public Employee() {}   // JPA 必須のデフォルトコンストラクタ

    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
    public Department getDepartment() { return department; }
    public void setDepartment(Department department) { this.department = department; }
}

package com.example.jpalearning.service;

import com.example.jpalearning.entity.Department;
import com.example.jpalearning.repository.DepartmentRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

    public DepartmentService(DepartmentRepository departmentRepository) {
        this.departmentRepository = departmentRepository;
    }

    public List<Department> findAll() {
        return departmentRepository.findAll();
    }

service/EmployeeService.java 
package com.example.jpalearning.service;

import com.example.jpalearning.entity.Employee;
import com.example.jpalearning.repository.EmployeeRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;

@Service
@Transactional
public class EmployeeService {

    private final EmployeeRepository employeeRepository;

    public EmployeeService(EmployeeRepository employeeRepository) {
        this.employeeRepository = employeeRepository;
    }

    public List<Employee> findAll() {
        return employeeRepository.findAll();
    }

    public Employee findById(Long id) {
        return employeeRepository.findById(id).orElseThrow();
    }

    // 名前の部分一致検索
    public List<Employee> searchByName(String keyword) {
        return employeeRepository.findByNameContaining(keyword);

    // id なし→INSERT、id あり→UPDATE（save が自動判定）

        employeeRepository.save(employee);

    }

controller/EmployeeController.java 
package com.example.jpalearning.controller;

import com.example.jpalearning.entity.Department;
import com.example.jpalearning.entity.Employee;
import com.example.jpalearning.service.DepartmentService;
import com.example.jpalearning.service.EmployeeService;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@Controller
@RequestMapping("/employees")
public class EmployeeController {

    private final EmployeeService employeeService;
    private final DepartmentService departmentService;

    public EmployeeController(EmployeeService employeeService,
                              DepartmentService departmentService) {
        this.employeeService = employeeService;
        this.departmentService = departmentService;
    }

    // 一覧＋検索（GET /employees。keyword があれば部分一致検索）
    @GetMapping
    public String list(@RequestParam(required = false) String keyword, Model model) {
        List<Employee> employees = (keyword != null && !keyword.isBlank())
            ? employeeService.searchByName(keyword)
            : employeeService.findAll();
        model.addAttribute("employees", employees);
        model.addAttribute("keyword", keyword);
        return "employees/list";
    }

    // 登録フォーム表示（GET /employees/new）
    @GetMapping("/new")
    public String newForm(Model model) {
        model.addAttribute("employee", new Employee());
        model.addAttribute("departments", departmentService.findAll());
        return "employees/form";
    }

    // 登録処理（POST /employees）
    @PostMapping
    public String create(@RequestParam Long departmentId,
                         @ModelAttribute Employee employee) {
        employee.setDepartment(departmentService.findById(departmentId));
        employeeService.save(employee);
        return "redirect:/employees";
    }

    // 編集フォーム表示（GET /employees/{id}/edit）
    @GetMapping("/{id}/edit")
    public String editForm(@PathVariable Long id, Model model) {
        model.addAttribute("employee", employeeService.findById(id));
        model.addAttribute("departments", departmentService.findAll());
        return "employees/form";   // 登録と同じフォームを再利用
    }

    // 編集処理（POST /employees/{id}/edit）
    @PostMapping("/{id}/edit")
    public String update(@PathVariable Long id,
                         @RequestParam Long departmentId,
                         @ModelAttribute Employee employee) {
        employee.setId(id);        // パスの id を設定して UPDATE 対象にする
        employee.setDepartment(departmentService.findById(departmentId));
        employeeService.save(employee);
        return "redirect:/employees";
    }

    // 削除処理（POST /employees/{id}/delete）
    @PostMapping("/{id}/delete")
    public String delete(@PathVariable Long id) {
        employeeService.delete(id);
        return "redirect:/employees";
    }
}

まとめ — Spring Data JPA 学習の振り返り

全11章を通じて、Spring Data JPA の基礎から実践的な Web アプリ開発まで学びました。習得した知識を振り返り、次のステップへの足がかりにしましょう。

この教材で習得したこと

  • @Entity / @Id / @GeneratedValue / @Column — Java クラスとデータベーステーブルを対応づけるアノテーション
  • JpaRepository によるCRUD — インターフェースを継承するだけで findAll / findById / save / deleteById が使える
  • クエリメソッド（条件検索） — メソッド名の命名規則で検索 SQL を自動生成（findByName / findByNameContaining 等）
  • @Query / JPQL — 命名規則では書けない複雑な検索を JPQL（エンティティ対象の問い合わせ言語）で記述
  • @ManyToOne / @JoinColumn（関連） — 多対1の関連テーブルを Java オブジェクトの参照として扱う
  • @Transactional — 複数の DB 操作をまとめてコミット・ロールバックできるトランザクション管理
  • Spring MVC 連携 — Controller → Service → Repository → DB → Thymeleaf の一連のフローを実装
