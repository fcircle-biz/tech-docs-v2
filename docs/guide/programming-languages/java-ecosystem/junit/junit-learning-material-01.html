第1章 所要時間: 約40分

テストって何だろう？ ソフトウェアテストの世界へようこそ

プログラムを「テストする」とはどういうことかを学びます。なぜテストが必要なのか、手動テストと自動テストはどう違うのか、そして JUnit とは何なのか。むずかしいコードはまだ書きません。料理のたとえを使いながら、テストの考え方をやさしくつかんでいきましょう。

この章で学ぶこと

  • プログラムを「テストする」とはどういうことかを、身近な例で説明できる
  • なぜテストが必要なのか、その理由を3つ挙げられる
  • 手動テストと自動テストの違いを説明できる
  • JUnit とは何か、どんな役割を持つ道具なのかを理解する

1.1 「テストする」ってどういうこと？

ソフトウェアの世界でテスト（test）とは、ひとことで言うと「プログラムが思ったとおりに正しく動くかどうかを確かめる作業」のことです。作ったプログラムが、期待した結果を返してくれるか。おかしな動きをしないか。それを一つひとつ確認していきます。

いきなりプログラムの話だとイメージしづらいので、料理でたとえてみましょう。あなたはレシピを見ながらカレーを作っています。完成したあと、お皿に盛る前に、こんな確認をするはずです。

たとえ話: カレーができたか確かめる

  • ・ひと口味見して、味がちょうどよいか確かめる
  • ・じゃがいもに火が通っているか、はしで刺して確かめる
  • ・分量どおりに材料を入れたか、レシピと見くらべる

この「できあがりが、思ったとおりかを確かめる」行為こそ、まさにテストです。料理人が味見をするように、プログラマーはプログラムをテストします。

テストの基本は、とてもシンプルです。「ある入力を与えたら、こういう結果になるはず」という予想（期待）を立て、実際にプログラムを動かして、予想どおりの結果になったかを見くらべる。たったこれだけです。

1. 入力を与える

プログラムに「2 と 3 を足して」のようにデータを渡します。料理なら材料を入れる工程です。

2. 実行して結果を得る

プログラムを動かし、実際の結果（たとえば 5）を受け取ります。料理なら煮込む工程です。

3. 期待と見くらべる

「結果は 5 になるはず」という期待と、実際の結果を見くらべます。これが味見にあたります。

この流れを図にすると、次のようになります。

flowchart LR A["入力を与える
（例: 2 と 3）"] --> B["プログラムを実行"] B --> C["実際の結果
（例: 5）"] C --> D{"期待した結果
と同じ？"} D -->|同じ| E["テスト成功"] D -->|ちがう| F["テスト失敗
＝バグ発見"]

1.2 なぜテストが必要なの？

「自分で書いたプログラムだから、ちゃんと動くはず」——そう思いたくなりますが、人間は必ずミスをします。プログラムの世界では、このミスや不具合のことをバグ（bug）と呼びます。テストは、このバグを見つけ出すための大切な仕組みです。テストが必要な理由は、大きく3つあります。

理由1: バグを早く見つけられる

バグは、見つかるのが遅いほど直すのが大変になります。料理で言えば、味見をせずにお客さんに出してしまってから「味が濃すぎた」と気づくようなもの。早めにテストして、こっそり直しておくのが一番です。

理由2: 安心して変更できる

プログラムは、あとから機能を追加したり直したりします。そのとき「別の場所を壊していないか」が心配になります。テストがあれば、変更後にもう一度テストを動かすだけで「ちゃんと動いている」と確認でき、安心して手を入れられます。

理由3: 仕様の説明書になる

よく書かれたテストは「このプログラムは、こう入力したら、こう動く」という具体例の集まりです。あとから読む人にとって、プログラムの使い方を示す生きた説明書になります。

バグは「見つかるのが遅いほど高くつく」

作っている最中に見つかったバグは、その場でサッと直せます。でも、利用者の手に渡ってから見つかったバグは、原因を調べ、修正し、再配布し……と手間も影響も大きくなります。だからこそ、できるだけ早い段階でテストする習慣が大切なのです。

1.3 手動テストと自動テスト

テストには、大きく分けて手動テストと自動テストの2つのやり方があります。

手動テストは、人が自分の手と目で確認する方法です。たとえば、画面のボタンを実際に押して、表示が正しいかを目で見てチェックします。一方の自動テストは、「確認のためのプログラム」をあらかじめ書いておき、コンピューターに代わりにチェックしてもらう方法です。本ガイドで学ぶ JUnit は、この自動テストを書くための道具です。

観点 手動テスト 自動テスト（JUnit など）
だれが確認する 人（自分の手と目） コンピューター
繰り返しやすさ 毎回手間がかかる。回数が増えると大変 何度でもボタン一つで実行できる
速さ ゆっくり（人の操作スピード） とても速い（一瞬で何百件も）
ミスの起きにくさ 見落としや勘違いが起きやすい いつも同じ基準で正確に判定する
向いている場面 見た目や使い心地の確認 計算やルールが正しいかの確認（何度も繰り返す確認）

手動テストが悪いわけではありません。見た目や操作感の確認など、人の目が向いている場面もあります。しかし、「計算が合っているか」「ルールどおりに動くか」といった何度も繰り返す確認は、自動テストの得意分野です。一度書いておけば、いつでも何度でも、正確に確認してくれます。

用語: 「単体テスト（ユニットテスト）」

プログラムを構成する小さな部品（メソッドやクラス）が、それぞれ正しく動くかを確かめるテストを単体テスト（またはユニットテスト / unit test）と呼びます。JUnit はこの単体テストを書くための代表的な道具で、名前の「Unit（ユニット）」もここから来ています。本ガイドでは、この単体テストを中心に学んでいきます。

1.4 JUnit とは何か

JUnit（ジェイユニット）は、Java のプログラムを自動テストするための道具（フレームワーク）です。Java の世界では「テストといえば JUnit」と言われるほど広く使われている、定番中の定番です。現在の最新シリーズは JUnit 5 で、本ガイドでもこの JUnit 5 を使って学んでいきます。

JUnit を使うと、先ほどの「入力を与えて、結果が期待どおりかを見くらべる」というテストを、Java のコードとして書いておけるようになります。たとえば、足し算をする add という機能をテストするコードは、次のようなイメージです。今はまだ意味が分からなくて大丈夫。「テストってこんな見た目なんだ」という雰囲気だけ感じてください。

CalculatorTest.java（イメージ） 
@Test  // これがテストですよ、という目印
void 足し算が正しく計算できる() {
    Calculator calc = new Calculator();
    int result = calc.add(2, 3);   // 2 + 3 を計算してみる
    assertEquals(5, result);       // 結果が 5 であることを確かめる
}

最後の行の assertEquals(5, result) が、まさに「味見」の部分です。「結果（result）は 5 と等しいはずだ」と確かめています。もし計算が間違って結果が 6 になっていたら、JUnit は「期待は 5 なのに、実際は 6 でした」と教えてくれて、テストは失敗になります。

重要ポイント

JUnit は「テストを書くための道具」であり、テストを実行して成功・失敗を自動で判定してくれます。あなたがやることは、「どんな入力で、どんな結果になるはずか」をコードで書いてあげること。判定はすべて JUnit にお任せできます。

次の第2章では、いよいよこの JUnit を実際に使える環境を準備し、あなたの手で最初のテストを書いて動かしてみます。その前に、ここまでの内容を簡単な練習とクイズで整理しておきましょう。

やってみよう 1-1: 身の回りの「確認作業」を見つけよう

テストの考え方は、プログラミングに限りません。まずは「入力 → 結果 → 期待と見くらべる」という形が、日常のどこに隠れているかを探してみましょう。手を動かす前に考え方に慣れることが上達の近道です。

手順

  1. 1 あなたの日常にある「できあがりを確かめる作業」を3つ書き出す（例: 料理の味見、持ち物チェック、計算の見直し）。
  2. 2 そのうちの1つについて、「どんな入力に対して」「どんな結果を期待していて」「どう見くらべているか」を言葉にしてみる。
  3. 3 それが本章で学んだテストの3ステップ（入力 → 実行 → 期待と見くらべる）と同じ形になっていることを確認する。

理解度確認クイズ

  1. Q1. ソフトウェアの「テスト」とは、ひとことで言うと何をする作業ですか？

    回答を表示

    プログラムが思ったとおりに正しく動くかどうかを確かめる作業です。具体的には「ある入力を与えたらこういう結果になるはず」という期待を立て、実際の結果と見くらべます。

  2. Q2. 自動テストが手動テストよりも得意なことを、2つ挙げてください。

    回答を表示

    「何度でも繰り返し実行できる」「とても速く、いつも同じ基準で正確に判定できる」ことです。そのため、計算やルールが正しいかを何度も確認する場面に向いています。

  3. Q3. JUnit とは、何のための道具ですか？

    回答を表示

    Java のプログラムを自動テストするための道具（フレームワーク）です。テストをコードとして書いておくと、JUnit がそれを実行し、成功・失敗を自動で判定してくれます。最新シリーズは JUnit 5 です。
