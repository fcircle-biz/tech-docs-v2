はじめてのJUnitテスト 環境構築と最初の一歩

JUnit を使うための環境を整え、人生で初めてのテストコードを書いて実行します。まるで料理を始める前にキッチンと道具をそろえるように、JDK・IDE・Maven を準備し、Maven プロジェクトを作成して JUnit 5 を導入します。最後には「グリーン（成功）」の画面を自分の手で見てみましょう。

この章で学ぶこと

  • JUnit を使うために必要な環境（JDK・IDE・Maven）を整えられる
  • Maven プロジェクトのディレクトリ構成（src/main/java と src/test/java）の意味を説明できる
  • pom.xml に JUnit 5 の依存関係を追加し、最初のテストクラスを書ける
  • IDE またはコマンドラインからテストを実行し、グリーン（成功）とレッド（失敗）の違いを確認できる

2.1 必要な道具をそろえよう（前提環境の確認）

料理を始める前に、キッチンの設備と道具がそろっているかを確認しますよね。JUnit を使うのも同じです。コードを書いて実行するために、まず3つの道具が必要です。

たとえ話: キッチンの3点セット

  • ・コンロ（JDK） ── Java のプログラムを動かす「エンジン」
  • ・調理台（IDE） ── コードを書いて整理する「作業スペース」
  • ・レシピ管理ノート（Maven） ── ライブラリを管理して手順通りにビルドする「段取り担当」

この3つがそろって初めて、JUnit のテストを書いて動かせるようになります。

JDK 17 以降

Java の開発キット。プログラムのコンパイルと実行に必須。java -version で確認できます。

IDE

IntelliJ IDEA（推奨）、Eclipse、VS Code のいずれか。テストの実行ボタン（緑の三角）が使えます。

Maven

Java のビルドツール。JUnit などのライブラリを自動でダウンロードして管理します。mvn -version で確認できます。

インストール確認コマンド（ターミナル／コマンドプロンプトで実行）

ターミナル 
java -version   # JDK のバージョン確認（17 以降が必要）
mvn -version    # Maven のバージョン確認

コマンドが見つからない場合は、JDK や Maven のインストールと PATH 設定が必要です。各公式サイトのインストール手順を参照してください。

2.2 Maven プロジェクトを作成する

環境がそろったら、Maven プロジェクトを作成します。Maven プロジェクトには決まったディレクトリ構成があり、これをしっかり理解することが最初の大事なステップです。

Maven プロジェクトでは「本番コード」と「テストコード」を別々のフォルダに分けて管理します。料理でたとえると、食材（本番コード）と確認用チェックリスト（テストコード）を別の引き出しに整理しておくイメージです。

Maven プロジェクトのディレクトリ構成

my-junit-project/          ← プロジェクトルート
├── pom.xml                ← ライブラリや設定を書く「レシピ帳」
└── src/
    ├── main/
    │   └── java/          ← 本番コードを置く場所
    │       └── com/example/
    │           └── Greeting.java    ← テスト対象のクラス
    └── test/
        └── java/          ← テストコードを置く場所
            └── com/example/
                └── GreetingTest.java ← テストクラス

2つのフォルダの役割を図で確認しましょう。

flowchart LR subgraph main["src/main/java（本番コード）"] A["Greeting.java
実際に動くプログラム"] end subgraph test["src/test/java（テストコード）"] B["GreetingTest.java
テスト専用コード"] end B -->|テスト対象として呼び出す| A A -->|結果を返す| B

重要ポイント: テストコードは本番に含まれない

src/test/java に置いたコードは、アプリとしてリリースする際には含まれません。Maven が「テスト専用」として扱うため、本番環境を汚さずに安全にテストを書けます。

IDE（IntelliJ IDEA）で新規 Maven プロジェクトを作成する場合は、メニューから File → New → Project を選び、「Maven Archetype」または「Java with Maven」を選択するだけで、このフォルダ構成が自動で作られます。

2.3 pom.xml に JUnit 5 を追加する

Maven プロジェクトには pom.xml（Project Object Model）という設定ファイルがあります。ここに「どのライブラリを使うか」を書いておくと、Maven が自動でダウンロードしてくれます。JUnit 5 を使うには、このファイルに設定を追加します。

pom.xml（JUnit 5 の依存関係を追加） 
<project>
  <!-- ... 他の設定 ... -->

  <dependencies>
    <!-- JUnit 5 本体: テストを書くためのライブラリ -->
    <dependency>
      <groupId>org.junit.jupiter</groupId>
      <artifactId>junit-jupiter</artifactId>
      <version>5.10.2</version>
      <scope>test</scope>  <!-- テスト専用: 本番には含まれない -->
    </dependency>
  </dependencies>

  <build>
    <plugins>
      <!-- Maven でテストを実行するプラグイン -->
      <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-surefire-plugin</artifactId>
        <version>3.2.5</version>
      </plugin>
    </plugins>
  </build>
</project>

2つのポイント

  • scope: test ── このライブラリはテスト時だけ使う、という宣言です。本番のリリースには含まれません。
  • maven-surefire-plugin ── コマンドライン（mvn test）からテストを実行するための仕組みです。JUnit 5 を正しく認識させるためにバージョン 3.x 以上を指定します。

pom.xml を保存したら、IDE が自動でライブラリをダウンロードします（IntelliJ IDEA なら右上に「Maven ツール」のインポートボタンが出ます）。これで JUnit 5 が使える状態になりました。

2.4 最初のテストを書いて実行しよう

準備が整いました。いよいよテスト対象のクラスと、テストクラスを書いていきます。今回はあいさつ文を返す Greeting クラスを例に使います。

ステップ1: テスト対象クラスを用意する

まず src/main/java/com/example/Greeting.java を作ります。このクラスが「テストされる側」です。

src/main/java/com/example/Greeting.java 
package com.example;

// あいさつ文を生成するクラス（テストされる側）
public class Greeting {

    // 名前を受け取って「Hello, 名前!」と返すメソッド
    public String greet(String name) {
        return "Hello, " + name + "!";
    }
}

ステップ2: テストクラスを書く

次に src/test/java/com/example/GreetingTest.java を作ります。テストクラスの名前は、対象クラス名の末尾に Test を付けるのが一般的な慣習です。

src/test/java/com/example/GreetingTest.java 
package com.example;

// JUnit 5 のアノテーションとアサーションをインポート
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

class GreetingTest {

    @Test  // このメソッドはテストですよ、という目印
    void 名前を渡すとHello名前が返る() {
        Greeting greeting = new Greeting(); // テスト対象を生成
        String result = greeting.greet("World"); // 実行
        assertEquals("Hello, World!", result); // 期待と見くらべる
    }
}

コードの3大ポイント

  • @Test ── このアノテーション（目印）を付けると、JUnit がそのメソッドをテストとして認識して実行します。
  • assertEquals（期待値, 実際の値） ── 「期待した値」と「実際に返ってきた値」が等しいかを確認します。違えばテスト失敗です。
  • import org.junit.jupiter.api.* ── JUnit 5 のクラスはこのパッケージに入っています。必ずインポートしましょう。

ステップ3: テストを実行する

テストを実行する方法は2つあります。どちらで実行しても、同じ結果が得られます。

方法A: IDE の実行ボタン

  1. 1. GreetingTest.java を開く
  2. 2. クラス名かメソッド名の横にある 緑の三角（▶） をクリック
  3. 3. 下部に「Tests passed」と表示されれば成功

方法B: コマンドライン

ターミナル 
# プロジェクトルートで実行
mvn test

動作確認: グリーンとレッドの見え方

成功のとき（グリーン）

IDE のテスト結果パネルに緑色のチェックが表示されます。

Tests run: 1, Failures: 0,
Errors: 0, Skipped: 0
BUILD SUCCESS

失敗のとき（レッド）

期待値と実際の値がどう違うかを教えてくれます。

expected: <Hello, World!>
 but was: <Hello, World>
BUILD FAILURE

テストが失敗したときは「何を期待していたか」と「実際に何が返ってきたか」が表示されるので、どこを直せばいいかがすぐわかります。

よくあるミスに注意

  • ・テストクラスを src/main/java に置いてしまう → src/test/java が正しい場所です
  • ・@Test を書き忘れる → JUnit がメソッドをテストとして認識しないため実行されません
  • ・import を忘れる → import org.junit.jupiter.api.Test; が必要です

やってみよう 2-1: テストを失敗させて、エラーメッセージを読んでみよう

テストが成功することだけでなく、「意図的に失敗させてメッセージを読む」練習も大切です。JUnit がどんな情報を教えてくれるか体験してみましょう。

手順

  1. 1 前のステップで作った GreetingTest.java の assertEquals の期待値を "Hello, World!" から "Hi, World!" に変える。
  2. 2 テストを実行して、赤いエラーと「expected: <Hi, World!> but was: <Hello, World!>」というメッセージを確認する。
  3. 3 期待値を元の "Hello, World!" に戻して再実行し、緑のグリーンを確認する。
  4. 4 余裕があれば、Greeting クラスに greet("")（空文字）を渡すテストも追加してみよう。何が返ってくるか予想してから実行する。

理解度確認クイズ

  1. Q1. Maven プロジェクトで、テストコードを置く正しいフォルダはどこですか？

    回答を表示

    src/test/java です。本番コードは src/main/java、テストコードは src/test/java と分けて管理します。テストコードはリリース時の成果物には含まれません。

  2. Q2. JUnit 5 のテストメソッドに付ける必須のアノテーションは何ですか？また、その役割を説明してください。

    回答を表示

    @Test です。このアノテーションを付けると、JUnit がそのメソッドをテストとして認識し、自動で実行します。付け忘れると、そのメソッドはテストとして実行されません。

  3. Q3. テストが失敗したとき、JUnit はどんな情報を表示しますか？それが役立つ理由を説明してください。

    回答を表示

    「期待した値（expected）」と「実際に返ってきた値（but was）」の両方を表示します。これにより、何が間違っているのかが一目でわかり、どのコードを直せばよいかをすぐに特定できます。
