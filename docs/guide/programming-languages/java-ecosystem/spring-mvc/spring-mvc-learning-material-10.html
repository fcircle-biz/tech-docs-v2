10.1 総合演習の概要

この章では、Spring MVC・Spring Data JDBC・Thymeleaf を使ってユーザー管理 Web アプリケーションを作成します。第1章から第9章まで少しずつ積み上げてきた知識を、ここで1つのアプリとして完成させます。

作るのは「一覧で見て、詳細を確認して、登録して、編集して、削除する」という CRUD（クラッド）操作の全機能です。CRUD とは Create（登録）・Read（参照）・Update（更新）・Delete（削除）の頭文字で、Web アプリケーションの基本的な操作パターンです。

10.2 URL 設計

Web アプリケーションを作るとき、最初に「どの URL でどの機能を提供するか」を決めます。これを URL 設計と呼びます。URL 設計を先に整理しておくと、Controller のマッピング（@GetMapping / @PostMapping）が作りやすくなります。

機能 HTTPメソッド URL 処理
一覧表示 GET /users ユーザー一覧を表示
詳細表示 GET /users/{id} ユーザー詳細を表示
登録フォーム GET /users/new 登録フォームを表示
登録処理 POST /users ユーザーを登録
編集フォーム GET /users/{id}/edit 編集フォームを表示
更新処理 POST /users/{id}/edit ユーザーを更新
削除処理 POST /users/{id}/delete ユーザーを削除

GETとPOSTの使い分け

GET は「画面を見るだけ」（データを変更しない）操作に使います。POST は「登録・更新・削除」などデータを変える操作に使います。削除も POST を使うのは、GET だとリンクをクリックするだけで削除が走ってしまう危険があるためです。

10.3 各機能の要件と画面の流れ

5つの機能ごとに、何を表示して何ができるのかを確認しましょう。画面の流れを頭に入れてから実装すると、どのファイルに何を書けばよいかが見えやすくなります。

一覧表示（GET /users）

  • /users にアクセスするとユーザー一覧を表示する
  • ID・名前・メールアドレスを一覧で表示する
  • 各行に詳細・編集・削除への導線を表示する
  • ページ上部に新規登録画面へのリンクを表示する

詳細表示（GET /users/{id}）

  • /users/{id} にアクセスするとそのユーザーの詳細を表示する
  • ID・名前・メールアドレスを表示する
  • 「一覧に戻る」リンクを表示する

新規登録（GET /users/new → POST /users）

  • /users/new で登録フォームを表示する
  • 名前・メールアドレスを入力する
  • 登録ボタンで /users に POST する
  • 登録後は一覧画面へリダイレクトする

編集（GET /users/{id}/edit → POST /users/{id}/edit）

  • /users/{id}/edit で編集フォームを表示する
  • 既存の名前・メールアドレスをフォームに表示した状態で開く
  • 更新ボタンで /users/{id}/edit に POST する
  • 更新後は一覧画面へリダイレクトする

削除（POST /users/{id}/delete）

  • 一覧画面の各行に削除ボタンを表示する
  • 削除ボタンで /users/{id}/delete に POST する
  • 削除後は一覧画面へリダイレクトする

10.4 UserController の全体像

UserController は、URL ごとのリクエストを受け取るクラスです。コンストラクタインジェクションで UserService を受け取り、各メソッドが機能ごとの処理を担います。メソッド単位で確認しましょう。

クラス宣言とコンストラクタ

UserController.java 
@Controller
@RequestMapping("/users")
public class UserController {

    // UserServiceをコンストラクタで受け取る（DI）
    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }
}

一覧表示

UserController.java — 一覧表示 
// GET /users：ユーザー一覧を表示する
@GetMapping
public String list(Model model) {
    // 全ユーザーを取得してModelに入れる
    model.addAttribute("users", userService.findAll());
    // list.htmlを表示する
    return "users/list";
}

詳細表示

UserController.java — 詳細表示 
// GET /users/{id}：指定IDのユーザー詳細を表示する
@GetMapping("/{id}")
public String detail(@PathVariable Long id, Model model) {
    // IDでユーザーを検索してModelに入れる
    model.addAttribute("user", userService.findById(id));
    // detail.htmlを表示する
    return "users/detail";
}

登録フォーム表示

UserController.java — 登録フォーム表示 
// GET /users/new：登録フォームを表示する
@GetMapping("/new")
public String newForm(Model model) {
    // 空のUserFormをModelに入れてThymeleafで使う
    model.addAttribute("userForm", new UserForm());
    // form.htmlを表示する
    return "users/form";
}

登録処理

UserController.java — 登録処理 
// POST /users：フォームの内容でユーザーを登録する
@PostMapping
public String create(@ModelAttribute UserForm form) {
    // Serviceに登録処理を依頼する
    userService.create(form.getName(), form.getEmail());
    // 登録後は一覧画面へリダイレクトする
    return "redirect:/users";
}

編集フォーム表示

UserController.java — 編集フォーム表示 
// GET /users/{id}/edit：編集フォームを表示する
@GetMapping("/{id}/edit")
public String editForm(@PathVariable Long id, Model model) {
    // 既存データを取得してフォームに入れる
    User user = userService.findById(id);
    UserForm form = new UserForm(user.getName(), user.getEmail());
    model.addAttribute("userForm", form);
    model.addAttribute("userId", id);
    return "users/edit";
}

更新処理

UserController.java — 更新処理 
// POST /users/{id}/edit：フォームの内容でユーザーを更新する
@PostMapping("/{id}/edit")
public String update(@PathVariable Long id,
                     @ModelAttribute UserForm form) {
    // Serviceに更新処理を依頼する
    userService.update(id, form.getName(), form.getEmail());
    // 更新後は一覧画面へリダイレクトする
    return "redirect:/users";
}

削除処理

UserController.java — 削除処理 
// POST /users/{id}/delete：指定IDのユーザーを削除する
@PostMapping("/{id}/delete")
public String delete(@PathVariable Long id) {
    // Serviceに削除処理を依頼する
    userService.delete(id);
    // 削除後は一覧画面へリダイレクトする
    return "redirect:/users";
}

アノテーションのまとめ

  • @PathVariable：URL の {id} 部分を引数に受け取る
  • @ModelAttribute：フォームの送信内容をオブジェクトにまとめて受け取る
  • redirect:/users：処理後にブラウザを /users へ転送する（二重送信防止）

10.5 View（Thymeleaf テンプレート）の要点

View は Thymeleaf テンプレートで作ります。Controller が Model に入れたデータを th:* 属性で HTML に表示します。各テンプレートの主要な部分を確認しましょう。

list.html — ユーザー一覧

templates/users/list.html（抜粋） 
<!-- th:eachでusersリストを繰り返し表示 -->
<tr th:each="user : ${users}">
    <td th:text="${user.id}"></td>
    <td th:text="${user.name}"></td>
    <td th:text="${user.email}"></td>
    <td>
        <a th:href="@{/users/{id}(id=${user.id})}">詳細</a>
        <a th:href="@{/users/{id}/edit(id=${user.id})}">編集</a>
        <form th:action="@{/users/{id}/delete(id=${user.id})}"
              method="post">
            <button type="submit">削除</button>
        </form>
    </td>
</tr>

form.html — 登録フォーム

templates/users/form.html（抜粋） 
<!-- th:object でフォームオブジェクトを指定 -->
<form th:action="@{/users}" method="post"
      th:object="${userForm}">
    <label>名前:
        <!-- *{name} で userForm.name にバインド -->
        <input type="text" th:field="*{name}">
    </label>
    <label>メール:
        <input type="email" th:field="*{email}">
    </label>
    <button type="submit">登録</button>
</form>

Thymeleaf 主要属性まとめ

  • th:text="${...}"：値をテキストとして表示する
  • th:each="変数 : ${リスト}"：リストを繰り返し表示する
  • th:href="@{/path}"・th:action="@{/path}"：URL を生成する
  • th:object="${form}"・th:field="*{name}"：フォームとオブジェクトを結びつける

総合演習 10-1: ユーザー管理 Web アプリケーションを完成させる

第1〜9章で作ってきたコードを統合して、一覧・詳細・登録・編集・削除の全機能を動作させましょう。

手順

  1. 1 データベースに usersテーブル（id, name, email）が作成されているか確認する。なければ schema.sql を確認して作成する。
  2. 2 User エンティティ・UserRepository・UserService・UserForm・UserController が揃っているか確認する。
  3. 3 templates/users/ フォルダに list.html・detail.html・form.html・edit.html の4ファイルを作成する。
  4. 4 アプリを起動して http://localhost:8080/users にアクセスし、一覧画面が表示されるか確認する。
  5. 5 「新規登録」でユーザーを1件追加し、一覧に表示されることを確認する。次に詳細表示・編集・削除の順に動作を確認する。

確認ポイント

  • 一覧に登録したユーザーが表示される
  • 詳細画面に ID・名前・メールが表示される
  • 編集フォームに既存の名前・メールが入った状態で開く
  • 削除後に一覧からそのユーザーが消える
  • 登録・更新・削除の後に必ず一覧へリダイレクトされる

10.6 本教材のまとめ

Spring MVC 入門の全10章を通じて学んだことを振り返りましょう。

Spring MVC 重要ポイント 10選

  1. 1 Spring MVC は Spring で Web アプリケーションを作るための仕組みである
  2. 2 Controller はブラウザからのリクエストを受け取る
  3. 3 Service は業務処理を担当する
  4. 4 Repository はデータベースアクセスを担当する
  5. 5 Model は画面に渡すデータを入れる「箱」である
  6. 6 View は HTML 画面を表示する。Thymeleaf を使うと Java 側のデータを HTML に埋め込める
  7. 7 GET は画面表示やデータ取得に使う
  8. 8 POST は登録・更新・削除などデータを変更する処理に使う
  9. 9 登録・更新・削除の後は redirect: で一覧画面へ転送して二重送信を防ぐ
  10. 10 Spring MVC と Spring Data JDBC を組み合わせることで、Web 画面からデータベースを操作できる

総合確認問題

  1. Q1. Spring MVC は何のために使いますか？

    回答を表示

    Spring で Web アプリケーションを作るための仕組みです。ブラウザからのリクエストを受け取り、Java の処理結果を HTML 画面として返します。

  2. Q2. Controller・Service・Repository・View はそれぞれ何を担当しますか？

    回答を表示

    Controller はブラウザからのリクエストを受け取り処理の流れを制御します。Service は業務処理（ビジネスロジック）を担当します。Repository はデータベースへのアクセスを担当します。View は HTML 画面を組み立ててブラウザに返します。

  3. Q3. Model は何のために使いますか？

    回答を表示

    Controller から View（画面）にデータを渡すための「箱」として使います。Controller が Model にデータを入れ、View がそのデータを使って HTML を組み立てます。

  4. Q4. @GetMapping と @PostMapping の違いは何ですか？

    回答を表示

    @GetMapping は HTTP の GET リクエストを受け付けます（画面表示・データ取得用）。@PostMapping は HTTP の POST リクエストを受け付けます（登録・更新・削除などデータを変更する処理用）。

  5. Q5. @PathVariable は何のために使いますか？

    回答を表示

    URL の一部（例: /users/{id} の {id}）を Java のメソッド引数として受け取るために使います。これにより「どのユーザーの詳細を見るか」などを URL から取得できます。

  6. Q6. @ModelAttribute は何のために使いますか？

    回答を表示

    フォームから送信されたデータ（name・email など）をまとめてオブジェクト（例: UserForm）として受け取るために使います。フィールド名とフォームの name 属性が自動的に対応します。

  7. Q7. "redirect:/users" は何を意味しますか？

    回答を表示

    処理完了後にブラウザを /users（一覧画面）へ転送（リダイレクト）する指示です。POST 処理の後にリダイレクトすることで、ブラウザの「更新」ボタンによる二重送信（同じ登録・削除が繰り返される）を防げます。

  8. Q8. Thymeleaf の th:each は何のために使いますか？

    回答を表示

    リスト（コレクション）を繰り返し表示するために使います。例えば th:each="user : ${users}" と書くと、users リストの件数分だけ同じ HTML を繰り返し生成します。ユーザー一覧の行を全員分表示するのに使います。

  9. Q9. 登録・更新・削除を POST で行う理由は何ですか？

    回答を表示

    GET はブラウザのアドレスバーやリンクから誰でも呼び出せるため、データを変更する操作に使うと意図せず変更されてしまう危険があります。POST はフォーム送信が必要なため、誤クリックや URL の直打ちでデータが変更されるリスクを防げます。

  10. Q10. Spring MVC と Spring Data JDBC はどのように連携しますか？

    回答を表示

    Spring MVC の Controller がリクエストを受け取り、Service を経由して Spring Data JDBC の Repository を呼び出します。Repository がデータベースを操作し、結果を Controller が Model に入れて View に渡します。これにより Web 画面からデータベースを操作できます。
