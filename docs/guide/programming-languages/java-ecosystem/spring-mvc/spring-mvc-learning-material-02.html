Spring MVCプロジェクトの基本構成

Spring MVC のアプリは、Java のクラスと HTML テンプレートを組み合わせて作ります。この章では、Controller・Service・Repository・Entity といった Java 側のクラスと、HTML テンプレートを置く templates フォルダがプロジェクトの中にどのように配置されているかを学びます。前教材（Spring 基礎・Spring Data JDBC）で出てきた構成の延長線上にあるので、すでに知っていることを確認しながら読み進めてください。

この章で学ぶこと

  • Spring MVC を使うプロジェクト構成を理解する
  • Controller・Service・Repository・View の配置場所を理解する
  • templates フォルダの役割を理解する

2.1 「Java側（処理）」と「HTML側（画面）」の2つに分かれる

Spring MVC のプロジェクトを大きく眺めると、「Java 側（処理）」と「HTML 側（画面）」の2つに分かれています。役割の違いでざっくり整理すると次のようになります。

Java 側（処理を担当）

  • Controller — リクエストを受け取る
  • Service — 業務ロジックを処理する
  • Repository — DB を読み書きする
  • Entity — DB のデータを Java オブジェクトで表す

配置先: src/main/java/

HTML 側（画面を担当）

  • Thymeleaf テンプレート（.html）
  • 一覧・詳細・登録・編集フォームなど
  • Java 側から渡されたデータを表示する

配置先: src/main/resources/templates/

料理店にたとえると、Java 側は「厨房（調理・仕込み）」、HTML 側は「ホール（お客さんに見える場所）」です。厨房でできあがった料理（データ）を、ホール（画面）がきれいに盛り付けてお客さん（ブラウザ）に届ける——この2層構造が Spring MVC プロジェクトの基本です。

前教材との連続性

前教材「Spring 基礎」「Spring Data JDBC」で学んだ Service / Repository / Entity の役割分担はそのまま引き継がれます。今回あらたに加わるのは「ブラウザからのリクエストを受け取る Controller」と「画面を表示する HTML テンプレート（templates/）」の2つです。知っているものに2つ足すだけ、と考えると整理しやすいです。

2.2 プロジェクトのフォルダ構成

Spring Boot プロジェクトでは、ソースコードは src/main/ 配下に置くのが決まりです。Java クラスは src/main/java/、設定ファイルや HTML テンプレートは src/main/resources/ に置きます。実際のフォルダ構成を見てみましょう。

プロジェクト構成（ファイルツリー） 
src
 └─ main
     ├─ java
     │   └─ com.example.demo
     │       ├─ DemoApplication.java        // 起動クラス
     │       ├─ controller
     │       │   └─ UserController.java     // リクエスト受付
     │       ├─ service
     │       │   └─ UserService.java        // 業務処理
     │       ├─ repository
     │       │   └─ UserRepository.java     // DB アクセス
     │       └─ entity
     │           └─ User.java               // DB の1行を表すクラス
     └─ resources
         ├─ application.properties          // 設定ファイル
         ├─ schema.sql                      // テーブル作成 SQL
         ├─ data.sql                        // 初期データ投入 SQL
         └─ templates
             └─ users
                 ├─ list.html              // 一覧画面
                 ├─ detail.html            // 詳細画面
                 ├─ form.html              // 新規登録フォーム
                 └─ edit.html              // 編集フォーム

少し多く見えますが、Java 側のパッケージ（フォルダ）は役割別に4つ、HTML テンプレートは機能別にまとめたサブフォルダに整理するだけです。それぞれを詳しく見ていきましょう。

2.3 Java 側の各パッケージの役割

src/main/java/com.example.demo/ の中には役割ごとにパッケージ（フォルダ）を分けてクラスを置きます。Spring MVC では、以下の4パッケージが基本セットです。

パッケージ 置くクラス 役割
controller UserController.java ブラウザからのリクエストを受け取り、Service を呼んで View へ渡す司令塔
service UserService.java 業務ロジック（バリデーションや計算など）を担当する
repository UserRepository.java データベースの読み書き（SELECT・INSERT・UPDATE・DELETE）を担当する
entity User.java データベースの1行を Java オブジェクトとして表すクラス
flowchart LR A["ブラウザ"] --> B["controller
UserController"] B --> C["service
UserService"] C --> D["repository
UserRepository"] D --> E[("DB")] E -.データ.-> D D -.結果.-> C C -.結果.-> B B --> F["templates/
users/list.html"] F --> A

図を見ると、パッケージの並び順がそのまま処理の流れになっていることが分かります。ブラウザ → controller → service → repository → DB → （逆流）→ controller → templates という一方向の流れです。各クラスは「自分の担当だけ」に集中し、隣のクラスに仕事を渡すだけなので、どこで何をしているかが見通しやすくなります。

重要ポイント

Controller は直接 Repository を呼ばず、必ず Controller → Service → Repository の順で呼び出します。「業務ロジックは Service に、DB アクセスは Repository に」という役割の境界を守ることで、コードが整理され、変更しやすくなります。

2.4 templates フォルダの役割

src/main/resources/templates/ フォルダは、Thymeleaf の HTML テンプレートを置く専用フォルダです。Spring Boot はこのフォルダを「View の置き場所」として自動で認識します。ここに置いたファイルが Controller から呼び出され、ブラウザへの HTML 画面として返されます。

HTML テンプレートは機能単位のサブフォルダにまとめるのが定番です。ユーザー管理機能なら templates/users/ というフォルダを作り、その中に list.html・detail.html・form.html・edit.html を置きます。

templates/ 配下の構成 
templates/
 └─ users/           // ユーザー機能のテンプレートをまとめる
     ├─ list.html    // ユーザー一覧画面
     ├─ detail.html  // ユーザー詳細画面
     ├─ form.html    // 新規登録フォーム
     └─ edit.html    // 編集フォーム
ファイル名 画面の役割 対応するアクション
list.html ユーザーの一覧を表示する 一覧取得（SELECT 全件）
detail.html 1件の詳細情報を表示する 1件取得（SELECT by ID）
form.html 新規登録フォームを表示する 登録（INSERT）
edit.html 既存データの編集フォームを表示する 更新（UPDATE）

なぜ templates フォルダを使うの？

Spring Boot は src/main/resources/templates/ を「View の置き場所」として自動的に認識します。Controller で return "users/list"; と書くだけで、Spring Boot が自動的に templates/users/list.html を見つけて返してくれます。フォルダを正しい場所に置くだけで動く、これが「設定より規約（Convention over Configuration）」という Spring Boot の考え方です。

2.5 schema.sql・data.sql・application.properties

src/main/resources/ 直下には、データベース関連のファイルも置きます。これらは前教材と同じです。

schema.sql

テーブルを作成する SQL を書く。Spring Boot 起動時に自動実行される。

data.sql

初期データ（サンプルユーザーなど）を投入する SQL を書く。schema.sql の後に実行される。

application.properties

DB の接続先やポート番号など、アプリ全体の設定を書くファイル。

schema.sql には CREATE TABLE users (...); のような DDL を書き、data.sql には INSERT INTO users VALUES (...); のような初期データを書きます。これらは前教材どおりで変わりません。Spring MVC を学ぶ本教材でも同じファイルを使ってテーブルを用意します。

まとめ: resources/ 配下の役割

templates/ ← Thymeleaf テンプレート（画面）
schema.sql ← テーブル定義（前教材と同じ）
data.sql ← 初期データ（前教材と同じ）
application.properties ← アプリ設定（前教材と同じ）

やってみよう 2-1: ファイルの場所を確認する

実際に Spring Boot プロジェクトを IDE（IntelliJ IDEA や Eclipse）で開き、2.2 のファイルツリーどおりにフォルダとファイルが存在することを確認しましょう。

手順

  1. 1 プロジェクトを IDE で開き、src/main/java/ の中に controller・service・repository・entity パッケージがあることを確認する。
  2. 2 src/main/resources/templates/ フォルダを確認し、users/ サブフォルダの中に 4 つの HTML ファイルがあることを確認する。
  3. 3 2.3 の表と見比べながら「このクラスはこの役割」と声に出して確認する。前教材で見た Service・Repository・Entity と同じ配置になっているか比べてみましょう。

理解度確認クイズ

  1. Q1. Controller クラスはどのような役割のクラスですか？

    回答を表示

    ブラウザからのリクエストを受け取り、処理の流れを制御する「司令塔」の役割を持つクラスです。Service を呼んで業務処理を依頼し、結果を Model に入れて View（HTML テンプレート）へ渡します。

  2. Q2. HTML テンプレートはどこに配置しますか？

    回答を表示

    src/main/resources/templates/ フォルダに配置します。機能単位でサブフォルダを作り（例: templates/users/）、その中に各画面の HTML ファイルを置くのが定番です。

  3. Q3. templates フォルダは何のために使いますか？

    回答を表示

    Thymeleaf の HTML テンプレート（View）を置く専用フォルダです。Spring Boot はこのフォルダを「View の置き場所」として自動認識します。Controller から View 名を返すと、Spring Boot が templates フォルダの中から対応する HTML ファイルを見つけてブラウザへ返します。
