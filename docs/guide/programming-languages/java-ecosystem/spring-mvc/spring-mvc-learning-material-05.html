ホーム Spring MVC 第5章
第5章 所要時間: 約60分

ユーザー一覧を表示する

この章では、ユーザー一覧画面を作ります。Controller が Service を呼び出してユーザー一覧を取得し、Model に入れて View へ渡す流れを学びます。View 側では Thymeleaf の th:each を使って一覧データを繰り返し表示します。前教材（Spring Data JDBC）で学んだ findAll() とのつながりも確認しながら、Spring MVC の「データを取得して画面に渡す」基本パターンを身につけましょう。

この章で学ぶこと

  • Service からユーザー一覧を取得する流れを理解する
  • Controller から View へ一覧データを渡す方法を理解する
  • Thymeleaf で繰り返し表示する方法を理解する

5.1 ユーザー一覧画面の全体像

ユーザー一覧画面とは、データベースに登録されたユーザー情報を表として一覧表示するページです。この画面を実現するには、次の3つの役割が協力します。

  • Controller: ブラウザから「/users へアクセスして」というリクエストを受け取る
  • Service → Repository: データベースから全ユーザーを取得する（findAll()）
  • View（Thymeleaf）: 受け取った一覧を th:each で繰り返し表示する
flowchart TD A["ブラウザ"] -->|"GET /users"| B["UserController
(リクエストの受付係)"] B --> C["UserService
（業務処理を担当）"] C --> D["UserRepository
（DBアクセスを担当）"] D --> E[("Database")] E -.全ユーザーを返す.-> C C -.Userのリストを返す.-> B B -->|"model.addAttribute で users を渡す"| F["View: users/list.html
（th:eachで繰り返し表示）"] F -->|HTMLを返す| A

前教材「Spring Data JDBC」で学んだ Repository の findAll() は、データベースの全レコードを取得するメソッドです。今回はそれを Service 経由で呼び出し、取得した結果を Model に入れて View へ渡します。前教材の知識がそのまま活きる構成です。

重要ポイント: 前教材との関係

Spring Data JDBC で学んだ Repository（findAll()）と Service の役割は変わりません。今回 Spring MVC で新しく加わるのは、「ブラウザからリクエストを受け取る Controller」と「データを一覧表示する View（Thymeleaf）」だけです。

5.2 Controller を作る — UserController

まず、ブラウザからのリクエストを受け取る UserController を作ります。Controller のポイントは2つあります。

  1. UserService を DI（依存性注入）で受け取る: Controller はコンストラクタ経由で UserService を受け取ります
  2. Model にデータを入れて View 名を返す: Service から取得した一覧を model.addAttribute() で渡します

用語: DI（依存性注入）とは

DI（Dependency Injection）とは、「クラスが必要な部品を自分で作らず、外から受け取る仕組み」です。new UserService() と書く代わりに、コンストラクタの引数として受け取るだけで Spring が自動的に用意してくれます。これにより、部品の差し替えやテストがしやすくなります。コンストラクタ経由で受け取る方法をコンストラクタインジェクションと呼びます。

では実際のコードを見てみましょう。

UserController.java 
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller  // Springに「これはControllerだ」と伝えるアノテーション
public class UserController {

    // UserServiceをコンストラクタで受け取る（コンストラクタインジェクション）
    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;  // Spring が自動的に渡してくれる
    }

    @GetMapping("/users")  // /users へのGETリクエストを受け取る
    public String list(Model model) {
        // Serviceからユーザー一覧を取得してModelに入れる
        model.addAttribute("users", userService.findAll());
        return "users/list";  // View名: templates/users/list.html を表示
    }
}

@Controller

「このクラスはブラウザのリクエストを受け取る Controller です」と Spring に伝えるアノテーション。

コンストラクタ DI

コンストラクタの引数に UserService と書くだけで Spring が自動注入してくれる。

model.addAttribute

Model に名前（"users"）とデータを入れると、View から参照できるようになる。

用語: @GetMapping と View 名の返し方

@GetMapping("/users") は「/users という URL へのGETアクセスが来たらこのメソッドを実行する」という意味です。メソッドが return "users/list"; を返すと、Spring が src/main/resources/templates/users/list.html（Thymeleaf テンプレート）を探して画面を組み立てます。

5.3 View を作る — th:each で繰り返し表示

次に、ユーザー一覧を表示する View（HTML テンプレート）を作ります。Thymeleaf の th:each 属性を使うと、リストのデータを1件ずつ繰り返して HTML に展開できます。

用語: th:each とは

th:each="user : ${users}" は「${users}（Model から受け取った一覧）から1件ずつ取り出して、user という名前で使う」という繰り返しの命令です。Java の for (User user : users) と同じ意味をHTMLで表現したものです。

実際の HTML コードを見てみましょう。

src/main/resources/templates/users/list.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ユーザー一覧</title>
</head>
<body>
    <h1>ユーザー一覧</h1>
    <table border="1">
        <tr>
            <th>ID</th>
            <th>名前</th>
            <th>メールアドレス</th>
        </tr>
        <!-- th:each で users から1件ずつ user として取り出して繰り返す -->
        <tr th:each="user : ${users}">
            <td th:text="${user.id}">1</td>
            <td th:text="${user.name}">田中太郎</td>
            <td th:text="${user.email}">tanaka@example.com</td>
        </tr>
    </table>
</body>
</html>

コードのポイントを整理します。

コード 意味
xmlns:th="..." Thymeleaf の th: 属性を使うための宣言
th:each="user : ${users}" ${users} から1件ずつ取り出し user という名前で繰り返す
${user.id} 取り出した user の id フィールドの値を表示する
th:text="${user.name}" <td> の中身を user.name の値で置き換える
田中太郎（仮テキスト） th:text が適用されると消えるダミーテキスト。ブラウザで HTML ファイルを直接開いたときの確認用

重要ポイント: th:text でタグの中身を差し替える

th:text="${user.name}" をタグに付けると、タグの中身（例: 「田中太郎」というダミーテキスト）が 実際のデータで自動的に置き換わります。Thymeleaf が HTML を生成するときに差し替えるため、元のダミーテキストはブラウザには表示されません。

5.4 データの流れをまとめて確認する

ここで、ユーザー一覧表示の全体の流れをもう一度まとめます。それぞれのコードがどのタイミングで動くか確認しましょう。

  1. 1

    ブラウザが /users にアクセスする

    ブラウザが「/users のページを見せて」というリクエストを Spring に送ります。

  2. 2

    UserController の list() メソッドが呼ばれる

    @GetMapping("/users") があるメソッドが実行されます。

  3. 3

    userService.findAll() でユーザー一覧を取得する

    Service → Repository → DB の順にアクセスして全ユーザーをリストとして取得します。

  4. 4

    model.addAttribute("users", ...) で View に渡す

    取得したリストを "users" という名前で Model に入れます。View（Thymeleaf）は ${users} でこのデータを受け取ります。

  5. 5

    View（users/list.html）が th:each で繰り返し表示する

    Thymeleaf が ${users} の件数分だけ <tr> を繰り返し生成し、テーブルとして一覧表示します。

用語: addAttribute の名前と ${} の関係

model.addAttribute("users", list) の第1引数（"users"）が、View 側で ${users} として参照できる名前になります。名前はどんな文字列でも構いませんが、Controller と View で必ず一致させる必要があります。

やってみよう 5-1: ユーザー一覧画面を動かしてみる

UserController と users/list.html を作成して、ブラウザでユーザー一覧が表示されることを確認しましょう。

手順

  1. 1 UserController.java を作成し、5.2 のコードを入力する。
  2. 2 src/main/resources/templates/users/list.html を作成し、5.3 のコードを入力する。
  3. 3 アプリを起動し、ブラウザで http://localhost:8080/users にアクセスする。
  4. 4 データベースに登録済みのユーザーがテーブルで一覧表示されることを確認する。

動作確認のポイント

  • • テーブルに ID・名前・メールアドレスが表示されている
  • • DB のレコード数だけ行（<tr>）が生成されている
  • • DB にデータがない場合はヘッダー行のみ表示される

理解度確認クイズ

  1. Q1. ユーザー一覧を取得するために Controller は何を呼び出しますか？

    回答を表示

    UserService の findAll() を呼び出します。Controller は直接 Repository を呼ばず、Service を経由してデータを取得します。コード例では userService.findAll() と記述します。

  2. Q2. th:each は何のために使いますか？

    回答を表示

    リスト（一覧データ）を1件ずつ繰り返して HTML に表示するために使います。th:each="user : ${users}" と書くと、${users} の件数分だけタグが繰り返し生成されます。Java の for-each ループと同じ役割をHTMLで表現したものです。

  3. Q3. 一覧データを HTML へ渡すには何を使いますか？

    回答を表示

    Model（model.addAttribute()）を使います。Controller で model.addAttribute("users", userService.findAll()) と書くと、View（Thymeleaf）側から ${users} としてデータを参照できるようになります。
