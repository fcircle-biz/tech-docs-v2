詳細画面を表示する

詳細画面では、URL に含まれる ID を使って、特定のユーザーを1件取得して表示します。この章では「URL の一部を変数として受け取る」という Spring MVC の仕組み（@PathVariable）を学び、一覧画面から詳細画面へリンクする流れを実装します。

この章で学ぶこと

  • URL に含まれる ID を Controller で受け取る方法を理解する
  • @PathVariable の役割を理解する
  • ID に対応するデータを画面に表示する流れを理解する

6.1 詳細画面とは — URL で「誰の情報か」を伝える

一覧画面では全ユーザーをまとめて表示しましたが、詳細画面では特定の1人だけを表示します。ここで問題になるのが「どのユーザーの詳細を見たいか」をサーバーに伝える方法です。Spring MVC では URL の中に ID を埋め込むという方法を使います。

たとえば /users/1 にアクセスすると「ID が 1 のユーザーの詳細を見たい」、/users/5 にアクセスすると「ID が 5 のユーザーの詳細を見たい」という意味になります。URL の数字の部分が変わるだけで、見えるユーザーが切り替わる仕組みです。

たとえで理解する: 「宅配便の伝票番号」

宅配便の追跡サイトで「伝票番号 12345 を追跡する」場合、URL が /tracking/12345 のようになっているのを見たことがあるでしょう。この 12345 の部分がまさに「どの荷物か」を指定するものです。Spring MVC の /users/{id} も同じ発想で、{id} の部分に実際の数字を入れることで「どのユーザーか」を指定します。

flowchart LR A["ブラウザ
/users/1 にアクセス"] -->|リクエスト| B["Controller
@PathVariable id=1"] B --> C["userService.findById（1）"] C --> D["ID=1 のユーザーを返す"] D --> B B -->|Modelに入れる| E["View
詳細画面を表示"]

この図を見ると、/users/1 という URL からリクエストが来ると、Controller が URL の 1 という数字を取り出し、それを使って Service からユーザーを1件取得し、画面に渡す流れがわかります。この「URL の一部を変数として受け取る」仕組みを担うのが @PathVariable です。

6.2 @PathVariable を使った Controller の実装

URLに含まれる値（たとえば /users/1 の 1）を Controller のメソッドで受け取るには、@PathVariable アノテーションを使います。URL のパターンに {id} のように波括弧で変数名を定義し、メソッド引数に @PathVariable Long id と書くと、その値が自動的にメソッドに渡されます。

UserController.java（詳細表示メソッド） 
import org.springframework.web.bind.annotation.PathVariable;

// /users/1 や /users/5 のようなURLを受け取る
@GetMapping("/users/{id}")
public String detail(@PathVariable Long id, Model model) {
    // URLの {id} 部分の値が id 変数に入る
    model.addAttribute("user", userService.findById(id));
    // templates/users/detail.html を表示する
    return "users/detail";
}
コード 意味
/users/{id} URL のパターン。{id} は可変部分を表す。
@PathVariable Long id URL の {id} の値をメソッド引数で受け取る。
userService.findById(id) Service に対して「この ID のユーザーを1件取得して」と依頼する。
model.addAttribute 取得したユーザーを "user" という名前で画面（View）に渡す。

重要ポイント: URL パターンと引数名は合わせる

@GetMapping("/users/{id}") の {id} と、引数の @PathVariable Long id の変数名 id は一致させます。名前が同じなら Spring が自動的に対応付けてくれます。別の名前にする場合は @PathVariable("id") Long userId のように明示する必要があります。

6.3 詳細画面の Thymeleaf テンプレート

Controller から Model に入れた user オブジェクトを、Thymeleaf テンプレートで表示します。th:text="${user.id}" のように書くと、Java 側で入れた値が HTML のテキストとして埋め込まれます。

templates/users/detail.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ユーザー詳細</title>
</head>
<body>
    <h1>ユーザー詳細</h1>
    <!-- th:text でJavaから渡された値を表示する -->
    <p>ID：<span th:text="${user.id}">1</span></p>
    <p>名前：<span th:text="${user.name}">田中太郎</span></p>
    <p>メール：<span th:text="${user.email}">tanaka@example.com</span></p>
    <!-- @{/users} でURLを生成する -->
    <a th:href="@{/users}">一覧へ戻る</a>
</body>
</html>

用語: th:text のプレースホルダーテキスト

<span th:text="${user.id}">1</span> のように、タグの中に書いてある 1 や 田中太郎 はデフォルト値（プレースホルダー）です。Thymeleaf が動く本番環境では th:text の値が優先されてこのテキストは置き換わりますが、ブラウザで HTML ファイルを直接開いたときの確認用としても使えます。

6.4 一覧画面から詳細画面へリンクする

ユーザー一覧画面で各ユーザーの「詳細」リンクをクリックしたときに、そのユーザーの ID を URL に埋め込んで詳細画面に遷移させるには、Thymeleaf の @{...} URL 記法を使います。

templates/users/list.html（一覧のリンク部分） 
<!-- @{/users/{id}(id=${user.id})} で動的にURLを生成する -->
<a th:href="@{/users/{id}(id=${user.id})}">詳細</a>

@{/users/{id}(id=${user.id})} は少し難しく見えますが、「URL テンプレートの {id} に ${user.id} の値を当てはめる」という意味です。たとえば user.id が 3 なら、生成される URL は /users/3 になります。

@{...}

Thymeleaf の URL 記法。コンテキストパス（アプリの場所）を自動で付加してくれる安全な URL 生成方法。

{id} の埋め込み

/users/{id} の {id} は URL テンプレートの変数。後ろの (id=${user.id}) で実際の値を指定する。

生成される URL

user.id が 1 なら /users/1、5 なら /users/5 が生成される。

全体の流れをまとめると

  • ① 一覧画面で th:href="@{/users/{id}(id=${user.id})}" を使い、各ユーザーへのリンクを生成する
  • ② リンクをクリックすると /users/1 のような URL でリクエストが送られる
  • ③ Controller の @PathVariable Long id が URL の数字（1）を受け取る
  • ④ userService.findById(id) で該当ユーザーを取得する
  • ⑤ Model に入れて詳細画面 View に渡し、th:text で値を表示する

やってみよう 6-1: 詳細画面を実装する

一覧画面で各ユーザーの「詳細」リンクをクリックすると詳細画面が表示されるよう実装してみましょう。

手順

  1. 1 UserController に @GetMapping("/users/{id}") を付けた detail メソッドを追加し、@PathVariable Long id でID を受け取るよう実装する。
  2. 2 templates/users/detail.html を作成し、th:text="${user.id}"・th:text="${user.name}"・th:text="${user.email}" でユーザー情報を表示する。
  3. 3 一覧画面 list.html の各ユーザー行に th:href="@{/users/{id}(id=${user.id})}" を使った詳細リンクを追加する。
  4. 4 アプリを起動して /users で一覧を表示し、「詳細」リンクをクリックして /users/1 などの詳細画面が正しく表示されることを確認する。

理解度確認クイズ

  1. Q1. @PathVariable は何のために使いますか？

    回答を表示

    URL の一部（可変部分）をメソッドの引数として受け取るために使います。たとえば /users/{id} の {id} の値を Controller のメソッドで取得するのに使います。

  2. Q2. /users/3 にアクセスした場合、id には何が入りますか？

    回答を表示

    3 が入ります。URL の /users/3 の数字部分がそのまま @PathVariable Long id の変数 id に代入されます。

  3. Q3. 詳細画面では Service のどのような処理を呼び出しますか？

    回答を表示

    ID を引数にとって1件のユーザーを取得する処理（findById(id)）を呼び出します。一覧表示で使う全件取得（findAll()）とは異なり、特定のユーザー1件だけを取得します。
