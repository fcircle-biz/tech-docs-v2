8.2 users テーブルの要件

ユーザー管理に使う users テーブルの列（カラム）構成は次のとおりです。schema.sql（第3章）にこの定義を書いてテーブルを作成します。

列名 型 内容
id BIGINT 主キー（PRIMARY KEY）。自動採番（IDENTITY や AUTO_INCREMENT）
name VARCHAR ユーザー名
email VARCHAR メールアドレス

第3章の振り返り: schema.sql と data.sql

schema.sql はアプリ起動時に実行される「テーブル定義（CREATE TABLE）」を書くファイルです。data.sql には初期データを登録する INSERT 文を書きます。どちらも src/main/resources/ に置きます（第3章参照）。

8.3 UserService に必要なメソッド

UserService には、CRUD に対応する5つのメソッドを用意します。第6章では Create・Read・Delete を作りました。この章では未実装の Update（更新） を追加して、CRUD をすべて揃えます。

メソッド 内容
findAll() 全件取得（Read）
findById(Long id) ID で1件取得（Read）
create(String name, String email) 新規登録（Create）
update(Long id, String name, String email) 更新（Update）← この章で追加
delete(Long id) 削除（Delete）

8.4 update メソッドの実装

第6章では create・findAll・findById・delete を実装しました。ここで残った update（更新） を追加します。

ポイントは save メソッドの挙動です。id が null なら「新規登録（INSERT）」、id を指定すれば「更新（UPDATE）」として動作します。create では id を null にしていましたが、update では id を指定することで同じ save メソッドを「更新」として使います。

UserService.java（update メソッド追加） 
// id を指定して save すると「更新」になる
public User update(Long id, String name, String email) {
    // id を持つ User オブジェクトを作る（id が null でないので UPDATE 扱い）
    User user = new User(id, name, email);
    // save は id があれば UPDATE、null なら INSERT を実行する
    return userRepository.save(user);
}

重要ポイント: save の登録と更新の使い分け

  • id = null → save が INSERT を実行（新規登録）
  • id = 指定あり → save が UPDATE を実行（更新）

同じ save メソッド1つで「登録」と「更新」の両方を賄えるのが CrudRepository の便利なところです。

手順

  2. 2 User エンティティを作成する（第4章参照）
    @Id を付けた id フィールドと、name・email を持つ Java クラスを作る。
  3. 3 UserRepository を作成する（第5章参照）
    CrudRepository<User, Long> を継承したインターフェースを定義する。メソッドの実装は Spring が自動生成するため、本文は空でよい。
  4. 4 UserService を作成する（第6章＋8.4 参照）
    @Service を付けたクラスに findAll・findById・create・update（8.4 参照）・delete の5メソッドを実装する。
  5. 5 AppRunner を作成する（第7章参照）
    CommandLineRunner を実装したクラスを作り、run メソッド内で create → findAll → update → findById → delete の順に呼び出して動作を確認する。

    コンソールに登録・取得・更新・削除のログが出ることを確認する。エラーが出た場合は application.properties の DB 接続設定（第2章）と schema.sql（第3章）を見直す。

8.5 教材全体のまとめ

本教材で学んだ内容を振り返りましょう。第1章から第7章で積み上げてきた知識が、この第8章の総合演習でひとつにまとまります。

本教材の重要ポイント

  • Spring Data JDBC は、Spring で DB 操作をシンプルに行うための仕組み
  • Entity はテーブルのデータを Java オブジェクトとして表す（@Id で主キーを指定）
  • Repository は DB アクセスを担当するインターフェース（CrudRepository を継承）
  • Service は業務処理を担当するクラス（@Service を付けて DI で受け取る）
  • CrudRepository を使うと基本的な CRUD 処理（findAll・save・deleteById 等）を簡単に扱える
  • application.properties には DB 接続情報（URL・ユーザー名・パスワード）を書く
  • schema.sql にはテーブル作成 SQL（CREATE TABLE）、data.sql には初期データ登録 SQL（INSERT）を書く
  • 次の教材では Spring MVC を使って Web 画面やリクエストから Service を呼び出す方法を学ぶ

総合確認問題

  1. Q1. Spring Data JDBC は何のために使いますか？

    回答を表示

    Spring で DB 操作をシンプルに行うためです。SQL の記述・接続管理・結果の取り出しといった「決まりきったコード」を Spring が肩代わりし、Repository を通じて基本的な CRUD 処理を簡単に実装できます。

  2. Q2. Entity、Repository、Service はそれぞれ何を担当しますか？

    回答を表示

    Entity はテーブルのデータを Java オブジェクトとして表します。Repository は DB アクセス（SQL の実行）を担当します。Service は業務処理（CRUD のロジック）を担当し、Repository を呼び出してデータ操作を行います。

  3. Q3. CrudRepository で使える主なメソッドを3つ挙げてください。

    回答を表示

    findAll()（全件取得）・findById(id)（ID で1件取得）・save(entity)（登録または更新）・deleteById(id)（ID で削除）・count()（件数取得）・existsById(id)（存在確認）のうち3つ。

  4. Q4. save メソッドは登録と更新のどちらに使えますか？

    回答を表示

    両方に使えます。渡す Entity の id が null の場合は INSERT（新規登録）、id が指定されている場合は UPDATE（更新）として動作します。

  5. Q5. schema.sql と data.sql の違いは何ですか？

    回答を表示

    schema.sql はテーブル定義（CREATE TABLE）を書くファイルで、テーブルの「構造」を作ります。data.sql は初期データの登録（INSERT）を書くファイルで、テーブルの「中身（初期値）」を入れます。

  6. Q6. Spring MVC に進むと、AppRunner の代わりにどのクラスから Service を呼び出すようになりますか？

    回答を表示

    Controller（コントローラー）から呼び出すようになります。Controller は Web のリクエスト（ブラウザからの操作）を受け取り、適切な Service メソッドを呼び出して処理を行う役割を担います。
