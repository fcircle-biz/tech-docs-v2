Spring Bootプロジェクトの基本構成

Spring Boot プロジェクトを開いたとき、「ファイルはどこにあるの？」「何から起動するの？」と迷うことがあります。この章では、基本的なファイル構成・アプリが起動する場所（main メソッドと @SpringBootApplication）・設定の書き場所（application.properties）を読めるようになることを目標にします。難しい設定はあとの章で学ぶので、まずは「どこに何があるか」のマップをつかみましょう。

この章で学ぶこと

  • Spring Boot プロジェクトの基本的なファイル構成を理解する
  • main メソッドの役割と @SpringBootApplication が何をしているかを理解する
  • application.properties の役割を理解する

2.1 基本的なファイル構成

Spring Boot プロジェクトを作成すると、最初から次のようなフォルダとファイルが生成されます。すべてを一度に覚える必要はありません。まずは「どこに何があるか」の地図をつかみましょう。

プロジェクト構成（基本） 
src
 └─ main
     ├─ java
     │   └─ com.example.demo
     │       └─ DemoApplication.java   ← 起動クラス（アプリの入口）
     └─ resources
         └─ application.properties     ← 設定ファイル

src/main/java

Java のプログラム（.java ファイル）を置く場所です。アプリのすべての処理はここに書きます。

DemoApplication.java — 起動クラス

src/main/resources

設定ファイルや静的リソースを置く場所です。Java コードには書かない設定を、ここにまとめます。

application.properties — 設定ファイル

パッケージ名（com.example.demo）について

com.example.demo はプロジェクト作成時に決めるパッケージ名です。フォルダの階層（com/example/demo/）と対応しています。実際の開発では会社名やプロジェクト名を使いますが、学習用なら com.example.demo のままでOKです。

2.2 起動クラスと main メソッド

Spring Boot アプリは DemoApplication.java から起動します。中身を見てみましょう。

DemoApplication.java 
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

// Spring Boot アプリの「開始点」を表すアノテーション
@SpringBootApplication
public class DemoApplication {

    // アプリを起動するための main メソッド
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
1

@SpringBootApplication — 開始点のしるし

このアノテーション（@ で始まる印）をクラスに付けると、「ここが Spring Boot アプリの開始点です」という意味になります。内部では「設定の自動化」「コンポーネント探索」など複数の設定を一括で有効にしてくれています。

2

main メソッド — アプリの起動口

Java プログラムは main メソッドから実行が始まります。Spring Boot でも同じで、ここからアプリが動き出します。

3

SpringApplication.run(...) — Spring を起動する命令

この1行が Spring Boot のエンジンを動かします。第1引数に自分のクラス名（DemoApplication.class）を渡すと、Spring はそのパッケージ以下を自動的にスキャンして準備を整えてくれます。

@SpringBootApplication がまとめてやってくれること

このアノテーション1つで、次の3つの設定をまとめて有効にしています。① 設定クラスとして認識（@Configuration）② 自動設定を有効化（@EnableAutoConfiguration）③ 同じパッケージ以下のクラスを自動スキャン（@ComponentScan）。難しい設定をまとめてくれているので、私たちは @SpringBootApplication を付けるだけで始められます。

2.3 application.properties の役割

application.properties は、アプリケーションの設定を書くファイルです。 アプリ名・サーバーのポート番号・ログの出力レベルなど、Java コードに直接書かずに外に出しておきたい設定をここにまとめます。

application.properties（記述例） 
# アプリケーション名
spring.application.name=demo

# サーバーのポート番号（デフォルトは 8080）
server.port=8080

# ログレベル（INFO / DEBUG / WARN など）
logging.level.root=INFO

「なぜ Java コードに書かないのか」というと、設定を変えるたびにコードを修正・再コンパイルしなくて済むからです。たとえばポート番号を変えたいとき、設定ファイルの1行を書き換えるだけで対応できます。「コードと設定は分離する」という考え方の入口がここにあります（くわしくは第6章で学びます）。

データベース設定は次の教材で

DB の接続先（URL・ユーザー名・パスワード）なども application.properties に書きます。これらは次の教材「Spring Data JDBC」でくわしく扱います。この章では「設定をまとめて書く場所がある」とだけ覚えておけば十分です。

2.4 アプリ起動の流れ（図解）

ここまで学んだことを組み合わせると、Spring Boot アプリが起動するまでの流れは次のようになります。

flowchart TD A["① main メソッドを実行"] --> B["② SpringApplication.run が呼ばれる"] B --> C["③ @SpringBootApplication が Spring を起動
（自動設定・コンポーネントスキャン）"] C --> D["④ application.properties を読み込み
（ポート番号・アプリ名など）"] D --> E["⑤ Bean（部品）を準備・管理"] E --> F["⑥ アプリが稼働開始！"]

⑤の「Bean（部品）を準備・管理」が Spring の核心です。この仕組み（DI）は第3章・第4章でくわしく学びます。今の段階では、「main メソッドを実行すれば、あとは Spring が勝手に準備を整えてくれる」というイメージをもてれば十分です。

理解度確認クイズ

  1. Q1. Spring Boot アプリケーションの開始点になるクラスはどれですか？

    回答を表示

    @SpringBootApplication が付いた起動クラスです。今回の例では DemoApplication クラスがそれにあたります。このクラスに main メソッドがあり、アプリの実行はここから始まります。

  2. Q2. @SpringBootApplication は何を表しますか？

    回答を表示

    Spring Boot アプリケーションの開始点を表すアノテーションです。設定の自動化（@EnableAutoConfiguration）・コンポーネントスキャン（@ComponentScan）・設定クラスとしての認識（@Configuration）という3つの機能をまとめて有効にしてくれます。

  3. Q3. application.properties にはどのような情報を書きますか？

    回答を表示

    アプリ名・サーバーのポート番号・ログレベルなどのアプリケーション設定を書きます。Java コードとは分離して書くことで、設定変更のたびにコードを修正しなくて済みます。DB接続情報など詳しい設定は次の教材（Spring Data JDBC）で扱います。
