JDBCとは？ データベース接続の基本

JDBC（Java Database Connectivity）は、Java プログラムからデータベースを操作するための「標準の窓口」です。この章では、そもそもデータベースとは何か、JDBC はどんな役割を果たすのか、なぜ JDBC を使うと便利なのか、そしてプログラムとデータベースがどうつながるのかという全体像をつかみます。

この章で学ぶこと

  • データベースとは何か、なぜ必要なのかを説明できる
  • JDBC の役割と「標準 API」であることの意味を理解する
  • JDBC を使う利点（移植性・統一された書き方）を知る
  • アプリ・JDBC・ドライバ・DB のつながり（全体像）をつかむ

1.1 データベースとは何か

データベース（Database）とは、大量のデータを整理して保存し、効率よく検索・更新できるようにした「データの保管庫」です。たとえば、会員情報・商品の在庫・注文履歴など、アプリが扱う情報の多くは、プログラムを終了しても消えないように、どこかに保存しておく必要があります。この「消えないデータの置き場所」がデータベースです。

本ガイドで扱うのは、もっとも広く使われているリレーショナルデータベース（RDB）です。RDB はデータを表（テーブル）の形で管理します。Excel の表をイメージすると分かりやすく、「行（1件分のデータ）」と「列（項目）」でできています。そして、この RDB に対して「どんなデータが欲しいか」を伝えるための専用の言語が SQL（エスキューエル）です。

id（列） name（列） email（列）
1 佐藤 太郎 taro@example.com
2 鈴木 花子 hanako@example.com

上の表は users という名前のテーブルの例です。横一列（佐藤 太郎の行）が「1件分のデータ＝レコード」、縦の name などが「項目＝カラム」です。代表的な RDB には、MySQL・PostgreSQL・Oracle Database・H2 などがあります。

用語: SQL と DBMS

SQL は「このデータを取り出して」「この値を更新して」とデータベースに指示するための言語です。そして、データベースを実際に動かして管理しているソフトウェアを DBMS（Database Management System）と呼びます。MySQL や H2 は、正確には「DBMS」の名前です。本ガイドでは、Java から SQL を送って DBMS を操作する方法を学びます。

1.2 JDBC とは何か

JDBC（Java Database Connectivity）は、Java プログラムからデータベースに接続し、SQL を実行するための標準 APIです。「API」とは、ある機能を使うための「決められた呼び出し方（窓口）」のことです。つまり JDBC は、Java とデータベースの間に立つ共通の窓口だと考えてください。

JDBC は Java 標準ライブラリの一部（java.sql パッケージ）として最初から用意されています。私たちプログラマは、Connection（接続）・Statement（SQL の実行役）・ResultSet（検索結果）といった、JDBC が決めた「部品（インターフェース）」を使ってデータベースを操作します。これらの部品は本ガイドを通して少しずつ学んでいきます。

つなぐ

Java とデータベースを接続し、やり取りの通り道を作ります。

送る

SQL 文をデータベースへ送り、検索や更新を実行させます。

受け取る

実行結果（検索したデータや件数）を Java 側で受け取ります。

用語: 「API」と「インターフェース」

API は「機能を使うための決められた呼び出し方」、インターフェースは Java における「こういうメソッドを用意します」という約束ごとです。JDBC は Connection などのインターフェースで「窓口の形」を決めており、その中身（実際の処理）は次の節で出てくるドライバが担当します。「形は JDBC が決め、中身はドライバが用意する」と覚えておきましょう。

1.3 JDBC を使う利点

JDBC の最大の利点は、「どのデータベースを使っても、Java 側の書き方がほぼ同じになる」ことです。MySQL でも PostgreSQL でも H2 でも、Connection を取得して SQL を実行する、という基本の流れは変わりません。データベースを乗り換えても、プログラムの大部分を書き直さずに済みます。

観点 JDBC がない世界 JDBC がある世界
書き方 DB ごとに独自の方法を覚える必要がある どの DB でも共通の書き方で操作できる
DB の乗り換え プログラムを大幅に書き直す 主にドライバと接続設定の変更で済む
学習コスト DB の数だけ覚えることが増える JDBC の基本を一度覚えれば応用が利く

このように「一度覚えれば、いろいろなデータベースで使い回せる」のが JDBC の強みです。本ガイドでは学習用に H2 という軽量データベースを使いますが、ここで身につけた知識は MySQL や PostgreSQL など実務でよく使う DB にもそのまま活かせます。

1.4 全体像 — アプリ・JDBC・ドライバ・DB のつながり

Java からデータベースを操作するとき、登場人物は4つです。あなたが書くアプリケーション、共通の窓口である JDBC API、データベースごとの「翻訳係」である JDBC ドライバ、そして実際にデータを保管するデータベースです。これらは次のようにつながっています。

flowchart LR A["Javaアプリケーション
(あなたのコード)"] --> B["JDBC API
(java.sql)"] B --> C["JDBCドライバ
(DBごとの翻訳係)"] C --> D[("データベース
(H2 / MySQL など)")] D -.結果.-> C C -.結果.-> B B -.結果.-> A

ポイントは「JDBC ドライバ」の存在です。JDBC API はあくまで「共通の窓口（形）」を決めているだけで、実際にそのデータベースと通信する具体的な処理はドライバが担当します。ドライバは使うデータベースごとに用意されており（H2 用、MySQL 用など）、あなたのプロジェクトに「部品（ライブラリ）」として追加して使います。

つまり、データベースを乗り換えたいときはドライバを差し替えるのが基本です。Java 側のコードは JDBC の共通の書き方のままでよいので、変更を最小限に抑えられます。これが 1.3 で説明した「利点」を支えている仕組みです。

参考までに、JDBC を使ったコードの「最小の流れ」を先に見ておきましょう。今は雰囲気をつかむだけで十分です（詳しくは第2章以降で学びます）。

JDBC の基本の流れ（イメージ） 
// 1. データベースに接続する
Connection conn = DriverManager.getConnection(url, user, pass);

// 2. SQL を実行する準備をする
Statement stmt = conn.createStatement();

// 3. SQL を実行して結果を受け取る
ResultSet rs = stmt.executeQuery("SELECT * FROM users");

// 4. 結果を1件ずつ取り出す
while (rs.next()) {
    System.out.println(rs.getString("name")); // name 列を表示
}

重要ポイント

JDBC は「形（共通の窓口）」、ドライバは「中身（DB ごとの翻訳係）」。アプリは JDBC の共通の書き方でデータベースを操作し、実際の通信はドライバに任せます。この役割分担を理解しておくと、第2章のセットアップがスムーズに進みます。

1.5 JDBC プログラムの基本ステップ

JDBC を使うプログラムは、データベースの種類や処理内容が変わっても、おおむね同じ流れで書けます。この「型」を最初に押さえておくと、以降の章がぐっと理解しやすくなります。基本は次の5ステップです。

flowchart TD A["1. ドライバを準備する
(ライブラリを用意)"] --> B["2. 接続する
(Connection を取得)"] B --> C["3. SQL を実行する
(Statement で送る)"] C --> D["4. 結果を処理する
(ResultSet で受け取る)"] D --> E["5. 後始末をする
(接続などを閉じる)"]
  • ステップ1（準備） — 使う DB のドライバをプロジェクトに用意します（第2章）。
  • ステップ2（接続） — DriverManager から Connection を取得します（第2章）。
  • ステップ3（実行） — Statement で SQL を送ります（第3章・第4章）。
  • ステップ4（結果処理） — 検索なら ResultSet で結果を取り出します（第3章）。
  • ステップ5（後始末） — 使い終わった接続などを閉じてリソースを解放します（第6章・第8章）。

注意: 後始末を忘れない

ステップ5の「後始末（接続を閉じる）」を忘れると、接続が残り続けてデータベースに負担がかかり、最終的にアプリが動かなくなることがあります。本ガイドでは、第6章・第8章で確実に閉じるための書き方（try-with-resources）をしっかり学ぶので安心してください。

やってみよう 1-1: 身近なデータベースを探す

インストール前のこの段階では、まず「データベースがどこで使われているか」を具体的にイメージしてみましょう。全体像を自分の言葉で説明できると、以降の学習がスムーズになります。

手順

  1. 1 普段使うアプリやサービスで「データが保存されていそうな場面」を3つ挙げる（例: SNS の投稿、通販の注文履歴、ゲームのスコア）。
  2. 2 それぞれが「テーブル（行と列）」だとしたら、どんな列がありそうか書き出してみる（例: 注文 = 注文ID・商品名・金額・日付）。
  3. 3 1.4 の図を見ながら、「アプリ → JDBC → ドライバ → DB」の流れを自分の言葉で説明してみる。

理解度確認クイズ

  1. Q1. JDBC とは何か、ひとことで説明してください。

    回答を表示

    Java プログラムからデータベースに接続し、SQL を実行するための「標準 API（共通の窓口）」です。java.sql パッケージとして Java に標準で用意されています。

  2. Q2. JDBC API と JDBC ドライバの役割の違いは何ですか？

    回答を表示

    JDBC API は「共通の窓口（形）」を決める役割で、どの DB でも同じ書き方ができるようにします。JDBC ドライバは「中身」を担当し、特定のデータベースと実際に通信する処理を提供する「翻訳係」です。DB を変えるときはドライバを差し替えます。

  3. Q3. JDBC プログラムの基本5ステップを順番に挙げてください。

    回答を表示

    1. ドライバを準備する → 2. 接続する（Connection を取得）→ 3. SQL を実行する（Statement）→ 4. 結果を処理する（ResultSet）→ 5. 後始末をする（接続を閉じる）。
