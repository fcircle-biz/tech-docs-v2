ホーム JavaScript 第3章
第3章 所要時間: 約60分

はじめてのプログラムを書こう

「Hello World!」という文字を画面やコンソールに表示する最初のプログラムを作成します。コードの書き方・実行方法・基本ルールを体験を通して習得し、「自分のコードが動いた！」という最初の成功体験を得ましょう。

この章で学ぶこと

  • 「Hello World」が最初のプログラムとして世界中で使われる理由を知る
  • console.log() を使ってブラウザのコンソールに文字を表示できる
  • HTML ファイルの <script> タグから JavaScript を実行できる
  • セミコロン・コメント・大文字小文字の区別など、コードの基本ルールを理解する

3.1 はじめての一歩「Hello World」

プログラミングを学ぶとき、ほぼすべての言語で最初に書くプログラムが 「Hello World」 です。画面に「Hello, World!」という文字を表示するだけのシンプルなプログラムですが、これは単なる慣習ではありません。「開発環境が正しく動いているか確認できる」「言語の基本的な書き方を最短で体験できる」という実用的な理由があります。

1978年にブライアン・カーニハンとデニス・リッチーが書いた C 言語の教科書でこのプログラムが紹介されて以来、世界中のプログラマーが Hello World からキャリアをスタートしてきました。あなたもこの章で、その伝統に加わります。

動作確認

コードを書いて実行するまでの一連の流れが正しく機能するかを確かめます。

基礎体験

その言語の「最低限の文法」をたった1行で体験できます。

最初の成功体験

「動いた！」という感動が、学習を続けるモチベーションになります。

3.2 コンソールに表示する

JavaScript で最もよく使う出力方法が console.log() です。ブラウザの「開発者ツール」の「コンソール」タブに文字や数値を表示します。デバッグ（プログラムの動作確認・問題調査）でも毎日使う基本中の基本です。

ブラウザの開発者ツールを開き（Windows: F12、Mac: Cmd+Option+I）、「コンソール」タブを選択して、次のコードを直接入力して Enter を押してみましょう。

ブラウザのコンソール 
// コンソールに文字を表示する
console.log("Hello World!");

// 日本語も表示できる
console.log("こんにちは、世界！");

実行すると、コンソールに Hello World! と表示されます。

console.log() のしくみ

console.log() は「コンソール（console）に表示（log）してください」という命令です。カッコ ( ) の中に表示したいものを渡す書き方を「関数の呼び出し」と言います。文字（文字列）を渡すときは、ダブルクォート " で囲みます。

重要ポイント: 文字列はクォートで囲む

文字のデータ（文字列）はダブルクォート " またはシングルクォート ' で囲みます。クォートで囲まないと、JavaScript は「変数の名前かな？」と解釈してエラーになります。本ガイドではダブルクォートを使います。

3.3 HTMLファイルから実行する

コンソールへの直接入力は手軽ですが、毎回入力し直すのは大変です。実際の開発では HTML ファイルの中に <script> タグを書いて JavaScript を実行します。ファイルをブラウザで開くだけでコードが自動的に実行されます。

VSCode でフォルダを作り、index.html というファイルを新規作成して、次のコードを書いてみましょう。

index.html 
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>はじめての JavaScript</title>
</head>
<body>
    <h1>Hello World ページ</h1>

    <!-- script タグの中に JavaScript を書く -->
    <script>
        console.log("Hello World!");
        alert("Hello World!");
    </script>
</body>
</html>

ファイルを保存してブラウザで開くと、alert() によってポップアップダイアログが表示され、コンソールにも「Hello World!」が出力されます。

<script> タグは </body> の直前に書くのがおすすめ

<script> タグはページのどこにでも書けますが、</body> 直前に書くと「HTML を先に読み込んでからスクリプトを実行する」ため、画面の要素を操作するコードが確実に動きます。本ガイドでもこの位置を使います。

また、document.body.textContent を使うと、ページ本文に直接テキストを書き込むこともできます。

index.html（抜粋） 
// ページの本文テキストを書き換える
// ※ body 内のすべてのテキストが置き換わるため確認用途向け
document.body.textContent = "Hello World!";

注意: alert() はテスト専用

alert() はポップアップを表示する便利な関数ですが、ボタンを押すまで処理が止まるため、実際の Web サービスではほとんど使いません。動作確認の練習用として覚えておきましょう。

3.4 コードの基本ルール

コードを書くときには、守るべき基本的なルールがあります。このルールを知っておくと、エラーを減らし、他の人が読みやすいコードを書けるようになります。

ルール 内容 例
セミコロン 1つの命令（文）の末尾に ; を付ける慣習 console.log("Hi");
1行コメント // から行末まではコメント（実行されない） // これはコメント
複数行コメント /* */ で囲んだ範囲がコメント /* 複数行 */
大文字小文字の区別 JavaScript は大文字と小文字を区別する（case sensitive） console ≠ Console
コメントとセミコロンの例 
// 1行コメント: この行はコードとして実行されない

/*
  複数行コメント:
  長い説明を書きたいときに使う
*/

// 文の末尾にセミコロンを付ける（慣習）
console.log("最初の文");
console.log("2番目の文");

// 大文字小文字を間違えるとエラーになる
// Console.log("エラー！"); // ← C が大文字なので動かない
console.log("正しく動く！"); // ← 小文字の c が正解

セミコロンは省略できるが、付けるのが無難

JavaScript には「自動セミコロン挿入（ASI）」という仕組みがあり、多くの場合セミコロンを省いても動きます。しかし、省略すると予期しないバグが起きる場合があります。本ガイドではセミコロンを付けるスタイルで統一します。

コードが実行されるまでの流れ

flowchart TD A["コードを書く
（エディタ／コンソール）"] --> B["ブラウザがコードを読む"] B --> C["上から1行ずつ実行"] C --> D["console.log → コンソールに表示"] C --> E["alert → ポップアップ表示"] D --> F["実行完了"] E --> F

やってみよう 3-1: 自分の名前を表示する

ブラウザのコンソールを開き、自分の名前を console.log() で表示してみましょう。

手順

  1. 1 ブラウザを開き、F12（Mac は Cmd+Option+I）で開発者ツールを開く。
  2. 2 「コンソール」タブを選択し、下のコードの ○○ を自分の名前に書き換えて入力する。
  3. 3 Enter を押して実行し、名前がコンソールに表示されることを確認する。
コンソールに入力する 
// ○○ を自分の名前に書き換えて実行しよう
console.log("私の名前は○○です");

確認: コンソールに「私の名前は〇〇です」と表示されれば成功です！

理解度確認クイズ

  1. Q1. console.log() はどこに文字を表示しますか？

    回答を表示

    ブラウザの「開発者ツール」の「コンソール」タブに表示します。Web ページ本文には表示されません。動作確認やデバッグで使う出力方法です。

  2. Q2. JavaScript で文字列を書くときは、文字をどのように囲みますか？

    回答を表示

    ダブルクォート "（または シングルクォート '）で囲みます。例: "Hello World!"。クォートで囲まないと JavaScript が変数名として解釈してエラーになります。

  3. Q3. 次の2つのコードのうち、正しく実行できるのはどちらですか？ また、もう一方はなぜ動かないのでしょうか？
    ① Console.log("Hi"); ／ ② console.log("Hi");

    回答を表示

    正しいのは ② です。JavaScript は大文字と小文字を区別するため、Console（C が大文字）は存在しない名前として扱われ、エラーになります。正しくは console（すべて小文字）です。
