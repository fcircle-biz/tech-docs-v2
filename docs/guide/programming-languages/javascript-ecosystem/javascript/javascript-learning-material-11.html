ホーム JavaScript 第11章
第11章 所要時間: 約90分

入力フォームを扱おう

テキストボックスやボタンなどのフォーム部品から値を取得し、入力内容に応じて画面を更新する方法を学びます。第9章の DOM 操作・第10章のイベント処理と組み合わせることで、ユーザーが入力した情報を受け取って画面に反映する、アプリらしい動きが実現できます。

この章で学ぶこと

  • フォーム部品（input・button）の役割と、id を付けて JS から扱う方法を理解する
  • .value プロパティで入力値を取得できる
  • ボタンクリックで入力値を取得し、別の要素に表示する統合例を作れる
  • 入力値を変数や配列に入れてためる流れを理解し、第12章への準備を整える

11.1 フォーム部品とは何か

フォーム部品とは、ユーザーが情報を入力するための HTML 要素です。代表的なものにテキストボックス（<input type="text">）とボタン（<button>）があります。JavaScript からこれらを操作するには、id 属性を付けて getElementById で取得します（第9章で学んだ方法と同じです）。

テキストボックス

<input type="text"> — 文字を入力できる一行テキスト欄。

ボタン

<button> — クリックするとイベントを発生させる。

id 属性で識別

id="myInput" を付けると JS から名前で取得できる。

HTML 側でフォーム部品を用意するときは、次のようなシンプルな構造になります。

index.html（フォーム部品の例） 
<!-- テキストボックスと、結果を表示する段落 -->
<input type="text" id="nameInput" placeholder="名前を入力">
<button id="greetBtn">あいさつ</button>
<p id="result"></p>

placeholder（プレースホルダー）とは

placeholder 属性には、ユーザーが何も入力していないときに薄く表示されるヒントテキストを指定します。実際の入力値ではないため、JavaScript で .value を取得すると空文字（""）が返ってきます。

11.2 入力値を取得する: .value プロパティ

input 要素を JavaScript で取得したら、.value プロパティにアクセスすることで、ユーザーが入力した文字列を取り出せます。まずは console.log で確認するところから始めましょう。

script.js（.value の基本） 
// id="nameInput" の要素を取得
const inputEl = document.getElementById("nameInput");

// .value で入力された文字列を取得
const text = inputEl.value;

// コンソールに表示して確認
console.log(text); // 例: "田中" と表示される

.value は常に文字列（string）を返します。数字を入力しても "42" のような文字列になるため、計算に使いたい場合は Number() で変換する必要があります（今回は文字列のまま使います）。

重要ポイント: .value は「今この瞬間」の値

.value にアクセスした時点でテキストボックスに入っている文字列が返ります。ボタンを押す前に取得しても空文字になるだけです。ボタンクリックのイベント内で取得するのが基本パターンです。

11.3 入力に応じて画面を更新する

第9章（DOM 操作）と第10章（イベント処理）で学んだことを組み合わせると、「ボタンを押した瞬間に入力値を取得して別の要素へ表示する」という処理が書けます。実際のコードで確認しましょう。

index.html（あいさつ表示の土台） 
<!-- 名前入力フォーム -->
<input type="text" id="nameInput" placeholder="名前を入力">
<button id="greetBtn">あいさつ</button>
<p id="result"></p>

<!-- JS ファイルを読み込む -->
<script src="script.js"></script>
script.js（あいさつ表示のロジック） 
const btn = document.getElementById("greetBtn");
const resultEl = document.getElementById("result");

// ボタンクリックで処理を実行
btn.addEventListener("click", function () {
  const inputEl = document.getElementById("nameInput");
  const name = inputEl.value; // 入力値を取得

  // 空入力チェック
  if (name === "") {
    resultEl.textContent = "名前を入力してください。";
    return; // 処理をここで終了
  }
  // 入力があれば画面に表示
  resultEl.textContent = "こんにちは、" + name + "さん！";
});

動作確認のポイント

  • 「田中」と入力してボタンを押すと 「こんにちは、田中さん！」 と表示される
  • 何も入力せずにボタンを押すと 「名前を入力してください。」 と表示される
  • 別の名前を入力してもう一度ボタンを押すと、表示が書き換わる

空入力チェックの書き方: return で早期終了

if (name === "") { ... return; } は「空だったらメッセージを出してそれ以上の処理を止める」パターンです。return によって関数の実行がそこで終了するため、以降のあいさつ表示は実行されません。このような書き方をガード節と呼ぶことがあります。

11.4 値を変数や配列に入れる

取得した入力値を変数に入れるだけでなく、配列に push して蓄積することもできます。これが次の第12章「配列と DOM の組み合わせ」の基礎になり、最終的にはToDo アプリでタスクをためる仕組みへとつながります。

script.js（入力値を配列にためる） 
// 入力値をためる配列（最初は空）
const names = [];

const btn = document.getElementById("addBtn");

btn.addEventListener("click", function () {
  const inputEl = document.getElementById("nameInput");
  const name = inputEl.value;

  if (name === "") return; // 空入力は無視

  names.push(name);    // 配列に追加
  inputEl.value = "";  // 入力欄を空にリセット
  console.log(names);  // 蓄積された名前を確認
});

inputEl.value = "" は入力欄の内容を空にするための代入です。ユーザーが次の入力をしやすくするため、ボタンクリック後に欄をリセットするのは定番パターンです。

第12章へのつながり

この章では「配列にためるところまで」を学びました。第12章では「配列の中身を HTML の一覧として画面に表示する」方法を学びます。

  • 配列をループして li 要素を動的に生成する
  • ToDo アプリの「タスク追加 → 一覧表示」の土台になる

よくある間違い: ページ読み込み直後に .value を取得する

スクリプトがページ読み込みと同時に実行されると、まだユーザーが何も入力していないため .value は空文字になります。必ずボタンクリックなどのイベント内で .value を取得するように書きましょう。

やってみよう 11-1: 好きな言葉を表示するミニ機能を作る

テキストボックスに好きな言葉を入力し、ボタンを押すと画面に表示される機能を作ってみましょう。空入力チェックも忘れずに実装してください。

手順

  1. 1 HTML ファイルに input・button・p を追加し、それぞれに id を付ける（例: wordInput・showBtn・output）。
  2. 2 JavaScript で showBtn にクリックイベントを設定し、wordInput.value を取得する。
  3. 3 空文字の場合は「言葉を入力してください。」と表示し、そうでなければ取得した言葉を output.textContent に設定する。
  4. 4 ブラウザで開いて動作を確認したら、入力欄をリセット（wordInput.value = ""）する処理も追加してみよう。

実装例

script.js（実装例） 
const showBtn = document.getElementById("showBtn");
const output = document.getElementById("output");

showBtn.addEventListener("click", function () {
  const wordInput = document.getElementById("wordInput");
  const word = wordInput.value;

  if (word === "") {
    output.textContent = "言葉を入力してください。";
    return;
  }
  output.textContent = word; // 入力された言葉を表示
  wordInput.value = "";      // 入力欄をリセット
});

理解度確認クイズ

  1. Q1. .value プロパティは何を表しますか？また、どんな型のデータが返りますか？

    回答を表示

    .value は input 要素にユーザーが入力している文字列を表します。返されるデータは常に文字列（string）型です。数字を入力しても "42" という文字列になるため、計算に使う場合は Number() などで変換が必要です。

  2. Q2. テキストボックスの入力値を取得して画面に表示するには、どのような順番で操作しますか？

    回答を表示

    ① HTML に id 付きの input 要素と表示先の要素を用意する → ② getElementById で input 要素を取得する → ③ ボタンのクリックイベント内で .value を取得する → ④ 表示先要素の textContent に代入する、という手順です。

  3. Q3. 空入力チェックはなぜ必要で、どのように書きますか？

    回答を表示

    何も入力されていない状態でボタンが押された場合に不正な処理（空のデータの登録など）を防ぐために必要です。if (text === "") { /* エラーメッセージ */ return; } のように、空文字と比較して一致すれば return で早期終了するのが基本パターンです。
