ホーム JavaScript 第4章
第4章 所要時間: 約90分

変数で情報を記憶しよう

変数という「箱」を使って、数値・文字列などの情報をプログラムの中で記憶・管理する方法を学びます。let と const の使い分け、データ型の基礎、テンプレートリテラルによる文字列の組み立てまで、初心者向けに丁寧に解説します。

この章で学ぶこと

  • 変数とは何か、なぜ必要なのかを「ラベル付きの箱」のたとえで理解する
  • let で変数を宣言・代入し、後から値を変える再代入を使える
  • const で変わらない値を宣言し、let と使い分けられる
  • 数値・文字列・真偽値の3つの基本データ型と、テンプレートリテラルを使える

4.1 変数とは — ラベル付きの箱

プログラムを書いていると、「名前」「年齢」「スコア」など、さまざまな情報を一時的に覚えておく必要があります。そのために使う仕組みが変数（へんすう）です。変数とは、値（データ）に名前を付けて保存する「ラベル付きの箱」のようなものです。

箱に「age（年齢）」というラベルを貼って、中に 20 という数字を入れておく——これが変数のイメージです。あとで箱の中身を取り出したり、別の値に入れ替えたりできます。

flowchart LR A["変数名: age
中身: 20"] -->|"後から変更可能"| B["変数名: age
中身: 21"] C["変数名: name
中身: 太郎"] -->|"中身を取り出して使う"| D["console.log（name）
→ 太郎 と表示"]

変数を使うと、次の3つのメリットがあります。

後で使える

一度保存した値を、プログラムの別の場所でも呼び出して使えます。

繰り返し使える

同じ値を何度も書かずに、変数名1つで参照できます。

意味が分かる

数字 20 より age と書いたほうが、何の値かが一目で分かります。

4.2 let で変数を作る

JavaScript で変数を作るには、let キーワードを使います。「let（レット）」は英語で「〜に〜させる」という意味で、「この名前に、この値を持たせる」と宣言しています。

sample.js 
// 変数の宣言と代入（同時に行う）
let age = 20;        // age という箱に 20 を入れる
let score = 85;      // score という箱に 85 を入れる

console.log(age);    // 結果: 20
console.log(score);  // 結果: 85

// 後から値を変える（再代入）
age = 21;            // age の中身を 21 に変える
console.log(age);    // 結果: 21

let 変数名 = 値; というパターンを宣言と代入といいます。= は「等しい」ではなく「右の値を左の変数に入れる」という意味です（算数の「＝」とは使い方が違います）。

変数名の付け方のルール

  • 英字・数字・_（アンダースコア）・$（ドル記号）が使える（数字から始めるのは禁止）
  • 何の値かが分かる意味のある名前にする（x より score のほうが分かりやすい）
  • 2単語以上のときは camelCase（キャメルケース）を使う — 例: firstName、totalScore
  • let・const・if などの予約語は変数名に使えない

4.3 const で変わらない値を宣言する

const（コンスト）は一度決めたら変更できない変数を宣言するキーワードです。「constant（コンスタント）＝定数」の略で、途中で値を変える必要がないものに使います。

sample.js 
// const で宣言した変数は変更できない
const name = "太郎";
console.log(name);  // 結果: 太郎

// 以下はエラーになる（再代入禁止）
// name = "次郎";   // TypeError: Assignment to constant variable.

// let は後から変更できる
let point = 0;
point = 100;        // OK: 再代入できる
キーワード 再代入 使う場面 例
const 不可 値が変わらないとき（基本はこちら） const name = "太郎";
let 可能 後で値を変える必要があるとき let score = 0;

重要ポイント: const を優先して使おう

迷ったらまず const を使い、後から値を変える必要が出てきたときだけ let に切り替える習慣をつけましょう。「変えるつもりのない値が意図せず変わってしまう」というバグを防ぐことができます。

4.4 値の種類（データ型）の基礎

変数に入れられる値にはいくつかの種類（データ型）があります。ここでは最初に知っておくべき3つの基本型を学びましょう。

型名 英語名 例 特徴
数値 number 20 / 3.14 整数・小数を区別しない
文字列 string "太郎" / 'Hello' クォート（" " または ' '）で囲む
真偽値 boolean true / false 「はい/いいえ」の2択。クォート不要
data-types.js 
// 数値型 (number) — クォートなし
const age = 20;
const price = 1980.5;

// 文字列型 (string) — クォートで囲む
const name = "太郎";
const greeting = 'こんにちは';

// 真偽値型 (boolean) — true または false
const isStudent = true;
const isAdult = false;

console.log(age);        // 20
console.log(name);       // 太郎
console.log(isStudent);  // true

文字列の連結とテンプレートリテラル

文字列どうしは + でつなぐことができます。しかし、変数が増えると読みにくくなります。そこで便利なのがテンプレートリテラルです。バッククォート（` `）で囲み、${変数名} で変数の値を埋め込めます。

template-literal.js 
const name = "太郎";
const age = 20;

// + による連結（少し読みにくい）
console.log("こんにちは、" + name + "さん。" + age + "歳ですね。");

// テンプレートリテラル（バッククォートで囲む）
console.log(`こんにちは、${name}さん。${age}歳ですね。`);
// 結果: こんにちは、太郎さん。20歳ですね。

バッククォートはどこにある？

バッククォート（`）は日本語キーボードではShift + @（アットマーク）で入力できます。シングルクォート（'）やダブルクォート（"）と見た目が似ているので、打ち間違えに注意しましょう。

よくある間違い: 数値と文字列の混在

const age = "20"; のように数値をクォートで囲むと文字列として扱われます。"20" + 5 は 25 ではなく "205"（文字列の連結）になります。計算に使う値はクォートなしで書きましょう。

やってみよう 4-1: 自己紹介文を作ろう

自分の名前・年齢・好きな食べ物を変数に入れて、テンプレートリテラルで自己紹介文を作り、コンソールに表示させましょう。

手順

  1. 1 VSCode で practice04.js を新規作成し、ブラウザの HTML ファイルから読み込む（第2〜3章の手順を参考に）。
  2. 2 下のサンプルコードをヒントに、変数に自分の情報を入れてみよう（値は自由に変えてOK）。
  3. 3 ブラウザの開発者ツール（コンソール）を開いて、自己紹介文が表示されるか確認しよう。
practice04.js 
// 自分の情報を変数に入れる
const myName = "太郎";       // 自分の名前に変えよう
const myAge = 20;             // 自分の年齢に変えよう
const favoriteFood = "ラーメン"; // 好きな食べ物に変えよう

// テンプレートリテラルで自己紹介文を作る
const intro = `はじめまして！${myName}です。${myAge}歳です。好きな食べ物は${favoriteFood}です。`;

console.log(intro);

期待される出力（例）

はじめまして！太郎です。20歳です。好きな食べ物はラーメンです。

理解度確認クイズ

  1. Q1. let と const の違いは何ですか？どちらを優先して使うべきですか？

    回答を表示

    let は後から値を変える（再代入する）ことができますが、const は一度宣言したら変更できません。基本的には const を優先して使い、後で値を変える必要があるときだけ let を使います。これにより意図しない値の変更を防げます。

  2. Q2. 変数を使う3つのメリットを答えてください。

    回答を表示

    ①後で使える（一度保存した値をプログラムの別の場所でも使える）、②繰り返し使える（同じ値を何度も書かずに変数名で参照できる）、③意味が分かる（20 より age と書いたほうが何の値か一目で分かる）の3つです。

  3. Q3. テンプレートリテラルとは何ですか？通常の文字列連結（+）と比べた利点を答えてください。

    回答を表示

    バッククォート（` `）で文字列を囲み、${変数名} で変数の値を文字列の中に直接埋め込める機能です。+ で連結するより記述がシンプルで読みやすく、変数が多くなるほど効果を発揮します。
