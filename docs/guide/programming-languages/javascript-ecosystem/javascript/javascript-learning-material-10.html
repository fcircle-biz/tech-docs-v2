ホーム JavaScript 第10章
第10章 所要時間: 約90分

ユーザーの操作に反応しよう（イベント処理）

ボタンのクリックやキーボード入力など、ユーザーの操作（イベント）に反応して処理を実行する方法を学びます。前章で学んだ DOM 操作と組み合わせることで、触ると反応するインタラクティブな Web ページを作れるようになります。

この章で学ぶこと

  • イベントとは何かを説明でき、身近な操作と結びつけて理解できる
  • addEventListener を使ってクリックイベントを登録できる
  • DOM 操作と組み合わせてボタン操作で画面を変化させることができる
  • click・input・mouseover など代表的なイベントの種類を知る

10.1 イベントとは何か

Web ページを使っていると、「ボタンをクリックするとメニューが開く」「文字を入力すると候補が表示される」といった動きを日常的に体験しています。こうした動きを実現するのがイベント処理です。

イベントとは、ユーザーの操作（クリック・入力・マウス移動など）や、ページの状態変化（読み込み完了など）が起きたときに発生する「合図」のことです。JavaScript は「この合図が来たら、この処理を実行する」という命令を事前に登録しておくことで、インタラクティブな動きを実現します。

操作の「合図」

クリック・入力・マウス移動など、ユーザーの行動が合図になります。

事前に登録する

「合図が来たらこの処理を実行する」と、あらかじめ JavaScript に伝えておきます。

画面が変わる

DOM 操作と組み合わせることで、操作に応じて画面の表示を変えられます。

イベント処理の流れ

flowchart TD A["ユーザーの操作
（クリック・入力など）"] --> B["イベント発生"] B --> C["登録した関数
（コールバック）を実行"] C --> D["DOM操作などの処理"] D --> E["画面の表示を更新"]

「インタラクティブ」とは？

英語の "interactive"（相互に作用する）から来た言葉で、ユーザーの操作に対してページが反応し、双方向にやりとりできる状態を指します。イベント処理を使うことで、ページが一方的に情報を表示するだけでなく、ユーザーの操作に応えられるようになります。

10.2 addEventListener でイベントを登録する

JavaScript でイベントを登録するには、addEventListener というメソッドを使います。「addEventListener」は「イベントリスナー（聞き役）を追加する」という意味で、「この要素でこのイベントが起きたら、この関数を実行して」と登録する命令です。

構文: addEventListener の基本形

flowchart LR A["要素"] --> B["addEventListener（イベント名, 実行する関数）"] B --> C["イベント発生時に関数が呼ばれる"]

addEventListener には2つの引数を渡します。第1引数はイベント名（文字列）、第2引数は実行したい関数です。

event-basic.js 
// ボタン要素を取得する
const btn = document.getElementById("myButton");

// クリックイベントを登録する（function式）
btn.addEventListener("click", function() {
    // クリックされたときに実行される処理
    alert("ボタンがクリックされました！");
});

第2引数の関数は、イベントが発生した時に自動で呼び出されます。このように「後で呼び出してもらうために渡す関数」のことをコールバック関数と呼びます。アロー関数を使って書くこともできます。

event-arrow.js 
const btn = document.getElementById("myButton");

// アロー関数を使って書くこともできる（動作は同じ）
btn.addEventListener("click", () => {
    alert("ボタンがクリックされました！");
});

用語：コールバック関数

「後から呼び出してもらうために渡す関数」のことです。レストランで「料理ができたら声をかけてください」と伝えるイメージです。addEventListener に渡した関数は、イベントが発生した瞬間に JavaScript が自動で呼び出してくれます。

10.3 クリックで DOM を操作する

イベント処理の本領は、第9章で学んだ DOM 操作と組み合わせることで発揮されます。「クリックされたら、ページの文字を変える・数を増やす・色を変える」といった処理をまとめて実装できます。

例1: ボタンでカウントアップ

ボタンをクリックするたびに数値が増えていく、シンプルなカウンターを作ってみましょう。HTML の土台と、JavaScript の処理をセットで確認します。

counter.html（抜粋） 
<!-- カウント表示エリアとボタン -->
<p id="count">0</p>
<button id="countBtn">+1 カウント</button>
counter.js 
// 現在のカウントを記憶する変数
let count = 0;

const countEl = document.getElementById("count");
const countBtn = document.getElementById("countBtn");

// クリックのたびに count を増やして表示する
countBtn.addEventListener("click", () => {
    count = count + 1;          // カウントを1増やす
    countEl.textContent = count; // 表示を更新する
});

例2: ボタンで文字色を変える

次は、ボタンをクリックするたびにテキストの色が切り替わる例です。DOM の classList と組み合わせると、スタイルの切り替えも簡単に行えます。

toggle-color.js 
const msg = document.getElementById("message");
const toggleBtn = document.getElementById("toggleBtn");

// クリックで文字色を赤⇔黒に切り替える
toggleBtn.addEventListener("click", () => {
    // classList.toggle: クラスがあれば外し、なければ追加する
    msg.classList.toggle("text-red");
});

DOM 操作 ＋ イベント処理 = インタラクティブなページ

第9章の DOM 操作は「画面を書き換える方法」、今章のイベント処理は「いつ書き換えるかのきっかけ」を担当します。この2つを組み合わせることが、動く Web ページを作る基本パターンです。

10.4 さまざまなイベントの種類

addEventListener の第1引数に渡す「イベント名」には、操作の種類によってさまざまな値が用意されています。代表的なものを確認しておきましょう。

イベント名 発生するタイミング よく使う場面
click 要素がクリックされたとき ボタン操作・メニュー開閉
input 入力欄の値が変わるたびに リアルタイム検索・入力プレビュー
mouseover マウスカーソルが要素に乗ったとき ツールチップ表示・ホバーエフェクト
mouseout マウスカーソルが要素から離れたとき ホバーエフェクトの解除
keydown キーが押されたとき ショートカットキー・ゲーム操作
submit フォームが送信されたとき 入力値の検証・送信処理

この中でも input イベントは、テキストボックスに1文字入力するたびに発火するため、リアルタイムに画面を更新したいときに便利です。次章の「入力フォームを扱おう」で詳しく活用します。

input-event.js 
const inputEl = document.getElementById("nameInput");
const previewEl = document.getElementById("preview");

// 入力のたびにプレビューを更新する（input イベント）
inputEl.addEventListener("input", () => {
    // event.target.value で入力値を取得できる（次章で詳しく学習）
    previewEl.textContent = "入力中: " + inputEl.value;
});

input と change の違い

input は1文字変わるたびに発火しますが、change は入力欄からフォーカスが外れた（確定した）タイミングで発火します。リアルタイム更新には input、確定後の処理には change を使い分けましょう。

やってみよう 10-1: クリックで変わるカウンターを作る

ボタンを押すたびに数字が増える（または減る）カウンターを自分で作ってみましょう。DOM 操作とイベント処理を組み合わせる基本パターンを体で覚えます。

手順

  1. 1 HTML に <p id="count">0</p> と <button id="countBtn">+1</button> を追加する。
  2. 2 JavaScript ファイルを作り、let count = 0; で変数を用意する。
  3. 3 ボタンに addEventListener("click", ...) を登録し、クリックで count を1増やして textContent を更新する。
  4. 4 動作確認できたら、「-1」ボタンも追加して count を減らす処理も実装してみよう。

参考: HTML の土台

<button id="plusBtn">+1</button>
<button id="minusBtn">-1</button>

理解度確認クイズ

  1. Q1. Web ページにおける「イベント」とは何ですか？身近な例を1つあげて説明してください。

    回答を表示

    イベントとは、ユーザーの操作やページの状態変化が起きたときに発生する「合図」のことです。例えば「ボタンをクリックする（click イベント）」「テキストボックスに文字を入力する（input イベント）」などが身近な例です。

  2. Q2. addEventListener の第1引数と第2引数にはそれぞれ何を渡しますか？

    回答を表示

    第1引数にはイベント名（文字列）（例: "click"）、第2引数にはイベント発生時に実行したい関数（コールバック関数）を渡します。

  3. Q3. input イベントはどのタイミングで発生しますか？click イベントとの違いも含めて説明してください。

    回答を表示

    input イベントは入力欄の値が1文字でも変わるたびに発生します。click イベントはボタンなどをクリック（タップ）したときに発生する点が異なります。input はリアルタイム検索やプレビュー表示によく使われます。
