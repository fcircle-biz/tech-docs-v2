処理をまとめよう（関数の基礎）

よく使う処理を「関数」としてまとめ、必要なときに呼び出して使う方法を学びます。引数で外から値を渡し、戻り値で結果を受け取るしくみを理解することで、読みやすくて再利用しやすいコードが書けるようになります。

この章で学ぶこと

  • 関数とは何か、なぜ使うのかを身近なたとえで説明できる
  • function キーワードで関数を定義し、呼び出せる
  • 引数（パラメータ）を使って関数に値を渡せる
  • return で計算結果や値を呼び出し元へ返せる
  • 関数を使うメリット（再利用・読みやすさ・修正の容易さ）を理解する

7.1 関数とは何か

関数（かんすう）とは、「よく使う処理に名前をつけてひとまとめにした箱」です。同じ処理を何度も書き直す代わりに、1回だけ定義しておいて、必要なときに名前で呼び出すことができます。

ボタンを押す

＝ 関数を呼び出す。「この処理を実行して！」という命令。

お金を入れる

＝ 引数（ひきすう）。処理に必要な値を外から渡す。

商品が出てくる

＝ 戻り値（もどりち）。処理の結果として返ってくる値。

自動販売機は「お金を入れてボタンを押すと、商品が出てくる」という処理を何度でも繰り返してくれます。中の仕組みを知らなくても使えるのがポイントです。関数もまったく同じで、「引数を渡して呼び出すと、戻り値として結果が返ってくる」しくみになっています。

用語の整理

関数を定義する＝処理の中身を書いておくこと。関数を呼び出す＝定義した処理を実際に実行すること。定義だけでは動かず、呼び出して初めて処理が実行されます。

7.2 関数の定義と呼び出し

JavaScript で関数を定義するには function キーワードを使います。定義した関数は、関数名に () を付けて呼び出します。

関数の基本構文 

function 関数名() {
    // ここに処理を書く
}

関数名(); // 呼び出し

実際に挨拶を表示する関数を書いてみましょう。

greet.js 
// 挨拶を表示する関数を定義する
function greet() {
    console.log("こんにちは！");
}

// 関数を呼び出す（何度でも使える）
greet(); // → こんにちは！
greet(); // → こんにちは！
greet(); // → こんにちは！

関数を1回定義しておけば、greet() と書くだけで何度でも同じ処理を実行できます。処理の中身を何十行書いても、呼び出し側はたった1行で済みます。

重要ポイント: 定義と呼び出しは別

function greet() {...} は「設計図を作る」だけです。greet() と書いて初めて処理が実行されます。設計図があっても呼び出さないと動きません。

7.3 引数（パラメータ）— 外から値を渡す

関数の () の中にパラメータ（引数を受け取る変数）を書くと、呼び出すときに値を渡せるようになります。渡した値はパラメータという名前の変数として関数の中で使えます。

greet-name.js 
// name という引数を受け取る関数
function greet(name) {
    console.log("こんにちは、" + name + "さん！");
}

// 呼び出し時に値を渡す（引数）
greet("田中");   // → こんにちは、田中さん！
greet("佐藤");   // → こんにちは、佐藤さん！
greet("鈴木");   // → こんにちは、鈴木さん！

引数は複数渡すこともできます。カンマ , で区切って並べましょう。

multi-params.js 
// 2つの引数を受け取る関数
function introduce(name, age) {
    console.log(name + "さんは" + age + "歳です。");
}

introduce("山田", 25); // → 山田さんは25歳です。
introduce("高橋", 30); // → 高橋さんは30歳です。

「パラメータ」と「引数」の違い

パラメータは関数を定義するときに書く受け取り用の変数名（例: name）。引数は関数を呼び出すときに実際に渡す値（例: "田中"）。どちらも「引数」と呼ぶことが多いので、あまり厳密に区別しなくて大丈夫です。

7.4 戻り値（return）— 結果を返す

return を使うと、関数の処理結果を呼び出し元へ「返す」ことができます。返された値は変数に入れたり、そのまま計算に使ったりできます。

引数 → 関数（処理） → 戻り値 の流れ

flowchart LR A["引数
（渡す値）"] -->|"add（2, 3）"| B["関数の処理
2 ＋ 3"] B -->|"return 5"| C["戻り値
（返ってくる値）"] C --> D["result = 5
（変数に代入）"]
add.js 
// 2つの数を足して結果を返す関数
function add(a, b) {
    return a + b; // return で値を返す
}

// 戻り値を変数に受け取る
const result = add(2, 3);
console.log(result); // → 5

// 戻り値をそのまま使う
console.log(add(10, 20)); // → 30

もう少し実用的な例として、消費税込みの金額を計算する関数を作ってみましょう。

tax.js 
// 消費税込み金額を計算する関数（税率10%）
function calcWithTax(price) {
    return Math.round(price * 1.1); // 小数点以下を四捨五入
}

console.log(calcWithTax(100));  // → 110
console.log(calcWithTax(980));  // → 1078
console.log(calcWithTax(1500)); // → 1650

注意: return の後は実行されない

return を実行すると、その時点で関数が終了します。return の後に書いたコードは実行されないので注意しましょう。

7.5 関数を使うメリット

関数にはなぜまとめるのか、3つの大きなメリットがあります。

再利用できる

同じ処理を何度でも呼び出せる。書く量が減り、コードが短くなります。

読みやすくなる

処理に名前が付くので「何をしているか」が一目でわかります。

修正が1箇所で済む

バグを直したり仕様を変えたりするとき、関数の中身だけ直せばOKです。

また、JavaScript には アロー関数という短い書き方もあります。以下のように書けます（動作はほぼ同じです）。

arrow.js 
// 通常の関数
function add(a, b) {
    return a + b;
}

// アロー関数（同じ処理を短く書ける）
const addArrow = (a, b) => a + b;

console.log(add(2, 3));       // → 5
console.log(addArrow(2, 3));  // → 5

アロー関数について

アロー関数は今後よく見かける書き方です。今は「こういう短い書き方があるんだな」と知っておくだけで十分です。基本的な動きは function で定義した関数と変わりません。詳しくは上級ガイドで学びます。

やってみよう 7-1: 自分で関数を作る

引数と戻り値を使った関数を自分で書いてみましょう。2つの課題のうち、どちらか1つでも動かせれば合格です。

課題 A: 大きいほうを返す関数

  1. 1 2つの数値 a、b を引数に受け取る関数 max を定義する。
  2. 2 if 文で大きいほうの数値を return する。
  3. 3 console.log(max(5, 8)) が 8 になることを確認する。

課題 B: 自己紹介文を返す関数

  1. 1 名前を引数に受け取る関数 selfIntro を定義する。
  2. 2 「はじめまして、〇〇です。よろしくお願いします。」という文字列を return する。
  3. 3 console.log(selfIntro("山田")) で自己紹介文が表示されることを確認する。

期待される出力例

// 課題 A
console.log(max(5, 8));   // → 8
console.log(max(10, 3));  // → 10

// 課題 B
console.log(selfIntro("山田"));
// → はじめまして、山田です。よろしくお願いします。

理解度確認クイズ

  1. Q1. 引数と戻り値の違いを説明してください。

    回答を表示

    引数は関数を呼び出すときに「外から渡す値」です（自動販売機でいうお金）。戻り値は関数の処理が終わったあとに「呼び出し元へ返す値」です（自動販売機から出てくる商品）。引数は「入力」、戻り値は「出力」と覚えると分かりやすいです。

  2. Q2. 関数を使う3つのメリットを挙げてください。

    回答を表示

    ①再利用できる（同じ処理を何度でも呼び出せる）、②コードが読みやすくなる（処理に名前が付くので意図が伝わりやすい）、③修正が1箇所で済む（バグや仕様変更があっても関数の中身だけ直せばよい）。

  3. Q3. return を書かなかった場合、関数の戻り値はどうなりますか？

    回答を表示

    return を書かなかった場合（または return; だけ書いた場合）、戻り値は undefined（値が存在しないことを示す特別な値）になります。console.log のように「結果を返す必要がない」関数では return を省略しても問題ありません。
