ToDoアプリを作ろう① 画面を作る

ToDoアプリの土台となるHTMLとCSSを作成します。入力欄・追加ボタン・タスク一覧エリアを用意し、JavaScriptで操作しやすい画面構成を整えます。この章ではまだロジックは書かず、画面の準備に集中します。

この章で学ぶこと

  • ToDoアプリ3章構成のロードマップを理解し、完成イメージをつかむ
  • タスク入力・追加ボタン・一覧エリアを含むHTMLの土台を作れる
  • CSSで入力欄・ボタン・一覧の見た目を整えられる
  • scriptタグを正しい位置（body末尾）に置く理由を説明できる

14.1 完成イメージと全体設計

第14〜16章の3章をかけて、ToDoアプリを一緒に作ります。ToDoアプリは「やることリスト」管理アプリで、第1〜13章で学んだHTML・CSS・JavaScriptの知識を総動員する総合演習です。

まず、完成形のイメージと3章の役割分担を確認しましょう。全体像を先に知っておくと、各章で自分が「今どこを作っているか」が分かり、迷わず進められます。

第14章は「画面の土台づくり」です。JavaScriptのロジックはまだ書きません。まずHTMLで画面の構造を作り、CSSで見た目を整え、「JavaScriptで操作しやすい状態」を準備することに集中します。

14.2 HTMLの土台を作る

ToDoアプリは1つのHTMLファイル（index.html）で完結します。JavaScriptから操作する要素には必ずid属性を付けておくことが重要です。idがあると、後でdocument.getElementById()で簡単に取得できます。

なぜidを付けるのか

HTMLだけでは画面を「表示する」だけです。JavaScriptで「値を取得する」「内容を書き換える」には、操作したい要素をJavaScriptが識別できる必要があります。id属性はその識別タグの役割を果たします。

ToDoアプリに必要なHTML要素は大きく3つです。

要素 id 役割
<input> taskInput タスク名を入力するテキストボックス
<button> addBtn タスクを追加するボタン
<ul> taskList タスクを一覧表示するリストエリア

実際のHTMLを見てみましょう。コードを2つに分けて確認します。まず<head>と<body>の骨格から始めます。

index.html（前半：head・bodyの開始） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>ToDoアプリ</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1>ToDoアプリ</h1>
  <!-- 14.2〜14.4 の内容をここに追加 -->
</body>
</html>

次に、入力欄・ボタン・リストエリアをbody内に追加します。

index.html（後半：入力・ボタン・リスト） 
  <div class="container">
    <!-- タスク入力欄（JSから値を取得するためidを付ける） -->
    <input id="taskInput" type="text" placeholder="タスクを入力">

    <!-- 追加ボタン（JSでクリックを検知するためidを付ける） -->
    <button id="addBtn">追加</button>

    <!-- タスク一覧（JSでli要素を動的に追加するためidを付ける） -->
    <ul id="taskList"></ul>
  </div>

todo-app/
├── index.html   ← 画面の構造（HTML）
├── style.css    ← 見た目（CSS）
└── app.js       ← 動き（JavaScript、第15章から記述）

14.3 CSSで見た目を整える

HTMLだけでは見た目が簡素なので、CSSで整えます。中央寄せ・余白・ボタンのスタイル・リストの装飾を加えると、より使いやすい画面になります。

style.css 
/* 全体を中央寄せ・余白を設定 */
.container {
  max-width: 480px;
  margin: 40px auto;
  padding: 24px;
  background: #f9fafb;
  border-radius: 8px;
}
/* 入力欄のスタイル */
#taskInput {
  width: 70%;
  padding: 8px;
  border: 1px solid #ccc;
  border-radius: 4px;
}
/* 追加ボタンのスタイル */
#addBtn {
  padding: 8px 16px;
  background: #3b82f6;
  color: #fff;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}
style.css（続き：リストのスタイル） 
/* タスクリストの装飾 */
#taskList {
  list-style: none;
  padding: 0;
  margin-top: 16px;
}
/* 各タスクアイテムのスタイル */
#taskList li {
  padding: 8px 12px;
  margin-bottom: 6px;
  background: #fff;
  border: 1px solid #e5e7eb;
  border-radius: 4px;
  display: flex;
  justify-content: space-between;
}

CSSの書き方のポイント

id属性に対するCSSは #taskInput のように先頭に # を付けて書きます。classに対しては .container のようにドット（.）を付けます。このルールは第3章で学んだHTMLの基礎の延長です。

14.4 JavaScriptを読み込む準備

最後に、JavaScriptファイル（app.js）を読み込む<script>タグを追加します。位置は</body>の直前が正解です。

なぜ </body> 直前に置くのか

ブラウザはHTMLを上から順番に読み込みます。<head>内にscriptを置くと、HTMLのinputやulがまだ読み込まれていない段階でJavaScriptが実行され、「要素が見つからない」エラーが起きます。</body>直前に置けば、全要素が読み込まれた後に実行されるので安全です。

index.html（scriptタグを追加） 
  <div class="container">
    <input id="taskInput" type="text" placeholder="タスクを入力">
    <button id="addBtn">追加</button>
    <ul id="taskList"></ul>
  </div>

  <!-- </body>直前に置くことでDOM読み込み後に実行される -->
  <script src="app.js"></script>
</body>

この章ではまだロジックを書きませんが、要素が正しく取得できるか確認するために、app.jsに以下のコードを書いてコンソールで確認してみましょう。

app.js（要素取得の確認コード） 
// 各要素を取得してコンソールに表示する（動作確認用）
const taskInput = document.getElementById('taskInput');
const addBtn    = document.getElementById('addBtn');
const taskList  = document.getElementById('taskList');

// nullでなければ取得成功
console.log(taskInput); // <input id="taskInput" ...> が表示される
console.log(addBtn);    // <button id="addBtn"> ...  が表示される
console.log(taskList);  // <ul id="taskList"></ul>   が表示される

動作確認の手順

  1. 1 index.htmlをブラウザで開く
  2. 2 F12キーで開発者ツールを開き「コンソール」タブを選ぶ
  3. 3 HTML要素（<input>・<button>・<ul>）が3行表示されれば成功。nullが出たらidのスペルを確認する

コンソールに null が表示されたら

nullはJavaScriptが「その要素が見つからなかった」ことを示します。idのスペルミス（大文字小文字も含む）か、scriptタグの位置（</body>より前に書かれているか）を確認しましょう。taskInputとTaskInputは別物です。

理解度確認クイズ

  1. Q1. HTMLの要素にid属性を付けるのはなぜですか？

    回答を表示

    JavaScriptからdocument.getElementById()を使って要素を取得するためです。idがないとJavaScriptはどの要素を操作すべきか識別できません。ToDoアプリでは入力欄・ボタン・リストすべてにidを付けることで、後でJSから自由に操作できるようにします。

  2. Q2. <script src="app.js">は</body>の直前に置くべきです。その理由を説明してください。

    回答を表示

    ブラウザはHTMLを上から順番に読み込みます。<head>内にscriptを置くと、inputやulなどのHTML要素がまだ読まれていない段階でJavaScriptが実行され「要素が見つからない（null）」エラーになります。</body>直前ならHTMLがすべて読み込まれた後に実行されるため、安全に要素を取得できます。

  3. Q3. ToDoアプリのHTML構造で使うul・li・inputはそれぞれ何の役割を担いますか？

    回答を表示

    inputはユーザーがタスク名を入力するテキストボックスです。ul（順不同リスト）はタスクの一覧を表示するコンテナで、JavaScriptが動的にli要素を追加していきます。li（リストアイテム）は各タスク1件を表す要素で、第15章以降でJavaScriptが自動生成します。
