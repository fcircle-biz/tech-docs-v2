エンティティクラス

前の章でテーブルを作り、データベースの準備ができました。今度は「テーブルの1行分のデータを Java で扱う」ための仕組みを作ります。それがエンティティクラスです。このクラスを定義することで、データベースの行と Java のオブジェクトをひもづけられるようになります。

この章で学ぶこと

  • エンティティクラスの役割を理解する
  • テーブルの1行を Java オブジェクトで表す考え方を理解する
  • @Table、@Id の基本を理解する

4.1 エンティティクラスとは

エンティティクラスとは、データベースのテーブルに対応する Java クラスのことです。たとえば users テーブルがあるなら、その1行分のデータを表す User クラスがエンティティです。

Spring Data JDBC は、このエンティティクラスを手がかりにして「テーブルのどの列をクラスのどのフィールドへ対応させるか」を自動的に判断します。難しい SQL や変換コードを自分で書かなくても、Spring がよしなにやってくれます。

重要ポイント

エンティティは、テーブルのデータを Java のオブジェクトとして表すためのクラスです。テーブルの1行が、そのクラスの1つのインスタンスに対応します。Spring Data JDBC はこのクラスを読んで、データベースとのやり取りを自動化します。

4.2 1行 = 1オブジェクトのイメージ

エンティティの考え方を「表で見る」と、イメージがつかみやすくなります。users テーブルの各行が、User クラスの1つのインスタンスに対応します。

usersテーブルの行 User オブジェクト（Java）
id name email インスタンスの状態
1 田中太郎 tanaka@example.com user1（id=1, name="田中太郎", ...）
2 佐藤花子 sato@example.com user2（id=2, name="佐藤花子", ...）

このように、テーブルの各行がそれぞれ1つの User インスタンスになります。Spring Data JDBC が「テーブルの行 ↔ Java オブジェクト」の変換を自動でやってくれるので、私たちはクラスの定義だけに集中できます。

4.3 User クラスを作る

それでは実際に User エンティティクラスを作りましょう。src/main/java/com/example/demo/ に User.java ファイルを新規作成します。コードは読みやすさのため2つのブロックに分けて見ていきます。

User.java（前半: アノテーション・フィールド・コンストラクタ） 
import org.springframework.data.annotation.Id;
import org.springframework.data.relational.core.mapping.Table;

// @Table: このクラスが users テーブルに対応することを示す
@Table("users")
public class User {

    // @Id: このフィールドが主キー（id列）であることを示す
    @Id
    private Long id;

    // テーブルの name 列に対応するフィールド
    private String name;

    // テーブルの email 列に対応するフィールド
    private String email;

    // コンストラクタ: 3つの値をまとめて受け取ってオブジェクトを作る
    public User(Long id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }
}
User.java（後半: getter メソッド群） 
    // getter: フィールドの値を外部から読み取るためのメソッド
    public Long getId() { return id; }
    public String getName() { return name; }
    public String getEmail() { return email; }

フィールドは private（外から直接さわれない）にしておき、値を読み取るには getId() などの getter メソッドを使います。Spring Data JDBC はこれらの getter を通じて値を扱います。

4.4 アノテーションの意味

コードに登場した @Table と @Id は、Spring へ「このクラスとテーブルの関係」を教えるためのアノテーション（目印）です。

アノテーション 付ける場所 意味
@Table("users") クラスの直前 このクラスが users テーブルに対応することを示す
@Id フィールドの直前 そのフィールドが主キー（id 列）であることを示す
  • フィールド id → users テーブルの id 列（@Id で主キーと宣言）
  • フィールド name → users テーブルの name 列
  • フィールド email → users テーブルの email 列
  • 1つの User オブジェクトが、users テーブルの 1行に対応する

用語: アノテーション

アノテーションとは、クラスやフィールドに付ける「目印（メタデータ）」のことです。@ から始まる記述がそれにあたります。Spring はこの目印を読み取って「どのクラスがどのテーブルに対応するか」「どのフィールドが主キーか」などを自動的に理解します。アノテーションを付けるだけで、面倒な設定ファイルを書かずに済みます。

import 文について

@Id は org.springframework.data.annotation.Id から、@Table は org.springframework.data.relational.core.mapping.Table からインポートします。これらは Spring Data JDBC が提供するアノテーションです。

重要ポイント

エンティティ = テーブルのデータを Java オブジェクトとして表すクラス。@Table でどのテーブルに対応するかを、@Id でどのフィールドが主キーかを Spring へ伝えます。テーブルの1行 = Java の1インスタンス、という対応関係を頭に入れておきましょう。

理解度確認クイズ

  1. Q1. エンティティクラスとは何ですか？

    回答を表示

    テーブルのデータを Java のオブジェクトとして表すクラスです。テーブルの1行が1つのインスタンスに対応します。たとえば users テーブルの1行は、User クラスの1つのインスタンスになります。

  2. Q2. @Table は何を表しますか？

    回答を表示

    そのクラスがどのテーブルに対応するかを Spring へ伝えるアノテーションです。@Table("users") と書けば、そのクラスが users テーブルに対応することを示します。

  3. Q3. @Id は何を表しますか？

    回答を表示

    主キーとなるフィールドを Spring へ伝えるアノテーションです。User クラスでは id フィールドに付けており、users テーブルの id 列が主キーであることを示します。
