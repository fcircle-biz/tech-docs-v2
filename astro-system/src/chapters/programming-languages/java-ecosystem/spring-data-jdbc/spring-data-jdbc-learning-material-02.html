データベース接続設定

第1章では Spring Data JDBC の全体像（Entity・Repository・Service の役割分担）をつかみました。この章では、いよいよアプリをデータベースに「つなげる」ための設定を行います。具体的には、Spring Boot が採用する設定ファイル application.properties の書き方と、学習用として最適な軽量データベース H2 Database の使い方を理解します。設定を正しく書くことで、次章以降のテーブル作成やデータ操作が動き始めます。

この章で学ぶこと

  • Spring Boot でデータベース接続情報を設定する方法を理解する
  • application.properties の役割と書き方を理解する
  • H2 Database を使った学習用環境のしくみを理解する

2.1 application.properties とは

Spring Boot には、アプリの動作に必要なさまざまな設定を1つのファイルにまとめて書くという考え方があります。そのファイルが application.properties（アプリケーション・プロパティーズ）です。

「プロパティ（property）」とは「設定項目」のことで、ファイル内では 「キー＝値」 の形で1行ずつ書きます。Spring Boot はアプリ起動時にこのファイルを自動的に読み込み、書かれた設定を反映させます。たとえば、接続するデータベースのアドレスやユーザー名も、このファイルに書いておくだけで Spring が読んでくれます。

「キー＝値」形式とは

キーは「何の設定か」を表す名前、値は「その設定に入れる内容」です。例えば spring.datasource.username=sa は、「データソースのユーザー名は sa」という意味になります。= の左がキー、右が値です。

第1章では「接続設定が必要」とお伝えしました。実際には、Spring Data JDBC が動くために最低限必要なのがこのファイルへの接続情報の記述です。設定がなければ Spring はどのデータベースに接続すれば良いのか分からず、アプリは起動できません。まずはこのファイルの役割をしっかり押さえましょう。

application.properties は、Maven/Gradle プロジェクトでは src/main/resources/ フォルダ内に置きます。下のファイル構成を見て、場所を確認しておきましょう。

プロジェクトのフォルダ構成（抜粋）

src/
└── main/
    ├── java/
    │   └── com/example/demo/
    │       └── DemoApplication.java
    └── resources/
        └── application.properties  ← ここに接続設定を書く

2.2 H2 Database を学習に使う

データベースには MySQL・PostgreSQL・Oracle などさまざまな種類がありますが、学習の最初からこれらを用意するのはセットアップが大変です。そこで本教材では H2 Database（エイチツー・データベース）という軽量なデータベースを使います。

H2 の最大の特徴は、Maven/Gradle の依存に追加するだけで使える点です。別途インストール作業は不要で、アプリを起動するとデータベースも自動的に起動します。学習中は接続先の切り替えを気にせず、コードそのものに集中できます。

用語: インメモリDB（In-Memory Database）

インメモリDB とは、データをコンピューターのメモリ（RAM）上に保持するデータベースです。アプリが起動している間だけ存在し、アプリを停止するとデータは消えます（ファイルには保存されません）。消えることがデメリットに聞こえるかもしれませんが、学習中は「毎回きれいな状態から試せる」というメリットになります。本番環境では永続化できる設定も可能ですが、まずはインメモリで学習を進めましょう。

H2 を使うためには、pom.xml（Maven の場合）の依存に以下を追加します。runtime スコープにすることで、コンパイル時ではなく実行時にのみ使われます。

pom.xml（H2 依存の追加） 


    com.h2database
    h2
    
    runtime

Spring Initializr（start.spring.io）でプロジェクトを作る際に「H2 Database」を依存として選んでおくと、自動で追加されます。

2.3 接続設定を書く

では、実際に application.properties に接続設定を書いてみましょう。以下が H2 Database を使う場合の基本的な設定例です。

src/main/resources/application.properties 
# 接続先データベースの URL（jdbc:h2:mem:testdb = メモリ上の testdb という名前のH2）
spring.datasource.url=jdbc:h2:mem:testdb

# 使用する JDBC ドライバのクラス名（H2 用）
spring.datasource.driver-class-name=org.h2.Driver

# データベースのユーザー名（H2 のデフォルトは sa）
spring.datasource.username=sa

# パスワード（H2 はデフォルトで空。本番では必ず設定すること）
spring.datasource.password=

# ブラウザから H2 コンソールにアクセスできるようにする
spring.h2.console.enabled=true

# 起動時に schema.sql / data.sql を自動実行する（次章で扱う）
spring.sql.init.mode=always

各設定が何を意味するのか、1つずつ確認しましょう。

  • spring.datasource.url — 接続先のデータベースを URL の形式で指定します。 jdbc:h2:mem:testdb は「JDBC を使って、H2 の、メモリ上の、testdb という名前のDB に接続する」という意味です。
  • spring.datasource.driver-class-name — Java がデータベースと通信するために使う「ドライバ」のクラス名を指定します。 データベースの種類ごとにドライバが異なり、H2 の場合は org.h2.Driver です。
  • spring.datasource.username — データベースにログインするためのユーザー名です。 H2 のデフォルトは sa（System Administrator の略）です。
  • spring.datasource.password — データベースにログインするためのパスワードです。 H2 のデフォルトは空（パスワードなし）です。実際の開発・本番環境では必ず設定してください。
  • spring.h2.console.enabled=true — ブラウザから H2 のデータベースを直接確認できる「H2 コンソール」を有効にします。 これをオンにすると /h2-console というアドレスでコンソールにアクセスできます（次のセクションで詳しく説明します）。
  • spring.sql.init.mode=always — アプリ起動時に schema.sql（テーブル定義）や data.sql（初期データ）を自動的に実行します。 これらのファイルは次章で作成します（今はこういう設定がある、とだけ覚えておけば大丈夫です）。
設定キー 設定した値 意味
spring.datasource.url jdbc:h2:mem:testdb 接続先DBのアドレス（メモリ上のH2）
spring.datasource.driver-class-name org.h2.Driver H2用JDBCドライバのクラス名
spring.datasource.username sa DBログインユーザー名（H2デフォルト）
spring.datasource.password （空） DBパスワード（H2デフォルトは空）
spring.h2.console.enabled true ブラウザからH2コンソールを利用可能にする
spring.sql.init.mode always 起動時に schema.sql / data.sql を実行（次章）

2.4 H2 コンソールで確認する

spring.h2.console.enabled=true を設定してアプリを起動すると、ブラウザから H2 コンソールにアクセスできます。H2 コンソールは、テーブルの中身を確認したり、手動で SQL を実行したりできる便利なツールです。

アクセス先は http://localhost:8080/h2-console です。開くとログイン画面が表示されるので、JDBC URL に jdbc:h2:mem:testdb、ユーザー名に sa、パスワードは空のまま「接続」をクリックしてください。

H2 コンソールのアクセス情報（まとめ）

  • URL: http://localhost:8080/h2-console
  • JDBC URL: jdbc:h2:mem:testdb（application.properties と一致させる）
  • ユーザー名: sa
  • パスワード: 空のまま

H2 コンソールはあくまで確認用のツールです。この章ではまだテーブルを作っていないので、次章以降に活用します。

この章では「H2 コンソールが使える」という事実を覚えておけば十分です。実際にテーブルの中身をコンソールで確認するのは、第3章以降で行います。

重要ポイント

接続情報は application.properties に集約します。将来 H2 から MySQL・PostgreSQL へ切り替えたくなったときも、このファイルの数行を書き換えるだけで済みます（Java コードには手を入れません）。「設定はファイル1つに集める」という Spring らしい考え方が、保守や環境切り替えを楽にしてくれます。

理解度確認クイズ

  1. Q1. データベース接続情報はどのファイルに書きますか？

    回答を表示

    application.properties（src/main/resources/ フォルダ内）に書きます。Spring Boot はアプリ起動時にこのファイルを自動で読み込み、設定を反映させます。

  2. Q2. spring.datasource.url は何を表しますか？

    回答を表示

    接続先のデータベースを URL の形式で指定する設定です。例えば jdbc:h2:mem:testdb は「JDBC を使って、H2 のメモリ上にある testdb という名前のデータベースに接続する」という意味になります。

  3. Q3. H2 Database を学習で使うメリットは何ですか？

    回答を表示

    依存に追加するだけで使えるため別途インストールが不要な点、およびメモリ上で動くインメモリDBのため毎回きれいな状態から試せる点が主なメリットです。セットアップの手間なく、すぐに Spring Data JDBC のコードを動かして学習に集中できます。
