処理をまとめる - メソッド

よく使う処理に名前をつけて「部品」として再利用する仕組みがメソッドです。この章では、メソッドの定義・呼び出し・戻り値の流れをレシピや自動販売機のたとえで丁寧に学びます。メソッドをうまく使うと、プログラムが読みやすくなり、同じコードを何度も書く必要がなくなります。

この章で学ぶこと

  • メソッドとは何か、なぜ使うと便利かを説明できる
  • 戻り値の型・メソッド名・引数（パラメータ）の構成を理解し、自分でメソッドを定義できる
  • メソッドを呼び出し、引数を渡して戻り値を受け取る流れを理解できる
  • void メソッド・static メソッド・オーバーロードの概念を理解できる

このページのコード例の読み方

各コード例の右上のラベルで「コードを書く場所」を示します。

  • 完全なプログラムそのまま .java ファイルに貼り付けて実行できます。
  • mainメソッド内に記述public static void main(String[] args) { ... } の中、またはクラス内に書いて実行します。

6.1 メソッドとは何か

これまでのコードで、同じ処理を何度も書いたことはありませんか？たとえば「2つの数を足す計算」を毎回 a + b と書いていると、同じロジックが何十行にも散らばってしまいます。メソッド（method）を使うと、処理に名前をつけて「部品」にすることができます。部品は何度でも呼び出して再利用できます。

料理にたとえると、メソッドはレシピカードです。「カレーの作り方」を1枚のカードにまとめておけば、「カレーを作れ」と一言言うだけで手順全体を実行してもらえます。毎回「まず玉ねぎを切って…」と全部説明し直す必要はありません。
また、自動販売機にもたとえられます。「お金を入れてボタンを押す（= 呼び出す）→ 飲み物が出てくる（= 戻り値）」という流れは、メソッドの動き方とそっくりです。

重複を減らせる

同じ処理を1か所にまとめることで、変更が1か所で済み、バグも減ります。

読みやすくなる

処理に名前がつくので「何をしているか」が一目でわかり、コードの意図が伝わります。

6.2 メソッドの定義 — 構成を覚えよう

メソッドは次の形式で定義します。各部品の意味を1つずつ確認しましょう。

メソッド定義の基本形 クラス内に記述（mainと並べる） 
// 戻り値の型  メソッド名（引数の型 引数名, ...）
static int add(int a, int b) {
    // ↑ static: mainから直接呼べるようにするキーワード（6.5で詳しく学ぶ）
    // ↑ int: この処理は「整数」を返すという宣言
    // ↑ add: メソッドの名前
    // ↑ int a, int b: 受け取る値（引数/パラメータ）
    return a + b; // returnで計算結果を返す
}
構成要素 役割 例
戻り値の型 処理の結果として返す値の型を指定する int、String、void
メソッド名 呼び出すときに使う名前（動詞で始めると分かりやすい） add、printHello
引数（パラメータ） メソッドに渡す入力値。なければ空の () でもOK int a, int b
return 文 処理結果を呼び出し元に返す。void の場合は不要 return a + b;

命名のコツ: 動詞＋目的語

メソッド名は「何をするか」が分かる動詞から始めるのがお作法です。add（足す）・printMessage（メッセージを表示する）・getMax（最大値を取得する） のように命名すると、コードが読みやすくなります。

6.3 メソッドの呼び出しと戻り値の流れ

定義したメソッドは、名前と引数を書くだけで呼び出せます。呼び出すと処理が実行され、return の値が呼び出し元に返ってきます。

MethodExample.java 完全なプログラム 
public class MethodExample {

    // 2数を足し算して結果を返すメソッド
    static int add(int a, int b) {
        return a + b; // 計算結果を返す
    }

    public static void main(String[] args) {
        // addメソッドを呼び出す（引数: 3と5を渡す）
        int result = add(3, 5); // 戻り値をresultに受け取る
        System.out.println("3 + 5 = " + result); // 8
    }
}

呼び出しから戻り値が返るまでの流れを図で確認しましょう。

flowchart LR A["main メソッド
add（3, 5）を呼び出す"] -->|"引数: a=3, b=5 を渡す"| B["add メソッド
a + b を計算"] B -->|"return 8（戻り値）"| C["main メソッド
result = 8 を受け取る"] C --> D["画面に表示
3 + 5 = 8"]

図の説明: main メソッドが add メソッドを引数つきで呼び出し、計算結果が戻り値として返ってきて画面に表示されるまでの流れを示しています。

重要ポイント: 引数と戻り値の型を合わせる

メソッドを定義するときに宣言した「引数の型」「戻り値の型」は、呼び出し側でも一致させる必要があります。型が合わない場合はコンパイル時にエラーになり、早めにミスを発見できます。これが Java の「型安全」の便利さです。

6.4 戻り値がないメソッド（void）

計算結果を返さず「処理をするだけ」のメソッドも作れます。その場合、戻り値の型を void（ボイド） と書きます。void は「空」を意味し、何も返さないことを表します。

VoidExample.java 完全なプログラム 
public class VoidExample {

    // voidメソッド: 戻り値はなく、表示するだけ
    static void printGreeting(String name) {
        // 引数で受け取った名前をあいさつ文に使う
        System.out.println("こんにちは、" + name + " さん！");
        // returnは書かなくてOK（省略できる）
    }

    public static void main(String[] args) {
        printGreeting("山田"); // 「こんにちは、山田 さん！」
        printGreeting("田中"); // 「こんにちは、田中 さん！」
    }
}

void メソッドの使い所

void メソッドは「画面への表示」「ファイルへの書き込み」「状態の更新」など、副作用（何かをする）が目的のときに使います。計算結果を返したい場合は int や String など具体的な型を指定します。

6.5 static メソッドと main メソッドの関係

第1章から書いてきた public static void main(String[] args) は、実はメソッドです。static（スタティック） キーワードに注目してみましょう。

static が付いたメソッドは、クラスからオブジェクトを作らなくても直接呼び出せるという特徴があります。本章のメソッド例では、main から呼び出せるよう、すべて static を付けています。

StaticExample.java 完全なプログラム 
public class StaticExample {

    // staticメソッド: mainから直接呼び出せる
    static int multiply(int x, int y) {
        return x * y; // 掛け算の結果を返す
    }

    // mainも staticメソッドの一種
    public static void main(String[] args) {
        int answer = multiply(4, 7); // staticメソッドを呼び出す
        System.out.println("4 × 7 = " + answer); // 28
    }
}

第7章への予告: static と インスタンスメソッド

static でないメソッドは「インスタンスメソッド」と呼ばれ、クラスからオブジェクト（インスタンス）を作って初めて呼べます。第7章でクラスとオブジェクトを学ぶと、static との違いがはっきり見えてきます。今は「main から呼ぶには static が必要」と覚えておけば十分です。

6.6 メソッドのオーバーロード

Javaでは、同じメソッド名でも引数の型や個数が違えば、別のメソッドとして定義できます。これをオーバーロード（overload）と言います。呼び出すときは引数に合ったメソッドが自動的に選ばれます。

OverloadExample.java 完全なプログラム 
public class OverloadExample {

    // int同士を足す addメソッド
    static int add(int a, int b) {
        return a + b;
    }

    // double同士を足す addメソッド（同名・別の引数型）
    static double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(3, 5));       // → 8（int版が呼ばれる）
        System.out.println(add(1.5, 2.3));   // → 3.8（double版が呼ばれる）
    }
}

オーバーロードの判別ルール

Javaコンパイラは「引数の型と個数（シグネチャ）」を見てどのメソッドを呼ぶか決めます。戻り値の型だけが違うオーバーロードは不可なので注意してください。引数の型・個数、または引数の順番が違う場合に限りオーバーロードできます。

やってみよう 6-1: 最大値・面積を求めるメソッドを作ろう

メソッドの定義・引数・戻り値の流れを実際に手を動かして体験しましょう。2つのうち好きなほうに挑戦してください（両方やってみると更に理解が深まります）。

課題A: 2数の最大値を返すメソッド

  • 1 static int getMax(int a, int b) というメソッドを定義する。if 文で a と b を比べ、大きい方を return する。
  • 2 main から getMax(10, 25) を呼び出し、結果を画面に表示する。
  • 3 引数を変えて何度か試し、常に大きい方が表示されることを確認する。

課題B: 長方形の面積を計算するメソッド

  • 1 static int calcArea(int width, int height) を定義し、width * height を返す。
  • 2 main から複数の縦横サイズで呼び出し、それぞれの面積を表示する。
  • 3 余裕があれば、引数を double に変えた版も追加し、オーバーロードにしてみよう。

理解度確認クイズ

  • Q1. メソッドを使うと何が嬉しいですか？ 2つ答えてください。

    回答を表示

    ①重複を減らせる: 同じ処理を1か所にまとめられるので、変更が1か所で済みバグも少なくなります。②読みやすくなる: 処理に名前がついているので「何をしているか」が一目でわかり、コードの意図が伝わります。

  • Q2. 次のメソッド定義 static int add(int a, int b) の「int」（最初のもの）は何を意味しますか？

    回答を表示

    戻り値の型です。このメソッドは処理の結果として「整数（int）」を返すことを宣言しています。何も返さない場合は void と書きます。

  • Q3. オーバーロードとは何ですか？ どんな場合に使えますか？

    回答を表示

    同じメソッド名で、引数の型や個数が異なるメソッドを複数定義することです。例えば add(int, int) と add(double, double) のように、扱う型ごとにメソッドを用意できます。呼び出し時に渡した引数に合わせてJavaが自動的に適切なメソッドを選びます。
