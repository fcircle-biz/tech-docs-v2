データをまとめて操作する - コレクション入門

配列よりも柔軟にデータを管理できる「コレクション」を学びます。要素を後から自由に追加・削除できる ArrayList、キーと値のペアで管理する HashMap、重複を許さない集合の HashSet という3つの強力な道具を、実践的なコード例とともにマスターしましょう。

この章で学ぶこと

  • コレクションとは何か、配列との違いをわかりやすく説明できる
  • ArrayList を使って要素の追加・取得・削除・反復ができる
  • HashMap でキーと値のペアを管理し、値を検索できる
  • HashSet を使って重複のない集合を作り、要素の存在チェックができる
  • ArrayList・HashMap・HashSet を使い分けの基準が説明できる

14.1 コレクションとは

これまで「複数のデータをまとめる」ために配列（Array）を使ってきました。しかし配列には不便な点があります。それはサイズが作った時点で固定されるという点です。たとえばアンケートで参加者が後から増えたり、リストから特定の人を途中で削除したりする場面では、固定サイズの配列だと対処が難しくなります。

そこで登場するのがコレクション（Collection）です。コレクションは「可変長の入れ物」であり、要素をいつでも追加・削除できます。Javaの標準ライブラリには豊富なコレクションクラスが用意されており、今回は最もよく使われる3つを学びます。

比較項目 配列（Array） コレクション
サイズ 作成時に固定（変更不可） 後から自由に変更できる
要素の追加 できない（新しい配列が必要） add() で簡単に追加できる
要素の削除 できない（null 代入で回避） remove() で削除できる
型指定 String[] など型を直接指定 ジェネリクス<String> で指定
主な用途 数が決まっているデータ 数が変わる・様々な操作が必要なデータ

用語: ジェネリクス（Generics）とは

ArrayList<String> の <String> の部分をジェネリクスと呼びます。「この入れ物には String 型だけ入れてください」という型の指定です。ジェネリクスを指定することで、間違った型のデータを入れようとした場合にコンパイルエラーで早期に気づけます。初心者のうちは「<型名> で中身の型を決める」と覚えておきましょう。

14.2 ArrayList（リスト）— 順番ありの可変リスト

ArrayList は、配列の可変版です。要素は順番通りに並び、重複する値も登録できます。ショッピングカートや名前のリストなど、「順番を保ちながら追加・削除したい」場面で大活躍します。

使い始めるには import java.util.ArrayList; が必要です。基本的な操作を見てみましょう。

ArrayListBasic.java 
import java.util.ArrayList;

public class ArrayListBasic {
    public static void main(String[] args) {
        // String 型の ArrayList を作成
        ArrayList<String> names = new ArrayList<>();

        // 要素を追加（add）
        names.add("田中");
        names.add("鈴木");
        names.add("佐藤");

        // サイズ（要素数）を確認
        System.out.println("人数: " + names.size()); // → 3

        // インデックスで取得（get・0始まり）
        System.out.println("先頭: " + names.get(0)); // → 田中
    }
}

続いて、要素の削除と拡張for文（for-each）を使った全要素のループを見てみましょう。拡張for文はコレクションとも相性抜群です。

ArrayListLoop.java 
import java.util.ArrayList;

public class ArrayListLoop {
    public static void main(String[] args) {
        ArrayList<String> fruits = new ArrayList<>();
        fruits.add("りんご");
        fruits.add("バナナ");
        fruits.add("みかん");

        // 拡張for文で全要素を順番に処理
        for (String fruit : fruits) {
            System.out.println("果物: " + fruit);
        }

        // インデックス指定で削除（remove）
        fruits.remove(1); // "バナナ" を削除
        System.out.println("削除後の数: " + fruits.size()); // → 2
    }
}
terminal — 実行結果 
果物: りんご
果物: バナナ
果物: みかん
削除後の数: 2
メソッド 説明 例
add（要素） 末尾に要素を追加する list.add("A")
get（i） i番目の要素を取得する（0始まり） list.get(0)
size（） 要素数を返す list.size()
remove（i） i番目の要素を削除する list.remove(1)
contains（要素） 要素が含まれるか確認する list.contains("A")

14.3 HashMap（マップ）— キーと値のペアで管理する

HashMap は「キー（Key）と値（Value）をセットで管理する」コレクションです。辞書をイメージしてください。「単語（キー）を調べると意味（値）が出てくる」ように、キーを使って値を素早く取り出せます。

たとえば「名前 → 点数」「商品名 → 価格」「都道府県コード → 名称」など、2つのデータを紐づけたいときに HashMap が便利です。

HashMapBasic.java 
import java.util.HashMap;

public class HashMapBasic {
    public static void main(String[] args) {
        // キー: String, 値: Integer の HashMap を作成
        HashMap<String, Integer> scores = new HashMap<>();

        // キーと値のペアを登録（put）
        scores.put("田中", 85);
        scores.put("鈴木", 92);
        scores.put("佐藤", 78);

        // キーを指定して値を取得（get）
        int tanaka = scores.get("田中");
        System.out.println("田中の点数: " + tanaka); // → 85

        // キーが存在するか確認（containsKey）
        if (scores.containsKey("鈴木")) {
            System.out.println("鈴木のデータあり");
        }
    }
}

全てのキーを取り出して繰り返す場合は keySet() を使います。登録した順番は保証されませんが、全データを一覧する処理がシンプルに書けます。

HashMapLoop.java（keySetで全件表示） 
import java.util.HashMap;

public class HashMapLoop {
    public static void main(String[] args) {
        HashMap<String, Integer> scores = new HashMap<>();
        scores.put("田中", 85);
        scores.put("鈴木", 92);
        scores.put("佐藤", 78);

        // keySet() で全キーを取り出してループ
        for (String name : scores.keySet()) {
            System.out.println(name + " の点数: " + scores.get(name));
        }
    }
}

重要ポイント: キーは重複しない

HashMap ではキーは一意（重複不可）です。同じキーで put すると、値が上書きされます。例えば scores.put("田中", 90) とすると、田中の点数は 90 に書き換わります。

14.4 HashSet（セット）— 重複を許さない集合

HashSet は重複のない集合を管理するコレクションです。同じ値を何度 add しても、一つしか保存されません。「既に登録済みかどうかを高速にチェックしたい」「重複を自動的に排除したい」ときに使います。

たとえばアクセスしたページのURL、参加者の名前（重複なし）、利用済みコードの管理などに向いています。

HashSetBasic.java 
import java.util.HashSet;

public class HashSetBasic {
    public static void main(String[] args) {
        HashSet<String> visited = new HashSet<>();

        // 要素を追加（add）。重複は自動的に無視される
        visited.add("東京");
        visited.add("大阪");
        visited.add("東京"); // ← 重複！自動的に無視される
        visited.add("名古屋");

        // サイズは重複を除いた数
        System.out.println("訪問都市数: " + visited.size()); // → 3

        // 存在チェック（contains）
        if (visited.contains("大阪")) {
            System.out.println("大阪は訪問済みです");
        }
    }
}
terminal — 実行結果 
訪問都市数: 3
大阪は訪問済みです

順番は保証されない

HashSet は要素の順番を保証しません。for-each でループすると、追加した順とは異なる順序で出力されることがあります。順番を保ちながら重複を排除したい場合は LinkedHashSet を使いましょう（使い方は HashSet とほぼ同じです）。

14.5 3つのコレクションを使い分ける

3つのコレクションは、それぞれ特徴が異なります。「どれを使うか」は次の観点で判断しましょう。

コレクション 順番 重複 キーと値 主な用途
ArrayList 保持する 許可する なし（値のみ） 順序のあるリスト・ログ・カート
HashMap 保証しない キーは重複不可 あり（キー→値） 検索・辞書・名前→点数
HashSet 保証しない 許可しない なし（値のみ） 重複排除・存在チェック

理解度確認クイズ

  1. Q1. 配列とコレクション（ArrayList）の最も大きな違いは何ですか？

    回答を表示

    配列はサイズが作成時に固定され、後から変更できません。一方、ArrayList は要素を後から自由に追加・削除できる可変長のコレクションです。これにより、データ数が実行中に変わる場面でも柔軟に対応できます。

  2. Q2. HashMap で「"田中" → 点数85」というデータを登録するには、どのメソッドを使いますか？また、"田中" の点数を取り出すには？

    回答を表示

    登録は scores.put("田中", 85) を使います。取り出しは scores.get("田中") で、戻り値として 85 が返ります。HashMap ではキーを指定することで対応する値を素早く検索できます。

  3. Q3. ["東京", "大阪", "東京", "名古屋"] という4つの値を HashSet に追加すると、最終的に何件のデータが格納されますか？その理由も説明してください。

    回答を表示

    3件です。HashSet は重複を許さないため、「東京」を2回 add しても内部では1つとして扱われます。結果として「東京」「大阪」「名古屋」の3件が格納されます。これが HashSet の最大の特徴「重複の自動排除」です。
