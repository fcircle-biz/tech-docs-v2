クラスの親子関係 - 継承

この章では、既存のクラスの機能をまるごと引き継いで新しいクラスを作る「継承」を学びます。動物クラスから犬・猫クラスを作るたとえを通じて、コードの再利用と拡張の仕組みを丁寧に理解していきましょう。

この章で学ぶこと

  • 継承の概念と「コードの再利用」というメリットを説明できる
  • extends キーワードを使ってスーパークラス・サブクラスを定義できる
  • 親のフィールド・メソッドを子が引き継ぎ、独自の機能を追加できることを理解する
  • @Override アノテーションを使ってメソッドをオーバーライドできる
  • super キーワードで親のコンストラクタやメソッドを呼び出せる

10.1 継承とは — 共通部分をまとめて再利用する

プログラムを書いていると、「似たような機能を持つクラスを何度も書いている」と気づくことがあります。たとえば「犬」クラスと「猫」クラスを別々に作ると、どちらにも「名前」「鳴く」「食べる」など同じような部分が登場します。このような重複をなくすのが継承（インヘリタンス）という仕組みです。

継承では、共通の機能を親クラス（スーパークラス）にまとめ、そこから新しい子クラス（サブクラス）を作ります。子クラスは親クラスの機能をすべて受け取ったうえで、自分だけの特別な機能を追加できます。

コードの再利用

同じコードを何度も書かずに済む。親クラスの機能はそのまま子クラスで使える。

整理・拡張しやすい

共通部分を親にまとめるだけで、新しい種類のクラスを素早く追加できる。

修正箇所が少ない

共通の修正は親クラスを1箇所直すだけ。子クラスに自動的に反映される。

たとえで理解: 動物の家系図

「動物（Animal）」という親から「犬（Dog）」「猫（Cat）」という子が生まれるイメージです。どの動物も「名前がある」「食べる」という共通点を持ちつつ、犬は「ワンと鳴く」、猫は「ニャアと鳴く」という独自の特徴を持っています。

10.2 スーパークラスとサブクラス — extends キーワード

Javaで継承を使うには extends（エクステンズ）キーワードを使います。「〜を拡張する」という意味で、「このクラスは〇〇クラスを継承します」と宣言します。

extends の書き方 
// 親クラス（スーパークラス）
class Animal {
    // 共通の機能をここに書く
}

// 子クラス（サブクラス）: Animal を継承する
class Dog extends Animal {
    // Dog 独自の機能をここに追加する
}

class Cat extends Animal {
    // Cat 独自の機能をここに追加する
}

継承の親子関係（クラス図）

classDiagram class Animal { +String name +eat() +sound() } class Dog { +bark() +sound() } class Cat { +purr() +sound() } Animal <|-- Dog : extends Animal <|-- Cat : extends

クラス図の矢印（<|--）は「継承」を表します。Dog と Cat はどちらも Animal を継承しているため、Animalのメンバー（フィールドとメソッド）を引き継ぎます。

用語まとめ

  • スーパークラス（親クラス・基底クラス）: 継承される側のクラス（ここでは Animal）
  • サブクラス（子クラス・派生クラス）: 継承する側のクラス（ここでは Dog・Cat）
  • is-a 関係: 「犬 is-a 動物（犬は動物の一種）」という関係が成り立つ場合に継承を使う

10.3 親の機能を引き継ぎ、子に独自機能を追加する

子クラスは親クラスのフィールドとメソッドを自動的に受け取ります。さらに、子クラス専用のフィールドやメソッドを自由に追加できます。実際のコードで確かめましょう。

Animal.java — 親クラス 
// 親クラス: すべての動物に共通する機能
class Animal {
    String name; // 動物の名前（子クラスでも使える）

    // 食べるメソッド（子クラスでそのまま使える）
    void eat() {
        System.out.println(name + "が食べています");
    }

    // 鳴くメソッド（子クラスで上書きする）
    void sound() {
        System.out.println("...");
    }
}
Dog.java — 子クラス 
// 子クラス: Animal を継承し、Dog 独自機能を追加
class Dog extends Animal {
    // Dog 専用のメソッドを追加
    void bark() {
        System.out.println(name + "が吠えています: ワン！");
    }
}

// 実行例
Dog dog = new Dog();
dog.name = "ポチ";      // 親から引き継いだフィールド
dog.eat();              // 親から引き継いだメソッド: ポチが食べています
dog.bark();             // Dog 独自のメソッド: ポチが吠えています: ワン！

重要ポイント: 引き継がれるもの・引き継がれないもの

  • 引き継がれる: public / protected のフィールドとメソッド
  • 引き継がれない: コンストラクタ（直接は受け継がない）、private のメンバー

10.4 メソッドのオーバーライド — 動きを子クラスで変える

親クラスから引き継いだメソッドの動きを、子クラスで上書きすることをオーバーライド（Override）といいます。たとえば Animal の sound() を Dog では「ワン！」に、Cat では「ニャア！」に変えることができます。

オーバーライドするときは @Override（アノテーション）をメソッドの直前に書く習慣をつけましょう。これにより「意図的に上書きしている」とJavaに伝え、タイプミスなどのバグを防げます。

オーバーライドの例 
class Dog extends Animal {
    @Override // 親の sound() を上書きする
    void sound() {
        System.out.println(name + ": ワン！");
    }
}

class Cat extends Animal {
    @Override // 親の sound() を上書きする
    void sound() {
        System.out.println(name + ": ニャア！");
    }
}
terminal — 実行結果 
Dog dog = new Dog(); dog.name = "ポチ"; dog.sound();
=> ポチ: ワン！

Cat cat = new Cat(); cat.name = "タマ"; cat.sound();
=> タマ: ニャア！

注意: @Override を省略しても動くが、必ず書こう

@Override を書かなくてもオーバーライドは機能します。しかしメソッド名を間違えた場合、上書きされずに新しいメソッドが増えてしまい、バグに気づきにくくなります。@Override があれば、親に同じメソッドが存在しない場合にコンパイルエラーが出るため、ミスをすぐに発見できます。

10.5 super キーワード — 親の機能を呼び出す

super（スーパー）キーワードを使うと、子クラスの中から親クラスのコンストラクタやメソッドを明示的に呼び出せます。特にコンストラクタで親の初期化処理を引き継ぐ場面でよく使います。

super(...) — 親のコンストラクタを呼ぶ

super でコンストラクタを引き継ぐ 
class Animal {
    String name;
    Animal(String name) { // 親のコンストラクタ
        this.name = name;
    }
}

class Dog extends Animal {
    String breed; // 犬種（Dog 独自フィールド）
    Dog(String name, String breed) {
        super(name); // 親のコンストラクタを呼び出す（必ず先頭行）
        this.breed = breed;
    }
}

super.メソッド名() — 親のメソッドを呼ぶ

super でメソッドを呼び出す 
class Dog extends Animal {
    @Override
    void sound() {
        super.sound();  // 親の sound() を先に呼ぶ
        System.out.println(name + ": ワン！（追加）");
    }
}

super(...) は必ず先頭行に書く

コンストラクタで super(...) を使う場合は、コンストラクタの先頭行に書くルールがあります。先頭に書かなければコンパイルエラーになります。明示的に書かない場合、Javaが自動的に super()（引数なし）を挿入します。

10.6 すべてのクラスは Object クラスを継承している

Javaでは、extends を書かなくても、すべてのクラスは自動的に java.lang.Object クラスを継承します。これはJavaの全クラスの「最上位の祖先」です。

全クラスの継承ツリー（イメージ）

flowchart TD O["Object
（全クラスの祖先）"] --> A["Animal"] A --> D["Dog"] A --> C["Cat"] O --> S["String"] O --> I["Integer"]

Object クラスには toString()（文字列に変換）や equals()（等価比較）など、すべてのクラスで使える便利なメソッドが用意されています。これらも必要に応じてオーバーライドして使います。

重要ポイント: Java の継承は単一継承

Javaでは1つのクラスが継承できる親クラスは1つだけです（単一継承）。複数の親クラスは持てません。ただし、複数の「インターフェース」を実装することはできます（第12章で学びます）。

やってみよう 10-1: 乗り物クラスで継承を試そう

親クラス Vehicle（乗り物）と、子クラス Car（車）・Bike（自転車）を作り、継承とオーバーライドを体験します。

手順

  1. 1 親クラス Vehicle を作成し、フィールド name（名前）とメソッド move()（「〇〇が移動します」と表示）を定義する。
  2. 2 子クラス Car を作り、@Override を使って move() を「〇〇がエンジンで走ります」に上書きする。
  3. 3 子クラス Bike を作り、move() を「〇〇がペダルで走ります」に上書きする。
  4. 4 Car と Bike のインスタンスを作り、move() を呼び出してそれぞれの結果が違うことを確認する。

期待する実行結果:

自動車がエンジンで走ります
自転車がペダルで走ります

理解度確認クイズ

  1. Q1. 継承を使う最大のメリットは何ですか？ また、Javaで継承を宣言するキーワードは何ですか？

    回答を表示

    最大のメリットはコードの再利用です。共通の機能を親クラスに1箇所にまとめることで、同じコードを何度も書く必要がなくなります。Javaで継承を宣言するキーワードは extends で、class Dog extends Animal のように書きます。

  2. Q2. 次のコードで @Override アノテーションの役割は何ですか？ 省略した場合との違いを説明してください。

    回答を表示

    @Override は「このメソッドは親クラスのメソッドを意図的に上書きしています」とJavaに伝えるアノテーションです。省略しても動作しますが、@Override があると、親クラスに同名メソッドが存在しない場合（タイプミスなど）にコンパイルエラーが発生してバグを早期発見できます。

  3. Q3. super() と super.メソッド名() の違いと、それぞれどんな場面で使うかを説明してください。

    回答を表示

    super() は親クラスのコンストラクタを呼び出します。子クラスのコンストラクタの先頭行に書き、親クラスの初期化処理を引き継ぐ際に使います。super.メソッド名() は親クラスのインスタンスメソッドを呼び出します。オーバーライドしたメソッドの中で「親の処理 ＋ 追加処理」を実現したいときなどに使います。
