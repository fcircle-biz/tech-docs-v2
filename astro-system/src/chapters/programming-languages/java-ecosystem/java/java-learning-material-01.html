1.1 プログラミングとは何か

プログラミングとは、コンピュータに「何を、どの順番でやってほしいか」を、コンピュータが理解できる言葉（プログラミング言語）で書き出す作業のことです。コンピュータはとても高速で正確ですが、自分で考えることはできません。人間が手順を1つずつ細かく指示してはじめて、計算をしたり、画面に文字を表示したり、ゲームを動かしたりできます。

この「手順を書いたもの」をプログラム（ソースコード）と呼びます。料理にたとえると、プログラムはレシピです。「材料を切る → 炒める → 味付けする」という手順を順番どおりに書いておけば、誰が読んでも同じ料理が作れます。コンピュータもレシピ（プログラム）どおりに、上から順番に処理を実行していきます。

用語: ソースコードとコンパイル

人間が書いたプログラム（ソースコード）は、そのままではコンピュータが直接実行できません。コンピュータが分かる形に「翻訳」する必要があり、この翻訳作業をコンパイルと呼びます。Javaでは、書いたコードをコンパイルしてから実行します（詳しくは 1.2 で解説します）。

1.2 Javaとはどんな言語か

Java（ジャバ）は、1995年に登場した歴史あるプログラミング言語で、今でも世界中で非常に多くのシステムに使われています。銀行のシステム、Androidアプリ、大規模なWebサービス、業務システムなど、活躍の場は実にさまざまです。「初心者が最初に学ぶ言語」としても、長年人気があります。

Javaには、初心者が学ぶうえでうれしい特徴がたくさんあります。代表的なものを見てみましょう。

どこでも動く

「一度書けば、どこでも動く」が合言葉。Windows でも Mac でも、同じプログラムが動きます。

オブジェクト指向

部品を組み合わせて大きなプログラムを作る考え方。整理しやすく、再利用しやすいのが特長です。

安全で頑丈

間違いを早めに教えてくれる仕組みがあり、大きなシステムでも安心して作れます。

「どこでも動く」を実現しているのが、JVM（Java仮想マシン）という仕組みです。書いたJavaコードはまず「バイトコード」という中間的な形にコンパイルされ、それを各パソコン上のJVMが実行します。OSごとの違いはJVMが吸収してくれるため、私たちは1つのコードを書くだけで済みます。

flowchart LR A["あなたが書く
Javaコード (.java)"] --> B["コンパイル"] B --> C["バイトコード (.class)"] C --> D["JVM
(Windows)"] C --> E["JVM
(Mac)"] C --> F["JVM
(Linux)"] D --> G["実行"] E --> G F --> G

図の説明: あなたが書いた Java コード（.java）はコンパイルでバイトコード（.class）に変換され、Windows・Mac・Linux それぞれの JVM が同じバイトコードを読み込んで実行する、という流れを表しています。

用語: JDK とは

JDK（Java Development Kit）は、Javaでプログラムを作るための道具一式です。コードをコンパイルする「コンパイラ」や、プログラムを動かす「JVM」などがまとめて入っています。Javaを学ぶには、まずこのJDKをインストールする必要があります（次の 1.3 で実施します）。

1.3 開発環境を準備する

Javaのプログラムを書いて動かすために、3つの道具を用意します。順番にインストールしていきましょう。

道具 役割
JDK Javaを動かす本体。コンパイラと実行環境が入っている。
VSCode コードを書くためのエディタ（高機能なメモ帳のようなもの）。
Extension Pack for Java VSCodeでJavaを快適に書くための拡張機能パック。

手順1: JDK のインストール

無償で使える OpenJDK（Adoptium / Temurin など）の配布サイトから、自分のOSに合った LTS版（長期サポート版・21以上を推奨）をダウンロードしてインストールします。インストール後、本当に入ったかを確認しましょう。ターミナル（Windowsならコマンドプロンプト／PowerShell、Macならターミナル）を開いて、次のコマンドを入力します。

terminal — JDKの確認 
# Javaのバージョンをチェックするコマンド
java -version

# 例: 次のように表示されればOK
# openjdk version "21.0.2" 2024-01-16 LTS

手順2: VSCode と Java拡張のインストール

Visual Studio Code（VSCode）の公式サイトからインストーラを入手し、インストールします。起動したら左側の拡張機能アイコン（四角が並んだマーク）をクリックし、検索欄に「Extension Pack for Java」と入力してインストールしてください。これでJavaを書く準備が整います。

準備ができたかチェック

  • java -version でバージョンが表示される
  • VSCode が起動し、Extension Pack for Java がインストール済みになっている

注意: 「java が見つかりません」と出たら

java -version でエラーが出る場合は、JDKのインストールが完了していないか、パソコンがJDKの場所を認識できていない可能性があります。インストーラを「管理者として実行」してやり直すか、パソコンを再起動してから再度確認してみてください。

1.4 最初のプログラム「Hello World」

いよいよ最初のプログラムを書きます。プログラミングの世界では、画面に「Hello, World!」と表示するプログラムを最初に作るのが伝統です。VSCodeで Hello.java という名前のファイルを作り、次のコードを入力しましょう。

Hello.java 完全なプログラム 
// これが最初のJavaプログラムです
public class Hello {
    // mainメソッド: プログラムはここから実行が始まる
    public static void main(String[] args) {
        // 画面に文字を表示する命令
        System.out.println("Hello, World!");
    }
}

まだ意味が分からない部分があっても大丈夫です。今は「こう書くと動く」という感覚をつかめれば十分です。それぞれの言葉の意味は、これからの章で少しずつ学んでいきます。各行のおおまかな役割だけ確認しておきましょう。

  • public class Hello — プログラムの入れ物（クラス）。ファイル名と同じ名前にします（第7章で学びます）。
  • main メソッド — プログラムの実行が始まる場所。Javaのお約束です（第6章で学びます）。
  • System.out.println(...) — 画面に文字を表示し、改行する命令です。

VSCodeでコードの上に表示される「Run（実行）」をクリックするか、ファイルを右クリックして実行すると、下のターミナルに結果が表示されます。

terminal — 実行結果 
Hello, World!

重要ポイント

画面に「Hello, World!」と表示されたら、あなたは初めて自分でコンピュータを動かしたことになります。プログラミング学習で大切なのは、小さくても自分の手で動かしてみること。これからも「読む」だけでなく、必ず「書いて動かす」を続けていきましょう。

やってみよう 1-1: 自己紹介プログラムを作ろう

Hello World が動いたら、次は表示する文章を自分で変えてみましょう。コードを少し書き換えるだけで、結果が変わることを体験します。

手順

  • 1 "Hello, World!" の部分を、自分の名前を含む文章（例: "私の名前は山田です"）に書き換える。
  • 2 System.out.println(...) の行をもう1行コピーして、好きな言葉（趣味など）を2行表示してみる。
  • 3 もう一度実行して、思ったとおりに2行表示されるか確認する。

理解度確認クイズ

  • Q1. プログラミングとは何をする作業ですか？ 料理にたとえて説明してください。

    回答を表示

    コンピュータにやってほしい手順を、コンピュータが分かる言葉（プログラミング言語）で書き出す作業です。料理でいえば、手順を書いた「レシピ」を用意することにあたります。コンピュータはそのレシピ（プログラム）どおりに上から順番に処理を実行します。

  • Q2. Javaの「一度書けば、どこでも動く」を支えている仕組みの名前は何ですか？

    回答を表示

    JVM（Java仮想マシン）です。Javaコードはまずバイトコードにコンパイルされ、各OS上のJVMがそれを実行します。OSごとの違いはJVMが吸収するため、同じプログラムがWindows・Mac・Linuxで動きます。

  • Q3. Javaでプログラムを作るために最初にインストールが必要な「道具一式」を何と呼びますか？

    回答を表示

    JDK（Java Development Kit）です。コードをコンパイルするコンパイラや、プログラムを実行するJVMなどがまとめて入っています。
