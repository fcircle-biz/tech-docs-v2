Springとは

Spring（スプリング）は、Java アプリケーション開発を支援するフレームワークです。この章では、「Spring とは何をしてくれるのか」という全体像を、いちばん大切なアイデア —— Spring がオブジェクト（部品）を管理し、必要な場所に自動で渡してくれる —— を中心につかみます。あわせて、なぜ Spring Boot を使うのか、そしてこの教材シリーズ全体の流れも確認します。

この章で学ぶこと

  • Spring が何をするフレームワークなのかを理解する
  • Spring を「使わない場合」と「使う場合」の違いをイメージできる
  • Spring Boot を使う理由を理解する
  • Spring 学習全体の流れ（本シリーズの位置づけ）をつかむ

1.1 Spring とは何か

Spring（スプリング）は、Java でアプリケーションを作るときに使うフレームワークです。ひとことで言うと、次のように説明できます。

この教材でいちばん大切な一文

Spring は、Java アプリケーションで使う部品（オブジェクト）を管理し、必要な場所に自動で渡してくれるフレームワークです。

「部品」とは、たとえば「ユーザー登録の処理を担当するクラス」や「データの保存を担当するクラス」など、アプリの中で役割を持つオブジェクトのことです。ふつうの Java では、こうした部品が必要になるたびに自分で new して作ります。Spring を使うと、その「作る」「渡す」という面倒な作業を Spring が肩代わりしてくれます。

用語: 「フレームワーク」とは？

フレームワークとは「アプリ開発の土台・骨組み」のことです。よく使う仕組みがあらかじめ用意されているので、開発者は本当に作りたい部分（業務の処理）に集中できます。家を建てるときに、土台や柱があらかじめ組まれている状態をイメージすると分かりやすいです。

Spring が提供する機能はとても幅広いのですが、その中心にあるのが「オブジェクトを管理して渡してくれる」という考え方です。この仕組みは DI（ディーアイ） と呼ばれ、本教材の主役になります（第3章でくわしく学びます）。まずはこの章で、その「うれしさ」のイメージをつかみましょう。

1.2 Spring を「使わない場合」と「使う場合」

Spring のありがたさは、「使わない場合」と比べると見えてきます。あるクラス A が、別のクラス B を必要としている場面を考えてみましょう。

Spring を使わない場合

クラス A が、自分でクラス B を new して用意します。A は「B の作り方」まで知っていなければなりません。

部品を「自分で」用意する

Spring を使う場合

Spring がクラス B を用意し、クラス A に渡してくれます。A は「B をどう使うか」だけを考えればよくなります。

部品を「Spring から受け取る」

図にすると、次のようになります。

flowchart LR subgraph NG["Springを使わない場合"] direction TB A1["クラスA"] -->|"自分で new する"| B1["クラスB"] end subgraph OK["Springを使う場合"] direction TB SP["Spring"] -->|"クラスBを用意する"| B2["クラスB"] B2 -->|"クラスAに渡す（注入）"| A2["クラスA"] end

「使う場合」では、クラス A はもう自分で new をしていません。必要な部品は Spring が用意して渡してくれるので、A は本来やりたい処理だけに集中できます。この「必要な部品を Spring に用意してもらう」考え方が、後の章で学ぶ DI の正体です。

1.3 Spring Boot を使う理由

Spring はとても多機能なため、昔はたくさんの設定を自分で書く必要がありました。その手間を大きく減らしてくれるのが Spring Boot（スプリング ブート）です。本教材でも、この Spring Boot を前提に学んでいきます。

設定が簡単

「よくある設定」を自動で用意してくれるので、難しい設定ファイルを大量に書かずに始められます。

すぐ起動できる

サーバーなどが内蔵されており、main メソッドを実行するだけでアプリを起動できます。

必要な機能をまとめて追加

「スターター」という単位で、関連するライブラリ一式をまとめて取り込めます。

まとめると、Spring Boot を使う理由は「設定を簡略化して、すぐにアプリ開発を始められるから」です。初心者にとっては「むずかしい準備を Spring Boot がやってくれるので、学びたい考え方（DI や AOP）に集中できる」点が大きなメリットです。

関係: Spring と Spring Boot

Spring が土台となる本体で、Spring Boot はその Spring を「すぐ使えるように準備してくれる仕組み」です。別物ではなく、Spring Boot は Spring の上に乗っているものと考えてください。本教材で学ぶ DI・Bean・AOP は、どちらの土台でも共通する Spring の中心概念です。

理解度確認クイズ

  1. Q1. Spring は何を助けるためのフレームワークですか？

    回答を表示

    Java アプリケーション開発を助けるためのフレームワークです。とくに、アプリで使う部品（オブジェクト）を管理し、必要な場所に自動で渡してくれることで、開発者が本来の処理に集中できるようにしてくれます。

  2. Q2. Spring Boot を使うメリットは何ですか？

    回答を表示

    設定を簡略化してくれるため、難しい準備をせずにすぐ Spring アプリの開発を始められる点です。サーバーが内蔵されていて main メソッドの実行だけで起動できることや、必要な機能を「スターター」でまとめて追加できることもメリットです。

  3. Q3. Spring が管理してくれるものは何ですか？

    回答を表示

    アプリケーションで使うオブジェクト（部品）です。Spring はこれらのオブジェクトを作成し、必要な場所へ自動で渡してくれます。Spring が管理するオブジェクトは「Bean（ビーン）」と呼ばれ、第4章でくわしく学びます。
