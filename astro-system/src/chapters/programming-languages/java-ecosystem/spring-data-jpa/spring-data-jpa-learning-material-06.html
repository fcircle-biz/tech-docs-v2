ホーム Spring Data JPA 第6章
第6章 所要時間: 約60分

条件検索を作成する

Spring Data JPA には、メソッドの名前を命名規則に従って書くだけで、その検索処理を自動生成してくれる仕組みがあります。この章では クエリメソッド（派生クエリ）の考え方とキーワード一覧を学び、名前・メール・部分一致・並び替えなど、よく使う検索パターンを Repository に追加します。SQL を自分で書かなくても検索ができる、Spring Data JPA の大きな魅力を体験しましょう。

この章で学ぶこと

  • クエリメソッド（派生クエリ）とは何か、なぜ実装コードを書かずに済むのかを説明できる
  • 命名規則（findBy + プロパティ名 + 条件キーワード）を理解し、自分でメソッド名を作れる
  • Containing・StartingWith・OrderBy・And/Or などのキーワードの意味を説明できる
  • EmployeeRepository にクエリメソッドを追加し、Service から呼び出せる

6.1 クエリメソッドとは — 名前を書くだけで検索ができる仕組み

第4章では findAll() や findById() など、JpaRepository が最初から用意しているメソッドを使いました。しかし実際のアプリケーションでは「名前で社員を探したい」「メールアドレスが一致する社員を取得したい」といった、条件つきの検索が必要になります。

Spring Data JPA には、クエリメソッド（派生クエリ）という仕組みがあります。Repository インターフェースに、決められた命名規則に従ったメソッドを宣言するだけで、Spring Data JPA がその名前を解析して SQL（JPQL）を自動的に生成・実行してくれます。実装クラスを書く必要はありません。

flowchart LR A["Repository に
メソッド名を書く"] --> B["Spring Data JPA が
名前を解析する"] B --> C["SQL を自動生成する
(JPQL)"] C --> D[("データベース")]

命名規則の基本形

findBy + エンティティのプロパティ名 + 条件キーワード という形式でメソッド名を作ります。プロパティ名の先頭は大文字にします（キャメルケース）。たとえば Employee の name フィールドで検索するなら findByName(String name) と書きます。

この仕組みの大きな特徴は、メソッド名が「仕様の文書」にもなる点です。findByNameContaining(String keyword) というメソッドを見れば、「name に keyword が含まれる社員を探す」と一目でわかります。コードが読みやすく、メンテナンスもしやすくなります。

6.2 よく使うクエリメソッドと生成される SQL

まず、本章で学ぶ主要なクエリメソッドを一覧で確認しましょう。Employee を題材に、メソッド名・引数・戻り値・生成される SQL の対応を整理します。

目的 メソッド名（例） 戻り値 生成される SQL イメージ
名前で検索 findByName(String name) List<Employee> WHERE name = ?
メールで検索 findByEmail(String email) List<Employee> WHERE email = ?
名前の部分一致 findByNameContaining(String keyword) List<Employee> WHERE name LIKE %?%
IDの降順で全件 findAllByOrderByIdDesc() List<Employee> ORDER BY id DESC
名前の前方一致 findByNameStartingWith(String prefix) List<Employee> WHERE name LIKE ?%
名前かつメール（AND） findByNameAndEmail(String name, String email) List<Employee> WHERE name = ? AND email = ?

補足: 「生成される SQL」は誰が作る？

正確には、Spring Data JPA がメソッド名を解析して検索条件を組み立て、最終的な SQL は Hibernate が発行します。上の表の「SQL イメージ」は理解のための概念図で、実際に DB に送られる SQL は spring.jpa.show-sql=true のログで確認できます（DB により細部は異なります）。なお、findByNameContaining("") のように空文字で検索すると LIKE %% となり全件がヒットする点にも注意しましょう。

表を見ると、「キーワードとプロパティ名を組み合わせる」というパターンがわかります。次のセクションで、各キーワードの意味を整理します。

6.3 条件キーワード一覧

Spring Data JPA が解釈できるキーワードには多くの種類がありますが、まず覚えておきたいものに絞って紹介します。これらを組み合わせるだけで、多くの検索パターンに対応できます。

キーワード 意味 SQL のイメージ 使用例
Containing 部分一致（前後ワイルドカード） LIKE %?% findByNameContaining
StartingWith 前方一致（後ろワイルドカード） LIKE ?% findByNameStartingWith
EndingWith 後方一致（前ワイルドカード） LIKE %? findByNameEndingWith
OrderBy ... Asc/Desc 並び替え（昇順/降順） ORDER BY ... ASC/DESC findAllByOrderByIdDesc
And 複数条件（すべて一致） WHERE ... AND ... findByNameAndEmail
Or 複数条件（どれか一致） WHERE ... OR ... findByNameOrEmail

重要ポイント: キーワードは組み合わせられる

これらのキーワードは組み合わせて使えます。たとえば findByNameContainingOrderByIdDesc(String keyword) と書けば「name に keyword が含まれる社員を id の降順で取得する」という検索になります。メソッド名を読むだけで意図がわかるのが強みです。

6.4 EmployeeRepository にクエリメソッドを追加する

実際に EmployeeRepository にクエリメソッドを追加してみましょう。インターフェースにメソッドを宣言するだけです。実装クラスは作りません。

EmployeeRepository.java — クエリメソッドを追加 
package com.example.jpalearning.repository;

import com.example.jpalearning.entity.Employee;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.List;

public interface EmployeeRepository extends JpaRepository<Employee, Long> {

    // 名前が一致する社員を検索（完全一致）
    List<Employee> findByName(String name);

    // メールアドレスが一致する社員を検索（完全一致）
    List<Employee> findByEmail(String email);

    // 名前に keyword が含まれる社員を検索（部分一致 LIKE）
    List<Employee> findByNameContaining(String keyword);

    // 全社員を id の降順で取得
    List<Employee> findAllByOrderByIdDesc();
}

これだけです。インターフェースにメソッドを宣言するだけで、Spring Data JPA がメソッド名を解析し、対応する SQL を自動で作成・実行してくれます。実装クラスを作る必要はありません。

注意: プロパティ名のスペルミスに気をつける

クエリメソッドのプロパティ名部分（findByName の Name）は、Entity クラスのフィールド名と完全に一致している必要があります。スペルミスがあると、アプリケーションの起動時にエラーが発生します（実行時ではなく起動時なので、早めに気づけます）。findByNaame のようなタイポは IDE の補完機能で防ぎましょう。

6.5 Service からクエリメソッドを呼び出す

Repository に追加したクエリメソッドを、EmployeeService から呼び出します。第5章と同じように、Service が Repository のメソッドを使うだけです。

EmployeeService.java — 検索メソッドを追加 
// 名前で社員を検索する
public List<Employee> searchByName(String name) {
    return employeeRepository.findByName(name);
}

// 名前の一部で社員を検索する（部分一致）
public List<Employee> searchByKeyword(String keyword) {
    return employeeRepository.findByNameContaining(keyword);
}

// 社員を id の降順で全件取得する
public List<Employee> findAllDesc() {
    return employeeRepository.findAllByOrderByIdDesc();
}

Service 側のコードはシンプルです。Repository に定義したメソッドを呼ぶだけで、検索処理が完成します。第10章で Spring MVC の Controller と接続し、画面から検索キーワードを受け取って結果を表示する完成形を作ります。

クエリメソッドが動く仕組み（処理フロー）

flowchart TD A["Service が Repository の
findByNameContaining を呼ぶ"] --> B["Spring Data JPA が
メソッド名を解析する"] B --> C["JPQL を自動生成する
(WHERE name LIKE ?)"] C --> D["Hibernate が SQL に変換する"] D --> E[("H2 Database で実行")] E --> F["List<Employee> として返す"]

クエリメソッドと @Query の使い分け

クエリメソッドはシンプルな検索に向いています。条件が増えてメソッド名が長くなりすぎるときや、JOINなど複雑な検索が必要なときは、次の第7章で学ぶ @Query アノテーションを使います。まずはクエリメソッドで書けるかを考え、難しければ @Query に切り替える、という判断基準が実務でもよく使われます。

やってみよう 6-1: クエリメソッドを追加して動作を確認する

実際に EmployeeRepository にクエリメソッドを追加し、コンソールログに出力される SQL を確認しましょう。application.properties で spring.jpa.show-sql=true が設定されていると、生成された SQL が確認できます。

手順

  1. 1 EmployeeRepository に 6.4 のコードを追加する。
  2. 2 EmployeeService に検索メソッドを追加し、CommandLineRunner などから呼び出して動作確認する。
  3. 3 コンソールに出力された SQL を確認し、findByNameContaining が LIKE %?% になっていることを確かめる。
  4. 4 （発展）findByNameAndEmail を追加し、2つの条件を組み合わせた SQL が生成されることも確認する。

確認ポイント

  • 起動時にエラーが出ないこと（スペルミスがないことの確認）
  • コンソールに期待通りの SQL が表示されること
  • 戻り値の List<Employee> に検索結果が入っていること

理解度確認クイズ

  1. Q1. クエリメソッドとは何ですか？実装クラスが不要な理由も含めて説明してください。

    回答を表示

    クエリメソッド（派生クエリ）とは、Repository インターフェースに命名規則に従ったメソッド名を宣言するだけで、Spring Data JPA がそのメソッド名を解析して SQL（JPQL）を自動生成・実行してくれる仕組みです。Spring Data JPA がプロキシ（実装）を自動で作るため、実装クラスを書く必要はありません。

  2. Q2. findByNameContaining(String keyword) と findByNameStartingWith(String prefix) の違いを、生成される SQL の観点で説明してください。

    回答を表示

    findByNameContaining は部分一致で LIKE %?%（前後ワイルドカード）のSQL が生成されます。findByNameStartingWith は前方一致で LIKE ?%（後ろのみワイルドカード）のSQL が生成されます。前者は指定した文字列が名前のどこかに含まれる検索、後者は名前が指定した文字列で始まる検索です。

  3. Q3. クエリメソッドのプロパティ名をスペルミスした場合、いつエラーが発生しますか？

    回答を表示

    アプリケーションの起動時にエラーが発生します。Spring Data JPA は起動時にリポジトリのメソッドを検査し、存在しないプロパティ名が含まれていると例外をスローします。実行時（メソッドを呼んだとき）ではなく起動時に気づけるため、早期に修正できます。
