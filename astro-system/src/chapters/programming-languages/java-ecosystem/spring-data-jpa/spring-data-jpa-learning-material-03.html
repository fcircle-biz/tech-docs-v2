Entityの基本

JPA で最初に書くのが Entity（エンティティ）クラスです。Entity とは、データベースのテーブルの「1行」に対応する Java クラスのことです。この章では、Entity を作るために欠かせない @Entity・@Table・@Id・@GeneratedValue・@Column の5つのアノテーションを学び、employees テーブルに対応する Employee クラスを実際に作ります。

この章で学ぶこと

  • Entity クラスが「テーブルの1行を表す Java クラス」であることを説明できる
  • @Entity・@Table・@Id・@GeneratedValue・@Column の役割をそれぞれ説明できる
  • Employee Entity クラスを正しく作成できる
  • Java クラスのフィールドと DB テーブルのカラムの対応関係を図で理解できる

3.1 Entity とは — テーブルの1行を表す Java クラス

JPA の世界では、データベースの「テーブル」と Java の「クラス」を対応させます。その対応関係を持つ Java クラスのことを Entity（エンティティ）と呼びます。

たとえば employees テーブルに次のようなデータが入っていたとします。このとき、テーブルの1行が Employee クラスの1オブジェクトに対応します。

id=1, name=山田太郎"] O2["Employee オブジェクト2
id=2, name=鈴木花子"] end R1 -- "1行 = 1オブジェクト" --> O1 R2 -- "1行 = 1オブジェクト" --> O2

この対応を JPA に教えるのが、次節で学ぶアノテーション（注釈）の役割です。クラスやフィールドにいくつかのアノテーションを付けるだけで、JPA が自動的に「このクラスはこのテーブル」「このフィールドはこのカラム」と判断してくれます。

重要ポイント

Entity は「テーブルの1行 = Java オブジェクトの1つ」という対応を持つクラスです。テーブルが変わっても Entity クラスを操作するだけでよく、SQL を直接書かずにすみます。これが JPA が「オブジェクト中心」と呼ばれる理由です。

3.2 Entity を作るアノテーション一覧

Entity クラスを作るために使うアノテーションは主に5つです。それぞれの役割を表で整理します。

アノテーション 付ける場所 役割
@Entity クラス 「このクラスは DB テーブルに対応する Entity だ」と JPA に伝える
@Table(name="...") クラス 対応するテーブル名を明示指定する（省略時はクラス名が使われる）
@Id フィールド このフィールドが主キー（Primary Key）であることを示す
@GeneratedValue(...) フィールド 主キーを DB が自動採番することを指定する
@Column(name="...") フィールド 対応するカラム名を明示したり、制約（NOT NULL など）を指定したりする

次のセクションから、それぞれのアノテーションを詳しく見ていきます。

3.3 @Entity と @Table — クラスとテーブルを対応させる

クラスの先頭に @Entity を付けると、「このクラスは JPA が管理する Entity だ」と宣言できます。JPA はこの目印があるクラスだけを Entity として扱います。

テーブル名はデフォルトでクラス名（Employee → employee）から推測されますが、クラス名とテーブル名が異なる場合は @Table(name = "employees") で明示します。本教材では複数形のテーブル名（employees）を使うため、必ず @Table を付けます。

@Table の省略について

@Table を省略すると、テーブル名はクラス名と同一（大文字・小文字の扱いは DB 依存）になります。H2 Database ではデフォルトで大文字に変換されるため、クラス名 Employee のままにすると EMPLOYEE テーブルが対象になります。命名規則を合わせるため @Table(name = "employees") と明示するのが安心です。

3.4 @Id と @GeneratedValue — 主キーと自動採番

テーブルの各行を一意に識別する主キー（Primary Key）のフィールドには、必ず @Id を付けます。これは JPA が「どのフィールドで行を識別するか」を知るために必須のアノテーションです。

通常、主キーは DB が自動で番号を割り当てます（自動採番）。これを指定するのが @GeneratedValue です。strategy 属性で採番方式を指定します。

IDENTITY（本教材で使用）

DB の AUTO_INCREMENT 機能に採番を任せる。H2・MySQL・PostgreSQL で利用可能。最もシンプルで初心者向け。

GenerationType.IDENTITY

SEQUENCE

DB のシーケンスオブジェクトを使って採番する。Oracle・PostgreSQL などで利用可能。

GenerationType.SEQUENCE

@Id がないと起動エラーになる

JPA は Entity に必ず主キーを要求します。@Id が付いたフィールドがないと、アプリ起動時に IdentifierGenerationException などのエラーが発生します。@Id を書き忘れないよう注意しましょう。

3.5 @Column — フィールドとカラムを対応させる

@Column はフィールドと DB カラムを対応づけるアノテーションです。フィールド名とカラム名が同じであれば省略できますが、名前が違う場合や制約（NOT NULL など）を付けたい場合に使います。

属性 意味 例
name 対応するカラム名を指定 @Column(name = "emp_name")
nullable NULL を許可するか（デフォルト true） @Column(nullable = false)
length VARCHAR の最大文字数（デフォルト 255） @Column(length = 100)
unique ユニーク制約を付ける @Column(unique = true)

@Column は省略できることが多い

フィールド名とカラム名が一致しており、特別な制約も不要な場合、@Column は省略できます。本教材の Employee クラスでは、name と email に nullable = false を指定して「必須項目」にします。制約の意味を学ぶため、あえて明示します。

3.6 Employee Entity クラスを作る

いよいよ実際に Employee クラスを作ります。第2章で作ったプロジェクトの entity パッケージにファイルを置きます。 

src/main/java/com/example/jpalearning/
└── entity/
    └── Employee.java  ← ここに作る
Employee.java 
package com.example.jpalearning.entity;

import jakarta.persistence.*;

@Entity                          // JPA の Entity クラスであることを宣言
@Table(name = "employees")       // 対応するテーブル名を指定
public class Employee {

    @Id                          // 主キー
    @GeneratedValue(strategy = GenerationType.IDENTITY) // DB が自動採番
    private Long id;

    @Column(name = "name", nullable = false, length = 100)  // NOT NULL
    private String name;         // 社員名

    @Column(name = "email", nullable = false, unique = true) // 重複不可
    private String email;        // メールアドレス

    // JPA はデフォルトコンストラクタを必須とする
    public Employee() {}

    // getter / setter（省略）…
}

コードのポイントを図で確認しましょう。クラスのフィールドと employees テーブルのカラムが1対1で対応しています。

flowchart LR subgraph Java["Employee クラス（Java）"] F1["Long id"] F2["String name"] F3["String email"] end subgraph DB["employees テーブル（DB）"] C1["id BIGINT PK AUTO_INCREMENT"] C2["name VARCHAR NOT NULL"] C3["email VARCHAR UNIQUE"] end F1 -- "@Id @GeneratedValue" --> C1 F2 -- "@Column nullable=false" --> C2 F3 -- "@Column unique=true" --> C3

デフォルトコンストラクタは必須

JPA は DB からデータを取得するとき、内部で 引数なしのコンストラクタ（デフォルトコンストラクタ）を使ってオブジェクトを生成します。独自のコンストラクタを定義したとき Java は自動生成を止めるため、必ず public Employee() {} を手書きしましょう。

3.7 Department Entity クラスも作る

本教材では Department（部署）クラスも使います。Employee と同じ手順で作りましょう。フィールドは id と name だけのシンプルな Entity です。

Department.java 
package com.example.jpalearning.entity;

import jakarta.persistence.*;

@Entity
@Table(name = "departments")     // departments テーブルに対応
public class Department {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;             // 部署ID

    @Column(name = "name", nullable = false, length = 100)
    private String name;         // 部署名

    // デフォルトコンストラクタ（JPA 必須）
    public Department() {}

    // getter / setter（省略）…
}

この章では関連を付けません

現時点の Employee クラスには Department への参照フィールドはまだ付けません。「社員が部署に属する」関連（@ManyToOne）は第8章で追加します。まずはそれぞれの Entity を単独で理解することに集中しましょう。

3.8 getter / setter の追加

Entity のフィールドは private で宣言するため、外部からアクセスするための getter / setter メソッドが必要です。JPA 自体は getter/setter を必須とはしませんが、Service や Controller からフィールド値を取り出すために実際には必須となります。

Employee.java — getter / setter を追加 
    // --- getter ---
    public Long getId()      { return id; }
    public String getName()  { return name; }
    public String getEmail() { return email; }

    // --- setter ---
    public void setId(Long id)         { this.id = id; }
    public void setName(String name)   { this.name = name; }
    public void setEmail(String email) { this.email = email; }

IDE の自動生成を活用しよう

IntelliJ IDEA では Alt + Insert → Getter and Setter（macOS は Cmd + N）で getter/setter を自動生成できます。Eclipse では右クリックメニューの「Source → Generate Getters and Setters」を使うと手書きの手間が省けます。

やってみよう 3-1: Employee と Department を作る

第2章で作ったプロジェクトに、Employee と Department の Entity クラスを追加しましょう。

理解度確認クイズ

  1. Q1. @Entity と @Table(name = "employees") の役割をそれぞれ説明してください。

    回答を表示

    @Entity は「このクラスが JPA の管理対象 Entity である」と宣言するアノテーションです。@Table(name = "employees") は対応するテーブル名を明示します。クラス名（Employee）とテーブル名（employees）が異なるため @Table で名前を指定しています。

  2. Q2. @Id を付け忘れるとどうなりますか？また、@GeneratedValue(strategy = GenerationType.IDENTITY) は何を指定していますか？

    回答を表示

    @Id を付け忘れると、アプリ起動時に JPA が主キーを認識できずエラーになります。@GeneratedValue(strategy = GenerationType.IDENTITY) は「主キーの値は DB の AUTO_INCREMENT 機能で自動採番する」ことを指定しています。

  3. Q3. JPA の Entity クラスにデフォルトコンストラクタが必要な理由を説明してください。

    回答を表示

    JPA は DB からデータを取得するとき、リフレクションを使って引数なしのコンストラクタでオブジェクトを生成し、その後 setter でフィールドに値を設定します。デフォルトコンストラクタがないと JPA がオブジェクトを生成できないため、実行時エラーになります。
