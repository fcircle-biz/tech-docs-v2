新規登録フォームを作る

Web アプリケーションでは、新しいデータを登録するときに「入力フォーム」を使います。この章では、まず GET リクエストで入力フォームを表示し、次に POST リクエストで入力内容をサーバーに送って登録する、という2段階の流れを学びます。GET と POST の違いを理解することが、この章の最重要ポイントです。

この章で学ぶこと

  • GET と POST の違いを理解する
  • 登録フォームを表示する方法を理解する
  • フォーム入力値を Controller で受け取る方法を理解する

7.1 GET と POST の違い

ブラウザからサーバーへリクエストを送る方法には、大きく分けて GET と POST の2種類があります。初心者向けにひとことで言うと、「GET は画面を表示するとき、POST はデータを登録・更新するときに使います」と覚えておきましょう。

種類 使う場面 主な特徴
GET 画面を表示する（データを取得する） ブラウザで URL を入力してアクセスするときや、リンクをクリックしたときに使われる
POST データを登録・更新する（データを送信する） フォームの「送信」ボタンを押したときに使われる。入力内容をサーバーへ送る

重要ポイント: GET と POST の使い分け

  • GET：画面を「見る」だけ。登録フォームの画面を表示するときに使う。
  • POST：データを「送る」。フォームに入力した内容をサーバーに送って登録・更新するときに使う。

新規登録の流れは「GET でフォーム表示 → 入力 → POST で送信・登録」の2段階です。

flowchart TD A["ブラウザ"] -->|"GET /users/new"| B["Controller
@GetMapping"] B -->|"フォーム画面を返す"| C["View
（入力フォーム）"] C -->|"ユーザーが名前・メールを入力"| D["フォーム送信ボタンをクリック"] D -->|"POST /users"| E["Controller
@PostMapping"] E -->|"登録処理を実行"| F["Service"] F -.登録完了.-> E E -->|"redirect:/users"| G["一覧画面へリダイレクト"]

この図のように、登録処理は 2つの Controller メソッドで成り立っています。最初の GET は「フォームの画面を表示するだけ」で、実際にデータを保存するのは後の POST です。Spring MVC では、@GetMapping と @PostMapping というアノテーションでそれぞれの担当を分けて書きます。

7.2 フォーム用クラス（UserForm）を作る

フォームの入力値（名前・メールなど）をまとめて受け取るために、フォーム用クラス（UserForm）を作ります。このクラスは「フォームに入力した値をまとめて入れておく箱」です。各フィールドとそれに対応する getter/setter メソッドで構成されます。

用語: getter と setter

getter はフィールドの値を「取得」するメソッド（getName() など）、setter はフィールドに値を「セット（代入）」するメソッド（setName(...) など）です。Spring MVC はこの getter/setter を使ってフォームの入力値をクラスに自動で入れてくれます。

まず、フィールドを定義します。

UserForm.java（フィールド定義） 
public class UserForm {
    // フォームの「名前」入力欄に対応するフィールド
    private String name;

    // フォームの「メール」入力欄に対応するフィールド
    private String email;
}

次に、getter と setter を追加します。

UserForm.java（getter/setter） 
    // name フィールドの getter（値を取得）
    public String getName() {
        return name;
    }

    // name フィールドの setter（値をセット）
    public void setName(String name) {
        this.name = name;
    }

    // email フィールドの getter（値を取得）
    public String getEmail() {
        return email;
    }

    // email フィールドの setter（値をセット）
    public void setEmail(String email) {
        this.email = email;
    }

UserForm は、フォームに入力された値を Spring MVC が自動的に詰め込んでくれる「入れ物」です。特別な処理を自分で書く必要はなく、フィールドと getter/setter を用意するだけで動作します。

7.3 フォーム表示用 Controller（GET）

登録フォームの画面を表示するには、@GetMapping を使った Controller メソッドを書きます。このメソッドは「フォームの画面を見せてください」というリクエスト（GET）を受け取り、空の UserForm を Model に入れて View に渡します。

UserController.java（GET: フォーム表示） 
// /users/new への GET リクエストを処理する
@GetMapping("/users/new")
public String newForm(Model model) {
    // 空の UserForm を Model に追加して View へ渡す
    model.addAttribute("userForm", new UserForm());
    // templates/users/form.html を表示する
    return "users/form";
}

@GetMapping("/users/new")

ブラウザが /users/new に GET リクエストを送ったとき、このメソッドが呼ばれます。「登録フォームを表示して」というリクエストに対応します。

new UserForm()

中身が空の UserForm を作って Model に入れます。Thymeleaf の th:object でこの UserForm を参照し、フォームの入力欄と紐づけます。

return "users/form" は、src/main/resources/templates/users/form.html を表示することを意味します。次のセクションでこの HTML ファイルを作ります。

7.4 フォームの HTML（Thymeleaf）

フォームの画面（View）は Thymeleaf を使った HTML で作ります。ポイントは th:object と th:field の2つです。th:object でフォームと UserForm を紐づけ、th:field で各入力欄とフィールドを紐づけます。

templates/users/form.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ユーザー登録</title>
</head>
<body>
    <h1>ユーザー登録</h1>
    <!-- th:object でフォームと userForm を紐づける -->
    <form th:action="@{/users}" th:object="${userForm}" method="post">
        <div>
            <label>名前：</label>
            <!-- th:field で name フィールドと紐づける -->
            <input type="text" th:field="*{name}">
        </div>
        <div>
            <label>メール：</label>
            <input type="email" th:field="*{email}">
        </div>
        <button type="submit">登録</button>
    </form>
    <a th:href="@{/users}">一覧へ戻る</a>
</body>
</html>
属性 意味 例
th:action="@{/users}" 送信先の URL を指定する /users へ POST する
th:object="${userForm}" フォームと紐づけるオブジェクトを指定する Model に入れた userForm を参照
th:field="*{name}" 入力欄と UserForm のフィールドを紐づける name フィールドと対応
method="post" フォーム送信に POST を使う 登録・更新操作なので POST を指定

用語: th:object と th:field の使い方

th:object="${userForm}" は「このフォームは userForm というオブジェクトと対応しています」という宣言です。th:field="*{name}" の *{...} は「th:object で指定したオブジェクトの〇〇フィールド」を意味します。つまり *{name} は ${userForm.name} と同じ意味です。

7.5 登録処理用 Controller（POST）

フォームの「登録」ボタンを押すと、ブラウザは /users へ POST リクエストを送ります。これを受け取るのが @PostMapping を使った Controller メソッドです。@ModelAttribute を使うと、フォームの入力値が自動的に UserForm に詰め込まれた状態でメソッドに渡されます。

UserController.java（POST: 登録処理） 
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PostMapping;

// /users への POST リクエストを処理する
@PostMapping("/users")
public String create(@ModelAttribute UserForm userForm) {
    // UserForm から名前とメールを取り出して登録する
    userService.create(userForm.getName(), userForm.getEmail());
    // 登録後は一覧画面へリダイレクトする
    return "redirect:/users";
}

@PostMapping("/users")

/users への POST リクエストを受け取ります。フォームの送信ボタンを押したときに呼ばれます。

@ModelAttribute

フォームの入力値を UserForm に自動で詰め込んでくれます。自分でリクエストパラメータを1つずつ取り出す必要はありません。

redirect:/users

登録完了後にブラウザを一覧画面 (/users) へリダイレクトします。「redirect:」プレフィックスで指定します。

重要ポイント: リダイレクトとは

リダイレクトとは、「この画面ではなく、あちらの画面へ移動してください」とブラウザに指示することです。return "redirect:/users" と書くと、登録処理が完了した後にブラウザが自動で /users へアクセスし直し、一覧画面が表示されます。登録後に同じフォームが再表示される代わりに、一覧画面へ移動するためにリダイレクトを使います。

7.6 登録フローのまとめ

この章で学んだ登録フローを整理しましょう。URL・HTTP メソッド・Controller・View の対応を表にすると次のようになります。

操作 URL HTTP メソッド Controller アノテーション 返す値
フォーム表示 /users/new GET @GetMapping "users/form"
登録処理 /users POST @PostMapping "redirect:/users"

URL が違う点と、HTTP メソッドが GET/POST で異なる点に注目しましょう。同じ Controller クラスの中に @GetMapping と @PostMapping の2つのメソッドを書くことで、フォーム表示と登録処理の両方に対応できます。

やってみよう 7-1: ユーザー登録フォームを動かす

この章で学んだ内容を実際にコードに書いて、ブラウザで登録フォームを動かしてみましょう。

手順

  1. 1 UserForm.java を作成し、name・email フィールドと getter/setter を追加する。
  2. 2 UserController.java に newForm メソッド（GET）と create メソッド（POST）を追加する。
  3. 3 templates/users/form.html を作成し、th:object と th:field を使ったフォームを書く。
  4. 4 アプリを起動して http://localhost:8080/users/new にアクセスし、名前とメールを入力して登録ボタンを押す。一覧画面へ戻れば成功！

理解度確認クイズ

  1. Q1. GET はどのような処理に使いますか？

    回答を表示

    画面を表示する（データを取得する）ときに使います。ブラウザで URL を入力してアクセスするときやリンクをクリックしたときに GET リクエストが送られます。登録フォームを表示するときに使うのが GET です。

  2. Q2. POST はどのような処理に使いますか？

    回答を表示

    データを登録・更新する（データを送信する）ときに使います。フォームの送信ボタンを押したときに POST リクエストが送られ、入力内容がサーバーへ届きます。

  3. Q3. UserForm は何のために作りますか？

    回答を表示

    フォームの入力値（名前・メールなど）をまとめて受け取るための「入れ物（クラス）」です。Spring MVC が自動でフォームの入力値を UserForm に詰め込んでくれるため、Controller で簡単に値を取り出せます。

  4. Q4. 登録後に一覧へ戻るには何を返しますか？

    回答を表示

    "redirect:/users" を返します。redirect: プレフィックスを付けることで、ブラウザが /users へ自動でアクセスし直し、一覧画面が表示されます。
