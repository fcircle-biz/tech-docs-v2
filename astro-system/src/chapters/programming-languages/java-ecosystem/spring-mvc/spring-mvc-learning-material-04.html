第4章 所要時間: 約55分

Modelを使って画面にデータを渡す

Controller が処理した結果を HTML 画面に表示するには、Model（モデル）という「データの箱」を使います。この章では、Controller から Model へデータを入れる方法と、Thymeleaf の HTML 側でそのデータを表示する方法を、ひとつひとつ丁寧に学んでいきます。

この章で学ぶこと

  • Model の役割（Controller から View へデータを渡す「箱」）を理解する
  • Controller から View へデータを渡す方法（model.addAttribute）を理解する
  • Thymeleaf の HTML でデータを表示する方法（th:text と ${...}）を理解する

4.1 Model とは — データを渡す「箱」

第3章では、Controller がブラウザのリクエストを受け取る方法を学びました。しかし、Controller が受け取ったデータや処理結果を画面（HTML）に表示するには、どうすればいいのでしょうか？

ここで登場するのが Model（モデル）です。Model は、Controller が View（HTML 画面）へデータを渡すための「箱」です。Controller がデータを箱に入れると、View 側でその箱の中身を取り出して画面に表示できます。

flowchart LR A["Controller
（処理の司令塔）"] -->|"model.addAttribute()
でデータを入れる"| B["Model
（データの箱）"] B -->|"${変数名} で
取り出す"| C["View
（HTML画面）"]

Model はいわばController と View をつなぐ「中継係」です。Controller は「画面に何を表示するか」を知っていますが、直接 HTML を書くことはしません。代わりに Model という箱にデータを詰め込んで、View に「あとはよろしく」と渡します。View はその箱の中身を見て、HTML を組み立てます。

用語: Model（モデル）

Spring MVC における Model は、org.springframework.ui.Model というインターフェースです。Controller のメソッド引数に書くだけで、Spring が自動的に用意してくれます。難しいことは考えず、「Controller が画面に渡したいデータを入れる箱」と覚えておけば大丈夫です。

4.2 Controller から Model にデータを入れる

Controller のメソッドに Model model という引数を追加するだけで、Spring が Model を自動的に用意してくれます。そして model.addAttribute("名前", 値) を呼び出すことで、データを Model に入れられます。

UserController.java 
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class UserController {

    @GetMapping("/users")
    public String list(Model model) {
        // Model にデータを追加する
        model.addAttribute("message", "ユーザー一覧画面です");
        // 表示するビュー（テンプレートファイル）の名前を返す
        return "users/list";
    }
}

コードのポイントを確認しましょう。

コード 意味
Model model 引数に書くだけで Spring が Model（箱）を用意してくれる
model.addAttribute("message", 値) 箱に "message" という名前でデータを入れる
return "users/list" src/main/resources/templates/users/list.html を表示する

重要ポイント

model.addAttribute("名前", 値) の「名前」は、後ほど HTML 側で値を取り出すときに使うキーです。同じ名前を HTML 側の ${名前} で指定することで、Controller が入れた値を表示できます。名前はわかりやすい英単語にしましょう。

4.3 Thymeleaf で Model のデータを表示する

Controller が Model に入れたデータは、Thymeleaf（タイムリーフ）という HTML テンプレートエンジンを使って画面に表示します。Thymeleaf では、HTML タグに特別な属性を書くことで、Java 側のデータを埋め込めます。

src/main/resources/templates/users/list.html 
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>ユーザー一覧</title>
</head>
<body>
    <!-- th:text="${message}" で Model の値を表示する -->
    <h1 th:text="${message}">メッセージ</h1>
</body>
</html>

Thymeleaf の記述のポイントを見てみましょう。

xmlns:th

xmlns:th="..." を <html> タグに書くことで Thymeleaf 属性が使えるようになる

th:text

タグの中身のテキストを Java の値で置き換える Thymeleaf 属性

${message}

Model に入れた "message" という名前のデータを参照する式

th:text の動作のしくみ

<h1 th:text="${message}">メッセージ</h1> と書いた場合、ブラウザには「メッセージ」ではなく、Controller が model.addAttribute で入れた値が表示されます。タグの中の「メッセージ」という文字は、Thymeleaf が動いていないとき（HTML ファイルをそのまま開いたとき）の仮テキストです。Thymeleaf が動作すると th:text の値で上書きされます。

4.4 データの流れをまとめる

ここまで学んだ内容を整理しましょう。Controller → Model → View という流れが Spring MVC のデータ受け渡しの基本です。

flowchart TD A["ブラウザ"] -->|"GET /users"| B["Controller
list（Model model）"] B -->|"model.addAttribute
（'message', '一覧画面です'）"| C["Model
（箱にデータが入る）"] C -->|"return 'users/list'"| D["View
users/list.html"] D -->|"th:text=${message}
でデータを表示"| E["ブラウザに
HTMLを返す"]

流れを言葉でまとめると次のようになります。

  1. 1 ブラウザが /users へリクエストを送る
  2. 2 Controller の list メソッドが呼ばれる。Spring が自動的に Model を用意して渡してくれる
  3. 3 model.addAttribute("message", "ユーザー一覧画面です") で箱にデータを入れる
  4. 4 return "users/list" で templates/users/list.html を View として指定する
  5. 5 Thymeleaf が th:text="${message}" を見て箱の中のデータをタグの中身に埋め込み、HTML をブラウザに返す

まとめ: Model の3つのルール

  • ① Controller のメソッド引数に Model model と書くだけで Spring が用意してくれる
  • ② model.addAttribute("名前", 値) でデータを入れる
  • ③ HTML 側では ${名前} でデータを取り出す

やってみよう 4-1: Model に複数のデータを追加してみる

model.addAttribute は1つのメソッドで1つのデータしか入れられませんが、何度でも呼び出せます。複数のデータを渡して画面に表示する練習をしてみましょう。

手順

  1. 1 UserController の list メソッドに model.addAttribute("title", "ユーザー管理") を1行追加する
  2. 2 list.html に <p th:text="${title}"></p> を追加して、title の値も表示されるようにする
  3. 3 アプリを起動して http://localhost:8080/users にアクセスし、2つの値が画面に表示されることを確認する

理解度確認クイズ

  1. Q1. Model は何のために使いますか？

    回答を表示

    Controller から View（HTML 画面）へデータを渡すための「箱」として使います。Controller が Model にデータを入れ、View がその中身を取り出して画面に表示します。

  2. Q2. model.addAttribute は何をするメソッドですか？

    回答を表示

    Model（箱）に「名前」と「値」のペアでデータを追加するメソッドです。model.addAttribute("message", "ユーザー一覧画面です") と書くと、"message" という名前で文字列データが箱に入ります。

  3. Q3. Thymeleaf で Model の値を表示するとき、どのように書きますか？

    回答を表示

    HTML タグに th:text="${名前}" という属性を付けて書きます。例えば "message" という名前で入れたデータを表示するには <h1 th:text="${message}"></h1> と書きます。th:text はタグの中身のテキストを Java の値で置き換える Thymeleaf 属性です。
