Controllerの基本

この章では、Spring MVC の中心的な存在である Controller の書き方を学びます。ブラウザからのリクエストを受け取るクラスをどう作るか、@Controller と @GetMapping という2つのアノテーションを使って、「リクエストを受け付けて画面を返す」最もシンプルな仕組みを手を動かしながら理解していきましょう。

この章で学ぶこと

  • @Controller の役割を理解する
  • @GetMapping の基本を理解する
  • Controller から View を返す流れを理解する

3.1 Controller とは何か

Controller（コントローラー）は、ブラウザからのリクエストを受け取るクラスです。第1章で学んだとおり、Web アプリケーションでは「ブラウザが何かお願い（リクエスト）を送り、サーバーが返事（レスポンス）を返す」というやり取りが基本です。この「受け取り窓口」の役割を担うのが Controller です。

Java のクラスを Spring MVC の Controller にするには、クラスに @Controller というアノテーション（設定のしるし）を付けます。これだけで Spring が「このクラスはリクエストを受け取る担当だ」と認識してくれます。

用語: アノテーションとは

アノテーションは、クラスやメソッドに付ける「設定のしるし」です。@Controller と書くだけで、Spring がそのクラスを「Web リクエストの受け取り役」として自動認識してくれます。コードに直接設定を書く代わりに、@ 記号ではじまる短い宣言で意図を伝えられるのが特徴です。

Controller はレストランのホール係のような存在です。お客さん（ブラウザ）から注文（リクエスト）を受け取り、厨房（Service）に伝え、できあがった料理（データ）を View（お皿に盛り付け）を通してお客さんに出します。ホール係は自分で料理を作るのではなく、受け取り・振り分け・返却を担当します。

3.2 @GetMapping — GETリクエストに対応する

ブラウザからサーバーへのリクエストには、いくつかの種類（メソッド）があります。最もよく使うのが GET リクエストです。

用語: GETリクエストとは

GETリクエストは、「画面を見る・データを取得するためのお願い」です。ブラウザのアドレスバーに URL を入力して Enter を押すと、ブラウザは自動的に GET リクエストを送ります。ショッピングサイトの商品一覧を見る、ニュース記事を開く——こうした「読むだけ・見るだけ」の操作はほぼすべて GET リクエストです。データを変更しない点が特徴で、画面表示に使うことが多いです。

Controller のメソッドに @GetMapping("/users") と書くと、「/users という URL への GET リクエストが来たとき、このメソッドを実行する」という設定になります。URL の /users の部分はパスと呼び、どのページへのリクエストかを示します。

flowchart LR A["ブラウザ"] -->|"GET /users"| B["DispatcherServlet
(Spring の受付)"] B --> C["UserController
list メソッド"] C -->|"users/list を返す"| D["View
(templates/users/list.html)"] D -->|"HTML を返す"| A

上の図のように、ブラウザが /users に GET リクエストを送ると、Spring の受付役（DispatcherServlet）が適切な Controller メソッドに処理を振り分けます。Controller メソッドが View の名前を返すと、対応する HTML テンプレートが画面としてブラウザに返ります。この流れが Spring MVC の基本です。

3.3 Controllerのコードを書く

実際にシンプルな Controller を作ってみましょう。ユーザー一覧画面を表示する UserController の例です。

UserController.java 
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class UserController {
    // /users にアクセスされたとき実行するメソッド
    @GetMapping("/users")
    public String list() {
        // templates/users/list.html を表示する
        return "users/list";
    }
}

このコードのポイントを1つずつ確認します。

@Controller

このクラスが Web 画面用の Controller であることを Spring に伝えます。このアノテーションがあることで、Spring はこのクラスをリクエストの受け取り役として管理します。

@GetMapping("/users")

ブラウザから /users への GET リクエストが来たとき、この list() メソッドを実行します。URL の /users の部分を変えることで、どのページのリクエストに対応するかを変えられます。

return "users/list"

戻り値の文字列がテンプレート名になります。"users/list" は src/main/resources/templates/users/list.html を指します。Spring がこのパスの HTML ファイルを探して画面として返します。

public String list()

メソッドの戻り値の型は String です。この文字列がテンプレート名として解釈されます。メソッド名（list）は自由に決められますが、処理の内容が分かりやすい名前をつけましょう。

重要ポイント: return 文とテンプレート名

return "users/list" と書くと、Spring は src/main/resources/templates/users/list.html を探して画面として返します。スラッシュ区切りのパスがそのままフォルダ構成に対応します。たとえば return "top" なら templates/top.html、return "admin/dashboard" なら templates/admin/dashboard.html が表示されます。

3.4 対応する View（HTML）を用意する

Controller が "users/list" を返すだけでは画面は表示されません。対応する HTML ファイルも必要です。ここでは Thymeleaf のシンプルな HTML ファイルを作成します。まずは Java のデータを表示しない、ただの HTML から始めましょう。

templates/users/list.html 
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>ユーザー一覧</title>
</head>
<body>
    <h1>ユーザー一覧</h1>
</body>
</html>

このファイルを src/main/resources/templates/users/list.html として保存します。Controller が return "users/list" を返すと、Spring が自動的にこのファイルを見つけて、ブラウザに HTML として返してくれます。

templates フォルダの場所

Thymeleaf テンプレートは src/main/resources/templates/ フォルダに置きます。Spring Boot はデフォルトでここを探します。users/ サブフォルダを作って list.html を置くことで、Controller の return "users/list" が対応するファイルを見つけられます。フォルダを作り忘れると 404 エラーになるので注意しましょう。

3.5 リクエストから画面表示までの流れ（まとめ）

ここまでの内容を整理します。ブラウザが /users にアクセスしてから画面が表示されるまでの流れを確認しましょう。

ステップ 誰が 何をするか
① ブラウザ /users に GET リクエストを送る
② Spring（DispatcherServlet） @GetMapping("/users") のついたメソッドを探して呼び出す
③ UserController.list() return "users/list" でテンプレート名を返す
④ Spring（ViewResolver） templates/users/list.html を見つけて HTML を組み立てる
⑤ ブラウザ 受け取った HTML を画面に表示する

この流れの中で、あなたが書くコードは Controller のメソッドと templates フォルダの HTML ファイルの2つだけです。Spring が間をつないで自動的に対応付けてくれます。

重要ポイント: 画面表示には GET リクエストを使うことが多い

一覧表示・詳細表示など「データを見るだけ」の処理は GET リクエスト（@GetMapping）を使います。データを新規作成・変更・削除するときは POST などの別のリクエストを使いますが、それは第7章で学びます。まずは「画面を表示する = GET + @GetMapping」と覚えましょう。

やってみよう 3-1: UserController を作成する

この章で学んだ内容をもとに、実際に Controller と View を作成してブラウザで確認してみましょう。

手順

  1. 1 src/main/java/.../controller/ フォルダに UserController.java を作成し、この章のコードをそのまま書く。
  2. 2 src/main/resources/templates/users/ フォルダを作成し、list.html を置く（この章の HTML をコピー）。
  3. 3 アプリを起動して、ブラウザで http://localhost:8080/users にアクセスする。「ユーザー一覧」という見出しが表示されれば成功！

確認ポイント

  • ・ ブラウザに「ユーザー一覧」と表示されるか
  • ・ 404 エラーが出る場合: templates/users/ フォルダと list.html の場所を確認
  • ・ アプリが起動しない場合: @Controller の import 文が正しいか確認

理解度確認クイズ

  1. Q1. @Controller は何を表しますか？

    回答を表示

    そのクラスが Web 画面用の Controller（ブラウザからのリクエストを受け取る担当）であることを Spring に伝えるアノテーションです。このアノテーションを付けることで、Spring がそのクラスをリクエストの受け取り役として自動認識します。

  2. Q2. @GetMapping は何に対応するアノテーションですか？

    回答を表示

    GET リクエストに対応するアノテーションです。@GetMapping("/users") と書くと、/users への GET リクエストが来たときに、そのメソッドが実行されます。GET リクエストは「画面を見る・データを取得するためのお願い」で、ブラウザで URL を入力してアクセスする操作がこれにあたります。

  3. Q3. return "users/list" はどのHTMLを表示しますか？

    回答を表示

    src/main/resources/templates/users/list.html を表示します。戻り値の文字列（"users/list"）がテンプレート名として解釈され、Spring が templates/ フォルダ内の対応するパスの HTML ファイルを探して画面として返します。
