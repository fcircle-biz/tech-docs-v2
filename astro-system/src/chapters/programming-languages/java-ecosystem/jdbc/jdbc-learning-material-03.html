データの検索（SELECT文）をマスターする

第2章で H2 データベースへの接続に成功しました。この章では、データベースから情報を取り出す「検索」の方法を学びます。Statement と PreparedStatement で SELECT 文を実行し、結果を ResultSet で受け取って処理する流れを、段階的にマスターします。

この章で学ぶこと

  • executeQuery() で SELECT 文を実行し ResultSet を取得できる
  • ResultSet のカーソルの仕組みを理解し、next() でループ処理できる
  • getInt() / getString() などで列の値を型に合わせて取り出せる
  • WHERE 句を使った条件検索を、安全な PreparedStatement で実装できる

この章の前提

第2章の環境が整っており、H2 データベースに接続できる状態を前提とします。また、users テーブル（id・name・email 列）がすでに作成・データ投入済みの状態で進めます。まだの場合は第2章に戻って確認してください。

3.1 検索の基本 — executeQuery() の役割

データベースからデータを取り出すには、SELECT 文を実行します。JDBC では、Statement というオブジェクトを使って SQL をデータベースへ送ります。SELECT 文の場合は executeQuery() メソッドを使います。このメソッドは検索結果を ResultSet（結果セット）として返します。

基本の流れは3ステップです。まず Connection.createStatement() で Statement を作ります。次に stmt.executeQuery(SQL文字列) で SQL を実行します。最後に、返ってきた ResultSet からデータを取り出します。

flowchart LR A["Connection"] -->|"createStatement()"| B["Statement"] B -->|"executeQuery(SQL)"| C["ResultSet
（検索結果）"] C -->|"next() でループ"| D["各行のデータ"]

executeQuery() は SELECT 専用のメソッドです。INSERT・UPDATE・DELETE などデータを変更する操作には使いません（それらは第4章で学ぶ executeUpdate() を使います）。SELECT でデータを取り出すときは必ず executeQuery() を使う、と覚えておきましょう。

用語: Statement とは？

Statement は「SQL 文を実行する役割を持つオブジェクト」です。Connection（接続）を通じて作成し、executeQuery() や executeUpdate() などのメソッドで SQL を送ります。実は JDBC には Statement の改良版として PreparedStatement があり、安全性と性能の面で優れています。後の 3.4 節で紹介します。

3.2 ResultSet の仕組み — カーソルで1行ずつ進む

ResultSet は「検索結果の表を指すカーソル（現在位置を示すポインタ）」です。最初、カーソルは先頭行の前にあり、まだどのデータも指していません。next() を呼ぶたびにカーソルが1行ずつ下に進みます。次の行が存在すれば true を返し、もう行がなければ false を返します。

flowchart TD START["ResultSet 初期状態
（先頭行の前）"] -->|"next() → true"| R1["1行目: id=1, name=佐藤太郎"] R1 -->|"next() → true"| R2["2行目: id=2, name=鈴木花子"] R2 -->|"next() → true"| R3["3行目: id=3, name=田中次郎"] R3 -->|"next() → false"| END["ループ終了"]

この仕組みにより、while (rs.next()) と書くだけで「結果が存在する間ループする」コードが書けます。結果が0件なら最初から false を返すので、ループ本体は1回も実行されません。

重要ポイント

ResultSet は最初は先頭行を指していない点が初心者に多い誤解です。next() を一度も呼ばずに getString() を呼ぶとエラーになります。必ず while (rs.next()) や if (rs.next()) で「カーソルを進めてから」取り出してください。

3.3 1件ずつ取り出す — getXxx() で列の値を取得する

カーソルが行を指している状態で、rs.getInt("id") や rs.getString("name") のように呼ぶと、その行の指定した列の値を取得できます。列名を文字列で指定するのが基本です（コードの可読性が高く、列の順番が変わっても壊れにくいため）。

列番号で指定することもできます（1から始まります）。例えば rs.getString(2) は2番目の列の値を返します。ただし、列の順番が変わるとバグになりやすいため、列名指定を推奨します。

SelectAllExample.java — 全件取得の基本 
// Statement を作成して SELECT を実行
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT id, name, email FROM users");

// next() で1行ずつ進みながら取り出す
while (rs.next()) {
    int    id    = rs.getInt("id");       // 列名で取得（推奨）
    String name  = rs.getString("name");
    String email = rs.getString("email");
    System.out.println(id + ": " + name + " <" + email + ">");
}

列名指定 vs 列番号指定

  • rs.getString("name") — 列名指定（推奨）。列順が変わっても動く
  • rs.getString(2) — 列番号指定（1始まり）。可読性が低く壊れやすい

迷ったら列名指定を使いましょう。コードが自己説明的になり、後から読んでも何の列か一目で分かります。

3.4 WHERE 句で条件検索 — PreparedStatement を使う

特定条件のデータだけを取り出すには、SQL の WHERE 句を使います。ただし、条件の値を SQL 文字列に直接つなげる方法は絶対に避けてください。

危険: 文字列連結は SQL インジェクションの原因

下記のような書き方は SQL インジェクション攻撃を受ける危険があります。悪意あるユーザーが name に ' OR '1'='1 などを入力すると、意図しない全件取得やデータ破壊が起きます。

// NG: 文字列連結で SQL を組み立てる（危険）
String sql = "SELECT * FROM users WHERE name = '" + userName + "'";

SQL インジェクションの詳細と完全な対策は第5章で学びます。本章では安全な方法として PreparedStatement を使います。

安全に条件検索を行うには PreparedStatement を使います。SQL の中に ?（プレースホルダ）を書いておき、後から値をバインドする方法です。値は自動的にエスケープされるため、SQL インジェクションを防げます。

SelectByNameExample.java — PreparedStatement で条件検索 
// ? をプレースホルダとして使う
String sql = "SELECT id, name, email FROM users WHERE name = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 1番目の ? に値をセット（setString の第1引数はプレースホルダの番号）
pstmt.setString(1, "佐藤太郎");

// executeQuery() で検索実行
ResultSet rs = pstmt.executeQuery();

if (rs.next()) {
    // 1件見つかった場合
    System.out.println("ID: " + rs.getInt("id"));
    System.out.println("メール: " + rs.getString("email"));
}

prepareStatement() と executeQuery() の順番

conn.prepareStatement(sql) でテンプレートを作り → setXxx(番号, 値) で穴埋めし → executeQuery() で実行、という順番を守ります。executeQuery() には SQL 文字列を渡しません（PreparedStatement の場合は引数なしで呼びます）。

3.5 getXxx() メソッドと Java の型

ResultSet から値を取り出すメソッドは、列のデータ型に合わせて使い分けます。型を間違えるとエラーや精度落ちの原因になります。よく使うメソッドをまとめました。

メソッド 返す Java 型 対応する SQL 型の例 用途
getInt() int INT, INTEGER ID・連番・整数値
getString() String VARCHAR, CHAR, TEXT 名前・メール・コード
getLong() long BIGINT 大きな整数・タイムスタンプ
getDouble() double DOUBLE, FLOAT 浮動小数点の数値
getBoolean() boolean BOOLEAN, BIT フラグ・有効/無効
getDate() java.sql.Date DATE 日付（年月日）
getTimestamp() java.sql.Timestamp TIMESTAMP, DATETIME 日時（年月日時分秒）
getBigDecimal() BigDecimal DECIMAL, NUMERIC 金額・正確な小数

NULL 値に注意

データベースの列が NULL の場合、getInt() は 0、getString() は null を返します。NULL かどうかを確認するには rs.wasNull() を使うか、getString() 後に != null をチェックします。

理解度確認クイズ

  1. Q1. executeQuery() の戻り値は何型ですか？また、何のために使いますか？

    回答を表示

    戻り値は ResultSet 型です。SELECT 文を実行した結果（検索結果の表）を受け取るために使います。この ResultSet を通して、1行ずつデータを取り出すことができます。

  2. Q2. ResultSet の next() メソッドはどんな役割を持ちますか？返り値は何ですか？

    回答を表示

    カーソルを次の行へ進める役割を持ちます。次の行が存在すれば true、もう行がなければ false を返します（boolean 型）。while (rs.next()) と書くことで、全行を順番に処理するループを作れます。

  3. Q3. ResultSet から int 型の列を取り出すメソッドと、String 型を取り出すメソッドをそれぞれ答えてください。列名指定と列番号指定の違いも説明してください。

    回答を表示

    int 型は rs.getInt()、String 型は rs.getString() を使います。
    引数は「列名（文字列）」または「列番号（1始まりの整数）」を指定できます。列名指定（例: rs.getString("name")）は列順が変わっても動き、コードが読みやすいため推奨です。列番号指定（例: rs.getString(2)）は列の順番が変わるとバグになりやすいため避けましょう。
