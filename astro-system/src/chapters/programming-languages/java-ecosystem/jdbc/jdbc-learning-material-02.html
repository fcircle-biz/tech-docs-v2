ホーム JDBC 第2章
第2章 所要時間: 約60分

開発環境の準備と初めての接続

JDBC を使ってデータベースに接続するには、JDK・JDBCドライバ・データベース本体の3つを揃える必要があります。この章では学習に最適な H2 データベースを使い、セットアップから「接続成功」を確認するプログラムの作成まで、ひとつひとつ丁寧に進めます。

この章で学ぶこと

  • JDBC プログラムに必要な3つの要素（JDK・JDBCドライバ・データベース）を理解する
  • H2 データベースが学習に最適な理由を説明できる
  • JDBCドライバをプロジェクトに追加する方法（Maven ／ jar 直接）を知る
  • 接続 URL の構造と H2 の書き方（ファイル型・インメモリ型）を読み解ける
  • DriverManager.getConnection() と try-with-resources で初めての接続プログラムを書ける
  • 第3章以降で使う users テーブルを作成し、初期データを投入できる（共通セットアップ）

2.1 開発環境の全体像

JDBC でデータベースに接続するプログラムを動かすには、最低限 3つの要素が必要です。それぞれの役割をざっくり押さえてから、実際のセットアップに進みましょう。

JDK

Java でプログラムを書いてコンパイル・実行するためのツールセット。javac と java コマンドを提供します。

JDBCドライバ（jar）

JDBC API と実際のデータベースをつなぐ「翻訳係」。データベースごとに専用のドライバ jar ファイルが存在します。

データベース本体

データを実際に保管・管理するソフトウェア（DBMS）。本ガイドでは学習用に H2 を使います。

第1章で学んだ「アプリ → JDBC → ドライバ → DB」の流れを、今回使う実際のソフトウェアにあてはめると次のようになります。

flowchart LR A["あなたのJavaコード"] --> B["JDBC API
(java.sql)"] B --> C["H2 JDBCドライバ
(h2-*.jar)"] C --> D[("H2データベース
(ファイル or インメモリ)")] D -.結果.-> C C -.結果.-> B B -.結果.-> A

H2 の場合、JDBCドライバとデータベース本体が同じ jar ファイルに入っています。そのため、H2 の jar 1つをプロジェクトに追加するだけで「ドライバ」と「データベース」の両方が揃うのです。これが H2 を学習用に使う大きな理由のひとつです。

2.2 学習用データベース H2 の準備

本ガイドでは、学習用データベースとして H2 Database を使います。H2 は Java で作られた軽量なリレーショナルデータベースで、初学者に非常に向いています。

特徴 内容
超軽量 jar 1つ（数MB）を追加するだけ。インストール不要で始められます。
インメモリ対応 ファイルを作らずメモリ上だけでDBを動かせます。プログラム終了と同時にデータが消えるので、何度でもゼロから試せます。
ドライバ込み JDBCドライバとDB本体が同じ jar に入っているため、jar 1つを追加するだけで環境が完成します。
標準 SQL 対応 MySQL や PostgreSQL で使う標準 SQL がほぼそのまま動くので、実務に直結した SQL を学べます。

重要ポイント: インメモリモードとファイルモード

H2 には2つのモードがあります。インメモリモードは「プログラム起動中だけデータが存在し、終了したら消える」モードで、1つのプログラム内で完結する実験に最適です。ファイルモードはディスクにファイルを作ってデータを保持します。本ガイドでは、章をまたいで同じ users テーブルを使い続けるため、第3章以降はファイルモード（jdbc:h2:~/testdb）を基本とします（理由は 2.6 で説明します）。

2.3 JDBCドライバをプロジェクトに追加する

JDBC でデータベースに接続するには、ドライバ（jar ファイル）をクラスパスに追加する必要があります。クラスパスとは「Java プログラムが参照できるライブラリの場所」のことです。追加方法は主に2つあります。

方法A: Maven を使う（推奨）

Maven を使ったプロジェクト（pom.xml がある）では、<dependencies> に H2 の依存を追加するだけです。Maven が自動的にインターネットから jar をダウンロードしてクラスパスに追加してくれます。

pom.xml（抜粋） 
<dependencies>
    <!-- H2 Database: JDBCドライバとDB本体が1つに入っている -->
    <dependency>
        <groupId>com.h2database</groupId>
        <artifactId>h2</artifactId>
        <version>2.2.224</version>
    </dependency>
</dependencies>

H2 のバージョンについて

本ガイドでは動作確認済みのバージョンとして 2.2.224 を使用します。学習中はバージョンを固定しておくと再現性が高く安心です。Maven Central にはより新しい版（例: 2.4.240）も公開されているので、最新版を使いたい場合は <version> の値を置き換えてください（jar を直接使う場合はファイル名のバージョン部分も合わせて読み替えます）。

方法B: jar を直接クラスパスに追加する

Maven を使わない場合は、H2 公式サイトから jar をダウンロードし、コンパイル・実行時にクラスパスへ指定します。

コンパイル・実行コマンド（jar を直接指定する例） 
# コンパイル（-cp でクラスパスに h2 jar を追加）
javac -cp h2-2.2.224.jar HelloJdbc.java

# 実行（カレントディレクトリと h2 jar を両方クラスパスに指定）
# macOS / Linux はクラスパス区切りが ":"（コロン）
java -cp .:h2-2.2.224.jar HelloJdbc

# Windows はクラスパス区切りが ";"（セミコロン）
java -cp .;h2-2.2.224.jar HelloJdbc

OS でクラスパスの区切り文字が違う

-cp で複数の場所を指定するときの区切り文字は OS によって異なります。macOS / Linux は :（コロン）、Windows は ;（セミコロン）です。Windows で .:h2-2.2.224.jar のようにコロンで書くと正しく認識されないので注意しましょう。

どちらを使えばいい？

実務では Maven（または Gradle）を使うのが一般的です。本ガイドのサンプルコードはどちらでも動きますが、IDE（IntelliJ IDEA や Eclipse）を使っている場合は Maven プロジェクトを作成して pom.xml に追記するのがもっともスムーズです。

2.4 接続 URL の読み方

JDBC でデータベースに接続するとき、接続 URL（JDBC URL）でどのデータベースに、どのように接続するかを指定します。形式は次のとおりです。

jdbc : サブプロトコル : データベース指定

固定の接頭辞 DBの種類（h2・mysql など） DBの場所や名前

H2 では、接続方法によって URL の形が変わります。以下の2パターンをまず覚えましょう。

モード 接続 URL 特徴
ファイルモード jdbc:h2:~/test ホームディレクトリに test.mv.db を作成。データが残る。
インメモリモード jdbc:h2:mem:testdb メモリ上のみ動作。プログラム終了でデータ消去。繰り返し練習に最適。

接続に必要な情報は URL だけではありません。ユーザー名とパスワードも必要です。H2 の初期設定では以下の値を使います。

パラメータ H2 の既定値 説明
URL jdbc:h2:mem:testdb インメモリモードで使う場合の URL 例
ユーザー名 sa 「System Administrator」の略。H2 の既定管理者ユーザー
パスワード ""（空文字） H2 は既定でパスワードなし。空文字列を渡す

MySQL・PostgreSQL の URL との違い

MySQL の URL は jdbc:mysql://localhost:3306/mydb のように「ホスト名:ポート/DB名」の形になります。H2 はファイルパスやメモリを直接指定する点が異なりますが、「jdbc:サブプロトコル:...」という基本形は同じです。

2.5 初めての接続プログラム

いよいよ実際にデータベースに接続するプログラムを書きましょう。最初のゴールは「接続が成功したことを確認する」だけです。シンプルに、接続して「接続成功！」と表示したら自動的に接続を閉じるプログラムを作ります。

try-with-resources で確実にクローズする

データベース接続（Connection）は使い終わったら必ず閉じる必要があります。うっかり閉じ忘れると、接続が残り続けてデータベースに負荷がかかります。Java の try-with-resources 構文を使うと、try ブロックを抜けたとき（正常でもエラーでも）に自動でクローズしてくれるので安全です。

HelloJdbc.java 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class HelloJdbc {
    public static void main(String[] args) {
        // 接続 URL・ユーザー名・パスワード（H2 インメモリの既定値）
        String url  = "jdbc:h2:mem:testdb";
        String user = "sa";
        String pass = "";

        // try-with-resources: ブロックを抜けると Connection が自動クローズ
        try (Connection conn = DriverManager.getConnection(url, user, pass)) {
            System.out.println("接続成功！ DB: " + conn.getMetaData().getURL());
        } catch (SQLException e) {
            System.err.println("接続失敗: " + e.getMessage());
        }
    }
}

ポイントは DriverManager.getConnection(url, user, pass) の1行です。この呼び出しが成功すると Connection オブジェクトが返ってきます。接続が確立された証拠です。

動作確認: 実行結果

プログラムを実行すると、コンソールに次のように表示されます。

コンソール出力 
接続成功！ DB: jdbc:h2:mem:testdb

この1行が表示されれば、JDBCドライバの追加・接続 URL・ユーザー名・パスワードがすべて正しく設定できています。

DriverManager と DataSource の違いについて

今回使った DriverManager は、シンプルな接続の取得に便利なクラスです。ただし、実務では接続を使いまわす「コネクションプール」に対応した DataSource インターフェースが推奨されます。DriverManager と DataSource の違いと使い分けは、第8章で詳しく扱います。

2.6 共通セットアップ — users テーブルの作成と初期データ投入

第3章以降では、users テーブル（id・name・email）にデータが入っている前提で学習を進めます。ここでテーブルを作成し、初期データを投入しておきましょう。これ以降の章はこのデータをそのまま使います。

注意: インメモリモードは章をまたぐと消える

インメモリモード（jdbc:h2:mem:testdb）のデータは、プログラムが終了すると消えます。さらに H2 では、最後の接続が閉じた時点でもインメモリ DB は破棄されます。そのため「セットアップ用プログラムでテーブルを作る → 別の検索プログラムを動かす」と、検索側では Table "USERS" not found になってしまいます。章をまたいで同じデータを使う本ガイドでは、ディスクに残るファイルモード（jdbc:h2:~/testdb）を基本にします。

まず、作成する users テーブルの定義と初期データ（SQL）です。

セットアップ SQL（users テーブル作成＋初期データ） 
CREATE TABLE IF NOT EXISTS users (
    id    INT AUTO_INCREMENT PRIMARY KEY,
    name  VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL
);

INSERT INTO users (name, email) VALUES
('佐藤太郎', 'taro@example.com'),
('鈴木花子', 'hanako@example.com'),
('田中次郎', 'jiro@example.com');

この SQL を JDBC から実行するセットアップ用プログラムが次です。接続先をファイルモード（jdbc:h2:~/testdb）にしているため、一度実行すればテーブルとデータがディスクに保存され、第3章以降でそのまま使えます。

SetupDatabase.java 
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.sql.Statement;

public class SetupDatabase {
    public static void main(String[] args) {
        // 章をまたいでデータを保持するためファイルモードを使う
        String url  = "jdbc:h2:~/testdb";
        String user = "sa";
        String pass = "";

        try (Connection conn = DriverManager.getConnection(url, user, pass);
             Statement stmt = conn.createStatement()) {

            // 1. テーブルを作成（すでにあれば何もしない）
            stmt.execute(
                "CREATE TABLE IF NOT EXISTS users (" +
                "  id    INT AUTO_INCREMENT PRIMARY KEY," +
                "  name  VARCHAR(100) NOT NULL," +
                "  email VARCHAR(255) NOT NULL)");

            // 2. 初期データを投入
            stmt.executeUpdate(
                "INSERT INTO users (name, email) VALUES " +
                "('佐藤太郎', 'taro@example.com')," +
                "('鈴木花子', 'hanako@example.com')," +
                "('田中次郎', 'jiro@example.com')");

            System.out.println("セットアップ完了: users テーブルを作成し、3件のデータを投入しました");
        } catch (SQLException e) {
            System.err.println("セットアップ失敗: " + e.getMessage());
        }
    }
}

2回目以降の実行に注意

上の INSERT は実行のたびに3件を追加します。2回実行するとデータが重複するので注意してください。最初からやり直したいときは、ホームディレクトリの testdb.mv.db ファイルを削除するか、INSERT の前に stmt.execute("DROP TABLE IF EXISTS users"); を追加してテーブルごと作り直しましょう。

どうしてもインメモリで試したい場合

インメモリモードを使うなら、テーブル作成・データ投入・検索を同じプログラムの中で続けて実行してください。加えて、複数回 getConnection() するなら jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1 のように DB_CLOSE_DELAY=-1 を付けると、JVM が動いている間は DB が保持されます。

やってみよう 2-1: 接続プログラムを実行して「接続成功」を確認する

2.5 で学んだ HelloJdbc.java を実際に作成して実行し、コンソールに「接続成功！」が表示されることを確認しましょう。

手順

  1. 1 Maven プロジェクト（または任意のフォルダ）を用意し、2.3 の方法で H2 の jar をクラスパスに追加する。
  2. 2 2.5 のコードをそのままコピーして HelloJdbc.java として保存する。
  3. 3 コンパイルして実行する。コンソールに 接続成功！ DB: jdbc:h2:mem:testdb と表示されれば完成。
  4. 4 （発展）URL を jdbc:h2:~/testdb に変えて実行し、ホームディレクトリに testdb.mv.db ファイルが作成されることを確認する。
  5. 5 2.6 の SetupDatabase.java を実行し、jdbc:h2:~/testdb に users テーブルと初期データ（3件）を作成しておく。第3章以降はこのデータを使う。

うまくいかない場合のチェックポイント

  • ClassNotFoundException が出た → H2 の jar がクラスパスに追加されていない
  • SQLException が出た → URL・ユーザー名・パスワードを確認する
  • Maven を使っている場合は mvn compile ＆ mvn exec:java で実行する

理解度確認クイズ

  1. Q1. JDBC の接続 URL の基本形式は何ですか？ H2 インメモリモードの URL を例として答えてください。

    回答を表示

    基本形式は jdbc:サブプロトコル:データベース指定 です。H2 インメモリモードの例は jdbc:h2:mem:testdb で、「jdbc」が固定の接頭辞、「h2」がサブプロトコル（DBの種類）、「mem:testdb」がデータベース指定（インメモリで testdb という名前）を意味します。

  2. Q2. DriverManager.getConnection() の3つの引数は何ですか？ H2 の既定値も合わせて答えてください。

    回答を表示

    3つの引数は ①接続 URL、②ユーザー名、③パスワード です。H2 の既定値は ①jdbc:h2:mem:testdb（インメモリの場合）、②sa、③""（空文字、パスワードなし）です。

  3. Q3. try-with-resources を使って Connection を扱う利点は何ですか？

    回答を表示

    try ブロックを抜けると（正常終了でも例外が発生しても）Connection が自動的にクローズされます。明示的に close() を呼ばなくても接続を確実に解放できるため、うっかり閉じ忘れてリソースが枯渇するバグを防げます。
