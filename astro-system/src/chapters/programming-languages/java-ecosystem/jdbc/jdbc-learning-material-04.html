第4章 所要時間: 約60分

データの登録・更新・削除（INSERT/UPDATE/DELETE文）

前章ではデータの「検索（SELECT）」を学びました。この章では、データベースに新しいデータを登録（INSERT）・既存データを変更（UPDATE）・不要なデータを削除（DELETE）する方法を習得します。これら「更新系」の SQL を JDBC で実行する際に使う executeUpdate() の使い方と、その戻り値（影響を受けた行数）の意味を理解しましょう。

この章で学ぶこと

  • SELECT と INSERT/UPDATE/DELETE のメソッドの違い（executeQuery vs executeUpdate）を説明できる
  • PreparedStatement を使って INSERT 文を安全に実行できる
  • WHERE 句を使って UPDATE・DELETE を正確に実行できる
  • executeUpdate() の戻り値（影響を受けた行数）を判定に活用できる

4.1 検索と更新系の違い — executeQuery と executeUpdate

JDBC では、SQL の種類によって使うメソッドが変わります。前章で学んだ SELECT 文は「データを取り出す」操作なので、結果（ResultSet）を返す executeQuery() を使いました。一方、INSERT・UPDATE・DELETE のような「データを変更する」操作は executeUpdate() を使い、返ってくるのは ResultSet ではなく「変更された行数（int）」 です。

SQL の種類 使うメソッド 戻り値 用途
SELECT executeQuery() ResultSet（検索結果） データを取り出す（読み取り専用）
INSERT executeUpdate() int（影響行数） 新しいデータを追加する
UPDATE executeUpdate() int（影響行数） 既存データを変更する
DELETE executeUpdate() int（影響行数） データを削除する

この違いを図で確認しましょう。

flowchart TD A["SQL を実行したい"] --> B{"SQL の種類は？"} B -->|"SELECT（検索）"| C["executeQuery()を使う"] B -->|"INSERT / UPDATE / DELETE"| D["executeUpdate()を使う"] C --> E["ResultSet で結果を受け取る"] D --> F["int で影響行数を受け取る"]

まとめて覚えるコツ

「Query（問い合わせ）＝ executeQuery ＝ ResultSet」「Update（更新）＝ executeUpdate ＝ int（行数）」と頭文字で覚えると混乱しません。SELECT は「取り出す」だけなのでデータを変更しないのに対し、INSERT/UPDATE/DELETE は「変更する」操作なのでデータが何件変わったか（行数）を返します。

4.2 INSERT（新規登録）

users テーブルに新しいユーザーを1件追加してみましょう。本章でも PreparedStatement（? プレースホルダを使う方法）を基本とします。なぜ PreparedStatement が安全なのかの詳細は第5章で学びますが、今の段階では「安全のために PreparedStatement を使う」と覚えておいてください。

流れは「SQL の ? に値をセット（bind）→ executeUpdate() で実行」です。

InsertExample.java 
// 1. ? プレースホルダで INSERT 文を準備
String sql = "INSERT INTO users (name, email) VALUES (?, ?)";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 2. ? に値をセット（1番目: name, 2番目: email）
pstmt.setString(1, "田中 次郎");   // 1番目の ? に文字列をセット
pstmt.setString(2, "jiro@example.com"); // 2番目の ? に文字列をセット

// 3. executeUpdate() で実行 → 影響を受けた行数が返る
int n = pstmt.executeUpdate();
System.out.println(n + "件登録しました"); // → "1件登録しました"

setString(インデックス, 値) の「インデックス」は ? の登場順（1始まり）です。数値を渡したい場合は setInt(index, value) を使います。以下に主要な setter の一覧を示します。

メソッド Java の型 SQL の型（例）
setString(n, value) String VARCHAR, TEXT
setInt(n, value) int INT, INTEGER
setDouble(n, value) double DOUBLE, DECIMAL
setBoolean(n, value) boolean BOOLEAN

4.3 UPDATE（既存データの更新）

UPDATE 文は「WHERE で対象を特定し、SET で値を変更する」のが基本です。JDBC では INSERT と同じく executeUpdate() を使い、変更された行数を受け取ります。

UpdateExample.java 
// id = 1 のユーザーのメールアドレスを更新する
String sql = "UPDATE users SET email = ? WHERE id = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 1番目 ? = 新しいメールアドレス
pstmt.setString(1, "jiro_new@example.com");
// 2番目 ? = 更新対象の id
pstmt.setInt(2, 1);

// 実行 → 更新された行数が返る
int n = pstmt.executeUpdate();
System.out.println(n + "件更新しました"); // → "1件更新しました"

? のインデックスは SQL に現れる順番

SET email = ? が1番目、WHERE id = ? が2番目、というように SQL 中の ? の登場順でインデックスが振られます。複数のカラムを更新したいときは SET col1 = ?, col2 = ? のように続け、それぞれに対応したインデックスで値をセットします。

4.4 DELETE（データの削除）

DELETE 文は「WHERE で対象を指定して削除する」のが基本です。使い方は INSERT・UPDATE と同じで executeUpdate() を呼び出します。

DeleteExample.java 
// id = 1 のユーザーを削除する
String sql = "DELETE FROM users WHERE id = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);

// 削除対象の id をセット
pstmt.setInt(1, 1);

// 実行 → 削除された行数が返る
int n = pstmt.executeUpdate();
System.out.println(n + "件削除しました"); // → "1件削除しました"

重大な注意: WHERE を付け忘れると全件削除されます！

DELETE FROM users のように WHERE を書かない DELETE 文を実行すると、テーブルのすべての行が削除されます。一度削除したデータは元に戻りません（トランザクションを使っていない場合）。DELETE 文を書く際は、必ず WHERE 句で対象を絞り込んでいることを確認してから実行してください。

危険な例（全件削除）:

DELETE FROM users; ← WHERE なし = 全件消える！

安全な例（指定した1件のみ削除）:

DELETE FROM users WHERE id = ?; ← WHERE あり = 安全

4.5 戻り値（影響行数）の活用

executeUpdate() が返す int は「その SQL によって変更（追加・更新・削除）された行数」です。この値を使えば、処理が本当に意図通りに実行されたかを確認できます。

影響行数の判定例 
int n = pstmt.executeUpdate();

if (n == 0) {
    // 対象レコードが見つからなかった（WHERE 条件に合うものがなかった）
    System.out.println("更新対象が見つかりませんでした");
} else {
    // n 件が変更された
    System.out.println(n + "件を正常に処理しました");
}

n > 0（正常）

n 件が正常に変更された。INSERT は通常 1 を返す。

n == 0（対象なし）

WHERE 条件に合うレコードが存在しなかった。変更は行われていない。

n > 1（複数件）

UPDATE/DELETE で WHERE 条件に複数件が一致し、まとめて変更された。意図した動作か確認を。

重要ポイント: 戻り値で処理結果を検証する

実務では executeUpdate() の戻り値を必ずチェックする習慣をつけましょう。「0 件だったらエラーメッセージを返す」「想定外に大量の行が変更されたら警告ログを出す」といった処理が、バグの早期発見につながります。

動作確認: 実行例

INSERT → UPDATE → DELETE の順に実行した場合、コンソールには以下のように出力されます。

実行結果（標準出力） 
1件登録しました（影響行数: 1）
1件更新しました（影響行数: 1）
1件削除しました（影響行数: 1）

それぞれ影響行数が 1 であれば、意図通り1件だけ変更されたことが確認できます。

やってみよう 4-1: 登録・検索・更新・削除を一連で試す

INSERT・SELECT・UPDATE・DELETE を順番に実行して、変更がデータベースに反映されることを手元で確認してみましょう。

手順

  1. 1 登録（INSERT）: users テーブルに自分の名前とメールアドレスで新規ユーザーを1件追加する。影響行数が 1 であることを確認する。
  2. 2 検索（SELECT）: 第3章で学んだ SELECT 文で全件または登録した名前を検索し、データが追加されていることを確認する。
  3. 3 更新（UPDATE）: 登録したユーザーの id を使って、メールアドレスを別の値に UPDATE する。再度 SELECT して変更されていることを確認する。
  4. 4 削除（DELETE）: 同じ id を使って DELETE する。SELECT で该当レコードが消えていることを確認する。WHERE を必ず付けること。
  5. 5 影響行数の確認: 存在しない id を WHERE に指定して UPDATE してみて、戻り値が 0 になることを確かめる。

理解度確認クイズ

  1. Q1. executeUpdate() の戻り値は何を表しますか？また、戻り値が 0 だった場合は何を意味しますか？

    回答を表示

    戻り値は「その SQL によって変更（追加・更新・削除）された行数（int）」を表します。戻り値が 0 の場合は、WHERE 条件に合うレコードが存在しなかったことを意味し、実際にはデータが何も変更されなかったことになります。

  2. Q2. SELECT 文を実行するメソッドと、INSERT/UPDATE/DELETE を実行するメソッドをそれぞれ答えてください。また、それぞれの戻り値の型も説明してください。

    回答を表示

    SELECT には executeQuery() を使い、戻り値は ResultSet（検索結果）です。INSERT・UPDATE・DELETE には executeUpdate() を使い、戻り値は int（影響を受けた行数）です。

  3. Q3. DELETE 文で WHERE 句を付け忘れるとどうなりますか？それを防ぐために実行前に確認すべきことは何ですか？

    回答を表示

    WHERE 句のない DELETE 文（DELETE FROM users）を実行すると、テーブルの全件が削除されます。削除したデータはトランザクションなしでは元に戻りません。防ぐためには、DELETE 文を実行する前に「WHERE 句が必ず記述されているか」「? プレースホルダに正しい値がセットされているか」を確認する習慣をつけることが大切です。
