はじめてのJSP - HTMLにJavaを埋め込む

この章では、JSP（JavaServer Pages）の基本を学びます。「HTML の中に Java を書く」という JSP の書き方、スクリプトレット・式・宣言などの基本構文、そして JSP がどのように動くかのライフサイクルを理解していきましょう。

この章で学ぶこと

  • JSP の概念と Servlet との関係（JSP は内部で Servlet に変換される）を理解する
  • スクリプトレット・式・宣言・コメントなど JSP の基本構文を理解する
  • if 文・for 文を使って動的な HTML ページを生成できる
  • JSP のライフサイクル（変換→コンパイル→実行）を理解する

4.1 JSPとは何か

第2・3章で Servlet を学びました。Servlet は「Java の中に HTML を書く」スタイルでしたが、HTML が増えてくると out.println("<p>..."); の羅列で読みにくくなります。 そこで登場するのが JSP（JavaServer Pages）です。JSP は「HTML の中に Java を書く」スタイルで、 画面の設計を HTML で行いながら動的な処理だけを Java で記述できます。

ServletとJSPの書き方の違い

同じ「Hello, World!」を表示するコードを Servlet と JSP で比べてみましょう。

Servlet の場合（Java の中に HTML）

HelloServlet.java 
// Javaの中でHTMLを文字列として出力
PrintWriter out = response.getWriter();
out.println("<html>");
out.println("<body>");
out.println("<h1>Hello, World!</h1>");
out.println("</body>");
out.println("</html>");

JSP の場合（HTML の中に Java）

hello.jsp 
<!-- HTMLを普通に書いて、動的部分だけJavaで -->
<html>
<body>
    <h1>Hello, World!</h1>
    <p>現在時刻: <%=new java.util.Date()%></p>
</body>
</html>

JSP は HTML がそのまま書けるので、画面のデザイン担当（フロントエンド担当）が読みやすく、 修正しやすいという利点があります。

JSPは内部でServletに変換される

JSP は便利な記法ですが、最終的にはブラウザから見れば普通の HTML が返ってくるだけです。 では JSP はどのように動いているのでしょうか？実は JSP は サーバー内部で自動的に Servlet に変換されています。 私たちが JSP を書くと、Tomcat が変換・コンパイルしてから実行しています。

flowchart LR A["hello.jsp
（私たちが書く）"] -->|"Tomcatが自動変換"| B["HelloJsp_jsp.java
（生成されたServlet）"] B -->|"コンパイル"| C["HelloJsp_jsp.class
（実行可能ファイル）"] C -->|"実行"| D["HTML
（ブラウザへ）"]

生成されたServletはどこにある？

Tomcat が変換した Java ファイルと .class ファイルは work/Catalina/localhost/[アプリ名]/org/apache/jsp/ に生成されます。中身を見ると JSP の記述がどのような Servlet コードになっているか確認できます。 名前空間は jakarta.servlet.*（Tomcat 10.1 以降）が使われています。

JSPを使う利点

HTMLが書きやすい

HTMLをそのまま書けるため、画面のレイアウト作成・修正が直感的です。

役割分担しやすい

HTML/CSSの担当者と Javaロジックの担当者で作業を分けやすくなります。

MVC の「V」になる

後に学ぶ MVC パターンで、View（表示）担当として活躍します。

4.2 JSPの基本構文

JSP で Java を埋め込む記法は4種類あります。それぞれ用途が異なるので、 「何を書くためのタグか」をしっかり理解しましょう。

種類 記法 用途
スクリプトレット <% ... %> Java のコードを実行する（if/for なども）
式 <%= ... %> 式の結果（値）を HTML に出力する
宣言 <%! ... %> フィールド変数・メソッドを定義する
JSPコメント <%-- ... --%> JSPレベルのコメント（HTMLには出力されない）

① スクリプトレット <% ... %>

最も基本となる記法です。<% %>（エスケープ不要で実際はJSPの記号）の中に Java のコードを書きます。変数の宣言・処理ロジック・制御構文などを記述できます。 ただし、式の結果を出力するには使えない（それは「式」タグの役割）点に注意してください。

scriptlet.jsp — スクリプトレットの基本 
<!-- スクリプトレット: <% %> の中にJavaコードを書く -->
<%
    // 変数の宣言と処理はここに書く
    String name = "田中さん";
    int score = 85;
%>
<p>名前: <%=name%></p>
<p>点数: <%=score%>点</p>

② 式 <%= ... %>

<%= %> は、式の結果（値）をそのまま HTML に埋め込みます。 末尾にセミコロンは不要です（書くとエラーになります）。 文字列だけでなく、数値・日付など toString() に変換できる値ならすべて出力できます。

expression.jsp — 式で値を出力する 
<!-- 式タグ: <%= 式 %> で値を出力（セミコロン不要） -->
<p>現在時刻: <%=new java.util.Date()%></p>
<p>合計: <%=10 + 20%></p>
<p>大文字: <%="hello".toUpperCase()%></p>

③ 宣言 <%! ... %>

<%! %> は、Servlet クラスのメンバー（フィールド変数やメソッド）を定義します。 スクリプトレットで定義した変数は _jspService() メソッド内のローカル変数になりますが、 宣言で定義した変数はクラスのフィールドになります。 メソッドを定義して JSP 内で呼び出すこともできます。

declaration.jsp — 宣言でメソッドを定義 
<!-- 宣言: <%! %> でクラスメンバーを定義 -->
<%!
    // メソッドをここで定義できる
    String greet(String name) {
        return "こんにちは、" + name + "さん！";
    }
%>
<!-- 定義したメソッドを式タグで呼び出す -->
<p><%=greet("山田")%></p>

④ JSPコメント <%-- ... --%>

JSP コメントは HTML コメント <!-- --> と混同しがちですが、大きな違いがあります。 HTML コメントはブラウザの「ソースを表示」で見られますが、 JSP コメントはサーバー側で除去され、クライアントには一切送られません。 セキュリティ上重要な情報のコメントには JSP コメントを使いましょう。

comment.jsp — コメントの違いを理解する 
<!-- HTMLコメント: ブラウザのソース表示で見える -->

<%-- JSPコメント: サーバーで除去。クライアントには届かない --%>

<!--
    重要: パスワードやDBの情報はここに書かない！
    JSPコメントを使うこと: <%-- --%>
-->

4つの構文の使い分けポイント

  • <% %>（スクリプトレット）→ 処理・ロジック・制御構文を書く
  • <%= %>（式）→ 値を HTML に出力する（セミコロン不要）
  • <%! %>（宣言）→ クラスのフィールドやメソッドを定義する
  • <%-- --%>（コメント）→ クライアントに見せたくないコメントを書く

4.3 JSPでの制御構文

JSP の醍醐味は「条件によって表示を変える」「繰り返しで一覧を表示する」などの動的な HTML 生成です。 スクリプトレットと HTML を組み合わせて制御構文を書くパターンを学びましょう。

if 文を使った条件分岐

スクリプトレットの中の { } で挟まれた範囲に HTML を「挿入」するイメージです。 開き括弧 { を1つ目のスクリプトレットに、 HTML を間に挟んで、閉じ括弧 } を2つ目のスクリプトレットに書きます。

condition.jsp — if 文で条件分岐 
<%
    int score = 75; // スコアの例
%>
<!-- if文: スクリプトレットで { } を開いてHTMLを挟む -->
<%if (score >= 80) {%>
    <p class="pass">合格です！</p>
<%} else {%>
    <p class="fail">もう少しです。</p>
<%}%>

for 文を使った繰り返し

for 文も同じパターンです。繰り返すたびに HTML が出力されるので、 テーブルの行や一覧リストを動的に生成できます。

loop.jsp — for 文でリスト生成 
<ul>
<!-- for文: ループのたびにHTMLが出力される -->
<%for (int i = 1; i <= 5; i++) {%>
    <li>アイテム <%=i%> 番目</li>
<%}%>
</ul>

可読性を保つコツ

JSP のスクリプトレットが増えると、Java と HTML が入り交じって読みにくくなります。 以下を意識して書くと、コードの品質が保てます。

コツ 理由
ロジックを JSP に増やしすぎない 複雑な計算や DB アクセスは Servlet や JavaBean に分離する
1つのスクリプトレットを短く保つ 長い Java コードは JSP の可読性を著しく下げる
EL/JSTL を使う（第10章） JSP の専用タグと式言語でスクリプトレットを大幅に減らせる

第10章への布石

<%for(...)%> のようなスクリプトレットは、 第10章で学ぶ JSTL の <c:forEach> タグに置き換えると、 Java を書かずに繰り返しが書けて可読性が大きく向上します。 今は「スクリプトレットで書ける」を理解しておけば十分です。

4.4 JSPのライフサイクル

JSP がリクエストを受け取って HTML を返すまでには、いくつかのフェーズがあります。 特に初回アクセス時の遅延を理解しておくと、開発中の「なぜ最初だけ遅い？」という疑問が解消されます。

JSPのライフサイクル

flowchart TD A["ブラウザがJSPにアクセス"] --> B{"初めてのアクセス？"} B -->|"Yes（初回）"| C["変換フェーズ
JSP → Servlet Java ファイル"] C --> D["コンパイルフェーズ
Java ファイル → .class ファイル"] D --> E["初期化フェーズ
jspInit（）が呼ばれる"] B -->|"No（2回目以降）"| E E --> F["実行フェーズ
_jspService（）がリクエストを処理"] F --> G["HTMLレスポンスを返す"] G --> H{"アプリ終了？"} H -->|"Yes"| I["破棄フェーズ
jspDestroy（）が呼ばれる"] H -->|"No"| F

各フェーズの詳細

1

変換フェーズ（Translation）

Tomcat が .jsp ファイルを読み込み、 対応する Java ファイル（Servlet クラス）に変換します。 hello.jsp は hello_jsp.java のような名前になります。 JSP の内容が変更されると、次のアクセス時に再変換されます。

2

コンパイルフェーズ（Compilation）

変換された Java ファイルを javac でコンパイルし、 .class ファイルを生成します。 Java のコードにエラーがあれば、このフェーズで コンパイルエラーが発生します。

3

実行フェーズ（Execution）

コンパイルされた Servlet がロードされ、 _jspService() メソッドがリクエストごとに呼ばれます。 2回目以降のアクセスはこのフェーズから始まるので、初回より大幅に速くなります。

初回アクセスの遅延と対策

JSP に初めてアクセスすると、変換とコンパイルが走るためわずかに遅くなることがあります。 本番環境では「事前コンパイル（Precompilation）」という手法で、 デプロイ時にあらかじめ変換・コンパイルを済ませておくことができます。 開発中は気にしなくて大丈夫です。

ライフサイクルを体感する方法

  • 初回アクセスと2回目アクセスで速度を比べてみましょう
  • Tomcat の work/ ディレクトリを開いて生成された Java ファイルを見てみましょう

4.5 つまずきやすいポイント

JSP を初めて使うとき、よく遭遇するトラブルをまとめました。困ったらここに戻ってきてください。

JSPが表示されない（404エラー）

  • 拡張子の確認: ファイル名が .jsp になっているか確認する（.html や .java になっていないか）。
  • 配置場所の確認: WEB-INF/ の中に置くと直接アクセスできません。JSP は webapps/アプリ名/ の直下または任意のサブフォルダに置きましょう（WEB-INF の外）。

Javaコードがそのまま画面に表示される

  • タグの記述ミス: <% や %> が正しく書かれているか確認する。特に <% と書いてもスクリプトレットになるが、IDE によっては JSP とみなさないことがある。
  • 式タグの = 付け忘れ: 値を出力したいのに <% name %> と書くとエラーになります。<%= name %> と = をつけましょう。

コンパイルエラーが起きる

  • エラーメッセージの読み方: ブラウザに表示されるエラーに「行番号」が書かれています。その行の前後をよく確認しましょう。
  • セミコロン忘れ: スクリプトレット内の Java 文には行末のセミコロン ; が必要です。式タグ <%= %> の中にはセミコロンを書かないこと。

やってみよう 4-2: 1から100を表示（偶数は赤字）

for 文と if 文を組み合わせて、偶数だけ赤く表示するページを作ってみましょう。

  1. 1
    numbers.jsp 
    <%@ page contentType="text/html; charset=UTF-8" %>
<html><body>
<%for (int i = 1; i <= 100; i++) {%>
    <%if (i % 2 == 0) {%>
        <span style="color:red"><%=i%></span>
    <%} else {%>
        <span><%=i%></span>
    <%}%>
<%}%>
</body></html>
  2. 2 ブラウザで表示して偶数が赤くなっていることを確認しましょう。

二重ループを使って九九の表を生成してみましょう。

  1. 1
    multiplication.jsp

<html><body>

理解度確認クイズ

  1. Q1. JSP と Servlet の最大の違いは何ですか？また、JSP は内部でどのように処理されますか？

    回答を表示

    Servlet は「Java の中に HTML を書く」スタイルで、JSP は「HTML の中に Java を書く」スタイルです。JSP は Tomcat によって内部で Servlet（Java ファイル）に自動変換され、さらにコンパイルされて実行されます。最終的には Servlet として動いています。

  2. Q2. <% %>（スクリプトレット）と <%= %>（式）の使い分けを説明してください。

    回答を表示

    スクリプトレット <% %> は Java のコードを「実行」するためのタグで、変数の定義や制御構文（if/for）などを書きます。式 <%= %> は式の結果（値）を HTML に「出力」するためのタグで、末尾にセミコロンは書きません。

  3. Q3. JSP の初回アクセスが2回目以降より遅い理由を説明してください。

    回答を表示

    初回アクセス時は JSP ファイルを Servlet（Java ファイル）に変換する「変換フェーズ」と、それをコンパイルして .class ファイルを生成する「コンパイルフェーズ」が走ります。2回目以降はすでにコンパイル済みの .class ファイルがあるため、これらのフェーズをスキップして実行フェーズに進めます。
