JavaBeansとスコープ - データの構造化

この章では、データを整理する仕組みである JavaBeans（データ専用クラス）の作り方と、JSPアクションタグを使った操作を学びます。さらに、データの「生存範囲」を管理する 4つのスコープ（page・request・session・application）を理解し、適切な場所にデータを格納する考え方を身につけます。

この章で学ぶこと

  • JavaBeans規約（引数なしコンストラクタ・privateフィールド・getter/setter）に従ったBeanクラスを作成できる
  • JSPの <jsp:useBean>・<jsp:setProperty>・<jsp:getProperty> アクションタグを使ってBeanを操作できる
  • page・request・session・applicationの4つのスコープの違いと生存期間を理解し使い分けられる
  • 要件に応じた適切なスコープを選択し、データを格納できる

9.1 JavaBeansとは

Webアプリでは、ユーザー情報・商品情報・注文情報など、関連するデータをひとまとめに扱いたい場面が多くあります。たとえば「名前・年齢・メールアドレス」を別々の変数で扱うと管理が大変です。そこで登場するのが JavaBeans です。JavaBeansは「データをまとめて持つための専用クラス」と考えてください。

JavaBeans規約の3つのルール

JavaBeansは単なる「特別なルールに従ったJavaクラス」です。難しい仕組みは何もありません。次の3つの規約を守るだけです。

引数なしコンストラクタ

引数を受け取らないコンストラクタを必ず用意する（何も書かなければ自動で作られるが、明示すると安全）。

privateフィールド

データを格納するフィールドは private で宣言し、外部から直接触れないようにする。

getter / setter

各フィールドにアクセスするための getXxx() / setXxx() メソッドを用意する。

BeanクラスとPOJOの関係

JavaBeansは POJO（Plain Old Java Object） の一種です。POJOは「特定のフレームワークや基底クラスに依存しない、普通のJavaクラス」のことを指します。JavaBeansはPOJOにgetter/setter規約を加えたものと理解しておくと整理しやすいです。

Beanクラスの配置場所

BeanクラスはServletと同様に、コンパイルした .class ファイルを WEB-INF/classes/ の下にパッケージ構造のまま配置します。パッケージ名を付けることが強く推奨されます。

Beanクラスの配置例 

WEB-INF/
└── classes/
    └── com/
        └── example/
            └── bean/
                └── UserBean.class   ← コンパイル済みBeanクラス

UserBeanクラスの例

実際のBeanクラスを見てみましょう。ユーザーの「名前」と「年齢」を持つシンプルな例です。

UserBean.java — JavaBeans規約に従ったクラス 
package com.example.bean;

public class UserBean {
    // ① privateフィールドでデータを保持
    private String name;
    private int age;

    // ② 引数なしコンストラクタ（必須）
    public UserBean() {}

    // ③ getter / setter（プロパティへのアクセス手段）
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

「プロパティ」という概念

JavaBeansの世界では、getter/setterのペアで定義される論理的な値を プロパティ と呼びます。たとえば getName()/setName() があると、「nameプロパティ」が定義されている、と言います。IDEの自動生成機能（Eclipse: Source → Generate Getters and Setters）を使うと楽に作れます。

9.2 JSPでのJavaBeans使用

JSPにはBeanをかんたんに扱うための アクションタグ（<jsp:...>）が用意されています。スクリプトレット（<% %>）でJavaコードをゴリゴリ書かなくても、宣言的にBeanを操作できます。

jsp:useBeanでBeanを取得・生成する

<jsp:useBean> は「指定したスコープにBeanがあれば取得、なければ新規作成」する宣言です。主要な属性は次のとおりです。

属性 意味 例
id JSP内でBeanを参照するときの変数名 id="user"
class 完全修飾クラス名（FQCN） class="com.example.bean.UserBean"
scope 保存するスコープ（省略時はpage） scope="request"

3つのアクションタグの使い方

3つのアクションタグ（useBean・setProperty・getProperty）を組み合わせてBeanを操作します。

userShow.jsp — アクションタグの基本的な使い方 
<%@ page contentType="text/html; charset=UTF-8" %>

<!-- ① Beanを取得または生成（requestスコープで管理） -->
<jsp:useBean id="user" class="com.example.bean.UserBean" scope="request" />

<!-- ② プロパティを個別に設定 -->
<jsp:setProperty name="user" property="name" value="山田太郎" />
<jsp:setProperty name="user" property="age"  value="25" />

<!-- ③ プロパティを取得して表示 -->
名前: <jsp:getProperty name="user" property="name" /><br>
年齢: <jsp:getProperty name="user" property="age"  />歳

property="*" によるフォームパラメータの自動設定

property="*" を指定すると、フォームのパラメータ名とBeanのプロパティ名が一致した場合に自動でセットされます。入力フォームとBeanを結びつける定番のテクニックです。

result.jsp — property="*" でフォーム入力を一括設定 
<%@ page contentType="text/html; charset=UTF-8" %>
<jsp:useBean id="user" class="com.example.bean.UserBean" scope="request" />

<!-- フォームパラメータ名とプロパティ名が同じなら自動でまとめてセット -->
<jsp:setProperty name="user" property="*" />

名前: <jsp:getProperty name="user" property="name" /><br>
年齢: <jsp:getProperty name="user" property="age"  />歳

property="*" の動作条件

①フォームの入力名（name="xxx"）と ②Beanのプロパティ名（setterの setXxx から派生）が完全に一致するとき自動セットされます。型変換（文字列→int）も自動で行われます。一致しないプロパティはスキップされます。

9.3 4つのスコープ

スコープとは「データが生き続ける範囲（期間と共有範囲）」のことです。JSP/Servletには4種類のスコープがあります。どのスコープにデータを入れるかによって、「いつまで」「誰と」共有されるかが変わります。

4つのスコープの範囲イメージ

flowchart TB subgraph APP["applicationスコープ（アプリ全体・全ユーザー共有）"] subgraph SES["sessionスコープ（ユーザーAのセッション期間中）"] subgraph REQ["requestスコープ（1回のリクエスト処理中）"] PAGE["pageスコープ
（現在のページ内のみ）"] end end SES2["ユーザーBの
sessionスコープ"] end

スコープ一覧

スコープ名 生存期間 共有範囲 主な用途
page 現在のJSPページの処理が終わるまで 同じJSPページ内のみ ページ内の一時的な計算結果
request 1回のリクエスト処理（forward先まで）が終わるまで 同じリクエストを処理するServlet/JSP全体 ServletからJSPへの結果渡し（MVC）
session セッションが維持される期間（タイムアウトまたはログアウトまで） 同じユーザーの複数リクエストにわたる ログイン情報・カート情報
application アプリケーションが起動している間（サーバー停止まで） 全ユーザー・全リクエスト共有 アクセスカウンター・共通設定値

各スコープに対応する暗黙オブジェクト

JSPではスコープごとに対応する暗黙オブジェクトがあります。データは setAttribute で入れ、getAttribute で取り出します。

各スコープへのデータ格納と取得 
// pageスコープ（同じJSPページ内だけ有効）
pageContext.setAttribute("msg", "pageスコープのデータ");

// requestスコープ（forward先のJSPとも共有）
request.setAttribute("msg", "requestスコープのデータ");

// sessionスコープ（ログイン中は保持し続ける）
session.setAttribute("loginUser", userBean);

// applicationスコープ（全ユーザー共有）
application.setAttribute("siteTitle", "MySite");

9.4 スコープの使い分け

スコープの選択ミスはバグの温床になります。「広いスコープ = 何でも入れていい」ではありません。広いスコープほどデータが長く残り続け、メモリを消費し続ける点に注意してください。「必要な期間だけ、必要な範囲で」が原則です。

スコープ選択の指針

request

よく使う基本スコープ

MVCパターンでServlet→JSPへデータを渡すときはこれ。1回の処理が終わったら消えるので安全。forward 先のJSPまで届く。

session

ユーザー固有の状態保持

ログイン情報・カートの中身など「ユーザーをまたいで保持したい」データに。ただしセキュリティに注意し、不要になったら削除する。

application

全ユーザー共有の定数的データ

サイト設定・アクセスカウンターなど「全員が同じ値を見る」データに。書き込み競合に注意が必要（同期処理）。

page

JSPページ内の一時変数

同一JSPファイル内だけで使う一時データに。forward 先に届かないため、実際に使う場面は少ない。

pageContextでスコープを明示指定する

pageContext の setAttribute は第3引数にスコープ定数を渡すことで、どのスコープに入れるかを明示できます。

pageContextでスコープを明示して操作する 
// 第3引数でスコープを定数で指定する
pageContext.setAttribute("key", "value", PageContext.PAGE_SCOPE);
pageContext.setAttribute("key", "value", PageContext.REQUEST_SCOPE);
pageContext.setAttribute("key", "value", PageContext.SESSION_SCOPE);
pageContext.setAttribute("key", "value", PageContext.APPLICATION_SCOPE);

// スコープを指定して取り出す
Object val = pageContext.getAttribute("key", PageContext.REQUEST_SCOPE);

メモリとパフォーマンスへの影響

sessionやapplicationスコープにBeanなどの大きなオブジェクトを格納すると、長期間メモリに残ります。sessionに不要なデータを詰め込みすぎるとサーバーのメモリを圧迫し、パフォーマンス低下につながります。ログアウト時に session.invalidate() を呼ぶなど、適切なタイミングで解放する習慣をつけましょう。

9.5 つまずきやすいポイント

JavaBeansとスコープを使い始めたときによくあるトラブルと対処法をまとめました。

Beanが見つからない（ClassNotFoundException）

主な原因は2つです。

  • 配置場所の間違い: .class ファイルが WEB-INF/classes/ の正しいパッケージ階層に置かれているか確認。
  • class属性のFQCN間違い: class="com.example.bean.UserBean" のようにパッケージ名を含む完全修飾クラス名で指定すること。

プロパティが設定されない（property="*" が効かない）

setterの命名規則が守られていないと自動設定されません。フィールド名が name なら setter は setName（先頭大文字）と定義されている必要があります。また、フォームの name属性とBeanのプロパティ名が完全に一致しているか確認してください。

スコープの選択ミス

「一時的な処理結果をsessionに入れた → 次のアクセスでも古いデータが残る」というミスはよくあります。一時データはrequest、ユーザー固有の状態はsessionという判断基準を意識してください。また、requestスコープのデータは redirect（別リクエスト）ではなく forward（同一リクエスト）先にしか届かない点も注意です。

やってみよう 9-1: UserBeanを作成してフォームと連携する

JavaBeansとJSPアクションタグを使って、フォーム入力をBeanに自動格納し画面に表示する一連の流れを体験します。

手順

  1. 1 com.example.bean.UserBean クラスを作成する。フィールドは name（String）と age（int）の2つ。getter/setterも定義する。
  2. 2 入力フォーム（form.html）を作成する。name="name" と name="age" の入力欄を持ち、result.jsp へPOSTする。
  3. 3 result.jsp で <jsp:useBean> + property="*" を使ってフォームの値を自動でBeanに格納し、<jsp:getProperty> で画面に表示する。

やってみよう 9-2: 各スコープの動作を比較する

同じキー名（例: "counter"）のデータをpage・request・session・applicationの各スコープに入れ、ページ遷移やブラウザ再読み込みでそれぞれの値がどう変化するか観察してみましょう。

  1. 1 scope-test.jsp を作成し、4つのスコープそれぞれに同じキーで異なる値（例: "page-data"・"request-data" など）をsetAttributeで格納・表示する。
  2. 2 ページを更新（F5）して、どのスコープの値が消えるか・残るかを確認する。
  3. 3 新しいブラウザタブを開いて同じURLにアクセスし、sessionとapplicationの違い（ユーザー固有か全ユーザー共有か）を確認する。

理解度確認クイズ

  1. Q1. JavaBeans規約の3つのルールを答えてください。

    回答を表示

    ① 引数なしのコンストラクタを持つ、② フィールドを private で宣言する、③ 各フィールドに対応する getter（getXxx）/ setter（setXxx） を用意する、の3つです。この規約を守ることで、JSPアクションタグやフレームワークがBeanを自動的に操作できるようになります。

  2. Q2. <jsp:setProperty name="user" property="*" /> はどのような動作をしますか？

    回答を表示

    フォームのパラメータ名（input要素のname属性）とBeanのプロパティ名（setterの名前から派生）が一致するものを自動的に検索し、まとめてsetterを呼び出してセットします。型変換（文字列→int等）も自動で行われます。一致しないプロパティはスキップされます。

  3. Q3. ログイン中のユーザー情報（ログインID・権限など）を保持するにはどのスコープが適切ですか？その理由も答えてください。

    回答を表示

    sessionスコープが適切です。理由は、ログイン情報はユーザーが複数のページを渡り歩く（複数リクエストをまたぐ）間も保持し続ける必要があるからです。requestスコープは1回のリクエストで消えてしまい、applicationスコープでは全ユーザーで共有されてしまうため不適切です。
