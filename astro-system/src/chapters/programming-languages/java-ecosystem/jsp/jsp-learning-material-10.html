第10章 所要時間: 約60分

EL式とJSTL - モダンなJSP開発

この章では、JSP をよりシンプルかつ安全に書くための2つの強力な道具、EL式（Expression Language）と JSTL（Jakarta Standard Tag Library）を学びます。スクリプトレット（Java コードの直書き）を排除し、保守性の高い「モダンなJSP」の書き方を習得しましょう。

この章で学ぶこと

  • EL式の構文（${...}）を理解し、スコープ変数・Beanプロパティ・各種演算子を使いこなせる
  • EL暗黙オブジェクト（param・cookie・各スコープ等）を活用できる
  • JSTLコアタグ（<c:out>・<c:forEach>・<c:if>等）を使いこなせる
  • スクリプトレットを使わないJSP開発（保守性の高いページ作成）ができる

10.1 EL式（Expression Language）とは

第9章までで、Servlet から request.setAttribute("user", user) のようにスコープに値を設定し、JSP 側で <%= request.getAttribute("user") %>（スクリプトレット）で取り出していました。しかし、これは JSP にJavaコードが混入して読みにくいという問題があります。EL式を使えば、同じことを ${user} と書けます。

スクリプトレットとEL式の比較

まず、スクリプトレットとEL式を並べて比較してみましょう。EL式がいかに簡潔で読みやすいかが一目でわかります。

書き方 スクリプトレット（旧来） EL式（モダン）
変数表示 <%= user.getName() %> ${user.name}
スコープから取得 <%= (String)request.getAttribute("msg") %> ${msg}
リクエストパラメータ <%= request.getParameter("name") %> ${param.name}

EL式の基本構文とスコープ探索

EL式は ${式} の形で書きます。単純な変数名を書いた場合（例: ${user}）、EL は page → request → session → application の順に自動で探してくれます。最初に見つかったスコープの値が使われます。

flowchart LR A["${user}と書いた"] --> B["pageスコープ
を探す"] B -->|"見つかった"| E["その値を表示"] B -->|"なし"| C["requestスコープ
を探す"] C -->|"見つかった"| E C -->|"なし"| D["sessionスコープ
を探す"] D -->|"見つかった"| E D -->|"なし"| F["applicationスコープ
を探す"] F -->|"見つかった"| E F -->|"なし"| G["空文字を表示
（エラーなし）"]

Beanプロパティへのアクセス

EL式ではドット演算子（.）でBeanのプロパティにアクセスできます。${user.name} と書くと、内部的に user.getName() が呼び出されます。ブラケット演算子（[]）でも同様にアクセスできます（変数キーや動的キーに便利）。

sample.jsp — EL式でプロパティ・配列・マップにアクセス 
<!-- ドット演算子: Beanの getName() を呼ぶ -->
<p>名前: ${user.name}</p>

<!-- ブラケット演算子: プロパティ名を文字列で指定 -->
<p>名前: ${user['name']}</p>

<!-- リスト/配列: インデックスでアクセス -->
<p>1番目: ${items[0]}</p>

<!-- Mapへのアクセス -->
<p>キーの値: ${map['key']}</p>

EL式の演算子

EL式では算術・比較・論理・特殊な演算子が使えます。比較演算子は記号（==・!=・< 等）とアルファベット（eq・ne・lt・gt 等）の両方が使えます。

種類 記号/キーワード 例
算術 + - * / % ${price * qty}
比較 == eq != ne < lt > gt <= le >= ge ${score gt 80}
論理 && and || or ! not ${a and b}
empty empty ${empty list}（nullまたは空のときtrue）
三項演算子 条件 ? 真 : 偽 ${score ge 60 ? '合格' : '不合格'}

empty演算子の便利な使い方

empty 演算子は、値が null・空文字・空リスト・空配列・空Map のいずれかであれば true を返します。Javaで if (list == null || list.isEmpty()) と書いていた条件を、${empty list} だけでシンプルに表現できます。

10.2 EL暗黙オブジェクト

EL式には、定義しなくてもそのまま使える「暗黙オブジェクト」が用意されています。これらを使うと、リクエストパラメータやCookieなどの値を ${param.name} のようにシンプルに取得できます。

暗黙オブジェクト 型 内容と使い方
param Map<String,String> リクエストパラメータ（単数値）。${param.name}
paramValues Map<String,String[]> 複数選択のパラメータ配列。${paramValues.hobby[0]}
header Map<String,String> HTTPヘッダー（単数値）。${header['User-Agent']}
cookie Map<String,Cookie> Cookie。${cookie.sessionId.value}
pageScope Map pageスコープの変数を明示指定。${pageScope.x}
requestScope Map requestスコープの変数を明示指定。${requestScope.user}
sessionScope Map sessionスコープの変数を明示指定。${sessionScope.loginUser}
applicationScope Map applicationスコープの変数を明示指定。${applicationScope.config}
pageContext PageContext ページコンテキスト。${pageContext.request.contextPath}

スコープを明示すべき場面

通常は ${user} と書けば自動探索してくれますが、同名変数が複数のスコープに存在する場合は ${sessionScope.user} のように明示するのが安全です。意図しないスコープの値が使われるバグを防げます。

10.3 JSTLコアタグ

JSTL（Jakarta Standard Tag Library）は、JSPでよく使う処理（条件分岐・繰り返し・変数操作・URL生成など）を独自タグとして提供するライブラリです。スクリプトレットで書いていた処理を、HTMLに近い読みやすいタグで表現できます。

JSTLの導入（JARとtaglibディレクティブ）

JSTLを使うには2つの準備が必要です。①JARファイルを WEB-INF/lib/ に配置し、②JSPのファイル先頭に taglib ディレクティブを書きます。

Jakarta EEでのURI変更（重要）

Tomcat 10以降（Jakarta EE 9+）では、taglibのURIが変わりました。旧来（javax時代）の http://java.sun.com/jsp/jstl/core は使えません。必ず新しいURIを使いましょう。

タグライブラリ 旧URI（javax）× 使わない 新URI（jakarta）○
コア http://java.sun.com/jsp/jstl/core jakarta.tags.core
フォーマット http://java.sun.com/jsp/jstl/fmt jakarta.tags.fmt
関数 http://java.sun.com/jsp/jstl/functions jakarta.tags.functions
sample.jsp — taglibディレクティブの宣言 
<!-- JSPファイルの先頭に書く -->
<%@ taglib prefix="c" uri="jakarta.tags.core" %>
<%@ taglib prefix="fmt" uri="jakarta.tags.fmt" %>
<%@ taglib prefix="fn" uri="jakarta.tags.functions" %>

<!-- 以降、<c:out>、<fmt:formatDate> 等が使える -->

c:out — 安全な出力（XSS対策）

<c:out> は値を出力するタグです。EL式 ${...} でも出力できますが、<c:out> は HTMLエスケープ（<→&lt; 等）を自動で行うため、ユーザー入力をそのまま表示する箇所では積極的に使うべきです。XSS（クロスサイトスクリプティング）攻撃を防げます。

sample.jsp — c:out の使い方 
<!-- HTMLエスケープして安全に出力（XSS対策） -->
<p><c:out value="${user.name}" /></p>

<!-- escapeXml="false" でエスケープを無効化（信頼できる値のみ） -->
<p><c:out value="${trustedHtml}" escapeXml="false" /></p>

<!-- defaultで値がnull/空のときの代替テキストを指定 -->
<p><c:out value="${user.name}" default="名前未設定" /></p>

c:set / c:remove — 変数操作

<c:set> でスコープに変数を設定し、<c:remove> で削除します。

sample.jsp — c:set / c:remove 
<!-- pageスコープに変数を設定（デフォルトはpage） -->
<c:set var="greeting" value="こんにちは" scope="page" />

<!-- EL式で表示: こんにちは -->
<p>${greeting}</p>

<!-- sessionスコープの変数を削除 -->
<c:remove var="loginUser" scope="session" />

c:if — 条件分岐

<c:if> で条件に応じた表示の出し分けができます。else節はなく、else が必要な場合は後述の <c:choose> を使います。

sample.jsp — c:if でログイン状態を切り替え 
<!-- sessionに loginUser がある場合だけ表示 -->
<c:if test="${not empty sessionScope.loginUser}">
    <p>ようこそ、${sessionScope.loginUser.name}さん！</p>
</c:if>

<!-- 数値比較 -->
<c:if test="${score ge 60}">
    <p class="pass">合格おめでとうございます！</p>
</c:if>

c:choose / c:when / c:otherwise — 多分岐

<c:choose>はJavaの if-else if-else に相当します。複数の条件を順番にチェックし、最初に true になった <c:when> の中身が表示されます。どれも該当しない場合は <c:otherwise> が使われます。

sample.jsp — c:choose で成績ランク表示 
<c:choose>
    <c:when test="${score ge 90}">
        <span class="rank-s">Sランク：優秀！</span>
    </c:when>
    <c:when test="${score ge 70}">
        <span class="rank-a">Aランク：よくできました</span>
    </c:when>
    <c:otherwise>
        <span class="rank-b">Bランク：もう少し頑張ろう</span>
    </c:otherwise>
</c:choose>

c:forEach — 繰り返し

<c:forEach>はJavaの拡張for文に相当し、リスト・配列・Mapを反復処理できます。varStatus 属性でインデックスや最初/最後の判定も取得できます。

sample.jsp — c:forEach でリストをテーブル表示 
<table>
    <tr><th>No.</th><th>名前</th><th>スコア</th></tr>
    <!-- itemsにリスト、varに各要素、varStatusにステータス -->
    <c:forEach items="${studentList}" var="s" varStatus="st">
    <tr>
        <td>${st.index + 1}</td> <!-- 0始まりなので+1 -->
        <td><c:out value="${s.name}" /></td>
        <td>${s.score}</td>
    </tr>
    </c:forEach>
</table>

varStatus で使えるプロパティ

プロパティ 内容 例（stの場合）
index 現在のインデックス（0始まり） ${st.index}
count 現在の繰り返し回数（1始まり） ${st.count}
first 最初の要素かどうか（boolean） ${st.first}
last 最後の要素かどうか（boolean） ${st.last}

c:forTokens — 文字列を分割して繰り返し

<c:forTokens> は、区切り文字で分割しながら繰り返し処理を行います。CSVや区切り付きの文字列を処理するときに便利です。

sample.jsp — c:forTokens でカンマ区切りを処理 
<!-- "Java,Python,Ruby" を , で分割して表示 -->
<c:forTokens items="Java,Python,Ruby" delims="," var="lang">
    <span class="badge">${lang}</span>
</c:forTokens>

<!-- EL式で取得した値も使える -->
<c:forTokens items="${csvData}" delims="," var="item">
    <li>${item}</li>
</c:forTokens>

10.4 JSTLその他のタグ

コアタグ以外にも、URLの生成・フォーマット・文字列関数など便利なタグが用意されています。

URL関連タグ（c:url / c:redirect / c:import）

<c:url>

コンテキストパスを自動付加してURLを生成。リンクやフォームのaction属性に使う。

<c:url value="/list" />

<c:redirect>

指定URLへリダイレクト（転送）する。処理後のページ遷移に使う。

<c:redirect url="/top" />

<c:import>

外部URL・他ファイルの内容を取り込む。JSP内でインクルードに使う。

<c:import url="/footer.jsp" />
sample.jsp — c:url でリンク生成 
<!-- コンテキストパスを自動で補完してリンクを生成 -->
<a href="<c:url value='/list' />">一覧へ</a>

<!-- パラメータを追加したURLを生成 -->
<c:url value="/detail" var="detailUrl">
    <c:param name="id" value="${item.id}" />
</c:url>
<a href="${detailUrl}">詳細を見る</a>

フォーマットタグ（fmt:formatDate / fmt:formatNumber）

日付や数値を見やすい形式に変換します。<%@ taglib prefix="fmt" uri="jakarta.tags.fmt" %> を先頭に追加してください。

sample.jsp — 日付・数値フォーマット 
<!-- 日付を "yyyy/MM/dd" 形式で表示 -->
<fmt:formatDate value="${order.createdAt}"
               pattern="yyyy/MM/dd HH:mm" />

<!-- 数値を3桁カンマ区切りで表示 -->
<fmt:formatNumber value="${product.price}"
                 pattern="#,##0" />円

関数タグ（fn:length / fn:contains 等）

文字列操作や長さの取得には関数タグを使います。<%@ taglib prefix="fn" uri="jakarta.tags.functions" %> を先頭に追加してください。

関数 説明 例
fn:length リストや文字列の長さを返す ${fn:length(list)}
fn:contains 文字列が指定文字列を含むか ${fn:contains(str,'Java')}
fn:toUpperCase 大文字に変換する ${fn:toUpperCase(str)}
fn:replace 文字列を置換する ${fn:replace(str,'A','B')}
fn:trim 前後の空白を除去する ${fn:trim(str)}

10.5 つまずきやすいポイント

EL式とJSTLを使い始めたときに多いつまずきをまとめました。困ったらここを確認してください。

JSTLが動作しない（タグがそのまま表示される）

以下を順番に確認してください。

  • JARが未配置: jakarta.servlet.jsp.jstl-3.x.x.jar（または同等品）が WEB-INF/lib/ にあるか確認。
  • taglibのURIが古い: http://java.sun.com/jsp/jstl/core（旧）ではなく jakarta.tags.core（新）になっているか確認。
  • taglibディレクティブの記述忘れ: JSPファイルの先頭に <%@ taglib prefix="c" uri="jakarta.tags.core" %> があるか確認。

EL式が評価されず ${...} がそのまま表示される

EL式が無効化されている可能性があります。

  • pageディレクティブの確認: JSPファイルの <%@ page isELIgnored="true" %> が「EL式を無視する」設定になっていないか確認。isELIgnored="false"（デフォルト）に直す。
  • web.xmlのバージョン確認: 非常に古い web.xml（Servlet 2.3 以前）が存在するとEL式が無効になる場合があります。Servlet 6.0 の名前空間（https://jakarta.ee/xml/ns/jakartaee）を使うよう更新しましょう。

nullと空文字の扱いに迷う

empty 演算子は null・空文字・空リスト・空配列・空Map のすべてを true として扱います。「値がない状態」をまとめてチェックしたいときは ${empty user.name} を活用しましょう。== null や eq '' を個別に書く必要はありません。

やってみよう 10-1: スクリプトレットをEL/JSTLに書き換え

スクリプトレットで書かれた既存のJSPを、EL式とJSTLを使ったモダンな書き方に書き換えてみましょう。

課題1: スクリプトレット → EL式に変換

  1. 1 Servletで request.setAttribute("userName", "田中太郎") を設定し、JSP側でスクリプトレット <%= request.getAttribute("userName") %> で表示するプログラムを用意する。
  2. 2 JSP側のスクリプトレットを ${userName}（EL式）に書き換えて、同じ結果が表示されることを確認する。
  3. 3 さらに <c:out value="${userName}" /> に変更して、XSSを試す（例: <script>alert(1)</script> を値に設定）と自動エスケープされることを確認する。

課題2: データ一覧をc:forEachでテーブル表示

  1. 1 Servletで商品リスト（名前・価格を持つBeanのList）を request.setAttribute("products", list) にセットする。
  2. 2 JSP側で <c:forEach items="${products}" var="p"> を使い、商品名と価格をテーブルに一覧表示する。
  3. 3 varStatus を使って行番号（1始まり）も表示してみる。

課題3: c:chooseで条件に応じた表示切り替え

  1. 1 Servletでスコア（0〜100の整数）を requestScope にセットする。
  2. 2 JSP側で <c:choose> を使い、90以上=「S」、70以上=「A」、60以上=「B」、60未満=「C」とランクを表示する。
  3. 3 Servletのスコア値をいくつか変えて、正しく表示が切り替わることを確認する。

理解度確認クイズ

  1. Q1. EL式 ${user} と書いたとき、どの順番でスコープが探索されますか？

    回答を表示

    page → request → session → application の順に探索されます。最初に見つかったスコープの値が使われます。スコープを明示したい場合は ${sessionScope.user} のように指定します。

  2. Q2. <c:out> タグを使う利点は何ですか？EL式の ${...} と何が違いますか？

    回答を表示

    <c:out> は出力時に <・>・& 等を自動でHTMLエスケープするため、XSS（クロスサイトスクリプティング）攻撃を防げます。EL式 ${...} はそのまま出力するため、ユーザー入力を表示する箇所では <c:out> を使うのが安全です。

  3. Q3. Tomcat 10以降でJSTLのコアタグを使うには、taglibディレクティブのURIに何を指定しますか？

    回答を表示

    jakarta.tags.core を指定します。旧来の http://java.sun.com/jsp/jstl/core（javax時代のURI）は Tomcat 10 以降では使えません。<%@ taglib prefix="c" uri="jakarta.tags.core" %> とJSPファイルの先頭に書きます。
