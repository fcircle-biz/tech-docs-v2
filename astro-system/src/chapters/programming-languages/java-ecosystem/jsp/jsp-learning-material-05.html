ホーム JSP/Servlet 第5章
第5章 所要時間: 約60分

JSPの構文をマスターする

この章では、JSP ページの動作をコントロールする ディレクティブ（ページ属性やファイル取り込みの宣言）と、ページ間の連携に使う 標準アクションタグ、そして Java コードを書かなくても使える 暗黙オブジェクト を学びます。これらを使いこなすことで、実用的な JSP ページが作れるようになります。

この章で学ぶこと

  • pageディレクティブでページ属性（文字コード・インポート・エラーページ）を設定できる
  • includeディレクティブで共通ヘッダー・フッターをファイル分割して組み込める
  • 標準アクションタグ（jsp:include / jsp:forward / jsp:param）を使用できる
  • 暗黙オブジェクト（request / response / session / application 等）を目的別に活用できる

5.1 pageディレクティブ

ディレクティブ（directive）とは、JSP エンジンへの「指示書き」です。レスポンスの内容には出力されませんが、そのページがどのように動くかをコントロールします。中でも最も使用頻度が高いのが pageディレクティブ（<%@ page ... %>）です。

基本構文

pageディレクティブは JSP ファイルの先頭付近に記述します。1行にまとめることも、複数に分けて書くこともできます。

JSP — pageディレクティブの基本構文 
<!-- JSPタグはHTMLとして表示するためエスケープしています -->
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page pageEncoding="UTF-8" %>
<%@ page import="java.util.List, java.util.ArrayList" %>
<%@ page errorPage="error.jsp" %>

主要な属性一覧

pageディレクティブには多くの属性がありますが、まず押さえるべきものを表にまとめます。

属性名 役割 代表的な値
contentType レスポンスのMIMEタイプと文字コードを指定する text/html; charset=UTF-8
pageEncoding このJSPファイル自体の文字コードを指定する UTF-8
import Javaのクラスをインポートする（カンマ区切りで複数指定可） java.util.List
errorPage 例外発生時に転送するエラーページのURLを指定する error.jsp
isErrorPage このJSPがエラーページであるかどうかを指定する true / false
session セッションを使うかどうかを指定する（デフォルトはtrue） true / false

contentTypeとpageEncodingの違い

初心者がよく混乱するのが、contentType と pageEncoding の違いです。図を使って整理しましょう。

（pageEncoding=UTF-8で保存）"] B["JSPエンジンが
Javaコードに変換"] C["ブラウザへ送信
（contentType=text/html; charset=UTF-8）"] A -->|"ファイル読み込み時の
文字コード"| B B -->|"レスポンスとして送る

実務では両方とも UTF-8 を指定する

ファイルの保存（pageEncoding）とレスポンス（contentType 内の charset）を どちらも UTF-8 に統一するのが最も安全です。どちらか一方でも違うコードが混入すると文字化けが起きます。

エラーページの設定

errorPage と isErrorPage を組み合わせると、例外発生時に専用のエラーページを表示できます。

shop.jsp — エラーページを指定する 
<%-- 通常ページ: 例外が起きたらerror.jspへ転送 --%>
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page errorPage="error.jsp" %>
error.jsp — エラーページ本体（isErrorPage=true） 
<%-- エラーページ: isErrorPage=trueで例外オブジェクトを受け取れる --%>
<%@ page contentType="text/html; charset=UTF-8" %>
<%@ page isErrorPage="true" %>
<p>エラーが発生しました: <%= exception.getMessage() %></p>

5.2 includeディレクティブ

Webサイトでは、ヘッダー・フッター・ナビゲーションなど複数のページで共通するパーツが多くあります。それを1ヶ所にまとめて管理できるのが includeディレクティブ（<%@ include file="..." %>）です。

静的インクルードの仕組み

includeディレクティブは「静的インクルード」と呼ばれます。JSP エンジンがJava コードに変換する前に、指定したファイルの内容をそのままこの位置に「貼り付ける」ように合体させます。

（includeディレクティブで
header.jspを取り込む）"] B["header.jsp
（共通ヘッダー）"] C["コンパイル前に合体
1つの大きなJSPとして変換"] D["Servletクラス
（1つのJavaコードになる）"] A -->|"変換前に内容を展開"| C B -->|"ここに貼り付け"| C C --> D

共通ヘッダー・フッターの作成例

実際にファイルを分割してみましょう。共通のヘッダーを header.jsp に切り出し、各ページからインクルードする構成です。

header.jsp — 共通ヘッダーパーツ 
<!-- 共通ヘッダー: 全ページで使い回す部品 -->
<header>
    <nav>
        <a href="/myapp/index.jsp">ホーム</a>
        <a href="/myapp/list.jsp">一覧</a>
    </nav>
</header>
index.jsp — ヘッダーをインクルードするページ 
<%@ page contentType="text/html; charset=UTF-8" %>
<html><body>
    <!-- 静的インクルード: コンパイル前にheader.jspの内容が展開される -->
    <%@ include file="header.jsp" %>
    <main>
        <h1>ホームページ</h1>
    </main>
</body></html>

変数スコープが共有される

静的インクルードでは、元の JSP とインクルードされたファイルが1つのコードに合体するため、変数を共有できます。たとえば index.jsp で宣言した変数を header.jsp 内から参照できます。これは後述の動的インクルード（jsp:include）とは異なる動作です。

パスの指定方法

file 属性には相対パスを指定します。同じフォルダの場合はファイル名のみ、上位フォルダの場合は ../ で移動します。

記法 意味
file="header.jsp" 同じフォルダにある header.jsp
file="../common/header.jsp" 1つ上のフォルダの common/ 内にある header.jsp
file="/common/header.jsp" 注意: includeディレクティブでは「/」始まりは使えない（コンテキストルートからの絶対パスはアクションタグで使用）

5.3 標準アクションタグ

標準アクションタグは、JSP の仕様に組み込まれた特別な XML タグです。プレフィックス jsp: で始まり、スクリプトレット（Javaコード）を書かずにページの動的な処理が記述できます。

jsp:include — 動的インクルード

<jsp:include> は動的インクルードと呼ばれます。インクルードされたページが別々に実行され、実行した結果（HTMLの出力）だけをこの位置に埋め込みます。includeディレクティブとは根本的に異なる仕組みです。

main.jsp — jsp:includeで動的に取り込む 
<%@ page contentType="text/html; charset=UTF-8" %>
<html><body>
    <!-- 動的インクルード: menu.jspは実行時に個別に処理される -->
    <jsp:include page="menu.jsp">
        <!-- jsp:paramでパラメータを渡すことができる -->
        <jsp:param name="activePage" value="home" />
    </jsp:include>
    <main><h1>メインコンテンツ</h1></main>
</body></html>

jsp:forward — フォワード処理

<jsp:forward> は、現在のリクエストを別のページに引き渡す（フォワードする）タグです。処理後に別のページで表示する、という流れでよく使います。フォワードが実行されると、それ以降のコードは実行されません。

process.jsp — 結果ページへフォワードする 
<%@ page contentType="text/html; charset=UTF-8" %>
<%
    // 何らかの処理（例: 入力チェック）
    String result = "OK";
%>
<!-- フォワード: result.jspへリクエストごと転送する -->
<jsp:forward page="result.jsp">
    <jsp:param name="status" value="OK" />
</jsp:forward>

静的インクルードと動的インクルードの比較

includeディレクティブ（静的）と jsp:include（動的）は、見た目は似ていますが動作が大きく異なります。使い分けを表で整理します。

比較項目 includeディレクティブ（静的） jsp:include（動的）
タイミング コンパイル時（変換前）に合体 実行時（リクエストのたび）に呼び出す
変数スコープ 変数を共有できる 別々のスコープ（独立したページ）
パラメータ渡し 不可 jsp:param で渡せる
変更の反映 再コンパイルが必要 常に最新のファイルを読む
向いている用途 変わらない共通部品（定型ヘッダーなど） 動的に内容が変わる部品・パーツ

5.4 JSP暗黙オブジェクト

JSP では、宣言しなくても最初から使えるオブジェクトがあります。これを 暗黙オブジェクト（implicit objects）と呼びます。Servlet と同じオブジェクトを、宣言なしで変数名そのまま参照できるのが JSP の便利なところです。

暗黙オブジェクト一覧

JSP には9つの暗黙オブジェクトがあります。特に重要な4つは request・response・session・application です。

オブジェクト名 型 スコープ（有効期間） 主な用途
request HttpServletRequest 1回のリクエスト パラメータ取得、属性の受け渡し
response HttpServletResponse 1回のリクエスト リダイレクト、ヘッダー操作
out JspWriter 1回のリクエスト レスポンスへのテキスト出力
session HttpSession セッション期間（ブラウザを閉じるまで等） ログイン情報など複数リクエストにまたがる情報
application ServletContext アプリケーション全体（サーバー起動中） 全ユーザー共通の情報、設定値
config ServletConfig ページ 初期化パラメータの取得
pageContext PageContext ページ 他のオブジェクトへのアクセス口
page Object（thisに相当） ページ ページ自身の参照（ほぼ使わない）
exception Throwable ページ（エラーページのみ） 発生した例外の情報取得（isErrorPage=trueのとき）

スコープの違いをイメージで理解する

request・session・application はそれぞれ「どのくらいの期間、情報を保持するか」が違います。用途に合ったオブジェクトを使うことが重要です。

request

1回のリクエスト（＝ブラウザの1回のアクセス）の間だけ有効。

例: フォームの入力値、Servletから転送された計算結果

session

セッション期間（ブラウザを閉じるか、タイムアウトまで）有効。

例: ログイン中のユーザー名、ショッピングカートの中身

application

アプリ全体（サーバー起動中）で全ユーザーが共有。

例: 全体のアクセスカウンター、共通設定値

暗黙オブジェクトの使用例

JSP では宣言不要でそのまま使えます。Servlet で書くコードより大幅に簡潔になります。

greeting.jsp — 暗黙オブジェクトを使う例 
<%@ page contentType="text/html; charset=UTF-8" %>
<%
    // request: リクエストパラメータを取得（宣言不要）
    String name = request.getParameter("name");

    // session: セッションに値を保存
    session.setAttribute("loginUser", name);

    // application: アプリ全体の属性を参照
    String appName = (String) application.getAttribute("appName");
%>
<p>ようこそ、<%= name %>さん！</p>

5.5 つまずきやすいポイント

ディレクティブとアクションタグを扱うとき、初心者が引っかかりやすいポイントを整理します。困ったら先にここを確認しましょう。

インクルードファイルが見つからない（404 / FileNotFoundException）

相対パスと絶対パスの混同が原因です。

  • <%@ include file="..." %> は相対パスのみ対応。/ 始まりは使えません。
  • <jsp:include page="..." /> は / 始まりの絶対パス（コンテキストルート基準）も使えます。

文字化けが起きる

pageディレクティブの contentType および pageEncoding の指定漏れが原因です。JSP ファイルのすべてのページで <%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" %> を先頭に書く習慣をつけましょう。

暗黙オブジェクトの混同（request / session / application）

「ログイン情報を保存したのに次のページで消えた」→ request.setAttribute に保存していませんか？ request は1リクエストで消えます。複数ページにまたがる情報には session、全ユーザー共通には application を使いましょう。

やってみよう 5-1: 共通ヘッダー・フッターを持つWebサイト構造の作成

includeディレクティブを使って、複数ページで共通パーツを再利用する構成を作ります。

手順

  1. 1 header.jsp にサイト名とナビゲーションリンクを書き、footer.jspにコピーライト文を書く。
  2. 2 index.jsp を作り、先頭に <%@ include file="header.jsp" %> を、末尾に <%@ include file="footer.jsp" %> を書く。
  3. 3 同じ構成で about.jsp も作る。ヘッダー・フッターが同一であることをブラウザで確認する。

やってみよう 5-2: エラーページの設定と動作確認

  1. 1 error.jsp を作り、<%@ page isErrorPage="true" %> を設定して例外メッセージを表示するようにする。
  2. 2 別のJSPに <%@ page errorPage="error.jsp" %> を設定し、意図的に int x = 1/0; など例外を発生させる。
  3. 3 ブラウザでアクセスし、error.jsp が表示され例外メッセージが出ることを確認する。

やってみよう 5-3: 複数JSPで構成されるページの作成

  1. 1 menu.jsp に現在選択中のメニュー項目を受け取るコードを書く（request.getParameter("current")）。
  2. 2 メインページから <jsp:include page="menu.jsp"><jsp:param name="current" value="home" /></jsp:include> でパラメータを渡してインクルードする。
  3. 3 ブラウザで確認し、menu.jsp がパラメータを受け取って動作していることを確認する。

理解度確認クイズ

  1. Q1. includeディレクティブ（静的インクルード）とjsp:include（動的インクルード）の最も大きな違いを説明してください。

    回答を表示

    includeディレクティブ（静的）は、コンパイル前に対象ファイルの内容を1つに合体させます。変数スコープが共有され、パラメータ渡しはできません。jsp:include（動的）は実行時に対象ページを別途実行し、その出力結果をこの位置に埋め込みます。変数スコープは独立していますが、jsp:paramでパラメータを渡せます。

  2. Q2. ログインしたユーザー名を複数のページにまたがって保持したい場合、どの暗黙オブジェクトを使うべきですか？また、その理由を答えてください。

    回答を表示

    session を使います。requestオブジェクトは1回のリクエストが終わると消えるため、次のページにはアクセスできません。sessionはブラウザが閉じられるかタイムアウトするまでの間、同じユーザーの複数リクエストにまたがって情報を保持できるからです。

  3. Q3. pageディレクティブで contentType="text/html; charset=UTF-8" と指定しても文字化けが起きました。他に確認すべき属性は何ですか？

    回答を表示

    pageEncoding="UTF-8" の指定を確認します。contentTypeはレスポンスで送る文字コードの宣言ですが、pageEncodingはJSPファイル自体を読み込む際の文字コードです。ファイルをUTF-8で保存しているのにpageEncodingが未指定だと、JSPエンジンがファイルを誤ったコードで読み込んで文字化けが起きます。
