ToDoアプリを作ろう③ 完了・削除機能を追加する

タスクの完了状態を切り替える機能と、不要なタスクを削除する機能を実装します。オブジェクトでタスクの状態を管理し、配列の更新に合わせて画面を再表示する一貫した設計を身につけ、ToDoアプリを完成させましょう。

この章で学ぶこと

  • タスクを文字列からオブジェクト（{ text, done }）に変えて状態を管理できる
  • task.done = !task.done で完了状態を反転し、見た目を切り替えられる
  • splice で配列からタスクを削除し、render() で再表示できる
  • 「データを変更 → render で再描画」という一貫したアプリ設計を理解する

16.1 データをオブジェクトに進化させる

第15章では、タスクを文字列の配列（例: ["買い物", "宿題"]）で管理していました。しかし、「完了したかどうか」という状態を一緒に持たせるには、文字列だけでは足りません。

そこで今章では、第8章で学んだ「配列の中にオブジェクト」を使います。タスク1件を { text: "買い物", done: false } というオブジェクトで表し、done プロパティで完了状態を管理します。

第15章まで（文字列）

["買い物", "宿題"]

テキストだけ。完了状態を持てない。

第16章から（オブジェクト）

{ text: "買い物", done: false }

テキスト＋完了状態を一緒に管理できる。

まずタスクを追加する部分と render 関数を更新しましょう。

todo.js（追加部分を変更） 
// タスクをオブジェクトの配列で管理する
const tasks = [];

function addTask() {
  const input = document.getElementById('taskInput');
  const text = input.value.trim();
  if (text === '') return;

  // 文字列ではなくオブジェクトとして追加
  tasks.push({ text: text, done: false });
  input.value = '';
  render(); // 画面を更新
}

復習: 配列の中のオブジェクト（第8章）

tasks[0].text でテキスト、tasks[0].done で完了状態を取り出せます。forEach や for ループで全件取り出すときも同じドット記法です。

16.2 完了の切り替え

タスクをクリックすると「完了 ↔ 未完了」を切り替える機能を実装します。手順は次の2ステップです。

  1. ① done を反転する — tasks[index].done = !tasks[index].done
  2. ② render() を呼ぶ — 画面を最新の配列に合わせて再描画する

!tasks[index].done は「今の値が true なら false、false なら true」を意味します。否定演算子 ! を使うと、真偽値を1行で反転できます（第5章の復習）。

render 関数では、done が true のタスクに取り消し線などの CSS クラスを付けて見た目を変えます。forEach の第2引数でインデックスを受け取るのがポイントです。

todo.js（完了トグルとrender） 
// 完了状態を反転する関数
function toggleDone(index) {
  tasks[index].done = !tasks[index].done; // true↔false を切り替え
  render();
}

// 配列を画面に表示する関数
function render() {
  const list = document.getElementById('taskList');
  list.innerHTML = ''; // いったん全消去

  tasks.forEach((task, index) => {
    const li = document.createElement('li');
    // doneがtrueなら取り消し線クラスを付ける
    li.className = task.done ? 'line-through text-slate-400' : '';
    li.textContent = task.text;
    li.onclick = () => toggleDone(index); // クリックで完了切り替え
    list.appendChild(li);
  });
}

重要ポイント: インデックスで対象タスクを特定する

forEach(task, index) の index は「配列の何番目か」です。クリックイベントに index を渡すことで、どのタスクが操作されたかを正確に特定できます。

16.3 削除機能

不要なタスクを配列から取り除く削除機能を追加します。配列からアイテムを削除するには splice(開始位置, 削除数) を使います。

例えば tasks.splice(1, 1) は「インデックス1から1件削除」を意味します。削除後は render() を呼んで画面を最新状態に更新します。

todo.js（削除機能を追加） 
// 指定インデックスのタスクを削除する
function deleteTask(index) {
  tasks.splice(index, 1); // 配列から1件取り除く
  render(); // 画面を再描画
}

// renderにも削除ボタンを追加する
function render() {
  const list = document.getElementById('taskList');
  list.innerHTML = '';
  tasks.forEach((task, index) => {
    const li = document.createElement('li');
    li.className = task.done ? 'line-through text-slate-400' : '';
    li.textContent = task.text;
    li.onclick = () => toggleDone(index);

    // 削除ボタンを作成して li に追加
    const btn = document.createElement('button');
    btn.textContent = '削除';
    btn.onclick = (e) => { e.stopPropagation(); deleteTask(index); };
    li.appendChild(btn);
    list.appendChild(li);
  });
}

e.stopPropagation() とは？

削除ボタンのクリックが親要素（li）にも伝わってしまうと、削除と同時に完了切り替えも起きてしまいます。e.stopPropagation() でイベントの伝播を止めることで、削除だけを実行できます。

16.4 完成と設計のまとめ

これで ToDo アプリの主要な機能がすべて揃いました。ここで、このアプリ全体の設計を振り返っておきましょう。

flowchart TD A["ユーザー操作
（クリック・入力）"] --> B{何の操作？} B -->|"タスク追加"| C["tasks.push（追加）"] B -->|"完了切り替え"| D["task.done = !task.done（反転）"] B -->|"タスク削除"| E["tasks.splice（削除）"] C --> F["render（）を呼ぶ"] D --> F E --> F F --> G["list.innerHTML = '' で画面をリセット"] G --> H["tasksをループして再描画"] H --> I["画面が最新の状態になる"]

このアプリの一貫したパターンは 「データ（配列＋オブジェクト）を変更 → render() で再描画」 です。画面を直接バラバラに書き換えるのではなく、必ずデータを先に正しい状態にしてから、render が画面全体を描き直すという流れです。この設計は React や Vue などのモダンフレームワークの考え方の基礎にもなっています。

発展: localStorage でデータを保存する

今のアプリはページを再読み込みするとタスクが消えてしまいます。ブラウザの localStorage を使うと、タスクをブラウザに保存して再読み込み後も残せます。localStorage.setItem('tasks', JSON.stringify(tasks)) で保存し、JSON.parse(localStorage.getItem('tasks')) で読み込めます。ぜひ挑戦してみてください。

理解度確認クイズ

  1. Q1. 第15章では文字列の配列でタスクを管理していましたが、第16章でオブジェクトの配列に変えた理由は何ですか？

    回答を表示

    タスクの「完了状態（done）」を一緒に管理するためです。文字列だとテキストしか持てませんが、{ text: "...", done: false } というオブジェクトにすることで、テキストと完了状態を1つのデータとしてまとめて扱えます。

  2. Q2. task.done = !task.done はどのような処理ですか？ ! の役割とあわせて説明してください。

    回答を表示

    ! は否定演算子で、真偽値を反転します。!true は false、!false は true になります。つまり task.done = !task.done は「今の done の値を反転して代入する」処理で、1行で完了 ↔ 未完了を切り替えられます。

  3. Q3. tasks.splice(index, 1) でタスクを削除した後、なぜ必ず render() を呼ぶ必要があるのですか？

    回答を表示

    splice は配列（データ）を変更するだけで、画面（DOM）は自動では更新されないからです。このアプリの設計では「データを変更した後に render() で画面を最新のデータから再描画する」という流れを守ることで、データと画面のズレを防いでいます。
