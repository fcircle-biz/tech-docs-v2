複数のデータを扱おう（配列とオブジェクト）

複数のデータをまとめて管理する「配列」と、関連する情報をひとまとまりにできる「オブジェクト」を学びます。ユーザー情報やタスク情報など、アプリで扱うデータの形をイメージし、後半の ToDo アプリ（第14〜16章）への土台を作ります。

この章で学ぶこと

  • 配列を使って複数の値をまとめて管理できる
  • インデックスで配列の要素にアクセスし、push で追加できる
  • オブジェクトのキーと値の仕組みを理解し、プロパティにアクセスできる
  • 「配列の中にオブジェクト」という、アプリでよく使うデータ構造をイメージできる

8.1 なぜ配列とオブジェクトが必要か

これまでの章では、変数1つに値を1つ入れてきました。たとえば名前なら const name = "太郎"; のように。しかしアプリを作るとき、値は1つだけではありません。

たとえば「クラス全員30人分の名前」を管理したいとき、変数を30個用意するのは大変です。後から人数が増えたら変数を増やすしかなく、全員のデータを一括で処理することもできません。

変数1つ → 値1つ（問題）

30人分なら変数が30個必要。人数が変わるたびにコードを書き直さなければなりません。

配列/オブジェクト → 値をまとめる（解決）

1つの変数で複数の値をまとめて管理。ループで一括処理もでき、コードがすっきりします。

この章では、複数の値を「順番で管理する配列」と「名前（キー）で管理するオブジェクト」の2種類を学びます。この2つを組み合わせることで、アプリが扱うほとんどのデータを表現できるようになります。

8.2 配列（Array）

配列とは、複数の値を「順番」で管理するデータ構造です。値を [ ]（角括弧）で囲み、カンマで区切って並べます。

配列 fruits のイメージ

インデックス 0 1 2
値 "りんご" "みかん" "ぶどう"

重要: インデックスは 0 から始まる

配列の「番号（インデックス）」は 0 からカウントします。最初の要素は fruits[0]、2番目は fruits[1] です。「1番目だから fruits[1]」と思いがちですが、正しくは fruits[0] です。初心者がよく間違えるポイントです。

配列でよく使う操作を確認しましょう。.length で要素の数（個数）を取得でき、.push() で末尾に要素を追加できます。

array-basics.js 
// 配列の作成（角括弧で値を並べる）
const fruits = ["りんご", "みかん", "ぶどう"];

// インデックスで要素にアクセス（0から始まる）
console.log(fruits[0]); // "りんご"
console.log(fruits[1]); // "みかん"

// 要素数を調べる
console.log(fruits.length); // 3

// 末尾に要素を追加する
fruits.push("もも");
console.log(fruits); // ["りんご", "みかん", "ぶどう", "もも"]
console.log(fruits.length); // 4

配列の全要素を処理するには、for 文や forEach メソッドが使えます。forEach は「各要素に対して〇〇する」という意味で、すっきり書けて読みやすいのが特徴です。

array-loop.js 
const fruits = ["りんご", "みかん", "ぶどう"];

// forEach: 各要素に対して処理を実行する
fruits.forEach(function(fruit) {
    console.log(fruit); // りんご, みかん, ぶどう と順番に表示
});

// for文でも同じことができる（どちらでもOK）
for (let i = 0; i < fruits.length; i++) {
    console.log(fruits[i]); // fruits[0], fruits[1], fruits[2]
}

const なのに push できる？

const で宣言した変数は「再代入禁止」ですが、配列の中身（要素）を変えることは許されています。fruits = ["新しい配列"] のように配列自体を別のものに差し替えることは禁止、という意味です。push で要素を追加するのは問題ありません。

8.3 オブジェクト（Object）

オブジェクトとは、関連する情報を「名前（キー）と値のペア」でまとめるデータ構造です。値を { }（波括弧）で囲み、キー: 値 の形式で書きます。

オブジェクト user のイメージ

キー（プロパティ名） 値 アクセス方法
name "太郎" user.name
age 20 user.age
hobby "読書" user.hobby
object-basics.js 
// オブジェクトの作成（波括弧でキー: 値を並べる）
const user = {
    name: "太郎",  // キー: name、値: "太郎"
    age: 20,       // キー: age、値: 20
    hobby: "読書"  // キー: hobby、値: "読書"
};

// ドット記法でプロパティにアクセス
console.log(user.name);  // "太郎"
console.log(user.age);   // 20

// プロパティの値を変更する
user.age = 21;
console.log(user.age);   // 21

配列とオブジェクトの違い

flowchart LR A["配列
fruits = ['りんご', 'みかん', 'ぶどう']"] B["インデックス（数字）で管理
fruits[0] → 'りんご'
fruits[1] → 'みかん'"] C["オブジェクト
user = { name: '太郎', age: 20 }"] D["キー（名前）で管理
user.name → '太郎'
user.age → 20"] A --> B C --> D

使い分けのポイント

「同じ種類のものが複数ある（名前のリスト、商品の一覧など）」→ 配列。「1つのものが複数の情報を持つ（ユーザー情報、商品の詳細など）」→ オブジェクト。この使い分けがアプリ設計の基本です。

8.4 配列の中にオブジェクト — アプリのデータ構造

実際のアプリでは、「配列」と「オブジェクト」を組み合わせて使います。たとえば ToDo アプリのタスク一覧は「複数のタスク（配列）」で、各タスクは「テキストと完了状態（オブジェクト）」を持ちます。

todo-data.js 
// ToDoアプリのタスク一覧（配列の中にオブジェクト）
const tasks = [
    { text: "買い物に行く", done: false },
    { text: "JavaScriptを勉強する", done: true },
    { text: "部屋を掃除する", done: false }
];

// 1番目のタスク（インデックス0）のテキストを表示
console.log(tasks[0].text); // "買い物に行く"
console.log(tasks[1].done); // true（勉強は完了済み）

// forEach で全タスクを確認する
tasks.forEach(function(task) {
    console.log(task.text, task.done ? "完了" : "未完了");
});

  3. 3 （発展）me.age を書き換えて、変更後の値が表示されることを確認する。

理解度確認クイズ

  1. Q1. const colors = ["赤", "青", "緑"]; のとき、colors[1] の値は何ですか？

    回答を表示

    "青" です。インデックスは 0 から始まるため、colors[0] が "赤"、colors[1] が "青"、colors[2] が "緑" になります。「1番目だから colors[1]」と思いがちですが、最初の要素は colors[0] です。

  2. Q2. 配列とオブジェクトの違いを、「管理方法」の観点から説明してください。

    回答を表示

    配列はインデックス（0, 1, 2 …の数字）で要素を管理し、順番が重要なデータ（同じ種類のものの一覧）に向いています。オブジェクトはキー（name, age などの名前）で値を管理し、1つのものに関する複数の情報をまとめるのに向いています。

  3. Q3. const book = { title: "JavaScript入門", price: 2800 }; のとき、価格を取得するコードを書いてください。

    回答を表示

    book.price と書きます。実行すると 2800 が返ります。オブジェクトのプロパティはドット記法（オブジェクト名.キー名）でアクセスします。
