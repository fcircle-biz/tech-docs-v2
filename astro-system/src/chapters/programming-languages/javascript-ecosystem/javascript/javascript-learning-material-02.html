ホーム JavaScript 第2章
第2章 所要時間: 約90分

開発環境を準備しよう

プログラムを書くためのエディタ（VSCode）と、実行・確認のためのブラウザの開発者ツールを準備します。この章では「エディタでコードを書き、ブラウザで実行し、エラーを確認して直す」という JavaScript 学習の基本サイクルを体験します。必要なものは無料のソフト2つだけです。

この章で学ぶこと

  • JavaScript の学習に必要な環境が「VSCode」と「Webブラウザ」の2つだけと分かる
  • Visual Studio Code をインストールし、学習用フォルダを開ける
  • ブラウザの開発者ツール（コンソール）を開き、JavaScript を直接実行できる
  • エラーメッセージが出ても慌てずに読んで対処する姿勢を身につける

2.1 開発環境とは

「開発環境」とは、プログラムを書いて動かすために必要な道具一式のことです。難しそうに聞こえますが、JavaScript の学習に必要なのは次の2つだけです。準備に時間はかかりません。

Visual Studio Code（VSCode）

コードを書くためのエディタ（メモ帳の高機能版）。無料で使えます。コードに色がついて読みやすく、入力ミスを指摘してくれます。

Webブラウザ（Chrome 推奨）

JavaScript を実行する場所。開発者ツールという機能が内蔵されており、コードのテストやエラー確認ができます。

重要ポイント

JavaScript はブラウザさえあれば動かせます。特別な有料ソフトや複雑なサーバー設定は不要です。「今すぐ試せる」のが JavaScript の大きな魅力の1つです。

2.2 VSCode の準備

Visual Studio Code（VSCode）は、Microsoft が無料で提供するコードエディタです。世界中のエンジニアが使っており、JavaScript との相性も抜群です。まずはインストールから始めましょう。

VSCode のインストール手順

  1. 1 https://code.visualstudio.com/ を開き、「Download for Windows」（または macOS / Linux）ボタンをクリックします。
  2. 2 ダウンロードしたインストーラーを実行し、画面の指示に従ってインストールします（基本的にすべて「次へ」で問題ありません）。
  3. 3 インストール完了後、VSCode を起動します。

フォルダ（作業場所）を開く

VSCode では「フォルダ」を作業単位として管理します。まずパソコンに学習用のフォルダを1つ作り、VSCode でそのフォルダを開く習慣をつけましょう。

学習用フォルダを作って開く

  1. 1 デスクトップや「ドキュメント」フォルダの中に javascript-study という名前のフォルダを作成します。
  2. 2 VSCode を起動し、メニューから「ファイル」→「フォルダーを開く…」を選択して、先ほど作成したフォルダを指定します。
  3. 3 左側のサイドバーに JAVASCRIPT-STUDY と表示されれば成功です。

便利な拡張機能（任意）

VSCode は「拡張機能」を追加してさらに便利にできます。学習に役立つ2つを紹介します。左サイドバーの四角いアイコン（拡張機能）から検索してインストールできます。

  • Japanese Language Pack for VS Code — メニューなどを日本語化します。
  • Live Server — HTML ファイルをブラウザで自動更新しながら確認できます。

2.3 ブラウザの開発者ツール

Chrome（または Edge・Firefox）には開発者ツールという強力な機能が内蔵されています。Web ページのコードを調べたり、JavaScript を直接実行したりできます。初心者の段階では、特に「コンソール（Console）」タブを使います。

キーボードショートカット

F12 キーを押す（最も素早い方法）

右クリックから開く

ページ上で右クリック →「検証」または「要素を検証」を選択

メニューから開く

Chrome の「…（メニュー）」→「その他のツール」→「デベロッパーツール」

開発者ツールが開いたら、上部のタブから「Console（コンソール）」をクリックしてください。コンソールは JavaScript の「対話式実行窓」です。コードを入力してEnterキーを押すと、すぐに結果が表示されます。

コンソールとは？

コンソールは、JavaScript のコードをその場で試せる「試験場」のようなものです。ファイルを保存しなくても、短いコードをすぐに実行して結果を確認できます。学習中は積極的に活用しましょう。

2.4 「書く → 実行 → エラーを確認」の基本サイクル

JavaScript の学習は「書いて、動かして、直す」の繰り返しです。このサイクルを早く回すほど、上達も早くなります。まずは一番シンプルなコードを実際に試してみましょう。

flowchart LR A["コードを書く
（VSCode）"] --> B["ブラウザで実行
（開発者ツール）"] B --> C{"結果を確認"} C -->|"成功（緑・白文字）"| D["次の学習へ進む"] C -->|"エラー（赤文字）"| E["エラー内容を読む"] E --> A

console.log を試してみよう

console.log() は、コンソールにメッセージを表示する命令です。JavaScript の「Hello World」として最初に覚えておきましょう。

コンソールに直接入力 
// コンソールにメッセージを表示する
console.log("こんにちは！");

// 数値の計算結果も表示できる
console.log(3 + 5);

ブラウザの開発者ツールを開き、Console タブを選択して上記のコードを1行ずつ入力し、Enterキーを押してみましょう。こんにちは！ と 8 が表示されれば成功です。

わざとエラーを出してみよう

プログラミングの学習でエラーは避けられません。でも、エラーは「どこが間違っているか」を教えてくれる大切なヒントです。怖がらず、むしろ積極的に読む習慣をつけましょう。

わざとエラーを起こすコード 
// 閉じカッコが足りない（構文エラー）
console.log("エラーのテスト"

上記のコードをコンソールに入力すると、赤い文字でエラーメッセージが表示されます。SyntaxError: Unexpected token（構文エラー）などと表示されます。これは「カッコが閉じていない」という意味です。

エラーの種類 主な原因 対処法
SyntaxError カッコの閉じ忘れ・クォートのペア不一致など エラー行番号を確認して記号を修正する
ReferenceError 存在しない変数・関数名を参照した 変数名のスペルミスを確認する
TypeError データの型が合っていない操作をした 変数の内容（型）を確認する

エラーは「失敗」ではありません

エラーメッセージが出ることはプログラミングでは日常茶飯事です。プロのエンジニアでも毎日エラーに向き合っています。「なぜエラーなのか」を読んで直す経験を積むことが、上達の一番の近道です。

やってみよう 2-1: 環境を整えてコンソールを試す

VSCode で学習フォルダを作り、ブラウザの開発者ツールで console.log を実行してみましょう。

手順

  1. 1 VSCode をインストールし、javascript-study という名前のフォルダを作成して開く。
  2. 2 Chrome を起動し、F12 キーで開発者ツールを開き、「Console」タブを選択する。
  3. 3 コンソールに console.log("はじめてのJavaScript！"); と入力してEnterキーを押し、はじめてのJavaScript！ と表示されることを確認する。
  4. 4 （チャレンジ）console.log(100 * 3); を実行し、300 が表示されることを確認する。

動作確認ポイント

  • コンソールにメッセージが白または黒の文字で表示された → 成功！
  • 赤い文字が出た → エラーメッセージを読んでカッコやクォートのペアを確認する。

理解度確認クイズ

  1. Q1. ブラウザの開発者ツールを開くキーボードショートカットを答えてください。また、JavaScript を実行・確認するには何というタブを使いますか？

    回答を表示

    キーボードショートカットは F12 キーです。また、JavaScript の実行や確認には開発者ツールの Console（コンソール） タブを使います。

  2. Q2. console.log() は何をするための命令ですか？

    回答を表示

    コンソールにメッセージや値を表示する命令です。文字列や数値の計算結果を確認したり、プログラムが正しく動いているかチェックしたりするために使います。

  3. Q3. コンソールに赤い文字（エラーメッセージ）が表示されたとき、どうすればよいですか？

    回答を表示

    慌てずにエラーメッセージを読みます。エラーの種類（SyntaxError なら記号のミス、ReferenceError なら変数名のスペルミスなど）と行番号を確認し、該当箇所を修正します。エラーは「どこを直せばよいか」を教えてくれるヒントです。
