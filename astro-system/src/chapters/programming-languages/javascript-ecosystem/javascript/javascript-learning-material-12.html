ホーム JavaScript 第12章
第12章 所要時間: 約90分

データを画面に反映しよう（配列とDOMの組み合わせ）

配列に入れたデータをもとに Web ページへ一覧表示する方法を学びます。「データを直接 HTML に書くのではなく、JavaScript で管理して画面に反映する」という、アプリ開発の核心となる重要な考え方を身につけましょう。

この章で学ぶこと

  • 「データ（配列）を真実とし、画面はそれを映す鏡」という考え方を理解できる
  • 配列を forEach でループして一覧を HTML 表示できる
  • 一覧表示を render() 関数にまとめ、データが変わるたびに呼び出せる
  • 配列に push してから render() を呼ぶ「データ変更 → 再描画」のサイクルを作れる

12.1 データと画面を分けて考える

Web ページにリストを表示するとき、最初に思いつく方法は HTML に直接 <li> タグを並べることかもしれません。しかしこの方法だと、アイテムを追加・削除するたびに HTML を書き直す必要があり、JavaScript からの操作も複雑になります。

そこで、アプリ開発では「データと画面を分けて考える」という重要な考え方を使います。配列がデータの真実（唯一の情報源）であり、画面はその配列を映す鏡にすぎない、というイメージです。

HTMLに直接書く（避けたい方法）

HTML の中に <li> を並べる。アイテムを増やすたびに HTML を書き直す必要があり、JavaScript との連携が難しい。

配列で管理する（おすすめの方法）

データは JavaScript の配列に入れ、画面は JavaScript が描画する。データを変えれば画面も自動で最新になる。

「データを真実とする」とは？

画面に何が表示されているかよりも、配列の中身が何であるかを「正解」とします。配列を変えれば、render を呼ぶだけで画面が最新状態に更新されます。この考え方は React などのフレームワークにも通じる、アプリ開発の基本原則です。

12.2 配列をループして一覧表示する

配列の各要素を HTML の <li> タグとして画面に表示する方法を2通り紹介します。まずは方法 (a) 「文字列を組み立てて innerHTML にまとめて入れる」を中心に学びましょう。

方法 (a)：innerHTML に文字列をまとめて入れる

forEach で配列を回し、HTML タグの文字列を連結していきます。最後に innerHTML に一括で代入するため、コードがシンプルになります。

innerHTML で一覧表示 
const fruits = ['りんご', 'バナナ', 'みかん'];
const list = document.getElementById('list');

let html = ''; // HTML文字列を蓄積する変数
fruits.forEach(function(item) {
  // テンプレートリテラルで<li>タグを組み立てる
  html += `<li>${item}</li>`;
});

list.innerHTML = html; // まとめて一括代入

方法 (b)：createElement で要素を作って追加する

document.createElement で要素を作り、appendChild で追加する方法です。HTML 文字列を直接組み立てないため、安全な操作ができます（発展的な内容として参考にしておきましょう）。

createElement で一覧表示 
const fruits = ['りんご', 'バナナ', 'みかん'];
const list = document.getElementById('list');

fruits.forEach(function(item) {
  const li = document.createElement('li'); // li要素を作成
  li.textContent = item;                   // テキストを設定
  list.appendChild(li);                   // リストに追加
});

どちらを使えばよい？

まずは方法 (a) innerHTML 方式を使いこなしましょう。コードが短く書けて分かりやすいです。方法 (b) は表示内容を細かく制御したいときや、ユーザー入力をそのまま HTML に埋め込む場合（セキュリティ上の理由）に有利です。

12.3 再描画する関数（render）

一覧表示の処理を render() 関数にまとめると、「データが変わったら render() を呼ぶだけで画面が最新になる」という仕組みが作れます。

render 関数の最初に innerHTML = "" で一覧をクリアするのがポイントです。これをしないと、呼び出すたびにリストが積み重なってしまいます。

render関数のパターン 
const fruits = ['りんご', 'バナナ', 'みかん'];
const list = document.getElementById('list');

function render() {
  list.innerHTML = ''; // まず画面をクリア
  let html = '';
  fruits.forEach(function(item) {
    html += `<li>${item}</li>`; // 各要素を文字列で組み立て
  });
  list.innerHTML = html; // まとめて反映
}

render(); // 最初に1回呼ぶ

render の2ステップ

  • ① クリア: innerHTML = "" で古い表示を消す
  • ② 描画: 配列を forEach でループして新しい HTML を作り、代入する

12.4 データを足して再表示する

配列に push してデータを追加した後に render() を呼ぶと、一覧が自動的に更新されます。この「データ変更 → 再描画」のサイクルが、第15・16章で作る ToDo アプリの心臓部になります。

データ追加 → 再描画 
const fruits = ['りんご', 'バナナ'];
const list = document.getElementById('list');
const btn = document.getElementById('addBtn');

function render() {
  list.innerHTML = '';
  fruits.forEach(function(item) {
    list.innerHTML += `<li>${item}</li>`;
  });
}

btn.addEventListener('click', function() {
  fruits.push('みかん'); // 配列にデータを追加
  render();             // 再描画で画面に反映
});
render(); // 初回表示

「データ変更 → 再描画」サイクルの重要性

この「配列を変えて render() を呼ぶ」流れは、第15・16章の ToDo アプリ（タスク追加・削除・完了）の核心になります。今の段階でこのパターンを体に覚えさせておきましょう！

やってみよう 12-1: 好きな食べ物リストを作ろう

好きな食べ物を配列で管理し、画面に箇条書きで表示します。さらにテキストボックスに入力した食べ物を追加できるようにしましょう。

手順

  1. 1 VSCode で food-list.html と food-list.js を作成する。HTML に <ul id="foodList">・<input id="foodInput">・<button id="addBtn"> を用意する。
  2. 2 JavaScript で好きな食べ物を3つ入れた配列 foods を定義し、render() 関数で <ul> に一覧表示する。
  3. 3 「追加」ボタンのクリックイベントで、foodInput.value を foods.push() して render() を呼ぶ。入力後はテキストボックスを空にする（value = ""）。
  4. 4 ブラウザで確認し、食べ物を入力して「追加」ボタンを押すとリストが増えることを確認する。

発展課題

入力欄が空のまま「追加」ボタンを押したときは何もしないように、if で入力チェックを追加してみましょう。

理解度確認クイズ

  1. Q1. 「データ（配列）を真実とし、画面はそれを映す鏡」とはどういう意味ですか？　なぜ HTML に直接 <li> を書くよりこの方法が優れているのでしょうか？

    回答を表示

    配列（データ）の内容を唯一の正解とし、画面はその配列を元に JavaScript が描画するという考え方です。データを変更するだけで render() を呼べば画面が最新になるため、HTML を直接書き換える手間がなく、アイテムの追加・削除が容易になります。

  2. Q2. render() 関数の冒頭で innerHTML = "" を実行するのはなぜですか？

    回答を表示

    前回の描画結果を消去するためです。クリアせずに render() を呼ぶたびにリストアイテムが重複して追加されてしまいます。まずクリアしてから配列の最新内容を描画することで、常に正しい一覧が表示されます。

  3. Q3. 「データを追加して render() を呼ぶ」流れを正しく示しているのはどれですか？
    ① render() を呼ぶ → push でデータを追加する
    ② push でデータを追加する → render() を呼ぶ

    回答を表示

    正解は ② です。先に push で配列にデータを追加し、その後 render() を呼んで画面を更新します。① の順番では render() が古いデータで描画してしまい、追加した項目が表示されません。
