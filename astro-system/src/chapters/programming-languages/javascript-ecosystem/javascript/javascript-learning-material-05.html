条件によって処理を変えよう（if文）

「もし〜なら」という条件分岐を使って、状況に応じて異なる処理を行う方法を学びます。信号機のたとえで条件分岐の概念をつかみ、比較演算子・真偽値・if/else/else if の構文、そして論理演算子を段階的に習得します。

この章で学ぶこと

  • 条件分岐とは何かを、身近なたとえ（信号機）で説明できる
  • 真偽値（true / false）と比較演算子を使って条件を表現できる
  • if / else / else if を使って複数の条件に応じた処理を書ける
  • 論理演算子（&& / || / !）で条件を組み合わせられる

5.1 条件分岐とは

プログラムはふつう、上から下へ順番に処理を実行します。しかし現実の問題を解くには「状況によって処理を変える」ことが欠かせません。これを条件分岐と呼びます。

信号機を思い浮かべてください。「赤なら止まる、青なら進む」のように、条件を満たすかどうかによって行動が変わります。プログラムでも同じ発想で、「もし〜なら A をする、そうでなければ B をする」という指示を書けます。

赤信号 → 止まる

条件が「赤」のとき、止まる処理を実行します。

青信号 → 進む

条件が「青」のとき、進む処理を実行します。

黄信号 → 注意

3つ目の条件（else if）として処理を分けられます。

JavaScript では、この「もし〜なら」の仕組みを if 文で表現します。条件分岐はプログラムの流れを制御する最も基本的な道具の一つです。

5.2 真偽値と比較演算子

条件分岐の「条件」は、最終的に true（真） か false（偽） のどちらかに評価されます。この2つの値を真偽値（ブーリアン）と呼び、JavaScript のデータ型の1つです。

2つの値を比べて true / false を導き出すのが比較演算子です。以下の表で代表的な演算子を確認しましょう。

演算子 意味 例 結果
=== 等しい（値と型が同じ） 5 === 5 true
!== 等しくない 5 !== 3 true
> より大きい 10 > 3 true
< より小さい 2 < 5 true
>= 以上 5 >= 5 true
<= 以下 3 <= 3 true

=== と == の違いに注意

== は値だけを比較し、型が違っても「等しい」と判断することがあります（例: 5 == "5" → true）。これは予期しないバグの原因になります。初心者のうちから ===（厳密等値） を使う習慣を付けましょう。

comparison.js 
// 比較演算子の結果を確認してみよう
const score = 75;

console.log(score === 75);  // true  ← 値と型が一致
console.log(score === "75"); // false ← 数値と文字列で型が違う
console.log(score >= 60);   // true  ← 75 は 60 以上
console.log(score < 60);    // false ← 75 は 60 より小さくない

5.3 if文の基本

条件分岐を書くには if 文を使います。基本的な形は次のとおりです。

if文の構文 
if (条件) {
    // 条件が true のときに実行される処理
} else if (別の条件) {
    // 最初の条件が false で、別の条件が true のとき
} else {
    // すべての条件が false のときに実行される処理
}

丸かっこ ( ) の中に条件を書き、波かっこ { } の中に実行したい処理を書きます。波かっこ内はインデント（字下げ）して読みやすくしましょう。

図: 点数による合否判定の流れ

flowchart TD A["点数を取得"] --> B{"score >= 60?"} B -->|"true（合格）"| C["「合格です」と表示"] B -->|"false（不合格）"| D["「不合格です」と表示"] C --> E["終了"] D --> E
admission.js 
// 年齢による入場可否の判定
const age = 20;

if (age >= 18) {
    // 18歳以上なら入場可
    console.log("入場できます");
} else {
    // 18歳未満は入場不可
    console.log("入場できません（18歳以上が対象です）");
}
grade.js 
// 点数による評価（else if で複数条件）
const score = 82;

if (score >= 90) {
    console.log("評価: A"); // 90点以上
} else if (score >= 70) {
    console.log("評価: B"); // 70点以上90点未満
} else if (score >= 50) {
    console.log("評価: C"); // 50点以上70点未満
} else {
    console.log("評価: D"); // 50点未満
}

else if は何個でも書ける

else if は必要なだけ続けて書けます。ただし上から順番に評価され、最初に true になった条件のブロックだけが実行されます。それ以降の条件は評価されません。

5.4 論理演算子（条件を組み合わせる）

「18歳以上かつ会員」のように、複数の条件を組み合わせたいときは論理演算子を使います。

&&

AND演算子 &&

両方の条件が true のときだけ true。「かつ」の意味。

||

OR演算子 ||

どちらか一方が true なら true。「または」の意味。

!

NOT演算子 !

true を false に、false を true に反転。「否定」の意味。

logic.js 
// 論理演算子の使用例
const age = 20;
const isMember = true;

// AND演算子: 18歳以上「かつ」会員の場合
if (age >= 18 && isMember) {
    console.log("割引が適用されます");
}

// OR演算子: 学生「または」シニアの場合
const isStudent = false;
const isSenior = true;
if (isStudent || isSenior) {
    console.log("特別料金が適用されます");
}

Mermaid図で論理の流れを整理しよう

条件が複雑になったら、頭の中でフロー図を描いてから書き始めると整理しやすくなります。AND演算子（かつ）は2つの条件が両方成立するかを確認し、OR演算子（または）はどちらか1つでも成立すればよいと覚えておきましょう。

flowchart TD A["開始"] --> B{"18歳以上?"} B -->|"true"| C{"会員?"} B -->|"false"| D["「対象外」と表示"] C -->|"true"| E["「割引適用」と表示"] C -->|"false"| F["「会員登録が必要」と表示"] D --> G["終了"] E --> G F --> G

やってみよう 5-1: 偶数・奇数を判定しよう

数値を変数に入れ、その数が偶数か奇数かを判定してコンソールに表示するプログラムを書いてみましょう。剰余演算子 %（割り算の余り）を使います。

手順

  1. 1 VSCode で新しいファイル odd-even.js を作成する。
  2. 2 const num = 7; のように好きな整数を変数に入れる。
  3. 3 num % 2 === 0 を条件に if 文を書き、偶数なら "偶数です"、奇数なら "奇数です" と表示する。
  4. 4 ブラウザの開発者ツール（またはターミナル）でコードを実行し、結果を確認する。num の値を変えて何度か試してみよう。
odd-even.js（ヒント） 
const num = 7; // ← ここを好きな数字に変えてみよう

if (num % 2 === 0) {
    // 2で割り切れる（余りが0）なら偶数
    console.log(num + " は偶数です");
} else {
    // それ以外は奇数
    console.log(num + " は奇数です");
}

理解度確認クイズ

  1. Q1. === と == の違いは何ですか？初心者がどちらを使うべきか理由とともに答えてください。

    回答を表示

    ===（厳密等値）は値と型の両方が一致するときだけ true を返します。一方、==（緩やかな等値）は型が違っても自動変換して比較するため、意図しない結果になることがあります（例: 5 == "5" → true）。バグを防ぐために、初心者は === を使う習慣を付けましょう。

  2. Q2. 以下のコードを実行すると何が表示されますか？理由も考えてみましょう。

    const score = 65;
if (score >= 90) {
    console.log("A");
} else if (score >= 70) {
    console.log("B");
} else if (score >= 50) {
    console.log("C");
} else {
    console.log("D");
}
    回答を表示

    C が表示されます。score は 65 なので、最初の条件 score >= 90 は false、次の score >= 70 も false（65 < 70）、3つ目の score >= 50 が true（65 >= 50）になり、"C" が表示されます。

  3. Q3. &&（AND演算子）と ||（OR演算子）の違いを日常的な言葉で説明してください。また、それぞれどんな場面に使いますか？

    回答を表示

    &&（AND演算子）は「かつ」の意味で、両方の条件がともに true のときだけ全体が true になります（例: 「18歳以上かつ会員」）。||（OR演算子）は「または」の意味で、どちらか一方でも true なら全体が true になります（例: 「学生または65歳以上」）。ANDは条件をより厳しくしたいとき、ORは条件を緩くしたいときに使います。
