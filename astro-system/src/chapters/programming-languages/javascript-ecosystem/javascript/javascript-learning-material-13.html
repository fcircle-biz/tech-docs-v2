小さな機能を組み合わせよう（ミニアプリ演習）

DOM 操作・イベント処理・フォーム入力・配列によるデータ管理と再描画——これまで個別に学んだ技術を1つの「アプリ」として組み合わせる練習をします。「カウンターアプリ」と「メモ表示アプリ」の2本立てで、次章からの ToDo アプリ総合演習への準備を整えましょう。

この章で学ぶこと

  • アプリ作りの基本パターン（HTML で土台 → 要素取得 → イベント登録 → データ更新 → 再描画）を説明できる
  • 変数で「現在の状態（カウント値）」を管理し、クリックのたびに画面へ反映できる
  • 配列に入力値を追加し、render 関数でリストを丸ごと再描画するパターンを使える
  • 空入力チェックなどの簡単なバリデーションをフォームに組み込める

13.1 これまでの知識をつなげる

第9〜12章では、Web ページを動かすための4つの技術をそれぞれ個別に学びました。

章 学んだ技術 できること

「状態」と「画面」を分けて考える

アプリ開発では「今どんなデータを持っているか（状態）」と「画面に何が表示されているか」を別々に管理します。状態（変数・配列）を正しく更新すれば、あとは画面を状態から再描画するだけ——この考え方が ToDo アプリ以降でも核心になります。

13.2 ミニアプリ①「カウンターアプリ」

「＋」ボタンで数値が増え、「－」ボタンで減るシンプルなカウンターを作ります。変数1つで現在値（状態）を管理し、ボタンを押すたびに変数を更新 → 画面に表示を反映するのが基本形です。

カウンターアプリの処理の流れ

flowchart TD A["ページ読み込み
count = 0 で初期化"] B{"ボタンをクリック"} C["＋ボタン
count++"] D["－ボタン
count--"] E["countDisplay.textContent = count
（画面を更新）"] A --> B B -->|"＋"| C B -->|"－"| D C --> E D --> E E --> B

ステップ1: HTML の土台を作る

まず、ボタンと数値を表示するエリアだけを HTML で用意します。JavaScript からアクセスしやすいよう id を付けておくのがポイントです。

counter.html（body部分） 
<!-- カウンター表示エリア -->
<p id="countDisplay">0</p>

<!-- ＋ / － ボタン -->
<button id="btnPlus">＋</button>
<button id="btnMinus">－</button>

<!-- JavaScript を読み込む -->
<script src="counter.js"></script>

ステップ2: 要素を取得してイベントを登録する

JavaScript 側では、まず3つの要素を取得します。次に各ボタンへクリックイベントを登録します。

counter.js（要素取得＋イベント登録） 
// 要素を取得する（第9章のおさらい）
const countDisplay = document.getElementById('countDisplay');
const btnPlus  = document.getElementById('btnPlus');
const btnMinus = document.getElementById('btnMinus');

// ＋ボタンのイベント登録（第10章のおさらい）
btnPlus.addEventListener('click', function() {
  count++;             // カウントを増やす
  updateDisplay();     // 画面を更新する
});

// －ボタンのイベント登録
btnMinus.addEventListener('click', function() {
  count--;             // カウントを減らす
  updateDisplay();     // 画面を更新する
});

ステップ3: 状態（count）を管理して再描画する

カウント値を変数 count で持ち、表示更新を updateDisplay 関数にまとめます。関数を1か所にまとめると、後から修正しやすくなります。

counter.js（状態管理＋再描画） 
// 現在のカウント値（状態）を変数で管理する
let count = 0;

// 表示を更新する関数（状態 → 画面に反映）
function updateDisplay() {
  countDisplay.textContent = count;
}

// ページ読み込み時に初期表示する
updateDisplay();

重要ポイント: 「変数を更新 → 表示関数を呼ぶ」を貫く

直接 countDisplay.textContent = count + 1 のように書くと、変数と表示が食い違いやすくなります。必ず変数を先に書き換えて、表示関数を呼ぶ流れを守りましょう。この習慣がアプリが大きくなっても崩れない設計の基礎です。

13.3 ミニアプリ②「メモ表示アプリ」

テキストボックスに入力した文字を「追加」ボタンで配列にためて、一覧として表示するアプリです。第12章の render パターン（配列 → innerHTML で丸ごと再描画）をそのまま使います。空文字チェックも加えて、実用的な入力バリデーションを体験します。

メモ表示アプリの処理の流れ

flowchart TD A["「追加」ボタンをクリック"] B{"入力欄は空?"} C["何もしない
（空入力チェック）"] D["memos.push（入力値）
配列にメモを追加する"] E["入力欄をクリアする"] F["renderMemos（）を呼ぶ
一覧を再描画する"] A --> B B -->|"空"| C B -->|"空でない"| D D --> E --> F

ステップ1: HTML の土台を作る

memo.html（body部分） 
<!-- テキスト入力欄（第11章のおさらい） -->
<input type="text" id="memoInput" placeholder="メモを入力">

<!-- 追加ボタン -->
<button id="addBtn">追加</button>

<!-- メモ一覧の表示エリア -->
<ul id="memoList"></ul>

<script src="memo.js"></script>

ステップ2: 配列でメモを管理し、render 関数を作る

memos 配列にメモ文字列をためていきます。renderMemos 関数は配列を見て ul 要素の中身を毎回丸ごと作り直します（第12章のパターン）。

memo.js（配列管理＋render 関数） 
// メモを保持する配列（状態）
const memos = [];

const memoList = document.getElementById('memoList');

// 配列の内容を元に一覧を再描画する関数
function renderMemos() {
  memoList.innerHTML = '';        // 一度リストをクリア
  memos.forEach(function(memo) {
    const li = document.createElement('li'); // li 要素を作る
    li.textContent = memo;        // メモ文字列をセット
    memoList.appendChild(li);     // リストに追加
  });
}

ステップ3: 追加ボタンに空入力チェックを入れる

入力欄が空のときは配列に追加しません。trim() を使うと、スペースだけの入力も弾けます。

memo.js（追加ボタン＋バリデーション） 
const memoInput = document.getElementById('memoInput');
const addBtn    = document.getElementById('addBtn');

addBtn.addEventListener('click', function() {
  const text = memoInput.value.trim(); // 前後の空白を除去

  // 空文字チェック（バリデーション）
  if (text === '') return;    // 空なら何もしない

  memos.push(text);           // 配列に追加（状態を更新）
  memoInput.value = '';       // 入力欄をクリア
  renderMemos();              // 画面を再描画
});

trim() でスペースだけの入力を防ぐ

memoInput.value はスペースキーを押しただけでも値が入ります。.trim() を使うと文字列の前後の空白をまとめて削除できるので、「スペースを入力して追加ボタンを押した」場合もきちんと弾けます。

やってみよう 13-1: 発展課題に挑戦する

2つのミニアプリが動いたら、以下の発展課題に挑戦してみましょう。どちらもこれまでの知識だけで実現できます。

カウンターに「リセット」ボタンを追加する

  1. 1 HTML に <button id="btnReset">リセット</button> を追加する。
  2. 2 JS で btnReset を取得し、クリック時に count = 0 → updateDisplay() を呼ぶ。
  3. 3 ボタンを押したら表示が 0 に戻ることを確認する。

メモアプリで件数を表示する

  1. 1 HTML に <p id="memoCount"></p> を追加する。
  2. 2 renderMemos 関数の中で memoCount.textContent = 'メモ件数: ' + memos.length を更新する。
  3. 3 メモを追加するたびに件数が増えることを確認する。

理解度確認クイズ

  1. Q1. アプリ作りの基本パターンを5つのステップで答えてください。

    回答を表示

    ① HTML で土台を作る（ボタン・入力欄・表示エリアに id を付ける）→ ② JS で要素を取得する → ③ イベントを登録する → ④ データ（変数や配列）を更新する → ⑤ 画面を再描画する。ユーザー操作のたびに④⑤が繰り返されます。

  2. Q2. カウンターアプリで、なぜ表示を直接書き換えるのではなく「変数を更新してから表示関数を呼ぶ」のですか？

    回答を表示

    「状態（変数）」と「表示（画面）」を常に一致させるためです。変数を正しく更新してから表示関数を呼ぶことで、どこかで表示だけが変わったり、変数だけが変わって表示が古いまま、という不整合を防げます。

  3. Q3. メモ表示アプリで memos.push(text) の後に必ず renderMemos() を呼ぶのはなぜですか？

    回答を表示

    配列（状態）を変えただけでは画面は自動で更新されないからです。renderMemos() は配列の現在の内容をもとに一覧を丸ごと作り直します。状態を変えたら必ず再描画関数を呼ぶことで、配列と画面を常に同期させられます。
