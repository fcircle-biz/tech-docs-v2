ToDoアプリを作ろう② タスクを追加・表示する

第14章で作った画面に、タスクを追加・表示する機能を実装します。配列をデータの「真実源」として管理し、データが変わるたびに画面を再描画する「データ→画面反映」の基本サイクルを体験しましょう。

この章で学ぶこと

  • 配列でタスクデータを管理する意味と方法を理解する
  • getElementById で要素を取得し、クリックイベントを登録できる
  • 入力値の取得・バリデーション・配列への追加を実装できる
  • render 関数を作り、配列の内容を一覧として画面に表示できる
  • 「入力→追加→再描画」の1サイクルを通して、データ変更が画面に反映される流れをつかむ

15.1 タスクを配列で管理する

アプリを作るとき、「画面に何を表示するか」は常にデータが決めます。ToDoアプリでは、タスクの文字列を配列に入れて管理します。この配列こそが「唯一の真実」です。

第12章で学んだ「データと画面を分ける」考え方を思い出しましょう。HTMLに直接タスクを書くのではなく、JavaScript の配列にデータを持たせ、その配列から画面を生成します。こうすることで、追加・削除・変更をしても「配列を更新してから再描画」するだけで済みます。

データ（配列）

タスクの文字列を配列に保持します。これが唯一の真実源です。

render関数

配列を読み取り、HTMLを生成して画面に書き出します。

画面（DOM）

render関数が生成したHTMLが表示されます。直接触りません。

まず、タスクを入れる配列を宣言します。最初は空の配列で構いません。

todo.js（抜粋） 
// タスクを管理する配列（唯一のデータ源）
let tasks = [];

// 例: タスクが2件ある状態のイメージ
// tasks = ["牛乳を買う", "メールを送る"];

なぜ let を使うの？

tasks は後から要素を追加するため、再代入可能な let で宣言します。ただし配列そのものを別の配列に入れ替えることはないので、const tasks = []; と書いても push は使えます。第16章以降では const に変更することも検討してみましょう。

15.2 要素の取得とイベント登録

第14章で作った HTML には、入力欄・追加ボタン・タスク一覧に id が付いています。getElementById でそれぞれを取得し、追加ボタンに addEventListener でクリック時の処理を登録します。

todo.js（抜粋） 
// HTML の id で各要素を取得する
const input    = document.getElementById("task-input");   // 入力欄
const addBtn   = document.getElementById("add-btn");      // 追加ボタン
const taskList = document.getElementById("task-list");    // タスク一覧

// 追加ボタンがクリックされたら addTask を呼ぶ
addBtn.addEventListener("click", addTask);

id は第14章で付けたもの

第14章の HTML では、入力欄に id="task-input"、ボタンに id="add-btn"、リストに id="task-list" を付けています。JavaScript から操作するための「名札」です。

15.3 タスクを追加する（addTask 関数）

追加ボタンが押されたときに呼ばれる addTask 関数を実装します。処理の流れは「入力値を取得 → 空ならスキップ → 配列に追加 → 入力欄をリセット → 画面を再描画」の5ステップです。

todo.js（抜粋） 
function addTask() {
    const text = input.value.trim(); // 前後の空白を除去して取得

    // 空文字ならスキップ（早期リターン）
    if (text === "") return;

    tasks.push(text);   // 配列にタスクを追加
    input.value = "";   // 入力欄を空に戻す
    render();           // 画面を再描画
}

早期リターンでコードをシンプルに

if (text === "") return; は「条件を満たさないときはすぐ終了する」書き方（ガード節）です。else でネストを深くするよりも読みやすくなります。.trim() を使うと空白だけの入力も弾けます。

15.4 一覧を表示する（render 関数）

render 関数は「配列の内容を読んで、画面の一覧エリアを丸ごと作り直す」役割を担います。ポイントは 表示前に innerHTML を空にしてからループすることです。こうしないと、追加するたびに重複して表示されてしまいます。

todo.js（抜粋） 
function render() {
    taskList.innerHTML = ""; // 一覧を空にしてからループ

    tasks.forEach(function(task) {
        const li = document.createElement("li"); // li 要素を作成
        li.textContent = task;                   // タスク文字列をセット
        taskList.appendChild(li);               // 一覧に追加
    });
}

なぜ innerHTML を空にするの？

render は「今の配列の状態を画面に反映する」関数です。毎回ゼロから作り直すことで、追加・削除・変更のすべてに対応できます。「追加専用の処理を別に書く」方法よりシンプルで、バグも少なくなります。

15.5 流れの確認: 1サイクルを通して理解する

実装した処理を「入力→追加→再描画」という1つの流れとして確認しましょう。このサイクルが理解できれば、第16章の完了・削除機能も同じ考え方で追加できます。

flowchart TD A["① 入力欄にタスクを入力"] --> B["② 追加ボタンをクリック"] B --> C["③ addTask が呼ばれる"] C --> D{"空文字チェック"} D -->|空なら| E["処理をスキップ（return）"] D -->|文字があれば| F["tasks.push でデータを追加"] F --> G["input.value を空にリセット"] G --> H["render を呼ぶ"] H --> I["innerHTML を空にクリア"] I --> J["tasks.forEach で li を生成・追加"] J --> K["④ 一覧に新しいタスクが表示される"]

図を見ると、データ（配列）が変わる → render が呼ばれる → 画面が更新されるという一方向の流れが分かります。「画面を直接書き換える」のではなく、「データを更新したら必ず render を呼ぶ」ことがポイントです。

ここまでの処理をまとめると、todo.js 全体はこのようになります。

todo.js（全体） 
let tasks = []; // タスクを管理する配列

const input    = document.getElementById("task-input");
const addBtn   = document.getElementById("add-btn");
const taskList = document.getElementById("task-list");

addBtn.addEventListener("click", addTask); // クリックで addTask を実行

function addTask() {
    const text = input.value.trim();
    if (text === "") return;  // 空ならスキップ
    tasks.push(text);         // 配列に追加
    input.value = "";         // 入力欄をリセット
    render();                 // 画面を更新
}

function render() {
    taskList.innerHTML = "";  // 一覧をクリア
    tasks.forEach(function(task) {
        const li = document.createElement("li");
        li.textContent = task;
        taskList.appendChild(li);
    });
}

この章で最も大切な考え方

「データ（配列）を変えたら必ず render を呼ぶ」というルールを守ることで、画面は常に配列と一致した状態になります。第16章で完了・削除機能を追加するときも、同じルールで拡張できます。

やってみよう 15-1: タスクを実際に追加してみよう

第14章で作った HTML ファイルに todo.js を読み込み、タスクを追加して画面に表示されることを確認しましょう。

手順

  1. 1 第14章の HTML ファイルを開き、</body> 直前に <script src="todo.js"></script> を追加する。
  2. 2 同じフォルダに todo.js を作成し、15.5節の全体コードを貼り付ける。
  3. 3 ブラウザで開き、入力欄にタスクを入力して追加ボタンを押す。一覧に表示されることを確認する。
  4. 4 空のまま追加ボタンを押しても一覧が変化しないことを確認する。

発展チャレンジ

  • Enterキーでも追加できるようにする: 入力欄に keydown イベントを追加し、event.key === "Enter" のときに addTask() を呼ぶ。
  • タスク件数を表示する: render の中で tasks.length を使って「現在 ○ 件」と表示する要素を更新する。
  • タスクが0件のとき「タスクはありません」と表示する: render の中で tasks.length === 0 を確認してメッセージを出す。

理解度確認クイズ

  1. Q1. タスクデータを HTML に直接書くのではなく、JavaScript の配列で管理するのはなぜですか？

    回答を表示

    配列をデータの「唯一の真実源」にすることで、追加・削除・変更のどの操作も「配列を更新してから render を呼ぶ」だけで対応できるからです。HTML を直接書き換えると操作ごとに個別の処理が必要になり、コードが複雑になります。

  2. Q2. render 関数の最初に taskList.innerHTML = "" を書く理由を説明してください。

    回答を表示

    一覧をいったん空にしてからループで再生成するためです。空にしないと、追加するたびに既存のリスト項目の後ろに新しい項目が重複して追加されてしまいます。毎回ゼロから作り直すことで、常に配列の現在の内容だけが表示されます。

  3. Q3. 「入力→追加ボタンクリック→再描画」の流れで、データ（配列）と画面（DOM）はそれぞれいつ更新されますか？

    回答を表示

    データ（配列）は addTask 内の tasks.push(text) で更新されます。画面（DOM）はその直後に呼ばれる render() の中で更新されます。データを先に変え、その後で画面を反映するという順序が重要です。
