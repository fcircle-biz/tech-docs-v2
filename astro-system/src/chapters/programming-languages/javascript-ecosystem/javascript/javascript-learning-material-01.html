プログラミングとJavaScriptを知ろう

プログラミングとは何か、JavaScript で何ができるのかを、身近な例を使って分かりやすく説明します。この章では、コードを書く前に「そもそもプログラムとは何をするものなのか」「JavaScript はどんな場面で活躍するのか」という全体像をつかみ、コンピュータと対話する第一歩を踏み出します。

この章で学ぶこと

  • プログラミングとは何か、なぜ必要なのかを自分の言葉で説明できる
  • JavaScript がどんな言語で、どこで動くのかを理解する
  • JavaScript でできることの具体例を知る
  • これから学ぶ学習の進め方（全体マップ）をイメージできる

1.1 プログラミングとは何か

プログラミングとは、ひとことで言えば「コンピュータに、やってほしい仕事の手順を順番どおりに教えること」です。コンピュータはとても速く正確に計算できますが、自分で考えて動くことはできません。「何を」「どんな順番で」やればよいかを、人間が細かく指示してあげる必要があります。この指示書のことをプログラム、指示を書くための言葉をプログラミング言語と呼びます。

身近なたとえで考えてみましょう。料理のレシピは「材料を切る → 炒める → 味付けする」のように手順が順番に書かれています。レシピのとおりに進めれば、誰が作っても同じ料理ができあがります。プログラムもこれと同じで、コンピュータはプログラムに書かれた手順を上から順番に実行していきます。

手順を書く

「何を、どの順番で」を1つずつ指示します。あいまいな指示はうまく動きません。

正確で高速

同じ作業を、何千回でも疲れず正確に繰り返してくれます。

指示どおりに動く

良くも悪くも、書いたとおりに動きます。だから手順が大切です。

用語: 「コード」と「プログラム」

プログラミング言語で書いた文章のことをコード（ソースコード）と呼びます。そして、ひとまとまりの仕事をするコードの集まりがプログラムです。「コードを書く＝プログラムを作る」とイメージしておけば大丈夫です。

1.2 JavaScript とはどんな言語か

JavaScript（ジャバスクリプト）は、世界中で最も広く使われているプログラミング言語の1つです。最大の特徴は、Webブラウザ（Chrome や Edge など）の中で動くという点です。みなさんが普段見ている Web ページの「ボタンを押すと表示が変わる」「入力に応じてメッセージが出る」といった動きの多くは、JavaScript で作られています。

Web ページは、大きく3つの技術が役割を分担して作られています。JavaScript はその中で「動き」を担当します。下の表でそれぞれの役割を確認しましょう。

技術 役割 たとえ
HTML 文章・見出し・画像などの「構造」を作る 建物の骨組み
CSS 色・大きさ・配置などの「見た目」を整える 内装・デザイン
JavaScript クリックや入力に反応する「動き」を加える 電気・自動ドアなどの仕掛け

つまり、HTML と CSS だけだと「見るだけ」のページですが、JavaScript を加えることで「触ると反応する、動くページ」になります。本ガイドでは、まず JavaScript 単体の基礎（第1〜8章）をしっかり学び、その後に Web ページを動かす方法（第9章以降）へ進みます。

注意: 「JavaScript」と「Java」は別物

名前がよく似ていますが、JavaScript と Java はまったく別のプログラミング言語です。歴史的な事情で名前が似ているだけで、用途も書き方も異なります。これから学ぶのは「JavaScript」のほうだと覚えておきましょう。

1.3 JavaScript でできること

JavaScript は、Web ページにさまざまな「動き」や「便利な機能」を加えられます。普段みなさんが使っている Web サイトにも、たくさんの JavaScript が使われています。代表的な例を見てみましょう。

  • 表示の切り替え — ボタンを押すとメニューが開く、画像が切り替わる
  • 入力チェック — フォームに「メールアドレスを正しく入力してください」と表示する
  • 計算・集計 — ショッピングカートの合計金額を自動で計算する
  • データの一覧表示 — ToDo リストや検索結果を画面に並べて表示する
  • 地図やグラフ — マウス操作で動く地図、リアルタイムに変わるグラフ

本ガイドの最後（第14〜16章）では、これらの基礎を組み合わせて、実際に動く「ToDo アプリ」を一緒に作ります。今はまだ難しそうに見えても、1章ずつ進めれば必ず作れるようになります。

重要ポイント

JavaScript の大きな魅力は、特別なソフトを買わなくても、ブラウザさえあればすぐに試せることです。書いたコードがその場で画面に反映されるので、「動かしながら学ぶ」のにとても向いた言語です。

本ガイドは、全16章を通して「JavaScript の基礎」から「動く Web アプリ作り」までを、無理のない順番で積み上げていきます。大きく4つのステップに分かれています。全体像を頭に入れておくと、今どこを学んでいるのかが分かりやすくなります。

学習のコツ

各章には「やってみよう」（実習）と「理解度確認クイズ」があります。読むだけでなく、必ず自分の手でコードを書いて動かしてみましょう。エラーが出ても大丈夫です。エラーは「どこを直せばよいか」を教えてくれる大切なヒントです。

やってみよう 1-1: 身のまわりの JavaScript を探す

コードを書く前に、まずは「JavaScript がどこで使われているか」を意識してみましょう。身近な Web サイトを観察することが、学習の第一歩になります。

手順

  1. 1 いつも使っている Web サイトを1つ開く（ニュース、ショッピング、SNS など）。
  2. 2 「クリックすると動く部分」「入力に反応する部分」を3つ探してメモする（例: メニューが開く、カートの金額が変わる）。
  3. 3 そのうちどれかを「自分でも作れるようになりたい」目標として書き留めておく。

理解度確認クイズ

  1. Q1. プログラミングとは何かを、料理にたとえて説明してください。

    回答を表示

    プログラミングは、料理のレシピのように「やってほしい仕事の手順を、順番どおりにコンピュータへ教えること」です。コンピュータはレシピ（プログラム）のとおりに上から順番に実行します。

  2. Q2. Web ページを作る HTML・CSS・JavaScript は、それぞれどんな役割を担当しますか？

    回答を表示

    HTML は文章や画像などの「構造」、CSS は色や配置などの「見た目」、JavaScript はクリックや入力に反応する「動き」を担当します。3つが役割分担して1つの Web ページを作ります。

  3. Q3. JavaScript を学ぶうえで「特別なソフトを買わなくてもすぐ試せる」のはなぜですか？

    回答を表示

    JavaScript は Web ブラウザの中で動く言語だからです。ブラウザさえあれば書いたコードをその場で実行・確認でき、「動かしながら学ぶ」ことができます。
