ホーム Codex 第4章
第4章 所要時間: 約75分

基本操作と最初のセッション

インストールが完了したら、いよいよ Codex と対話してみましょう。この章では、codex コマンドでの起動方法からターミナル UI（TUI）の見方、プロンプトの送り方、処理の中断・再開まで、日々の操作の基本を学びます。最初の簡単な依頼を出して、AI がどのように作業を進めるかを観察しましょう。

この章で学ぶこと

  • codex コマンドでセッションを起動し、ターミナル UI の各部を説明できる
  • プロンプト（指示）を送り、AI の応答・進行状況を読み取ることができる
  • 処理を途中で中断する方法と、その後の選択肢を理解する
  • codex resume でセッションを再開する流れを理解する

4.1 Codex の起動

第3章でインストールと初期設定が完了していれば、ターミナルで codex と入力するだけでセッションが始まります。まずは起動の基本形を確認しましょう。

terminal — 基本的な起動コマンド 
# 作業したいプロジェクトのフォルダへ移動してから起動する
cd ~/my-project

# 基本形: Codex を対話モードで起動
codex

# 最初から指示を渡して起動することもできる
codex "README を読んで、このプロジェクトの概要を教えて"

起動前に 作業対象のフォルダへ移動（cd）してから codex を実行することが重要です。Codex は起動したカレントディレクトリを「作業場所」として認識し、そのフォルダ内のファイルを読み書きします。

ヒント: プロジェクトフォルダで起動する理由

Codex はカレントディレクトリを起点にファイルを探します。プロジェクトのルートフォルダで起動することで、関連ファイルをすべて参照できる状態になります。ホームディレクトリなど無関係な場所から起動しても動作しますが、正しいファイルを見つけられない原因になります。

主な起動オプション

オプション 意味 例
（なし） 対話モードで起動（最もよく使う） codex
"プロンプト" 最初の指示を渡して起動 codex "〇〇して"
--model 使用するモデルを指定（/model でも切替可） codex --model gpt-5.5
--ask-for-approval, -a 承認ポリシーを指定（第7章で詳説） codex -a on-request
--sandbox, -s サンドボックスモードを指定（第7章で詳説） codex -s read-only

4.2 ターミナル UI（TUI）の見方

Codex が起動すると、ターミナルに専用の対話画面（TUI: Terminal User Interface）が表示されます。画面は大きく3つのエリアに分かれています。

flowchart TD A["ヘッダーエリア
（モデル名・承認/サンドボックス・作業ディレクトリ）"] B["会話エリア
（あなたの入力と AI の応答が流れる）"] C["ステータスエリア
（AI が今何をしているか・ツール使用の進捗）"] D["入力エリア
（次のプロンプトをここに入力する）"] A --> B --> C --> D

Codex TUI の画面構成（イメージ）

ヘッダーエリア

使用中のモデル名（例: gpt-5.5）、現在の承認ポリシー・サンドボックス、作業ディレクトリが表示されます。設定確認に使います。

会話エリア

あなたの入力と AI の応答がここに表示されます。Markdown 形式で整形されることが多く、コードブロックや箇条書きも表示されます。

ステータスエリア

AI がファイルを読む・コマンドを実行するなどの「道具の使用」の状況がリアルタイムで流れます。AI が今何をしているか確認できます。

入力エリア

次のプロンプトをここに入力します。複数行の入力も可能で、Enter で送信します。

覚えておきたいキー操作

キー 動作
Enter プロンプトを送信する
Ctrl + C 実行中の処理を中断する（詳しくは 4.4 節）
Ctrl + D セッションを終了してターミナルに戻る
↑ / ↓ 入力履歴を前後に移動する

4.3 プロンプトの送り方と応答の読み方

Codex への指示（プロンプト）は、ふつうの日本語や英語で書いて Enter を押すだけです。送信後、AI は「計画 → ファイルの読み込み → 編集・実行 → 報告」という流れで作業を進め、ステータスエリアに進捗が流れます。

terminal — セッションの例（応答の流れ） 
$ codex
> このプロジェクトの README を読んで、概要を3行でまとめて

  ● README.md を読み込んでいます...         ← ステータス（ツール使用）
  ● 分析中...

  このプロジェクトは〇〇を行う Python ライブラリです。
  主な機能は〇〇と〇〇で、pip でインストールできます。
  対応バージョンは Python 3.9 以上です。

> ありがとう。次は src/main.py のバグを直して
  ...

応答の読み方のポイント

AI の応答には、作業の進捗と最終的な報告の2種類があります。それぞれの読み方を押さえましょう。

ステータス行（作業中）

「● ファイル名 を読み込んでいます」「● コマンドを実行中」などの行が流れます。AI が今どの作業をしているかが分かります。

これらは処理中のログであり、最終報告ではありません。

最終報告（完了時）

すべての作業が終わると、AI が自然言語で結果を説明します。変更点や確認すべき点が整理されて表示されます。

ここで「変更を適用しますか？」と確認を求める場合もあります。

重要ポイント: 承認プロンプトに気を付ける

承認モードが Auto 以外（デフォルト）の場合、AI がファイルを編集したりコマンドを実行する前に「承認しますか？ y/n」と確認を求めます。内容をよく読んでから y か n を入力してください。

初めてのプロンプトに向く依頼

最初のセッションでは、小さく具体的な依頼から始めると AI の動きが分かりやすくなります。

向いている依頼（小さく・具体的）

  • 「README.md を読んで概要を教えて」
  • 「src/utils.py にある関数の一覧を教えて」
  • 「このフォルダ構成を説明して」

慣れてから試す依頼（大きい・曖昧）

  • 「このプロジェクト全体をリファクタリングして」
  • 「なんかいい感じにして」
  • 「バグを全部直して」

4.4 処理の中断

AI が処理している最中に「やっぱり止めたい」と思ったときは、Ctrl + C を押すことで処理を中断できます。中断後は、そのまま別のプロンプトを入力して新しい指示を出すか、セッション自体を終了するかを選べます。

terminal — 処理の中断例 
> 全ファイルを読み込んで詳細な解析レポートを作って
  ● src/ を読み込んでいます...
  ● tests/ を読み込んでいます...
^C                                   ← Ctrl+C で中断
  処理を中断しました。
> （次のプロンプトを入力するか、Ctrl+D で終了）

注意: 中断しても既に行われた変更は残る

Ctrl+C で中断しても、中断前にすでに実行された編集・コマンドは取り消されません。承認が必要な操作については承認前に止まりますが、Auto モードで動かしているときは特に注意が必要です。不安な場合は Git で変更を管理しておきましょう（第10章）。

中断後にできること

別の指示を出す

そのままプロンプトを入力して、別の作業を依頼できます。同じセッションが継続されます。

指示を修正して再試行

プロンプトの表現を変えて、より具体的に指示し直すと良い結果が出ることがあります。

セッションを終了

Ctrl+D または exit でターミナルに戻ります。

4.5 セッションの再開（codex resume）

Codex はセッションの会話履歴を保存します。一度セッションを終了したあとでも、codex resume コマンドを使うことで前回の続きから再開できます。長い作業を複数回に分けて進めるときに便利です。

terminal — セッションの再開 
# 最後のセッションを再開する（最もよく使う）
codex resume

# 特定のセッション ID を指定して再開する
codex resume --session-id <セッションID>

# 過去のセッション一覧を表示する
codex sessions
flowchart LR A["codex を起動"] --> B["作業・対話"] B --> C["Ctrl+D でセッション終了"] C --> D["履歴が自動保存される"] D --> E["codex resume で再開"] E --> B

セッションの終了と再開の流れ

ヒント: セッション履歴の保存先

セッションの履歴は ~/.codex/sessions/ ディレクトリに保存されます（設定で変更可能）。再開時には AI が前回の会話内容を記憶した状態から続きを始められます。コンテキストの詳しい管理方法は第8章で学びます。

やってみよう 4-1: 最初のセッションを体験する

実際に codex を起動して、AI とやり取りしてみましょう。作業対象のプロジェクトがなければ、練習用のフォルダを作って試せます。

手順

  1. 1 練習フォルダを作り、移動する。
    mkdir ~/codex-practice && cd ~/codex-practice
  2. 2 codex を起動し、TUI の各エリアを確認する。
  3. 3 「hello.py という Python ファイルを作って、Hello, Codex! と出力するコードを書いて」と入力する。
  4. 4 ステータスエリアの流れと最終報告を観察し、承認を求められたら y を押す。
  5. 5 Ctrl+D でセッションを終了し、codex resume で再開できることを確認する。

確認ポイント

  • TUI の4つのエリア（ヘッダー・会話・ステータス・入力）が確認できた
  • AI がファイルを作成し、hello.py が生成された
  • codex resume でセッションが再開し、前回の会話が表示された

理解度確認クイズ

  1. Q1. Codex を起動する前に、まず行うべき操作は何ですか？

    回答を表示

    作業したいプロジェクトのフォルダへ移動（cd）してから codex を実行します。Codex は起動したカレントディレクトリを「作業場所」として認識するため、正しいフォルダで起動することが重要です。

  2. Q2. Codex の TUI（ターミナル UI）は大きく何つのエリアに分かれていますか？それぞれの役割を説明してください。

    回答を表示

    4つのエリアに分かれています。①ヘッダーエリア（モデル名・承認モード・作業ディレクトリを表示）、②会話エリア（入力と AI の応答が流れる）、③ステータスエリア（AI が今何をしているか・ツール使用の進捗が表示される）、④入力エリア（次のプロンプトを入力する場所）です。

  3. Q3. セッションを終了した後、前回の続きから再開するにはどのコマンドを使いますか？

    回答を表示

    codex resume コマンドを使います。特定のセッションを再開するには codex resume --session-id <ID> で指定できます。また codex sessions で過去のセッション一覧を確認できます。
