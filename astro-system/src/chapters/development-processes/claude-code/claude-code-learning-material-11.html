MCPによる外部ツール連携

Model Context Protocol（MCP）を使うと、Claude Code の能力をデータベース・外部 API・ブラウザ操作など、さまざまな外部ツールへ拡張できます。この章では MCP とは何か、どのように接続・活用するのかを学びます。

この章で学ぶこと

  • MCP（Model Context Protocol）が何のためのプロトコルかを説明できる
  • クライアント（Claude Code）と MCPサーバーの関係を図で理解する
  • MCPサーバーの追加・確認の基本操作を理解する
  • 代表的な MCP 活用例と、利用時のセキュリティ上の注意を把握する

11.1 MCP（Model Context Protocol）とは

MCP（Model Context Protocol） は、AI アシスタントと外部のツール・データソースを安全に接続するためのオープンな標準プロトコルです。Anthropic が提唱し、オープンソースで公開されています（modelcontextprotocol.io）。

プロトコルとは「共通のやり取りのルール」です。MCP はこのルールを定めることで、Claude Code（クライアント）がさまざまな外部サービスと共通の方法で通信できるようにしています。

オープン標準

特定のサービスに縛られず、MCP に対応したツールならどれとでも接続できる共通規格です。

能力を拡張

データベース、外部 API、ブラウザ操作、社内ツールなど、Claude Code 単体では扱えない機能を後付けで追加できます。

用語：「プロトコル」とは

HTTP が Web ブラウザとサーバーの通信ルールを定めているように、MCP は「AI と外部ツールの通信ルール」を定めています。USB のように「この規格に対応していればつながる」という仕組みをイメージするとわかりやすいです。

11.2 クライアントとMCPサーバーの関係

MCP の仕組みは、大きく3つの役割で成り立っています。

クライアント

Claude Code がこの役割です。「何かをして」と MCPサーバーに依頼します。

MCPサーバー

外部ツールへの橋渡し役です。MCP プロトコルに沿ってリクエストを受け取り、処理して結果を返します。

データソース・外部ツール

データベース、GitHub、Web ブラウザなど、実際の処理を担うサービスや情報源です。

3つの関係をMermaid図で表すと次のようになります。

flowchart LR A["Claude Code
（クライアント）"] -->|"MCP プロトコルで依頼"| B["MCPサーバー"] B -->|"API・コマンドで操作"| C["外部ツール ＆
データソース"] C -->|"結果を返す"| B B -->|"結果を返す"| A

Claude Code は「どんなデータベースか」「どんな API か」を細かく知らなくても、MCPサーバーに「〇〇のデータを取得して」と依頼するだけで作業を進められます。複雑な接続の詳細は MCPサーバーが隠蔽してくれます。

重要ポイント

MCPサーバーは Claude Code と別プロセスで動くため、外部ツールへの接続をサーバー側だけに閉じ込められます。Claude Code 本体の設定を変えることなく、接続できるツールを柔軟に追加・削除できます。

11.3 MCPサーバーの追加と確認

Claude Code には MCPサーバーを追加するためのコマンド claude mcp add が用意されています。書き方は接続方式（トランスポート）によって異なります。リモートのサービスに接続する場合は HTTP が推奨、自分の PC でツールを動かす場合は stdio を使うのが基本です。

terminal — MCPサーバー追加の例（トランスポート別） 
# ① リモートの HTTP サーバーを追加する（リモート接続の推奨方式）
claude mcp add --transport http <名前> <URL>
# 例: claude mcp add --transport http notion https://mcp.notion.com/mcp

# ② ローカルの stdio サーバーを追加する（-- の後に起動コマンドを書く）
claude mcp add <名前> -- <起動コマンド> [引数...]
# 例: claude mcp add airtable -- npx -y airtable-mcp-server

# 現在接続中の MCPサーバーを確認する（セッション中のスラッシュコマンド）
/mcp

追加したサーバーはセッションをまたいで記憶されます。接続状況を確認したいときは、Claude Code 起動中に /mcp と入力すると、現在登録されている MCPサーバーの一覧と状態を確認できます。

トランスポートは4種類。リモートは原則 HTTP

MCP の接続方式には HTTP・stdio・SSE・WebSocket があります。

  • HTTP（推奨）: クラウド上のリモートサービス向け。--transport http
  • stdio: ローカルで動かすサーバー向け。-- の後に起動コマンドを書く
  • SSE: 非推奨（deprecated）。可能な限り HTTP を使う
  • WebSocket: --transport では指定できず、claude mcp add-json で設定する

引数や認証ヘッダーの詳細は、利用するサーバーの公式ドキュメントを参照してください。

MCPサーバーの設定は Claude Code の設定ファイル（settings.json など）にも記録されます。第10章で学んだ設定管理の仕組みと組み合わせて管理できます。

11.4 代表的なMCP活用例

MCP に対応したサーバーは、公式・サードパーティを含めてさまざまなものが公開されています。代表的な活用パターンを見ていきましょう。

GitHub 連携

GitHub の MCPサーバーを追加すると、Claude Code から Issue の作成・確認、Pull Request の操作などを自然言語で依頼できます。

例: 「このブランチの PR をドラフトで作成して」

データベース連携

PostgreSQL・SQLite などのDBに接続する MCPサーバーを使うと、スキーマの確認やクエリの実行を Claude Code から行えます。

例: 「usersテーブルの構造を教えて」

ブラウザ自動化

Playwright などのブラウザ自動化ツールと連携する MCPサーバーを使うと、Web ページの操作やスクリーンショット取得を指示できます。

例: 「このページのログイン画面を確認して」

社内ツール連携

社内の API やナレッジベースに接続するカスタム MCPサーバーを自分で実装すれば、組織固有のシステムとも連携できます。

例: 「社内Wikiの〇〇ページを参照して実装して」

MCPサーバーのリポジトリ

Anthropic が管理する公式サーバー一覧やコミュニティ製サーバーは、MCP の公式サイト（modelcontextprotocol.io）や GitHub 上で探すことができます。まずは用途に合ったサーバーを探してみましょう。

11.5 セキュリティ上の注意

MCPサーバーは外部ツールへのアクセスを Claude Code に与えます。便利な一方で、信頼できるサーバーだけを追加することが重要です。

MCPサーバー追加時の注意点

  • 出所不明なサーバーは追加しない: 悪意のある MCPサーバーは、データの読み取りや意図しない操作を行う可能性があります。
  • 権限の範囲を確認する: どのリソースへのアクセス権をサーバーに与えているか把握しておきましょう。
  • 不要になったら削除する: 使わなくなった MCPサーバーは接続を解除し、不必要な接続を残さないようにしましょう。

公式のオープンソースサーバーや、組織内で管理・審査されたサーバーを選ぶのが安全です。第10章で学んだ「最小権限の原則」と同じ考え方を MCP にも適用してください。

やってみよう 11-1: /mcp で接続状況を確認する

Claude Code を起動して、現在の MCP 接続状況を確認してみましょう。

手順

  1. 1 ターミナルで claude を起動する。
  2. 2 /mcp と入力して Enter を押し、登録済みの MCPサーバー一覧を確認する。初期状態では「接続中のサーバーなし」と表示される場合があります。
  3. 3 （任意）興味があれば、MCP 公式サイトから用途に合ったサーバーを探し、ドキュメントに沿って claude mcp add で接続を追加してみる。
  4. 4 追加後に再度 /mcp を実行し、サーバーが一覧に表示されることを確認する。

確認ポイント

/mcp を実行したとき、サーバーの名前と接続状態が表示されれば成功です。何も登録されていない場合は「現在 MCPサーバーは登録されていません」のようなメッセージが表示されます。

理解度確認クイズ

  1. Q1. MCP（Model Context Protocol）とは何ですか？誰が提唱しましたか？

    回答を表示

    MCP は、AI アシスタントと外部のツールやデータソースを接続するためのオープンな標準プロトコルです。Anthropic が提唱し、オープンソースで公開されています。共通の「通信ルール」を定めることで、さまざまな外部サービスと同じ方法で連携できるようにします。

  2. Q2. MCPサーバーを追加するコマンドと、接続状況を確認する方法をそれぞれ答えてください。

    回答を表示

    MCPサーバーの追加は claude mcp add で行い、書き方は接続方式によって変わります。リモートは claude mcp add --transport http <名前> <URL>（HTTP 推奨）、ローカルは claude mcp add <名前> -- <起動コマンド>（stdio）です。接続状況の確認は、Claude Code 起動中に /mcp と入力することで一覧表示できます。

  3. Q3. MCPサーバーを追加するときに守るべきセキュリティ上の注意を1つ挙げてください。

    回答を表示

    「信頼できる出所のサーバーのみ追加する」こと。悪意ある MCPサーバーは、データの読み取りや意図しない操作を行う可能性があります。公式・オープンソース・組織で審査済みのサーバーを利用するのが基本です。（他にも「権限の範囲を確認する」「不要なサーバーは削除する」が正解として挙げられます。）
