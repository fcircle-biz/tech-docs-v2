Claude Codeとは

Claude Code は、ターミナル上で動作する Anthropic 公式の「エージェント型コーディングツール」です。この章では、Claude Code が何で、何ができ、従来のツールとどう違うのか、そして AI がどのように作業を進めるのかという全体像をつかみます。

この章で学ぶこと

  • Claude Code とは何か、誰が提供しているのかを理解する
  • 従来のコード補完ツールやチャット型 AI との違いを説明できる
  • Claude Code でできることの具体例を知る
  • 「計画→編集→検証」というエージェントの働き方の全体像をつかむ

1.1 Claude Code とは何か

Claude Code は、AI 企業 Anthropic が提供する公式のコーディング支援ツールです。最大の特徴は、ターミナル（コマンドライン）上で動作し、自然言語（ふだんの日本語や英語）で指示するだけで、AI が自分でコードを読み、考え、ファイルを編集し、コマンドを実行して、開発作業を進めてくれる点にあります。

単にコードの続きを提案したり（補完）、質問に答えたり（チャット）するだけのツールとは異なり、Claude Code は「やってほしいこと」を伝えると、その完了に向けて一連の作業を主体的に進めることができます。このような AI を「エージェント（agent）」と呼びます。

ターミナルで動く

使い慣れたコマンドラインから起動。特別な画面に切り替える必要がありません。

自然言語で指示

「このバグを直して」のように、ふだんの言葉で依頼できます。

作業を完了させる

調査・編集・実行・確認まで、一連の作業を自分で進めます。

用語: 「Claude」と「Claude Code」の違い

Claude は Anthropic が開発した AI（大規模言語モデル）そのものの名前です。Claude Code は、その Claude を「開発作業のためのエージェント」として使えるようにしたツールを指します。本ガイドで学ぶのは後者の「ツール」の使い方です。

1.2 従来ツールとの違い

AI を使った開発支援ツールには、大きく3つのタイプがあります。それぞれの違いを理解すると、Claude Code の立ち位置がはっきりします。

タイプ 代表例 できること
コード補完型 GitHub Copilot など 入力中のコードの続きを提案する。あくまで「補助」。
チャット型 ブラウザで使う対話 AI など 質問に答える・コード例を出す。ファイルの編集や実行は自分で行う必要がある。
エージェント型 Claude Code 指示に基づき、自分でファイルを読み・編集し、コマンドを実行し、結果を確認して作業を完了させる。

つまり Claude Code は、「提案してくれる」「教えてくれる」を超えて、実際に手を動かして作業を進めてくれるところに大きな違いがあります。もちろん、変更を適用する前に内容を確認し、許可を求める仕組みも備わっているため、安心して使えます（詳しくは第10章で学びます）。

1.3 Claude Code でできること

Claude Code は、開発に関わるさまざまな作業を任せられます。代表的な活用例は次のとおりです。

  • 新機能の実装 — 「ログイン画面を追加して」のような依頼からコードを作成
  • バグ修正 — エラーの原因を調べて修正
  • リファクタリング — 動作を変えずにコードを整理・改善
  • コードの調査・説明 — 「この処理の流れを説明して」に回答
  • テストの作成・実行 — テストコードを書き、実行して結果を確認
  • Git 操作 — 変更のコミットやプルリクエストの作成（第9章）

実際のセッション（やり取り）は、次のようなイメージで進みます。

terminal — Claude Code セッション例 
$ claude
> ログイン処理にバグがあるようなので、原因を調べて直してほしい

● src/auth/login.js を読み込んでいます…
● 原因はトークンの有効期限チェックが抜けていることでした。修正します。
● テストを実行して動作を確認しました（5件すべて成功）。
● 修正が完了しました。変更内容を確認しますか？

1.4 エージェントの働き方（全体像）

Claude Code は、指示を受け取ってから完了するまで、人間の開発者と同じように「計画 → 調査 → 編集・実行 → 検証」というステップを繰り返します。検証で問題が見つかれば、計画に戻ってやり直します。このループを理解しておくと、AI が何をしているのかが見えやすくなります。

flowchart TD A["あなたの指示（自然言語）"] --> B["計画を立てる"] B --> C["コードベースを読む・調べる"] C --> D["ファイルを編集する／コマンドを実行する"] D --> E["結果を検証する（テスト・確認）"] E -->|うまくいった| F["変更内容を報告する"] E -->|問題があった| B

重要ポイント

Claude Code は「魔法の箱」ではなく、あなたの指示と確認のもとで動く協働パートナーです。良い指示を出し、変更内容を確認しながら使うことで、その力を最大限に引き出せます。

やってみよう 1-1: Claude Code を使える場面を考える

インストール前のこの段階では、まず「自分のどの作業に Claude Code を活かせそうか」をイメージしてみましょう。手を動かす前に活用イメージを持つことが、上達の近道です。

手順

  1. 1 自分が普段行っている開発・作業を3つ書き出してみる（例: バグ修正、ドキュメント作成、調査）。
  2. 2 そのうち、1.3 の活用例に当てはまり「Claude Code に任せられそう」なものに印を付ける。
  3. 3 第3章でインストールしたら、その作業を実際に依頼してみることを目標にする。

理解度確認クイズ

  1. Q1. Claude Code が、コード補完ツールやチャット型 AI と最も異なる点は何ですか？

    回答を表示

    指示に基づいて、自分でファイルを読み・編集し、コマンドを実行し、結果を確認するところまで「作業を完了させる」エージェント型である点。補完は入力の提案、チャットは質問への回答が中心で、編集や実行は人が行います。

  2. Q2. Claude Code は何の上で（どこで）動作しますか？

    回答を表示

    ターミナル（コマンドライン）上で動作します。使い慣れたターミナルから claude コマンドで起動します。

  3. Q3. エージェントの基本的な働き方のループを、順番に説明してください。

    回答を表示

    「計画を立てる → コードベースを調べる → ファイルを編集・コマンドを実行する → 結果を検証する」というステップを繰り返します。検証で問題があれば計画に戻り、うまくいけば変更内容を報告します。
