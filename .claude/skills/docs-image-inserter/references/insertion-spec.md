# 差し込み指示Markdownの読み取り

`work/<名称>/` に置かれた「差し込み指示Markdown」（例: `差し込み箇所.md`）を真実源として、何を・どこへ差し込むかを確定する。指示書の体裁は案件ごとに揺れるため、**意味で読む**こと（厳密フォーマットを要求しない）。

## 入力ディレクトリの判別

`work/<名称>/` の中身は典型的に次で構成される。

- 画像ファイル: `*.png` `*.jpg` `*.jpeg` `*.webp` `*.svg` `*.gif`（ファイル名は任意・UUID 可）。
- 差し込み指示Markdown: `*.md`（`差し込み箇所.md` 等）。1ファイルを想定。

`<名称>` 省略時は `work/` 直下のうち「画像 ＋ 指示 `.md`」を含むディレクトリを対象とする。

## 指示書から抽出する項目（画像ごと）

各画像について次を読み取る。欠けていれば本文・画像内容から補う。

| 項目 | 説明 | 補完方法 |
|------|------|----------|
| 論理名 | 配置時の正式ファイル名（例 `git-github-overview.png`） | 指示書の「差し込みイメージ」内の `images/<name>.png` を採用。無ければ役割から命名 |
| 差し込み箇所 | 対象の見出し・アンカーテキスト・「○○の直後」等 | 「差し込み箇所」節と表から特定 |
| 推奨キャプション | `figcaption` に入れる文（例 `図1-1 …`） | 「推奨キャプション」を採用。無ければ役割から作文 |
| 役割 | その図が示す内容（同定と alt 作成に使う） | 「役割」列・本文から |
| alt | 本文に沿った日本語の代替テキスト | 役割・画像内容から作文（装飾でなく内容を述べる） |

## 対象ガイド（分類パス／slug）と章番号の特定

指示書は対象章をファイル名・URL で示すことが多い。

- 例: `.../guide/development-processes/git-github/git-github-learning-material-01.html`
  - **分類パス** = `development-processes`
  - **slug** = `git-github`
  - **章番号 NN** = `01`
- 対象の本文断片は `astro-system/src/chapters/<分類パス>/<slug>/<slug>-learning-material-NN.html`。

URL/ファイル名が無い場合の確定手順:

1. `work/<名称>` の `<名称>` を手がかりに、`astro-system/src/chapters/` 配下の slug と照合（例 `github` → `development-processes/git-github`）。
2. 候補が複数あるときは、指示書中の見出し文言（例 `# Git/GitHubで何ができるのか`）を各章 HTML から grep して一致する章に確定する。
3. 複数章にまたがる指示なら、画像ごとに対象章を割り当てる。

## 差し込み位置のアンカー特定

指示書は「`# 見出し` の直後」「`## 1.1 …` 内のコード例の直後」のように位置を示す。章 HTML（本文断片）側で対応する箇所を特定する際の対応関係:

- Markdown の見出し `#`/`##` → HTML の `<h1>`/`<h2 ...>` のテキスト一致で探す。
- 「コード例の直後」→ 該当 `<pre><code>…</code></pre>` を含む `code-block-wrapper` の閉じ `</div>` の直後。
- 「表の直後」→ 該当 `<table>` を包む `overflow-x-auto` の閉じ `</div>` の直後。
- 「既存フロー図の直後／前」→ `<div class="mermaid">…</div>` を包む枠の前後。既定は **説明 → 画像 → 既存図** の順（画像を既存図の直前に置く）。

挿入は既存の段落・カード等のブロック境界に対して行い、要素の途中に割り込ませない。
