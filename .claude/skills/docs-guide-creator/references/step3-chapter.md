# step3: 単一章の本文断片生成（並列サブエージェント用）

技術分野の初心者向けHTMLベース教育コンテンツ作成の専門家として、**指定された1章分の本文断片を生成する**。このファイルは生成フェーズで `general-purpose`・`model: sonnet` のサブエージェントとして並列起動される。

> **旧フローからの変更**: 生成するのは**本文断片**（`max-w-5xl` ラッパー内の本文のみ）。フル HTML（head/ヘッダー/フッター/スクリプト）は生成しない。共通部品（CSS/JS/サイドバー）・データ.ts・index.ts には触れない。

## 重要な実行ルール

**【絶対遵守】以下をすべて完了するまで終了してはいけない:**

1. カリキュラム/README の読み込みと分析
2. 第1章断片の読み込み（構造ベース）
3. **指定章の本文断片を生成（Write で保存）**
4. 生成ファイルの確認と報告

**途中で「これから生成します」と報告して終了することは禁止。実際にファイルが生成されるまで処理を継続すること。**

## 前提（呼び出し元から渡される情報）

- カリキュラム設計（または `public/guide/<category>/<slug>/README.md`）のフルパス
- 第1章断片（`<slug>-learning-material-01.html`）のフルパス ← 構造ベース（継承元）
- 生成する章番号
- 出力先フルパス（`src/chapters/<category>/<slug>/<slug>-learning-material-NN.html`）

技術フォルダには既に以下が存在する（step1/step2 で作成済み・**触らない**）:

- `src/data/guides/<category>/<slug>.ts`（`primary`・`chapters[]`）
- `src/data/guides/index.ts`（登録済み）
- `<slug>-learning-material-01.html`（第1章断片＝構造ベース）

共通部品（`GuideChapterLayout.astro`・`_shared/styles.css`・`_shared/main.js`・`_shared/drawing-tool.js`）は全技術共有でレイアウトが供給する。**これらは生成・編集しない。**

## 実行手順

### 1. カリキュラム分析
渡されたカリキュラム/README を読み込み、指定章番号の情報（学習目標・内容）を抽出する。

### 2. 第1章断片の構造を確認
第1章断片を Read で読み込み、以下を把握する。

- パンくずリストの構造
- 章ヘッダー（章番号バッジ・h1・説明文）
- 学習目標カード
- 本文セクションの構造・クラス名
- 前章／次章ナビゲーションの構造

参照ドキュメント（必ず参照）:
- カードコンポーネント: `astro-system/templates/v1/snippets/components.html`
- Mermaid図表パターン: `astro-system/templates/v1/reference/mermaid-patterns.md`
- 共通HTMLルール（断片構造・Mermaid記法・コンポーネント・ダーク可読性・コード方針）: references/html-rules.md

### 3. 指定章の本文断片を生成【最重要ステップ】

1. 第1章断片を Read で読み込み、その構造をベースとする。
2. カリキュラム/README から該当章の情報を読み取る。
3. 章番号・タイトル・内容を該当章のものに書き換える。
4. Write で保存する（ファイル名: `<slug>-learning-material-NN.html`、NN は2桁）。

#### 書き換える箇所

1. パンくずリストの章番号
2. 章ヘッダー（章番号バッジ・h1タイトル・説明文）
3. 学習目標カードの内容
4. 本文セクション（カリキュラム/README の該当章の内容に基づく）
5. 前章／次章ナビゲーションのリンクと章番号（相対 `<slug>-learning-material-NN.html`）

#### 守る箇所（第1章断片の構造を継承）

- パンくず・章ヘッダー・学習目標カード・前後ナビの**構造とクラス名**
- 技術色は `primary` クラス（`text-primary-600` 等）で参照する（生 hex を書かない）

### 4. 完了報告

```
## 生成完了

以下のファイルを生成しました：
- src/chapters/<category>/<slug>/<slug>-learning-material-NN.html
```

## 生成ルール

1. カリキュラム/README から該当章の学習目標・内容を読み取り反映する。
2. コードブロックは15行以内、日本語コメント必須。
3. Mermaid図表はデフォルトテーマ（dark テーマ禁止）。記法詳細は references/html-rules.md（CLAUDE.md 方式）。
4. **第1章断片の構造・スタイルを完全に継承すること。**
5. **断片のみ生成**（`<head>`・グローバルヘッダー・フッター・スクリプト・`tailwind.config` を書かない）。

## ダーク可読性（標準クラスのみ）

`_shared/styles.css` のダーク補正は標準 Tailwind クラス（`bg-*-50〜300`／`text-*-600〜950`）を自動補正する。**生color指定（`text-[#333]` 等）・インライン `style="color/background"`・半透明背景（`bg-white/70` 等）は禁止**（ダーク事故源）。内側ボックスは同系色 `-100` 背景＋`-900` テキスト。

## 重要事項

- **データ.ts・index.ts・共通部品（レイアウト／`_shared/`）を生成・上書きしないこと。**
- **第1章断片の構造から逸脱しないこと。**
- フル HTML（head/ヘッダー/フッター/スクリプト）を書かないこと。

## 初心者向け重点事項

- 理論・背景説明優先。用語解説充実。視覚的理解促進（図表多用）。段階的理解構築。

## 禁止事項

- 「これから生成します」と報告して終了すること。
- ファイルを生成せずに終了すること。
- 共通部品（レイアウト／`_shared/` の CSS/JS）・データ.ts・index.ts を上書きすること。
- 断片にフル HTML 構造（head/ヘッダー/フッター/スクリプト）を混入すること。
